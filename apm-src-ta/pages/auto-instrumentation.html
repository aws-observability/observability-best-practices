Auto Instrumentation

"Auto instrumentation" என்றால் என்ன

Auto instrumentation இயக்க நேரத்தில் OpenTelemetry instrumentation-ஐ சேர்க்கிறது (உங்கள் பயன்பாட்டு code-ஐ மாற்றாமல்), இதனால் சேவை Application Signals கண்டறிதல் மற்றும் ஆரோக்கிய கண்காணிப்புக்கு பயன்படுத்தும் telemetry-ஐ வெளியிடுகிறது.

பொதுவான AWS அமைப்பு கூறுகள்

  • AWS Distro for OpenTelemetry (ADOT) SDK மற்றும்/அல்லது collector
  • CloudWatch Agent ஆதரிக்கப்படும் அமைப்புகளில் telemetry-ஐ சேகரிக்க/ஏற்றுமதி செய்ய
  • Platform enablement (EKS add-on / ECS sidecar / EC2 agent அமைப்பு / Lambda layers)

இங்கே தொடங்கவும்: Application Signals மேலோட்டம்.

ஆதரிக்கப்படும் மொழிகள் மற்றும் அமைப்புகள்

Application Signals Java, Python, Node.js மற்றும் .NET பயன்பாடுகளை ஆதரிக்கிறது மற்றும் Amazon EKS, Kubernetes, Amazon ECS மற்றும் Amazon EC2-ல் ஆதரிக்கப்படுகிறது/சோதிக்கப்படுகிறது.

பாருங்கள்: ஆதரிக்கப்படும் அமைப்புகள்.

குறிப்புகள் மற்றும் கவனிக்க வேண்டியவை

  • Node.js ESM: ESM ஆதரவு வரையறுக்கப்பட்டது; ஆவணங்களில் ESM-குறிப்பிட்ட வழிகாட்டுதலைப் பின்பற்றாவிட்டால் CommonJS பரிந்துரைக்கப்படும் பாதையாகும்.
  • Containers-ல் Python: OTel autoinstrumentation-க்கு PYTHONPATH கட்டமைக்கப்பட வேண்டியிருக்கலாம்.
