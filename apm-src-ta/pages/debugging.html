CloudWatch APM ஆவணங்கள்

பிழைத்திருத்த வழிகாட்டிகள்

விரைவான பகுப்பாய்வு பணிப்பாய்வு (பரிந்துரைக்கப்படுகிறது)

  1. SLO-களில் தொடங்குங்கள் (வரையறுக்கப்பட்டிருந்தால்): ஆரோக்கியமற்ற சேவைகள்/செயல்பாடுகள்/சார்புகளை அடையாளம் காணுங்கள்.
  2. சேவைகளைப் பயன்படுத்துங்கள்: முக்கிய குற்றவாளிகளைக் கண்டறிய அதிக தாமதம் அல்லது தவறு விகிதத்தின்படி வரிசைப்படுத்துங்கள்.
  3. சேவை விவரத்தைத் திறங்கள்: செயல்பாடுகள் மற்றும் சார்பு விளிம்புகளை ஆராயுங்கள்; தாமதம்/தவறு தரவுப்புள்ளியைத் தேர்ந்தெடுங்கள்.
  4. தொடர்புடைய traces/spans-க்கு செல்லுங்கள்: மெதுவான/தோல்வியடைந்த கீழ்நிலை அழைப்பைக் கண்டறிய trace timeline-ஐ ஆராயுங்கள்.
  5. லாக்கள்/runtime மெட்ரிக்ஸுடன் உறுதிப்படுத்துங்கள்: பொருத்தமான போது பிழை லாக்கள் மற்றும் JVM/CPU/GC-ஐ தொடர்புபடுத்துங்கள்.

பார்க்கவும்: Application Signals மூலம் செயல்பாட்டு நிலையை கண்காணிக்கவும்.

"அரிய உச்சங்களுக்கு" Transaction Search பயன்படுத்துதல்

இடையிடையே ஏற்படும் உச்சங்களுக்கு, Transaction Search பண்புக்கூறு அடிப்படையில் spans-ஐ பகுப்பாய்வு செய்ய உதவுகிறது மற்றும் பாதிக்கப்பட்ட பயனர்கள்/கோரிக்கைகளை ஆராய உதவுகிறது. பொதுவான வடிவங்கள்:

  • குழு பகுப்பாய்வு வணிக பண்புக்கூறின்படி (எடுத்துக்காட்டாக, வாடிக்கையாளர் ID) யார் பாதிக்கப்பட்டுள்ளனர் என்பதைக் கண்டறியவும்.
  • வடிகட்டுதல் service/operation மற்றும் duration (p99) அடிப்படையில் விலகல்களை தனிமைப்படுத்தவும்.
  • ஒரு trace-ஐ திறங்கள் விலகல் span-இலிருந்து மற்றும் சார்பு அழைப்பு பாதையைப் பின்தொடரவும்.

பார்க்கவும்: பயன்பாட்டு சிக்கல்களை பிழைத்திருத்துதல் (Transaction Search).

பொதுவான "ஏன் தரவு தெரியவில்லை?" சோதனைகள்

  • கணக்கு இயக்கம்: Application Signals ஒவ்வொரு கணக்கு/பகுதிக்கும் ஒரு முறை இயக்கப்பட வேண்டும் (service-linked role உருவாக்கப்படும்).
  • நேர வரம்பு: தேர்ந்தெடுக்கப்பட்ட நேர சாளரத்தில் செயல்பாடு இருந்தால் மட்டுமே சேவைகள்/செயல்பாடுகள் காண்பிக்கப்படும்.
  • சேவை பெயரிடுதல்: service.name / OTEL_RESOURCE_ATTRIBUTES சரியாக அமைக்கப்பட்டுள்ளதை உறுதிப்படுத்துங்கள் (EKS அல்லாதவற்றுக்கு பெரும்பாலும் வெளிப்படையான service/environment பெயர்கள் தேவை).
  • Pipeline நிலை: CloudWatch Agent/collector இயங்குகிறது மற்றும் OTLP பெறுகிறது; உள்ளூர் endpoints மற்றும் IAM அனுமதிகளை சரிபாருங்கள்.

பார்க்கவும்: உங்கள் Application Signals நிறுவலை பிழைத்திருத்துதல்.
