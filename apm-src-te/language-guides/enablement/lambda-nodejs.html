Overview

This guide provides step-by-step instructions to deploy a Node.js Lambda function with AWS Application Signals enabled. The simplest approach using AWS Lambda console with built-in layer support.

What you'll accomplish:
  • Deploy a Node.js Lambda function with Application Signals
  • Enable AWS Application Signals auto-instrumentation via Lambda Layer
  • View your service in CloudWatch Application Signals console
  • Monitor service metrics, traces, and operations
Time to complete: 15-20 minutes

Prerequisites

Required Tools

  • AWS CLI v2 (latest version)
  • Node.js 18+
  • npm
  • AWS Account with appropriate permissions
  • Text editor

AWS Permissions Required

  • Lambda: Create and manage functions
  • IAM: Create roles and policies
  • CloudWatch: Logs and Application Signals access
  • S3: Basic read access (for testing)

Architecture

Your Lambda function will be automatically instrumented using:

  1. AWS Lambda Layer for OpenTelemetry - Provides auto-instrumentation
  2. Application Signals Integration - Built into the layer
  3. Automatic Tracing - HTTP requests, AWS SDK calls, function execution

Step 1: Enable Application Signals in Your Account

Important: Application Signals must be enabled once per AWS account per region.

Option 1: Enable via AWS Console (Recommended)

  1. Go to AWS Console → CloudWatch → Application Signals
  2. Select your region (e.g., us-east-1)
  3. Click "Get Started" or "Enable Application Signals"
  4. Wait 2-3 minutes for activation

2.1 Create Application Directory

mkdir -p ~/my-lambda-nodejs-appsignals
cd ~/my-lambda-nodejs-appsignals

2.3 Create index.js

Create the main Lambda function code with various endpoints for testing:

const AWS = require('aws-sdk');
const axios = require('axios');

const s3 = new AWS.S3();

exports.handler = async (event, context) => {
    console.log('Received event:', JSON.stringify(event, null, 2));
    
    try {
        const path = event.path || '/';
        const httpMethod = event.httpMethod || 'GET';
        
        const response = await routeRequest(path, httpMethod, event, context);
        
        return {
            statusCode: 200,
            headers: {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            body: JSON.stringify(response)
        };
        
    } catch (error) {
        console.error('Error processing request:', error);
        
        return {
            statusCode: 500,
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
                error: 'Internal Server Error',
                message: error.message,
                timestamp: Date.now(),
                function_name: context.functionName
            })
        };
    }
};

async function routeRequest(path, httpMethod, event, context) {
    if (path === '/' || path === '') {
        return await rootHandler(context);
    } else if (path === '/health') {
        return await healthHandler(context);
    } else if (path.startsWith('/api/hello/')) {
        const name = path.substring('/api/hello/'.length);
        return await helloHandler(name, context);
    } else if (path === '/api/slow') {
        return await slowHandler(context);
    } else if (path === '/api/error') {
        return await errorHandler(context);
    } else if (path === '/api/external') {
        return await externalHandler(context);
    } else if (path === '/api/s3') {
        return await s3Handler(context);
    } else if (path === '/api/database') {
        return await databaseHandler(context);
    } else if (path === '/api/complex') {
        return await complexHandler(context);
    } else {
        return {
            error: 'Not Found',
            path: path,
            timestamp: Date.now()
        };
    }
}

// Handler implementations...
async function rootHandler(context) {
    return {
        message: 'AWS Application Signals Lambda Node.js Demo',
        timestamp: Date.now(),
        function_name: context.functionName,
        request_id: context.awsRequestId,
        service: 'lambda-nodejs',
        runtime: 'nodejs18.x'
    };
}

async function healthHandler(context) {
    return {
        status: 'healthy',
        timestamp: Date.now(),
        function_name: context.functionName,
        memory_limit: context.memoryLimitInMB,
        remaining_time: context.getRemainingTimeInMillis(),
        runtime: 'nodejs18.x'
    };
}

async function helloHandler(name, context) {
    await new Promise(resolve => setTimeout(resolve, Math.random() * 100 + 10));
    
    return {
        message: `Hello, ${name}!`,
        timestamp: Date.now(),
        function_name: context.functionName,
        greeting_id: Math.floor(Math.random() * 10000),
        runtime: 'nodejs18.x'
    };
}

async function slowHandler(context) {
    const delay = Math.random() * 1000 + 500;
    await new Promise(resolve => setTimeout(resolve, delay));
    
    return {
        message: 'Slow operation completed',
        timestamp: Date.now(),
        function_name: context.functionName,
        duration_ms: Math.floor(delay),
        runtime: 'nodejs18.x'
    };
}

async function errorHandler(context) {
    if (Math.random() > 0.5) {
        throw new Error('Simulated error for Application Signals testing');
    }
    
    return {
        message: 'No error occurred',
        timestamp: Date.now(),
        function_name: context.functionName,
        runtime: 'nodejs18.x'
    };
}

async function externalHandler(context) {
    try {
        const response = await axios.get('https://httpbin.org/delay/1', {
            timeout: 10000
        });
        
        return {
            message: 'External call completed',
            external_status_code: response.status,
            timestamp: Date.now(),
            function_name: context.functionName,
            runtime: 'nodejs18.x'
        };
        
    } catch (error) {
        throw new Error(`External call failed: ${error.message}`);
    }
}

async function s3Handler(context) {
    try {
        const response = await s3.listBuckets().promise();
        
        const result = {
            message: 'S3 call completed successfully',
            bucket_count: response.Buckets.length,
            timestamp: Date.now(),
            function_name: context.functionName,
            runtime: 'nodejs18.x'
        };
        
        if (response.Buckets.length > 0) {
            result.first_bucket = response.Buckets[0].Name;
        }
        
        return result;
        
    } catch (error) {
        return {
            error: 'S3 call failed',
            message: error.message,
            timestamp: Date.now(),
            function_name: context.functionName,
            runtime: 'nodejs18.x'
        };
    }
}

async function databaseHandler(context) {
    await new Promise(resolve => setTimeout(resolve, Math.random() * 200 + 50));
    
    return {
        message: 'Database query completed',
        timestamp: Date.now(),
        function_name: context.functionName,
        records_returned: Math.floor(Math.random() * 100) + 1,
        runtime: 'nodejs18.x'
    };
}

async function complexHandler(context) {
    await new Promise(resolve => setTimeout(resolve, Math.random() * 50 + 10));
    await new Promise(resolve => setTimeout(resolve, Math.random() * 100 + 20));
    await new Promise(resolve => setTimeout(resolve, Math.random() * 150 + 30));
    
    return {
        message: 'Complex operation completed successfully',
        timestamp: Date.now(),
        function_name: context.functionName,
        steps_completed: 3,
        total_processing_time_ms: Math.floor(Math.random() * 300) + 60,
        runtime: 'nodejs18.x'
    };
}

Step 4: Create IAM Role for Lambda

Create an IAM role with the following policies:

  • AWSLambdaBasicExecutionRole
  • CloudWatchLambdaApplicationSignalsExecutionRolePolicy (CRITICAL!)
  • AmazonS3ReadOnlyAccess (for testing)

4.1 Create IAM Role

aws iam create-role \
  --role-name lambda-nodejs-execution-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'

4.2 Attach Required Policies

aws iam attach-role-policy \
  --role-name lambda-nodejs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

aws iam attach-role-policy \
  --role-name lambda-nodejs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLambdaApplicationSignalsExecutionRolePolicy

aws iam attach-role-policy \
  --role-name lambda-nodejs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

4.3 Get Role ARN

ROLE_ARN=$(aws iam get-role \
  --role-name lambda-nodejs-execution-role \
  --query 'Role.Arn' \
  --output text)

echo "Role ARN: $ROLE_ARN"

Step 5: Deploy Lambda Function

5.1 Create Lambda Function

aws lambda create-function \
  --function-name lambda-nodejs \
  --runtime nodejs18.x \
  --role "$ROLE_ARN" \
  --handler index.handler \
  --zip-file fileb://lambda-nodejs-appsignals.zip \
  --timeout 30 \
  --memory-size 512

Note: If you plan to enable Application Signals via the Lambda Console (recommended), you can skip the manual layer and environment variable configuration steps below. The console will automatically handle these for you.

Step 7: Generate Traffic for Testing

To test Application Signals, you need to generate traffic to your Lambda function. We provide a Node.js traffic generator:

7.1 Node.js Traffic Generator

Create a traffic-generator.js file:

const AWS = require('aws-sdk');

async function generateTraffic(functionName, region, profile, numRequests) {
    console.log(`Starting traffic generation for ${functionName}`);
    
    AWS.config.update({ region: region });
    if (profile !== 'default') {
        AWS.config.credentials = new AWS.SharedIniFileCredentials({ profile: profile });
    }
    const lambdaClient = new AWS.Lambda();
    
    const endpoints = [
        { path: '/', method: 'GET', name: 'Root', weight: 20 },
        { path: '/health', method: 'GET', name: 'Health', weight: 15 },
        { path: '/api/hello/User', method: 'GET', name: 'Hello', weight: 30 },
        { path: '/api/slow', method: 'GET', name: 'Slow', weight: 10 },
        { path: '/api/error', method: 'GET', name: 'Error', weight: 10 },
        { path: '/api/external', method: 'GET', name: 'External', weight: 8 },
        { path: '/api/s3', method: 'GET', name: 'S3', weight: 5 },
        { path: '/api/database', method: 'GET', name: 'Database', weight: 2 }
    ];
    
    const weightedEndpoints = [];
    endpoints.forEach(endpoint => {
        for (let i = 0; i < endpoint.weight; i++) {
            weightedEndpoints.push(endpoint);
        }
    });
    
    let successCount = 0;
    let errorCount = 0;
    
    for (let i = 1; i <= numRequests; i++) {
        const endpoint = weightedEndpoints[Math.floor(Math.random() * weightedEndpoints.length)];
        
        let path = endpoint.path;
        if (endpoint.name === 'Hello') {
            const userName = `User${Math.floor(Math.random() * 1000) + 1}`;
            path = `/api/hello/${userName}`;
        }
        
        const payload = { path: path, httpMethod: endpoint.method };
        
        try {
            const response = await lambdaClient.invoke({
                FunctionName: functionName,
                InvocationType: 'RequestResponse',
                Payload: JSON.stringify(payload)
            }).promise();
            
            const responsePayload = JSON.parse(response.Payload);
            const statusCode = responsePayload.statusCode || 200;
            
            if (statusCode === 200) {
                successCount++;
                console.log(`SUCCESS Request ${i}: ${endpoint.name} | Status: ${statusCode}`);
            } else {
                errorCount++;
                console.log(`WARNING Request ${i}: ${endpoint.name} | Status: ${statusCode}`);
            }
            
            await new Promise(resolve => setTimeout(resolve, Math.random() * 400 + 100));
            
        } catch (error) {
            errorCount++;
            console.log(`ERROR Request ${i}: ${endpoint.name} | Error: ${error.message}`);
        }
    }
    
    console.log(`Summary: ${successCount} successful, ${errorCount} errors`);
}

// Usage
const functionName = process.argv[2] || 'lambda-nodejs';
const numRequests = parseInt(process.argv[3]) || 50;
generateTraffic(functionName, 'us-east-1', 'eksworkstream', numRequests);

7.2 Shell Script Traffic Generator

Alternative shell script for traffic generation:

#!/bin/bash

# Node.js Lambda Traffic Generator for Application Signals Testing
# Generates various types of requests to test Application Signals metrics and traces

FUNCTION_NAME="lambda-nodejs"
NUM_REQUESTS=50
REGION="us-east-1"
PROFILE="eksworkstream"

# Parse arguments
if [ $# -ge 1 ]; then
    FUNCTION_NAME="$1"
fi
if [ $# -ge 2 ]; then
    NUM_REQUESTS="$2"
fi
if [ $# -ge 3 ]; then
    REGION="$3"
fi
if [ $# -ge 4 ]; then
    PROFILE="$4"
fi

echo "Starting traffic generation for ${FUNCTION_NAME}"
echo "   Region: ${REGION}"
echo "   Profile: ${PROFILE}"
echo "   Requests: ${NUM_REQUESTS}"
echo "--------------------------------------------------"

# Test endpoints with different characteristics
declare -a ENDPOINTS=(
    "root:/:GET:20"
    "health:/health:GET:15"
    "hello:/api/hello:GET:30"
    "slow:/api/slow:GET:10"
    "error:/api/error:GET:10"
    "external:/api/external:GET:8"
    "s3:/api/s3:GET:5"
    "database:/api/database:GET:2"
)

success_count=0
error_count=0

echo "Generating ${NUM_REQUESTS} requests..."
echo

for i in $(seq 1 $NUM_REQUESTS); do
    # Select random endpoint
    endpoint_info=${ENDPOINTS[$((RANDOM % ${#ENDPOINTS[@]}))]}
    IFS=':' read -r name path method weight <<< "$endpoint_info"

    if [ "$name" = "hello" ]; then
        user_name="User$((RANDOM % 1000 + 1))"
        full_path="/api/hello/$user_name"
    else
        full_path="$path"
    fi

    payload="{\"path\": \"$full_path\", \"httpMethod\": \"$method\"}"

    # Invoke Lambda function
    response=$(aws --profile "$PROFILE" --region "$REGION" lambda invoke \
        --function-name "$FUNCTION_NAME" \
        --payload fileb://<(echo "$payload") \
        /tmp/lambda-response.json 2>&1)

    if [ $? -eq 0 ]; then
        # Parse response
        if [ -f /tmp/lambda-response.json ]; then
            status_code=$(cat /tmp/lambda-response.json | jq -r '.statusCode // 200' 2>/dev/null || echo "200")

            if [ "$status_code" = "200" ]; then
                success_count=$((success_count + 1))
                status_icon="SUCCESS"
            else
                error_count=$((error_count + 1))
                status_icon="WARNING"
            fi
        else
            error_count=$((error_count + 1))
            status_icon="ERROR"
            status_code="ERROR"
        fi

        # Print progress
        printf "%s Request %2d: %-8s | Status: %s | Path: %s\n" \
            "$status_icon" "$i" "$name" "$status_code" "$full_path"

        # Clean up
        rm -f /tmp/lambda-response.json

    else
        error_count=$((error_count + 1))
        printf "ERROR Request %2d: %-8s | Error: %s\n" "$i" "$name" "$response"
    fi

    # Add small delay between requests
    sleep $(echo "scale=2; $RANDOM/32767 * 0.4 + 0.1" | bc -l)
done

# Print summary
echo
echo "=================================================="
echo "TRAFFIC GENERATION SUMMARY"
echo "=================================================="
echo "Total Requests: $NUM_REQUESTS"
echo "Successful:     $success_count ($(echo "scale=1; $success_count * 100 / $NUM_REQUESTS" | bc -l)%)"
echo "Errors:         $error_count ($(echo "scale=1; $error_count * 100 / $NUM_REQUESTS" | bc -l)%)"
echo "Timestamp:      $(date '+%Y-%m-%d %H:%M:%S')"
echo
echo "Next Steps:"
echo "1. Wait 5-10 minutes for Application Signals to process data"
echo "2. Check Application Signals console:"
echo "   https://us-east-1.console.aws.amazon.com/cloudwatch/home?region=us-east-1#application-signals:services"
echo "3. Look for 'lambda-nodejs' in the services list"
echo

if [ $success_count -gt 0 ]; then
    echo "Traffic generation completed successfully!"
    exit 0
else
    echo "Traffic generation failed!"
    exit 1
fi

Step 5.3: Enable Application Signals via Lambda Console (Recommended)

RECOMMENDED METHOD! This is the simplest and most reliable approach. The console automatically handles all the complex configuration.

After creating your Lambda function via CLI (Step 5.1), enable Application Signals via the console:

  1. Go to AWS Lambda Console: https://us-east-1.console.aws.amazon.com/lambda/home?region=us-east-1#/functions/lambda-nodejs
  2. Navigate to your lambda-nodejs function
  3. Go to Configuration → Monitoring and operations tools
  4. Click "Edit"
  5. Enable both checkboxes:
    • ✅ "Automatically collect application traces and standard application metrics with Application Signals"
    • ✅ "Automatically collect Lambda service traces for end to end visibility with X-Ray"
  6. Click "Save"

That's it! The console automatically:

  • Adds the AWS Lambda Layer for OpenTelemetry
  • Sets the AWS_LAMBDA_EXEC_WRAPPER environment variable
  • Attaches the required IAM policy

Troubleshooting

Issue: Shell Environment Problems

Problem: Commands fail with errors like parse error near 'cursor_snap_ENV_VARS...' or command not found: dump_zsh_state

Cause: Cursor IDE environment variable injection is corrupting the shell environment

Solutions:

  1. Use a fresh terminal window outside of Cursor IDE
  2. Restart Cursor IDE completely and try again
  3. Use VS Code instead of Cursor IDE
  4. Use the Lambda Console method (recommended for shell issues)

Issue: Role ARN is Empty

Problem: aws: error: argument --role: expected one argument

Cause: The ROLE_ARN variable is empty, indicating IAM role creation failed

Debug Steps:

  1. Check if role exists: 
    aws iam get-role --role-name lambda-nodejs-execution-role
  2. Create role if missing: 
    aws iam create-role \
  --role-name lambda-nodejs-execution-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'
  3. Get role ARN manually: 
    aws iam get-role \
  --role-name lambda-nodejs-execution-role \
  --query 'Role.Arn' \
  --output text
  4. Use ARN directly in create-function command

Issue: Command Syntax Errors

Common mistakes:

  • Running multiple commands on the same line with periods
  • Using duplicate parameters (e.g., --role twice)
  • Missing quotes around variables

❌ WRONG - Don't do this:

# WRONG: Commands on same line with period
aws iam create-role \ ... }. aws lambda create-function \ ...

# WRONG: Duplicate --role parameter
aws lambda create-function \
  --role ${ROLE_ARN} \
  --role lambda-nodejs-execution-role

# WRONG: Missing quotes around variable
--role $ROLE_ARN

✅ CORRECT - Do this instead:

# Step 1: Create IAM role (separate command)
aws iam create-role \
  --role-name lambda-nodejs-execution-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'

# Step 2: Attach policies (separate commands)
aws iam attach-role-policy \
  --role-name lambda-nodejs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

aws iam attach-role-policy \
  --role-name lambda-nodejs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLambdaApplicationSignalsExecutionRolePolicy

# Step 3: Get role ARN
ROLE_ARN=$(aws iam get-role \
  --role-name lambda-nodejs-execution-role \
  --query 'Role.Arn' \
  --output text)

# Step 4: Create Lambda function (with quotes around variable)
aws lambda create-function \
  --function-name lambda-nodejs \
  --runtime nodejs18.x \
  --role "$ROLE_ARN" \
  --handler index.handler \
  --zip-file fileb://lambda-nodejs-appsignals.zip \
  --timeout 30 \
  --memory-size 512

Issue: Service Not Appearing in Application Signals

Check:

  • Verify Application Signals is enabled in your account
  • Check if the OpenTelemetry layer is attached
  • Verify environment variables are set correctly
  • Wait 5-10 minutes after generating traffic
  • Ensure the IAM policy CloudWatchLambdaApplicationSignalsExecutionRolePolicy is attached

Issue: Lambda Function Errors

Common issues:

  • Missing IAM policy - Ensure CloudWatchLambdaApplicationSignalsExecutionRolePolicy is attached
  • Wrong layer ARN - Verify the layer ARN for your region
  • Missing environment variable - Ensure AWS_LAMBDA_EXEC_WRAPPER=/opt/otel-instrument
  • Handler mismatch - Ensure handler is index.handler
  • Zip file not found - Ensure npm package completed successfully

Issue: Permission Denied Errors

Check your AWS credentials:

  • Verify AWS profile: aws configure list
  • Check caller identity: aws sts get-caller-identity
  • Ensure you have the required IAM permissions
