CloudWatch APM డాక్యుమెంటేషన్

ట్రబుల్‌షూటింగ్

సాధారణ సమస్యలు

ఈ విభాగం CloudWatch Application Signals ను అమలు చేసేటప్పుడు తరచుగా ఎదుర్కొనే సమస్యలను మరియు వాటిని ఎలా పరిష్కరించాలో వివరిస్తుంది.

CloudWatch లో ట్రేస్‌లు కనిపించడం లేదు

సాధ్యమైన కారణాలు:

  • సేవ కోసం Application Signals ఎనేబుల్ చేయబడలేదు
  • తప్పు IAM అనుమతులు
  • ఏజెంట్ సరిగ్గా కాన్ఫిగర్ చేయబడలేదు లేదా రన్ అవడం లేదు
  • శాంప్లింగ్ రేట్ చాలా తక్కువగా సెట్ చేయబడింది
  • CloudWatch కు నెట్‌వర్క్ కనెక్టివిటీ సమస్యలు

పరిష్కారాలు:

  1. AWS కన్సోల్‌లో Application Signals ఎనేబుల్ అయ్యిందని ధృవీకరించండి
  2. IAM అనుమతులలో cloudwatch:PutMetricData మరియు xray:PutTraceSegments ఉన్నాయని చెక్ చేయండి
  3. ఏజెంట్ రన్ అవుతోందని మరియు సరిగ్గా కాన్ఫిగర్ చేయబడిందని నిర్ధారించండి
  4. పరీక్ష కోసం శాంప్లింగ్ రేట్‌ను పెంచండి (తాత్కాలికంగా 100% కు సెట్ చేయండి)
  5. VPC సెక్యూరిటీ గ్రూప్‌లు మరియు NACL లు CloudWatch కు అవుట్‌బౌండ్ ట్రాఫిక్‌ను అనుమతిస్తాయని చెక్ చేయండి

తప్పిపోయిన లేదా అసంపూర్ణ ట్రేస్‌లు

సాధారణ సమస్యలు:

  • సేవల మధ్య సందర్భం ప్రచారం కాలేదు
  • కొన్ని సేవలు ఇన్‌స్ట్రుమెంట్ చేయబడలేదు
  • శాంప్లింగ్ స్పాన్‌లను డ్రాప్ చేస్తోంది
  • ఏజెంట్ కాన్ఫిగరేషన్ సమస్యలు

డీబగ్గింగ్ దశలు:

  1. ఇన్‌స్ట్రుమెంట్ చేయబడని సేవలను గుర్తించడానికి సర్వీస్ మ్యాప్ చెక్ చేయండి
  2. ట్రేస్ కాంటెక్స్ట్ హెడర్‌లు పాస్ అవుతున్నాయని ధృవీకరించండి (ఉదా., x-amzn-trace-id)
  3. కాన్ఫిగరేషన్ ఎర్రర్‌ల కోసం ఏజెంట్ లాగ్‌లను సమీక్షించండి
  4. శాంప్లింగ్ సమస్యలను తొలగించడానికి 100% శాంప్లింగ్‌తో పరీక్షించండి
  5. పాక్షిక ట్రేస్‌లను కనుగొనడానికి ట్రేస్ సెర్చ్ ఉపయోగించండి

అధిక ఇన్‌స్ట్రుమెంటేషన్ ఓవర్‌హెడ్

లక్షణాలు:

  • పెరిగిన CPU వినియోగం
  • అధిక మెమరీ వినియోగం
  • పెరిగిన లేటెన్సీ
  • అప్లికేషన్ పనితీరు క్షీణత

ఆప్టిమైజేషన్ వ్యూహాలు:

  • శాంప్లింగ్ రేట్‌ను తగ్గించండి (ప్రొడక్షన్ కోసం 10% తో ప్రారంభించండి)
  • టెయిల్-ఆధారిత బదులు హెడ్-ఆధారిత శాంప్లింగ్ ఉపయోగించండి
  • తగిన స్పాన్ పరిమితులను కాన్ఫిగర్ చేయండి
  • బ్యాచ్ స్పాన్ ఎక్స్‌పోర్ట్‌లు
  • ఇన్‌స్ట్రుమెంటేషన్ మెట్రిక్స్‌ను మానిటర్ చేయండి

RUM డేటా కనిపించడం లేదు

చెక్‌లిస్ట్:

  • RUM వెబ్ క్లయింట్ సరిగ్గా ఇన్‌స్టాల్ చేసి లోడ్ చేయబడింది
  • అప్లికేషన్ ID మరియు కాన్ఫిగరేషన్ సరైనది
  • RUM డొమైన్‌ల కోసం CORS హెడర్‌లు కాన్ఫిగర్ చేయబడ్డాయి
  • Content Security Policy RUM స్క్రిప్ట్‌లను అనుమతిస్తుంది
  • బ్రౌజర్ కన్సోల్ JavaScript ఎర్రర్‌లు చూపించదు

ధృవీకరణ:

// Check if RUM is loaded
console.log(window.cwr);

// Verify configuration
console.log(cwr('getConfig'));

Synthetic కెనరీలు విఫలమవుతున్నాయి

సాధారణ కారణాలు:

  • కెనరీ VPC నుండి నెట్‌వర్క్ కనెక్టివిటీ సమస్యలు
  • ప్రమాణీకరణ లేదా ఆథరైజేషన్ వైఫల్యాలు
  • టెస్ట్ స్క్రిప్ట్‌లను విచ్ఛిన్నం చేసే అప్లికేషన్ మార్పులు
  • కెనరీ రన్‌టైమ్ లేదా డిపెండెన్సీలు పాతవి

డీబగ్గింగ్:

  1. CloudWatch Logs లో కెనరీ లాగ్‌లను సమీక్షించండి
  2. కెనరీ స్క్రిప్ట్‌లను స్థానికంగా పరీక్షించండి
  3. VPC రూటింగ్ మరియు సెక్యూరిటీ గ్రూప్‌లను చెక్ చేయండి
  4. అవసరమైతే కెనరీ రన్‌టైమ్‌ను అప్‌డేట్ చేయండి
  5. అప్లికేషన్ ఎండ్‌పాయింట్‌లు యాక్సెస్ చేయగలవని ధృవీకరించండి

డీబగ్గింగ్ గైడ్‌లు

సంక్లిష్ట సమస్యలను నిర్ధారించడానికి దశల వారీ గైడ్‌లు.

ట్రేస్ కాంటెక్స్ట్ ప్రచారాన్ని డీబగ్ చేయడం

దశ 1: ఇన్‌స్ట్రుమెంటేషన్ ధృవీకరించండి

మీ కాల్ చైన్‌లోని అన్ని సేవలు సరిగ్గా ఇన్‌స్ట్రుమెంట్ చేయబడ్డాయని చెక్ చేయండి:

# Check for trace headers in requests
curl -H "x-amzn-trace-id: Root=1-5759e988-bd862e3fe1be46a994272793;Parent=53995c3f42cd8ad8;Sampled=1" \
     http://your-service/endpoint

దశ 2: ఏజెంట్ కాన్ఫిగరేషన్ చెక్ చేయండి

ఏజెంట్ కాన్ఫిగరేషన్ ఫైల్‌లు మరియు ఎన్విరాన్‌మెంట్ వేరియబుల్‌లను ధృవీకరించండి.

దశ 3: ఎండ్-టు-ఎండ్ పరీక్ష

ఒక టెస్ట్ రిక్వెస్ట్ చేసి అన్ని సేవల ద్వారా దాన్ని ట్రేస్ చేయండి.

పనితీరు సమస్య నిర్ధారణ

అడ్డంకిని గుర్తించండి:

  1. అధిక-లేటెన్సీ సేవల కోసం సర్వీస్ మ్యాప్ చెక్ చేయండి
  2. నెమ్మదైన స్పాన్‌ల కోసం ట్రేస్ టైమ్‌లైన్‌లను పరిశీలించండి
  3. రిసోర్స్ మెట్రిక్స్ (CPU, మెమరీ, డిస్క్ I/O) సమీక్షించండి
  4. లాగ్‌లలో ఎర్రర్ నమూనాల కోసం చూడండి
  5. డేటాబేస్ క్వెరీ పనితీరును చెక్ చేయండి

సాధారణ అడ్డంకులు:

  • డేటాబేస్ క్వెరీ ఆప్టిమైజేషన్ అవసరం
  • బాహ్య API కాల్‌లు టైమ్ అవుట్ అవుతున్నాయి
  • రిసోర్స్ పరిమితులు (CPU/మెమరీ)
  • థ్రెడ్ పూల్ నిండిపోవడం
  • సేవల మధ్య నెట్‌వర్క్ లేటెన్సీ

ఏజెంట్ ట్రబుల్‌షూటింగ్

ఏజెంట్ స్థితిని చెక్ చేయండి:

# For Java applications
jps -l | grep opentelemetry

# For Python applications
ps aux | grep opentelemetry

# Check agent logs
tail -f /var/log/aws-opentelemetry-agent.log

సాధారణ ఏజెంట్ సమస్యలు:

  • JVM కోసం తప్పు Java ఏజెంట్ వెర్షన్
  • తప్పిపోయిన డిపెండెన్సీలు లేదా లైబ్రరీలు
  • కాన్ఫిగరేషన్ ఫైల్ సింటాక్స్ ఎర్రర్‌లు
  • లాగ్ ఫైల్‌లకు రాయడంలో అనుమతి సమస్యలు

సాధనాలు మరియు కమాండ్‌లు

ట్రబుల్‌షూటింగ్ కోసం ఉపయోగకరమైన సాధనాలు మరియు కమాండ్‌లు.

CloudWatch CLI కమాండ్‌లు

# List Application Signals services
aws application-signals list-services

# Get service metrics
aws cloudwatch get-metric-data --metric-data-queries '[
  {
    "Id": "m1",
    "MetricStat": {
      "Metric": {
        "Namespace": "AWS/ApplicationSignals",
        "MetricName": "Latency"
      },
      "Period": 300,
      "Stat": "Average"
    }
  }
]'

# Query traces
aws xray get-trace-summaries --start-time 2024-01-01T00:00:00Z --end-time 2024-01-02T00:00:00Z

లాగ్ విశ్లేషణ

టెలిమెట్రీ డేటాను విశ్లేషించడానికి CloudWatch Logs Insights ఉపయోగించండి:

# Find errors in application logs
fields @timestamp, @message
| filter @message like /ERROR/
| sort @timestamp desc
| limit 100

# Correlate traces with logs
fields @timestamp, traceId, @message
| filter ispresent(traceId)
| sort @timestamp desc
