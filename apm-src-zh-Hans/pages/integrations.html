CloudWatch APM 文档

集成

AWS 服务集成

CloudWatch Application Signals 与 AWS 服务无缝集成，在您的整个云基础设施中提供全面的 Observability。

API Gateway

监控 API 性能并跟踪从 API Gateway 到后端服务的请求。

集成功能：

  • API 调用自动追踪
  • API 端点延迟监控
  • 错误率跟踪
  • 请求/响应关联

设置：

在 API Gateway 阶段配置中启用 X-Ray 追踪。

Application Load Balancer (ALB)

跟踪通过负载均衡器的流量并与应用程序性能相关联。

集成优势：

  • 请求路由可见性
  • 目标组性能指标
  • 健康检查监控
  • 负载分配分析

DynamoDB

监控数据库操作并识别数据访问模式中的性能瓶颈。

追踪功能：

  • 查询操作计时
  • 限流和错误跟踪
  • 表和索引性能
  • 跨区域复制监控

Amazon RDS

获取数据库性能和连接池的洞察。

数据库监控：

  • 查询执行时间
  • 连接池利用率
  • 数据库实例指标
  • 慢查询识别

Amazon S3

跟踪对象存储操作和数据传输性能。

存储操作：

  • Put/Get 对象追踪
  • 传输延迟监控
  • 错误率分析
  • 跨区域传输跟踪

SNS/SQS

在事件驱动架构中监控消息处理和队列性能。

消息洞察：

  • 消息发布/订阅追踪
  • 队列深度监控
  • 消息处理延迟
  • 死信队列分析

AWS Step Functions

可视化工作流执行并识别状态机处理中的瓶颈。

工作流监控：

  • 状态转换追踪
  • 执行时间分析
  • 错误状态识别
  • 并行执行监控

Amazon EventBridge

跟踪事件驱动系统中的事件路由和处理。

事件处理：

  • 事件路由可见性
  • 规则评估计时
  • 目标调用追踪
  • 事件转换监控

AWS Lambda

监控无服务器函数性能和冷启动影响。

无服务器洞察：

  • 函数调用追踪
  • 冷启动持续时间
  • 内存和 CPU 利用率
  • 错误和超时跟踪

Amazon ECS/EKS

容器编排平台集成，用于微服务监控。

容器监控：

  • 服务网格追踪
  • Pod 间通信
  • 资源利用率
  • 部署跟踪

第三方集成

使用第三方工具和平台扩展 Application Signals。

OpenTelemetry

利用 OpenTelemetry 生态系统实现与供应商无关的 Observability。

集成选项：

  • 自定义检测库
  • 第三方代理支持
  • 多后端数据导出
  • 社区收集器贡献

APM 供应商

从其他 APM 解决方案迁移或与之集成。

支持的集成：

  • 向第三方 APM 工具导出数据
  • 混合云 Observability
  • 多供应商关联
  • 数据迁移工具

监控和告警

将 Application Signals 与您现有的监控堆栈连接。

集成合作伙伴：

  • PagerDuty：事件响应和告警
  • Slack：团队通知
  • Datadog：统一监控平台
  • New Relic：应用程序性能监控
  • Splunk：日志分析和关联

CI/CD 集成

将 Observability 纳入您的开发流水线。

DevOps 工具：

  • GitHub Actions：自动化测试和部署
  • Jenkins：构建流水线集成
  • GitLab CI：DevSecOps 工作流
  • Terraform：基础设施即代码
  • AWS CDK：云基础设施配置

自定义集成

使用 API 和 SDK 构建您自己的集成。

CloudWatch API

使用 CloudWatch API 围绕 Application Signals 发布到 CloudWatch 的遥测数据构建自定义仪表板、告警和基于指标的工作流。

可用 API：

  • PutMetricData - 发送自定义指标
  • GetMetricData - 检索指标数据
  • PutDashboard / GetDashboard - 管理仪表板
  • PutMetricAlarm / DescribeAlarms - 管理告警

AWS X-Ray API

Application Signals 使用 AWS X-Ray 作为追踪存储。如果您需要直接的追踪摄取/查询集成，请使用 X-Ray API。

  • PutTraceSegments - 摄取追踪段
  • GetTraceSummaries / BatchGetTraces - 查询追踪

Webhook 和回调

接收实时通知并与自定义工作流集成。

集成模式：

  • 用于自定义通知系统的告警 Webhook
  • 使用 Lambda 的事件驱动处理
  • 自定义仪表板更新
  • 自动化修复工作流

SDK 和库

检测通常通过 OpenTelemetry 和/或 AWS Distro for OpenTelemetry（ADOT）完成。对于 AWS API 调用，请使用带有支持的 OpenTelemetry 检测的 AWS SDK。

支持的语言：

  • JavaScript/Node.js：AWS SDK for JavaScript
  • Python：Boto3 AWS SDK
  • Java：AWS SDK for Java
  • .NET：AWS SDK for .NET
  • Go：AWS SDK for Go
