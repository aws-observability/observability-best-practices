CloudWatch APM ドキュメント

API リファレンス

Application Signals API

Application Signals のデータと機能へのプログラムによるアクセス。

サービスオペレーション

ListServices

Application Signals によって検出されたサービスの一覧を返します。

リクエスト構文:
GET /services?nextToken=string&maxResults=integer&startTime=long&endTime=long
パラメータ:
  • nextToken - ページネーション用トークン
  • maxResults - 結果の最大数 (1〜100)
  • startTime - Unix タイムスタンプ形式の開始時刻
  • endTime - Unix タイムスタンプ形式の終了時刻
レスポンス:
{
  "services": [
    {
      "name": "string",
      "namespace": "string",
      "logGroupName": "string"
    }
  ],
  "nextToken": "string"
}

GetService

特定のサービスに関する詳細情報を返します。

リクエスト構文:
GET /services/{serviceName}?startTime=long&endTime=long
パスパラメータ:
  • serviceName - サービスの名前
レスポンス:
{
  "service": {
    "name": "string",
    "namespace": "string",
    "logGroupName": "string",
    "telemetry": {
      "apm": {
        "status": "ENABLED|DISABLED",
        "lastUpdated": "timestamp"
      },
      "rum": {
        "status": "ENABLED|DISABLED",
        "lastUpdated": "timestamp"
      },
      "synthetics": {
        "status": "ENABLED|DISABLED",
        "lastUpdated": "timestamp"
      }
    }
  }
}

ListOperations

サービスのオペレーション一覧を返します。

リクエスト構文:
GET /services/{serviceName}/operations?nextToken=string&maxResults=integer&startTime=long&endTime=long
レスポンス:
{
  "operations": [
    {
      "name": "string",
      "metricName": "string"
    }
  ],
  "nextToken": "string"
}

GetOperation

特定のオペレーションに関する詳細情報を返します。

リクエスト構文:
GET /services/{serviceName}/operations/{operationName}?startTime=long&endTime=long
レスポンス:
{
  "operation": {
    "name": "string",
    "metricName": "string",
    "telemetry": {
      "apm": {
        "status": "ENABLED|DISABLED"
      }
    }
  }

Traces API

分散トレースデータへのアクセスと分析。

GetTraceSummaries

トレースサマリーの一覧を取得します。

リクエスト構文:
POST /TraceSummaries
リクエストボディ:
{
  "startTime": "timestamp",
  "endTime": "timestamp",
  "sampling": true|false,
  "filterExpression": "string"
}
レスポンス:
{
  "traces": [
    {
      "id": "string",
      "duration": number,
      "startTime": "timestamp",
      "endTime": "timestamp",
      "hasError": true|false,
      "hasFault": true|false,
      "hasThrottle": true|false
    }
  ],
  "nextToken": "string"
}

BatchGetTraces

リクエスト構文:
POST /Traces
リクエストボディ:
{
  "traceIds": ["string"],
  "nextToken": "string"
}
Response:
{
  "traces": [
    {
      "id": "string",
      "duration": number,
      "segments": [
        {
          "id": "string",
          "name": "string",
          "startTime": number,
          "endTime": number,
          "annotations": {},
          "metadata": {},
          "subsegments": [...]
        }
      ]
    }
  ],
  "nextToken": "string",
  "unprocessedTraceIds": ["string"]
}

SDK Reference

言語固有のSDKドキュメント。

JavaScript SDK

インストール：

npm install @aws-sdk/client-application-signals

使用方法：

import { ApplicationSignalsClient, ListServicesCommand } from "@aws-sdk/client-application-signals";

const client = new ApplicationSignalsClient({ region: "us-east-1" });

const command = new ListServicesCommand({
  startTime: new Date(Date.now() - 24 * 60 * 60 * 1000),
  endTime: new Date()
});

const response = await client.send(command);

Python SDK（Boto3）

インストール：

pip install boto3

使用方法：

import boto3
from datetime import datetime, timedelta

client = boto3.client('application-signals', region_name='us-east-1')

response = client.list_services(
    startTime=datetime.utcnow() - timedelta(days=1),
    endTime=datetime.utcnow()
)

Java SDK

Maven依存関係：

<dependency>
  <groupId>software.amazon.awssdk</groupId>
  <artifactId>application-signals</artifactId>
  <version>2.20.0</version>
</dependency>

使用方法：

ApplicationSignalsClient client = ApplicationSignalsClient.builder()
    .region(Region.US_EAST_1)
    .build();

ListServicesRequest request = ListServicesRequest.builder()
    .startTime(Instant.now().minus(1, ChronoUnit.DAYS))
    .endTime(Instant.now())
    .build();

ListServicesResponse response = client.listServices(request);

.NET SDK

NuGetパッケージ：

dotnet add package AWSSDK.ApplicationSignals

使用方法：

using Amazon.ApplicationSignals;
using Amazon.ApplicationSignals.Model;

var client = new AmazonApplicationSignalsClient(RegionEndpoint.USEast1);

var request = new ListServicesRequest
{
    StartTime = DateTime.UtcNow.AddDays(-1),
    EndTime = DateTime.UtcNow
};

var response = await client.ListServicesAsync(request);

エラーコード

一般的なエラーコードとその意味。

Application Signalsエラー

  • ValidationException - 無効な入力パラメータ
  • ResourceNotFoundException - リクエストされたリソースが存在しない
  • ThrottlingException - リクエストレートの超過
  • InternalServerException - 内部サービスエラー
  • AccessDeniedException - 権限が不足しています

X-Rayエラー

  • InvalidRequestException - 不正なリクエスト
  • ThrottledException - レート制限の超過
  • ResourceNotFoundException - トレースが見つからない

CloudWatchエラー

  • InvalidParameterValueException - 無効なパラメータ値
  • MissingRequiredParameterException - 必須パラメータが不足しています
  • LimitExceededException - クォータの超過
