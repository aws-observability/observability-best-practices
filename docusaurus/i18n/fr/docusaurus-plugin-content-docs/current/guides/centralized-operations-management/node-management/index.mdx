# Gestion des noeuds

Que les clients choisissent d'opérer sur AWS, sur site ou dans des environnements multicloud et à travers les comptes et les régions, AWS Systems Manager fournit un endroit centralisé pour gérer facilement, diagnostiquer et remédier l'agent SSM. Offrant une visibilité complète de l'infrastructure, tout en augmentant l'efficacité opérationnelle et la productivité indépendamment de l'endroit où résident les noeuds.

## Console unifiée Systems Manager

La [console unifiée Systems Manager](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-unified-console.html) est une expérience consolidée qui combine divers outils pour vous aider à accomplir des tâches courantes de noeuds à travers plusieurs comptes AWS et régions AWS dans une organisation AWS Organizations, ou un seul compte et une seule région. Dans la console unifiée, vous disposez d'informations détaillées sur vos noeuds. Vous pouvez générer des rapports pour vos noeuds, diagnostiquer et remédier les problèmes courants qui empêchent les noeuds d'être rapportés comme gérés par Systems Manager, comme les problèmes de connectivité. En plus des résumés sur vos noeuds, vous pouvez voir des détails spécifiques sur un noeud comme l'inventaire logiciel et le statut de mise à jour.

## Prérequis pour l'enregistrement des noeuds

Il y a trois prérequis pour enregistrer des noeuds avec AWS Systems Manager.

Premièrement, l'agent SSM doit être installé. L'agent SSM est [préinstallé sur certaines Amazon Machine Images (AMI)](https://docs.aws.amazon.com/systems-manager/latest/userguide/ami-preinstalled-agent.html) fournies par AWS et des tiers de confiance.

Deuxièmement, l'[agent SSM](https://docs.aws.amazon.com/systems-manager/latest/userguide/ssm-agent.html) a besoin des permissions nécessaires pour effectuer des actions sur le noeud en votre nom. Vous pouvez automatiser le processus d'attachement et de remédiation des permissions IAM en [activant la console unifiée Systems Manager pour une AWS Organization](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-setting-up-organizations.html). Alternativement, vous pouvez ajouter le rôle IAM et les permissions nécessaires via IaC lorsque vous déployez vos ressources.

Et enfin, l'agent SSM doit avoir une connectivité réseau vers les [points de terminaison du service Systems Manager](https://docs.aws.amazon.com/general/latest/gr/ssm.html) via Internet ou en utilisant des [points de terminaison VPC.](https://docs.aws.amazon.com/systems-manager/latest/userguide/setup-create-vpc.html)

![Agent AWS Systems Manager](/img/cloudops/guides/centralized-operations-management/node-management/BP-Node-Mgmt-1.png "Agent AWS Systems Manager")

L'agent SSM permet à Systems Manager de mettre à jour, gérer et configurer ces ressources. L'agent traite les requêtes du service Systems Manager dans le AWS Cloud, puis les exécute comme spécifié dans la requête. L'agent SSM envoie ensuite le statut et les informations d'exécution au service Systems Manager.

## Gérer les permissions IAM pour les noeuds à grande échelle

Il existe plusieurs façons de gérer les permissions IAM pour les noeuds. Si vous avez une stratégie Infrastructure as Code (IaC) pour déployer des ressources AWS, vous devriez attacher le profil d'instance IAM lors du lancement des instances EC2.

Si vous n'avez pas de stratégie IaC ou que passer par la voie IaC n'est pas une option, vous pouvez [activer la console unifiée Systems Manager](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-setting-up-organizations.html) qui est alimentée par Quick Setup.

[Quick Setup](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-quick-setup.html) simplifie la configuration des services AWS, y compris Systems Manager, en automatisant les tâches courantes ou recommandées dans votre AWS Organization à travers les comptes AWS et les régions. Ces tâches incluent la création des rôles de profil d'instance AWS Identity and Access Management (IAM) requis et la mise en place des meilleures pratiques opérationnelles, telles que les scans périodiques de correctifs et la collecte d'inventaire.

## Améliorer la visibilité à travers l'ensemble de votre environnement d'infrastructure

AWS Systems Manager vous aide à mettre à l'échelle l'efficacité opérationnelle en simplifiant la gestion des noeuds, rendant plus facile la gestion des noeuds. Vous pouvez maintenant voir tous les noeuds gérés et non gérés à travers les comptes AWS et les régions de votre organisation depuis un seul endroit. Vous pouvez également identifier, diagnostiquer et remédier les noeuds non gérés.

Une fois remédiés, c'est-à-dire gérés par Systems Manager, vous pouvez tirer parti de la suite complète d'outils Systems Manager pour exécuter efficacement des tâches opérationnelles critiques, telles que l'application de correctifs de sécurité, l'initiation et la journalisation de sessions, ou l'exécution de commandes opérationnelles, et obtenir une visibilité complète sur l'ensemble de votre flotte.

## Créer un administrateur délégué AWS Organizations pour Systems Manager

Vous pouvez configurer un compte administrateur délégué pour Systems Manager pour gérer de manière centralisée vos noeuds à travers l'organisation lors de la configuration de la console unifiée. Cela inclut Quick Setup pour vous aider à déployer et gérer des configurations à travers les comptes et les régions en utilisant AWS Organizations. Un administrateur délégué pour Quick Setup peut créer, mettre à jour, visualiser et supprimer les ressources du gestionnaire de configuration dans votre organisation. Systems Manager enregistre un administrateur délégué pour Quick Setup dans le cadre du processus de configuration de l'expérience de console intégrée. Vous offrant une vue complète et centralisée pour voir tous les noeuds gérés et non gérés à travers les comptes AWS et les régions de votre organisation depuis un seul endroit.
