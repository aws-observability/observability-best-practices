# Meilleures pratiques pour l'enrichissement d'événements CloudTrail Lake

L'enrichissement d'événements CloudTrail vous permet d'améliorer les événements de gestion et de données en ajoutant des clés de tags de ressources et des clés de condition globales IAM, incluant les clés de tags de principal, lors de la création ou de la mise à jour d'un magasin de données d'événements. Cet enrichissement permet une meilleure catégorisation, recherche et analyse des événements CloudTrail en fonction du contexte commercial, comme l'allocation des coûts, la gestion financière, les opérations et les exigences de sécurité des données. Les événements enrichis incluent un champ eventContext qui fournit des informations contextuelles pour les actions API, que vous pouvez analyser en exécutant des requêtes dans CloudTrail Lake ou via la fédération avec Amazon Athena. Cette fonctionnalité aide les organisations à mieux comprendre et suivre l'utilisation de leurs ressources AWS et leur posture de sécurité en incorporant des métadonnées supplémentaires sur les ressources et les conditions d'autorisation dans leurs journaux d'événements.

### Enrichissement par tags de ressources

* Configurez les clés de tags de ressources lors de la création ou de la mise à jour du magasin de données d'événements pour catégoriser et analyser les événements en fonction du contexte commercial
* Considérez que les tags de ressources ajoutés après la création de la ressource peuvent connaître des délais avant d'apparaître dans les événements CloudTrail
* Sachez que les tags de ressources pour les ressources supprimées pourraient ne pas inclure les informations de tags
* Comprenez que les tags appliqués au moment de la création de la ressource ne connaissent que peu ou pas de délais

### Enrichissement par clés de condition globales IAM

* Sélectionnez les clés de condition globales IAM pertinentes pour inclure un contexte supplémentaire sur les processus d'autorisation
* Rappelez-vous que les clés de condition configurées peuvent ne pas apparaître dans chaque événement si elles n'étaient pas pertinentes pour l'évaluation de la politique IAM
* Envisagez d'utiliser les [clés de condition supportées](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#condition-keys-supported-services) telles que :
    * aws:FederatedProvider
    * aws:MultiFactorAuthPresent
    * aws:PrincipalAccount
    * aws:PrincipalType
    * aws:ViaAWSService
    * aws:SecureTransport
    * Et bien d'autres ([Clés de condition globales IAM supportées pour les événements enrichis](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#condition-keys-supported-services))

### Considérations relatives aux services

* Soyez conscient des [services](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#resource-tags-supported-services) qui peuvent connaître des délais dans les mises à jour de tags de ressources, incluant :
    * Amazon S3
    * AWS CloudTrail
    * AWS Lambda
    * Amazon DynamoDB
    * AWS Organizations
    * AWS KMS
    * Amazon SQS
    * Et bien d'autres (la [documentation AWS](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#resource-tags-supported-services) liste des services supplémentaires)

### Gestion du contexte d'événement

* Surveillez le champ eventContext dans les événements CloudTrail pour les informations enrichies
* Notez que le champ eventContext ne sera présent que dans les événements pour les magasins de données d'événements configurés avec l'enrichissement
* Sachez que les événements retardés n'incluront pas les données eventContext. Les événements qui ont une livraison retardée contiennent un champ « addendum » qui affiche des informations sur la raison du retard de l'événement. En savoir plus sur les [enregistrements d'événements CloudTrail](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-event-reference-record-contents.html).
* Maintenez le rôle lié au service AWSServiceRoleForCloudTrailEventContext pour assurer le bon peuplement des tags
* L'ajout du contexte d'événement peut augmenter la taille globale de l'événement, ce qui peut entraîner des coûts d'ingestion supplémentaires pour CloudTrail Lake

### Considérations opérationnelles

* Planifiez les pannes de service potentielles qui peuvent causer des délais dans les mises à jour de tags de ressources
* Surveillez les champs addendum dans les événements CloudTrail qui incluent des informations sur les changements de tags de ressources après les pannes de service
* Comprenez que les événements dans Event history, EventBridge et les trails n'incluront pas le champ eventContext
* Considérez les exigences commerciales pour l'allocation des coûts, les opérations et la sécurité lors de la sélection des clés de tags et des clés de condition pour l'enrichissement

### Exemples de requêtes SQL pour CloudTrail Lake

#### **Tags de ressources pour l'enrichissement d'événements**

```
SELECT eventtime, eventName, substr(userIdentity.arn, strpos(userIdentity.arn, '/') +1) as IAM,
eventContext.tagContext.resourceTags[1].tags as tags,
eventContext.tagContext.resourceTags[1].arn as resourceArn, 
element_at(requestParameters, 'key') as S3Object
FROM $EDS_ID
WHERE eventContext IS NOT NULL 
and eventSource = 's3.amazonaws.com'
and eventname in ('DeleteObject', 'PutObject')
AND eventtime >= '2025-06-05 00:00:00'
AND eventtime <= '2025-06-05 23:59:59'
```

#### **Tags de ressources pour l'enrichissement d'événements sur le tag de classification des données**

```
SELECT eventtime, eventName, substr(userIdentity.arn, strpos(userIdentity.arn, '/') +1) as IAM,
element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') as DataClassification,
eventContext.tagContext.resourceTags[1].arn as resourceArn, 
element_at(requestParameters, 'key') as S3Object
FROM $EDS_ID
WHERE eventContext IS NOT NULL
and element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') = 'sensitive'
and eventSource = 's3.amazonaws.com'
and eventname in ('DeleteObject', 'PutObject')
AND eventtime >= '2025-06-05 00:00:00'
AND eventtime <= '2025-06-05 23:59:59'
```

#### **Exemple de requête pour créer un graphique de visualisation pour l'enrichissement d'événements**

```
SELECT count(*) as count, eventName, substr(userIdentity.arn, 
strpos(userIdentity.arn, '/') +1) as IAM,
element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') as DataClassification,
eventContext.tagContext.resourceTags[1].arn as resourceArn, 
element_at(requestParameters, 'key') as S3Object
FROM $EDS_ID
WHERE eventContext IS NOT NULL
and element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') = 'sensitive'
and eventSource = 's3.amazonaws.com'
and eventname in ('DeleteObject', 'PutObject')
AND eventtime >= '2025-06-05 00:00:00'
AND eventtime <= '2025-06-05 23:59:59'
Group BY eventName,
substr(userIdentity.arn, strpos(userIdentity.arn, '/') +1),
element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification'),
eventContext.tagContext.resourceTags[1].arn,
element_at(requestParameters, 'key')
```
