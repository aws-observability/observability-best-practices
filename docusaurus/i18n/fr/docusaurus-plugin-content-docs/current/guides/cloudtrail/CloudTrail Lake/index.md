# CloudTrail Lake
CloudTrail Lake agrege, stocke de maniere immuable et interroge les evenements ingeres, offrant un lac de donnees gere pour votre organisation. Votre organisation peut utiliser ces evenements pour l'audit, les investigations de securite et les investigations operationnelles. CloudTrail Lake simplifie les flux de travail d'analyse en integrant la collecte, le stockage, la preparation et l'optimisation pour l'analyse et l'interrogation dans CloudTrail. Ce qui suit presente quelques bonnes pratiques autour de CloudTrail Lake.

### Choisir la bonne option de tarification
Lors de la creation de votre magasin de donnees d'evenements, vous souhaitez choisir une option de tarification adaptee au type et a la quantite d'evenements que vous prevoyez d'ingerer mensuellement. Pour la plupart, l'option de tarification avec retention extensible d'un an est l'approche la plus rentable. Cependant, si vous prevoyez d'ingerer plus de 25 To de donnees par mois, l'option de tarification avec retention de sept ans peut etre un meilleur choix.

![Option de tarification CloudTrail Lake](/img/cloudops/guides/cloudtrail-lake/price-option-eds.png "Option de tarification CloudTrail Lake")

### Federation de requetes Lake
Nous recommandons d'activer la federation de requetes Lake, qui vous permettra de configurer votre magasin de donnees d'evenements pour une analyse zero-ETL depuis Athena. Cela aide a eliminer les complexites operationnelles de construction de pipelines de traitement de donnees pour pouvoir correler les journaux d'activite avec les journaux applicatifs ou les donnees d'utilisation et de cout stockees dans S3. Cela vous permettra egalement d'executer des requetes de jointure croisee dans Athena sur vos autres jeux de donnees. L'activation de cette fonctionnalite eliminera egalement le besoin de repliquer ou deplacer vos donnees CloudTrail puisqu'elle fournit un lien federe vers votre magasin de donnees d'evenements via LakeFormation. Avec cette fonctionnalite, vous pouvez egalement utiliser LakeFormation pour creer des filtres de donnees et partager un sous-ensemble de vos donnees au sein de votre magasin de donnees d'evenements avec d'autres comptes de votre organisation. Pour plus d'informations, consultez le blog : [Securely share AWS CloudTrail Lake logs across accounts without replicating data](https://aws.amazon.com/blogs/mt/securely-share-aws-cloudtrail-lake-logs-across-accounts-without-replicating-data/)

### Configurer des politiques basees sur les ressources
Vous pouvez configurer des politiques basees sur les ressources pour accorder des permissions a d'autres principaux IAM. Cela vous permettrait de partager votre EDS avec d'autres comptes membres pour interroger les donnees CloudTrail. Cela peut aider a eviter la necessite de dupliquer ou copier des evenements vers d'autres comptes puisque vous pouvez accorder a des principaux IAM specifiques l'acces pour interroger le magasin de donnees d'evenements.

### Configurer des etiquettes pour votre magasin de donnees d'evenements
L'ajout d'etiquettes a votre magasin de donnees d'evenements peut vous permettre de suivre les couts d'interrogation et d'ingestion de vos donnees d'evenements CloudTrail Lake si vous ajoutez ces etiquettes comme etiquettes d'allocation de couts definies par l'utilisateur. Un autre cas d'utilisation des etiquettes pour votre magasin de donnees d'evenements serait l'ajout d'une politique IAM basee sur les ressources definissant qui peut gerer ou interroger un magasin de donnees d'evenements.

### Ingerer des evenements de donnees pour votre magasin de donnees d'evenements
Les evenements de donnees offrent une visibilite sur les operations de ressources effectuees sur ou dans la ressource. Les evenements de donnees CloudTrail prennent en charge divers types de ressources ; pour une liste complete des types de ressources pris en charge, consultez la documentation [Evenements de donnees CloudTrail](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/logging-data-events-with-cloudtrail.html#logging-data-events). Ces evenements sont egalement connus sous le nom d'operations du plan de donnees. Les evenements de donnees sont souvent des activites a volume eleve, surtout si vous stockez des donnees sensibles sur S3 ou si vous avez des operations commerciales cles effectuees via des fonctions Lambda. La visibilite sur tout acces inattendu aux donnees sensibles vous permet de prendre des mesures correctives pour proteger vos donnees. Parce que certains rapports de conformite (par exemple, FedRAMP et PCI-DSS) necessitent que les evenements de donnees soient actives, AWS recommande d'utiliser les regles gerees AWS Config ou un modele de Conformance Pack appropriate pour verifier qu'au moins une piste enregistre les evenements de donnees S3 pour tous les compartiments S3.

### Importer les evenements historiques de CloudTrail Trail
Lors de la migration de CloudTrail Trails vers CloudTrail Lake, utilisez la fonctionnalite Copier les evenements de piste pour copier les evenements de piste existants vers votre magasin de donnees d'evenements CloudTrail Lake qui ont ete enregistres avant la creation du magasin de donnees d'evenements. Cela vous permettra d'importer des evenements depuis un compartiment Amazon Simple Storage Service (Amazon S3) correspondant a une piste CloudTrail dans un EDS CloudTrail Lake. Cependant, lors de l'utilisation de cette fonctionnalite, il est recommande de specifier une plage de dates d'importation, afin que vous n'importiez que le sous-ensemble de journaux necessaires pour le stockage a long terme et l'analyse dans Lake. Cela aidera a prevenir les couts supplementaires lies a l'importation de journaux en dehors de la plage de dates specifiee.

![Copie d'evenements de piste CloudTrail Lake](/img/cloudops/guides/cloudtrail-lake/copy-trail-eds.png "Copie d'evenements de piste CloudTrail Lake")

### Options de filtrage ameliorees pour les evenements CloudTrail ingeres dans les magasins de donnees d'evenements
Les capacites de filtrage d'evenements ameliorees vous donnent un plus grand controle sur les evenements CloudTrail qui sont ingeres dans vos magasins de donnees d'evenements. Ces options de filtrage ameliorees fournissent un controle plus strict sur vos donnees d'activite AWS, ameliorant l'efficacite et la precision des investigations de securite, de conformite et operationnelles. De plus, les nouvelles options de filtrage vous aident a reduire les couts de votre flux de travail d'analyse en n'ingerant que les donnees d'evenements les plus pertinentes dans vos magasins de donnees d'evenements CloudTrail Lake.

Vous pouvez filtrer les evenements de gestion en utilisant la collecte d'evenements avancee et filtrer les evenements pour les inclure ou les exclure en fonction d'attributs tels que eventSource, eventType, eventName, userIdentity.arn et sessionCredentialFromConsole.

Lorsque vous utilisez des evenements de donnees, nous recommandons d'utiliser des selecteurs d'evenements avances qui offrent un plus grand controle sur les evenements CloudTrail ingeres dans vos magasins de donnees d'evenements. Avec les selecteurs d'evenements avances, vous pouvez inclure ou exclure des valeurs sur des champs tels que EventSource, EventName, userIdentity.arn et ResourceARN. Les selecteurs d'evenements avances prennent egalement en charge l'inclusion ou l'exclusion de valeurs avec correspondance de motifs sur des chaines partielles.

L'utilisation de filtres ameliores pour CloudTrail aidera a augmenter l'efficacite et la precision de vos investigations de securite, de conformite et operationnelles tout en aidant a reduire les couts. Par exemple, vous pouvez filtrer les evenements CloudTrail en fonction de l'attribut userIdentity.arn pour exclure les evenements generes par des roles ou utilisateurs IAM specifiques. Vous pouvez exclure un role IAM dedie utilise par un service qui effectue des appels API frequents a des fins de surveillance. Cela vous permet de reduire significativement le volume d'evenements CloudTrail ingeres dans CloudTrail Lake, diminuant les couts tout en maintenant la visibilite sur les activites pertinentes des utilisateurs et du systeme.

### Requetes SQL
Lors de l'execution de requetes SQL, nous recommandons de contraindre les requetes en ajoutant des horodatages eventTime de debut et de fin aux requetes. Cela aidera a minimiser le cout lorsque les donnees sont analysees pour les requetes. Vous pouvez le faire en ajoutant le champ eventtime dans la clause where, en ajoutant la plage de temps qui sera recherchee avec la plage de temps que vous souhaitez utiliser. La chaine de date specifiee apres eventTime > est l'horodatage d'evenement le plus ancien qui sera inclus, tandis que la chaine de date specifiee apres eventTime < est l'horodatage d'evenement le plus recent qui sera inclus. La requete suivante est un exemple montrant l'utilisation du champ eventtime.

```sql
SELECT eventTime, useridentity.arn, awsRegion FROM $EDS_ID WHERE eventTime > '2024-07-20 00:00:00' AND eventTime < '2024-07-23 00:00:00' AND awsRegion in ('us-east-1') AND eventName = 'ConsoleLogin'
```

### Invites en langage naturel
Utilisez le processeur de requetes en langage naturel pour vous aider a demarrer l'analyse de vos journaux d'activite (evenements de gestion et de donnees) stockes dans CloudTrail Lake sans avoir a ecrire une requete SQL ou passer du temps a comprendre la syntaxe SQL necessaire pour interroger vos evenements d'activite. NLQ peut egalement vous aider a obtenir des informations plus rapidement sur vos donnees en demandant ce que vous souhaitez interroger, puis en fournissant la requete SQL a utiliser.

### Validation de l'integrite des resultats de requete CloudTrail
La validation de l'integrite des resultats de requete CloudTrail Lake vous permet de savoir si les resultats de la requete ont ete modifies, supprimes ou changes lorsque les resultats ont ete exportes. En validant les resultats de la requete, vous pouvez affirmer qu'aucune modification n'a ete apportee au fichier de resultats exporte livre par CloudTrail. Pour valider les resultats, vous pouvez utiliser la commande AWS CLI **verify-query-results** pour verifier la valeur de hachage de chaque fichier de resultats de requete par rapport a la valeur de hachage dans le fichier signe.

### Configurer des alertes CloudWatch pour surveiller l'utilisation de CloudTrail Lake
Vous pouvez creer des alarmes et des notifications sur les metriques CloudWatch prises en charge pour CloudTrail Lake afin d'aider a suivre l'utilisation de votre magasin de donnees d'evenements sur une periode donnee. Vous pouvez ensuite configurer des alertes pour vous notifier lorsqu'un certain seuil a ete depasse. Avec CloudWatch, vous pouvez surveiller des metriques telles que HourlyDataIngested, TotalDataRetained, TotalStorageBytes et TotalPaidStorageBytes pour vous aider a obtenir une meilleure visibilite sur votre utilisation globale des donnees pour CloudTrail Lake. Par exemple, vous pouvez creer un tableau de bord CloudWatch montrant la taille du magasin de donnees d'evenements CloudTrail Lake.

```sql
SORT(SEARCH('{AWS/CloudTrail,"Event data store ID","Lake Metrics"} MetricName="TotalPaidStorageBytes" NOT "Lake Metrics"="IngestionMetrics"',"Sum"),SUM, DESC)
```

### Tableaux de bord CloudTrail Lake
Nous recommandons d'activer les tableaux de bord CloudTrail Lake pour visualiser les tendances d'evenements des donnees stockees dans vos magasins de donnees d'evenements dans CloudTrail Lake. Le tableau de bord Points forts fournira un resume global facile a consulter des donnees capturees dans CloudTrail Lake. Cela fournira un tableau de bord pour identifier et comprendre rapidement les informations importantes au sein de votre magasin de donnees d'evenements, telles que les principaux appels API echoues, les tendances des tentatives de connexion echouees et les pics de creation de ressources. Les tableaux de bord CloudTrail Lake fournissent egalement d'autres tableaux de bord geres specifiques a un service AWS qui fourniront des informations supplementaires autour de ce service. Vous pouvez egalement creer un tableau de bord personnalise montrant vos propres widgets ou des widgets specifiques de n'importe lequel des tableaux de bord geres.
