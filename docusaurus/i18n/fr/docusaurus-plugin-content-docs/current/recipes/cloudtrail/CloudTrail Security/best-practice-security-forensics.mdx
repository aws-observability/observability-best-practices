# Bonnes pratiques pour l'analyse forensique de securite

L'analyse forensique de securite et la reponse aux incidents sont des pratiques essentielles de cybersecurite pour identifier, investiguer et attenuer les incidents de securite informatique. L'analyse forensique collecte et analyse systematiquement les preuves numeriques, telles que les journaux et les artefacts systeme, pour decouvrir les details d'une violation. La reponse aux incidents se concentre sur la detection, le confinement et la remediation des incidents pour minimiser les dommages et prevenir la recurrence. Ensemble, elles aident les organisations a comprendre les violations, attribuer les actions et assurer la conformite avec les reglementations. La mise en oeuvre des bonnes pratiques CloudTrail ameliore la journalisation, l'analyse et la remediation en temps opportun pour une analyse forensique de securite et une reponse aux incidents efficaces.

:::

## Configuration complete de la journalisation

### Activer la journalisation complete dans toutes les regions et tous les comptes

Configurez CloudTrail pour journaliser les evenements de gestion, de donnees et d'activite reseau dans toutes les regions AWS et tous les comptes de votre organisation. Utilisez des trails multi-regions ou des trails organisationnels pour capturer toutes les activites API, y compris les evenements de services globaux (par ex., IAM, Route 53).

**Avantages** : Garantit qu'aucun evenement n'est manque, permettant la detection d'actions non autorisees dans des regions ou comptes inutilises.

**Exemple** : Une activite inattendue dans une region dormante peut indiquer une reconnaissance par un attaquant.

### Utiliser les selecteurs d'evenements avances pour se concentrer sur les evenements de donnees a haut risque

Configurez des selecteurs d'evenements avances pour journaliser les evenements de donnees a haut risque, tels que les appels S3 DeleteObject ou Lambda InvokeFunction, tout en excluant les evenements a volume eleve et faible risque pour reduire les couts.

**Avantages** : Concentre l'analyse forensique sur les activites critiques tout en controlant les couts.

**Exemple** : Utilisez les selecteurs pour journaliser uniquement les evenements DeleteObject pour des compartiments S3 specifiques contenant des donnees sensibles, comme montre dans la requete :
```sql
SELECT * FROM <event-data-store-ID> 
WHERE eventName = 'DeleteObject' 
AND resources.ARN LIKE 'arn:aws:s3:::sensitive-bucket%'
```

## Analyse et interrogation des journaux

### Utiliser CloudTrail Lake pour les requetes avancees

Exploitez CloudTrail Lake pour executer des requetes SQL sur les magasins de donnees d'evenements, permettant une analyse rapide de grands volumes de journaux.

**Avantages** : Permet un filtrage efficace des evenements par champs comme eventName, userIdentity.arn ou sourceIPAddress pour identifier les activites malveillantes.

**Exemple de requete** :
```sql
SELECT eventTime, userIdentity.arn, eventName 
FROM <event-data-store-ID> 
WHERE eventName = 'CreateAccessKey'
```

**Bonne pratique** : Stockez les evenements dans CloudTrail Lake pendant plus de 90 jours (la valeur par defaut) pour soutenir les investigations prolongees.

### Integrer avec Amazon Athena pour une analyse efficace des journaux

Utilisez Amazon Athena pour interroger les journaux CloudTrail stockes dans S3, en exploitant des tables partitionnees pour reduire les couts de requete et ameliorer les performances.
:::tip Automatisez l'analyse CloudTrail
**Explorez les [AWS CloudTrail Athena Automation Scripts](https://github.com/aws-samples/sample-automation-cloudtrail-athena)** sur GitHub pour rationaliser les investigations d'incidents avec des requetes pre-construites et l'automatisation pour l'integration CloudTrail et Athena.
:::

**Avantages** : Permet des requetes ciblees sur des comptes, regions ou periodes specifiques.

**Exemple de requete** :
```sql
SELECT * FROM trail_123456789012 
WHERE eventName = 'DeleteObject' 
AND eventTime > '2025-05-01'
```

**Mise en oeuvre** : Creez des tables specifiques par compte et a l'echelle de l'organisation partitionnees par compte, region et date.

:::note Accedez aux requetes de securite enregistrees
**Ameliorez vos investigations d'incidents** avec des requetes Athena pre-construites pour AWS CloudTrail. **[Explorez les requetes d'incidents de securite enregistrees](https://github.com/awslabs/aws-security-analytics-bootstrap/blob/main/AWSSecurityAnalyticsBootstrap/sql/dml/analytics/cloudtrail/cloudtrail_demo_queries.sql)** sur GitHub pour rationaliser l'analyse des journaux CloudTrail dans les tables Athena.
:::



## Surveillance et detection

### Activer CloudTrail Insights pour la detection d'anomalies

Activez CloudTrail Insights pour detecter les activites API inhabituelles, telles que des pics d'appels CreateUser ou AssumeRole, qui peuvent indiquer une compromission.

**Configuration** : Configurez Insights pour livrer les evenements a un compartiment S3 ou un magasin de donnees d'evenements CloudTrail Lake, et integrez avec Amazon EventBridge pour declencher des alertes ou des fonctions Lambda pour des reponses automatisees.

**Exemple** : Un evenement Insights pour des appels GetCallerIdentity excessifs pourrait declencher une notification par email a l'equipe de securite.

### Surveiller et alerter sur les activites suspectes en temps reel

:::tip Diffuser les evenements CloudTrail vers CloudWatch
**Integrez AWS CloudTrail avec Amazon CloudWatch Logs** pour une surveillance quasi temps reel des evenements de securite. **[Apprenez a configurer la diffusion d'evenements dans notre documentation](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/send-cloudtrail-events-to-cloudwatch-logs.html)**.
:::

**Mise en oeuvre** : Utilisez CloudWatch Logs Insights pour interroger les journaux et configurez des alarmes pour les evenements a haut risque, tels que les connexions Root ou les tentatives d'authentification echouees.

**Exemple d'alarme CloudWatch utilisant les filtres de metriques** : Creez une alarme pour notifier l'equipe de securite lorsque :
```
{ ($.eventName = ConsoleLogin) && ($.errorMessage = "Failed authentication") }
```

:::tip Surveiller les modifications de politiques IAM
**Configurez des alarmes CloudWatch pour AWS CloudTrail** pour detecter les modifications de politiques IAM. [Apprenez comment dans la documentation AWS](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudwatch-alarms-for-cloudtrail.html#cloudwatch-alarms-for-cloudtrail-iam-policy-changes).
:::


## Securite et integrite des journaux

### Securiser les journaux CloudTrail avec des controles robustes

Protegez les journaux CloudTrail pour assurer leur integrite et leur disponibilite pour l'analyse forensique :

1. **Stockage centralise** : Stockez les journaux dans un compartiment S3 centralise dans un compte AWS separe avec un acces restreint pour empecher les modifications non autorisees.

2. **Protection contre la suppression** : Activez la suppression MFA ou le verrouillage d'objets S3 et le versionnage sur le compartiment S3 pour empecher la suppression accidentelle ou malveillante des journaux.

3. **Chiffrement** : Chiffrez les journaux au repos en utilisant des cles gerees par AWS KMS (SSE-KMS) pour proteger les donnees sensibles. Utilisez les cles de compartiment Amazon S3 pour reduire les couts de requetes KMS.

4. **Validation de l'integrite** : Activez la validation de l'integrite des fichiers journaux en utilisant le hachage SHA-256 et la signature RSA pour detecter les alterations ou suppressions, garantissant que les journaux sont fiables pour les investigations forensiques.
