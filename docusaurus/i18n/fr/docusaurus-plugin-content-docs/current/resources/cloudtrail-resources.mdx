## Ateliers AWS CloudTrail

Ces ateliers pratiques simulent des evenements de securite courants et demontrent la detection et la reponse en utilisant AWS CloudTrail et les services associes. Chaque atelier inclut un template CloudFormation ou un script pour reproduire des scenarios reels.

### Liste des ateliers
- **[Unauthorized IAM Credential Use](https://catalog.us-east-1.prod.workshops.aws/workshops/6a8ad836-10a6-4694-9a3b-f53f193041de/en-US/setup/cloudformation)**  
  **Description** : Simulez l'utilisation non autorisee d'identifiants IAM via un script CloudShell, imitant la reconnaissance et l'escalade de privileges.  
  **Pourquoi c'est important** : Apprenez a detecter et attenuer les acces non autorises, un vecteur d'attaque courant dans les environnements AWS.

- **[Ransomware on S3](https://catalog.workshops.aws/aws-cirt-ransomware-simulation-and-detection/en-US)**  
  **Description** : Utilisez un template CloudFormation pour creer des buckets S3 et simuler un ransomware via l'exfiltration et la suppression de donnees dans CloudShell.  
  **Pourquoi c'est important** : Protegez-vous contre la perte ou l'exposition de donnees en detectant les modifications S3 non autorisees.

- **[Cryptominer Security Events](https://catalog.workshops.aws/aws-cirt-cryptominer-simulation-and-detection/en-US)**  
  **Description** : Deployez des instances EC2 via CloudFormation pour simuler le cryptominage par des requetes DNS vers des domaines connus.  
  **Pourquoi c'est important** : Identifiez le cryptominage gourmand en ressources pour reduire les couts et securiser les instances EC2.

- **[SSRF on IMDSv1](https://catalog.us-east-1.prod.workshops.aws/workshops/6a8ad836-10a6-4694-9a3b-f53f193041de/en-US/setup/ssrf)**  
  **Description** : Simulez une attaque de type Server-Side Request Forgery (SSRF) sur les identifiants d'instance EC2 via les vulnerabilites IMDSv1.  
  **Pourquoi c'est important** : Empecez les acces non autorises aux identifiants d'instance grace a l'attenuation SSRF.
