# AWS Config

AWS Config एक पूर्णतः प्रबंधित सेवा है जो आपको सुरक्षा और शासन के लिए संसाधन इन्वेंट्री, कॉन्फ़िगरेशन इतिहास, और कॉन्फ़िगरेशन परिवर्तन सूचनाएँ प्रदान करती है। AWS Config के साथ, आप मौजूदा AWS संसाधनों की खोज कर सकते हैं, तृतीय-पक्ष संसाधनों के लिए कॉन्फ़िगरेशन रिकॉर्ड कर सकते हैं, सभी कॉन्फ़िगरेशन विवरणों के साथ अपने संसाधनों की पूर्ण इन्वेंट्री निर्यात कर सकते हैं, और यह निर्धारित कर सकते हैं कि किसी भी समय पर एक संसाधन कैसे कॉन्फ़िगर किया गया था। ये क्षमताएँ अनुपालन ऑडिटिंग, सुरक्षा एनालिसिस, संसाधन परिवर्तन ट्रैकिंग, और समस्या निवारण का उपयोग करती हैं। AWS Config को निम्नलिखित परिदृश्यों में आपके एप्लिकेशन संसाधनों की निगरानी करने में मदद करने के लिए डिज़ाइन किया गया है:

1. **संसाधन प्रशासन**: AWS Config संसाधन कॉन्फ़िगरेशन में दृश्यता प्रदान करता है और संसाधन परिवर्तनों के बारे में स्वचालित रूप से सूचित करता है, AWS Config नियमों के माध्यम से गलत कॉन्फ़िगरेशन का पता लगाने और अनुपालन का मूल्यांकन करने में मदद करता है।
2. **ऑडिटिंग और अनुपालन**: AWS Config आपके AWS संसाधनों का ऐतिहासिक कॉन्फ़िगरेशन डेटा प्रदान करके अनुपालन प्रदर्शन को सक्षम बनाता है।
3. **कॉन्फ़िगरेशन परिवर्तनों का प्रबंधन और समस्या निवारण**: AWS Config संसाधन संबंधों को दिखाकर कॉन्फ़िगरेशन परिवर्तनों के प्रभाव का आकलन करने में मदद करता है और समस्या निवारण के लिए ऐतिहासिक कॉन्फ़िगरेशन प्रदान करता है।
4. **सुरक्षा एनालिसिस**: AWS Config आपको संभावित सुरक्षा कमजोरियों का एनालिसिस करने के लिए ऐतिहासिक IAM अनुमतियों और security group कॉन्फ़िगरेशन की समीक्षा करने की अनुमति देता है।
5. **पार्टनर समाधान**: AWS Config तृतीय-पक्ष लॉगिंग और एनालिसिस विशेषज्ञों के साथ एकीकृत होता है ताकि संसाधन प्रबंधन के लिए उन्नत समाधान प्रदान किए जा सकें।


निम्नलिखित अनुभागों में हम AWS Config के लिए कुछ बेस्ट प्रैक्टिसेज़ की रूपरेखा तैयार करेंगे।
