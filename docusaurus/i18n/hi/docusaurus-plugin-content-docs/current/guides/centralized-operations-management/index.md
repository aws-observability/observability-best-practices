# केंद्रीकृत संचालन प्रबंधन

## केंद्रीकृत संचालन प्रबंधन क्या है?

AWS एक [केंद्रीकृत संचालन प्रबंधन](https://aws.amazon.com/cloudops/centralized-operations-management/) समाधान प्रदान करता है जिसका उपयोग आप AWS, ऑन-प्रिमाइसेस, हाइब्रिड वातावरण और एज पर अपने एप्लिकेशन प्रबंधित और संचालित करने के लिए कर सकते हैं। ऑटोमेशन, इंटीग्रेशन, बिल्ट-इन सर्वोत्तम प्रथाओं और हाइब्रिड क्षमताओं के साथ एक केंद्रीय स्थान से अपने एप्लिकेशन संचालित करें। यदि आप दक्षता और स्थिरता में सुधार करने के लिए अपने IT Service Management (ITSM) टूल को बढ़ाना चाहते हैं, तो आप संचालन के लिए एक ऑल-इन-वन टूल का उपयोग करते हुए अपने वर्तमान इंटीग्रेशन और निवेशों को ऑटोमेट करने के लिए AWS का उपयोग कर सकते हैं।

ग्राहक ऑन-प्रिमाइसेस, हाइब्रिड और AWS के लिए बड़े पैमाने पर अपने संसाधनों का प्रबंधन और संचालन करने के लिए [AWS Systems Manager](https://aws.amazon.com/systems-manager/) का उपयोग करते हैं। Systems Manager नोड्स (जैसे, Amazon EC2 इंस्टेंस, अन्य क्लाउड पर नोड्स, और ऑन-प्रिमाइसेस नोड्स) पर परिचालन कार्यों को सुविधाजनक बनाता है जैसे सुरक्षा-संबंधित अपडेट के साथ पैचिंग, SSH कुंजी प्रबंधित किए बिना या bastion hosts बनाए रखे बिना नोड्स से कनेक्ट करना, और बड़े पैमाने पर परिचालन कमांड को ऑटोमेट करना। AWS में, एक नोड को managed तब माना जाता है जब उसमें ऑन-प्रिमाइसेस, हाइब्रिड और AWS पर पूरी तरह से कार्यशील SSM Agent काम कर रहा हो।

Systems Manager की मुख्य कार्यक्षमता उपयोग मामलों पर केंद्रित है। Agent AWS Systems Manager सुविधाओं का लाभ उठाने का मुख्य घटक है। एक बार जब कोई नोड Systems Manager द्वारा managed हो जाता है, तो आप Remote Management, Patch Management, और Session Management जैसी अन्य सुविधाओं को अनलॉक कर सकते हैं और साथ ही परिचालन कार्यों को ऑटोमेट कर सकते हैं।
