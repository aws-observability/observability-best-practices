### One ऑब्ज़र्वेबिलिटी Workshop

यह हैंड्स-ऑन कार्यशाला AWS द्वारा monitoring और ऑब्ज़र्वेबिलिटी के लिए पेश किए गए विभिन्न प्रकार के टूल सेट के साथ व्यावहारिक अनुभव प्रदान करती है। कार्यशाला अत्यधिक अनुकूलन योग्य है - आप अपनी रुचि और उपलब्धता के आधार पर किसी भी मॉड्यूल को चुन सकते हैं।

[कार्यशाला शुरू करें](https://catalog.workshops.aws/observability/)


### AWS ऑब्ज़र्वेबिलिटी बेस्ट प्रैक्टिसएँ गाइड

इस ऑब्ज़र्वेबिलिटी बेस्ट प्रैक्टिसेज़ की गाइड का होम पेज। यह व्यापक संसाधन AWS पर ऑब्ज़र्वेबिलिटी लागू करने के लिए बेस्ट प्रैक्टिसेज़, पैटर्न और रेसिपी को कवर करता है।

[गाइड पर जाएँ](https://aws-observability.github.io/observability-best-practices/)

### AWS Cloud Operations बेस्ट प्रैक्टिसएँ गाइड

Cloud Operations बेस्ट प्रैक्टिसेज़ की गाइड का होम पेज। यह व्यापक संसाधन AWS पर अपने Cloud Operations वर्कलोड को कॉन्फ़िगर और संचालित करने के लिए अनुशंसित दृष्टिकोणों को कवर करता है।

[गाइड पर जाएँ](https://aws-samples.github.io/cloud-operations-best-practices/)

### AWS Distro for OpenTelemetry (ADOT)

ADOT के साथ अपने कोड को इंस्ट्रुमेंट करने के बारे में व्यापक गाइड। OpenTelemetry के AWS-समर्थित डिस्ट्रीब्यूशन का उपयोग करके अपने एप्लिकेशन से डिस्ट्रीब्यूटेड ट्रेस और मेट्रिक्स एकत्र करना सीखें।

[ADOT प्रलेखन देखें](https://aws-otel.github.io/docs/introduction)

### AWS CloudTrail संसाधन

अपनी **AWS CloudTrail** सुरक्षा कार्यप्रणालियों को मजबूत करने, इंसिडेंट प्रतिक्रिया में सुधार करने और निगरानी क्षमताओं को बढ़ाने के लिए क्यूरेट किए गए संसाधनों का अन्वेषण करें।

[गाइड पर जाएँ](https://aws-samples.github.io/resources/cloudtrail-resources)

### Application Signals डेमो

AWS Application Signals क्षमताओं को प्रदर्शित करने वाला एक पूर्ण डेमो एप्लिकेशन। यह डेमो एक कार्यशील उदाहरण (Pet Clinic) प्रदान करता है जिसे आप डिप्लॉय करके देख सकते हैं कि Application Signals व्यवहार में कैसे काम करता है।

[GitHub पर देखें](https://github.com/aws-observability/application-signals-demo)
