# Unified Data Store のセットアップ

クロスアカウントオブザーバビリティに加えて、ログを中央リージョンと中央アカウントに集約（コピー）することもできます。

CloudWatch Unified Data Store は、組織向けに大規模な統合リポジトリを構築するのに役立ちます。ここからテレメトリを収集、整形、分析できます。

## 概要

CloudWatch Unified Data Store を使用すると、アプリケーション、運用、セキュリティ、コンプライアンスのデータを 1 か所に集中管理できます。つまり、複数の AWS アカウント/リージョンからのログデータや、サードパーティツールからのログデータを単一のアカウントとリージョンに集中管理し、一元的なクエリと分析を行うことができます。

![Unified Data Store](../../images/GettingStarted/UDS.png)

Unified Data Store は、クロスアカウントオブザーバビリティと組み合わせて使用することも、単独で使用することもできます。

## 主な利点

- **すべてのオブザーバビリティデータを一元化** – AWS サービスとサードパーティソースからの運用、セキュリティ、コンプライアンスデータを、複数のアカウントとリージョンにわたって 1 つの統合ストアに集約します

- **データサイロと重複を排除** – 不要な ETL パイプラインを削除し、ストレージコストを削減し、データを単一の場所に一度保存することで管理を簡素化します

- **トラブルシューティングを加速し、MTTR を削減** – CloudWatch Logs Insights と Amazon OpenSearch Service による直感的なファセットクエリ、自然言語検索、高度な可視化により、より迅速なインサイトを取得できます

- **柔軟な分析でビジネスインテリジェンスを実現** – Amazon Athena、QuickSight、SageMaker、Apache Spark、サードパーティの Iceberg 互換プラットフォームなど、お好みのツールを使用して、データの重複なしに統合データを分析できます

## 設定手順

これは組織レベルで設定されます。

### ステップ 1: ルートアカウントを設定する

ルートアカウントで以下を実行します。
1. 信頼されたアクセスを有効にします
2. 一元化されたデータストアの委任アカウントを特定します

### ステップ 2: 集約アカウントを設定する

集約アカウントで、以下を含む集約ルールを作成します。
- 組織 ID またはソースアカウント
- ソースリージョン

## 集約ルール

次の内容を決定して設定できます。

1. **ソースアカウント** – データをコピーするソースアカウントを選択し、組織、OU、またはアカウント ID でフィルタリングします
2. **ソースリージョン** – データをコピーするソースリージョンを選択します
3. **送信先アカウントとリージョン** - 一元化されたテレメトリデータを保存する AWS アカウントとリージョンを指定します
4. **バックアップリージョン** – データの 2 番目のコピー用にバックアップリージョンを設定します (オプション)
5. **ロググループフィルター** – 複数の演算子を使用して、名前、プレフィックス、またはキーワードでロググループをフィルタリングします
6. **複数のルール** – 要件に基づいて複数の一元化ルールを設定します
7. **KMS 暗号化ロググループオプション** - KMS 暗号化されたロググループの一元化動作を選択します

一元化されたログ記録のためのアカウント構造は、次のようになります。

![Centralized Logs](../../images/GettingStarted/centralised-logs.png)

## まとめ

要約すると、次のようになります。
1. ルートアカウントで信頼されたアクセスを有効にし、委任アカウントを特定します
2. 集約アカウントで集約ルールを作成します
3. ソースアカウント、リージョン、ログ グループ フィルターを設定します
4. 冗長性のためにバックアップ リージョンをオプションで設定します
