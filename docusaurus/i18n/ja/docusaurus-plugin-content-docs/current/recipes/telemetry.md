# テレメトリ

テレメトリとは、独自のアプリやインフラストラクチャを含むさまざまなソースからシグナルを収集し、それらが消費される宛先にルーティングする方法のすべてです。

:::info
    各テレメトリタイプのベストプラクティスの詳細については、[データタイプ](../signals/logs.md)セクションを参照してください。
:::

## ソース

シグナルの発生元をソースと呼びます。ソースには 2 種類あります。

1. 制御下にあるもの、つまりインストルメンテーションを通じたアプリケーションのソースコード。
1. マネージドサービスなど、（直接的な）制御下にない、使用する可能性のあるその他すべてのもの。
   これらのタイプのソースは通常 AWS によって提供され、API を通じてシグナルを公開します。

## エージェント

シグナルをソースから宛先に転送するには、エージェントと呼ばれる仲介役が必要です。これらのエージェントはソースからシグナルを受信またはプルし、通常は設定を通じてシグナルの送信先を決定します。また、オプションでフィルタリングや集約もサポートしています。

:::note
    「エージェント？ルーティング？シッピング？インジェスティング？」
    ソースからデスティネーションへシグナルを転送するプロセスを指す用語には、ルーティング、シッピング、アグリゲーション、インジェスティングなど多くのものがあります。それぞれ若干異なる意味を持つ場合がありますが、ここではこれらを同義語として使用します。正式には、これらの中間転送コンポーネントをエージェントと呼びます。
:::

## 送信先

シグナルが最終的に消費のために到達する場所です。後で消費するためにシグナルを保存したい場合、ダッシュボードに表示したい場合、特定の条件が真のときにアラートを設定したい場合、またはシグナルを相関させたい場合など、エンドユーザーとしてあなたに役立つすべてのコンポーネントがデスティネーションです。
