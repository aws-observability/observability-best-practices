# テレメトリー

テレメトリとは、独自のアプリケーションやインフラストラクチャを含むさまざまなソースからシグナルがどのように収集され、それらが消費される宛先にルーティングされるかに関するものです。

:::info
    各タイプのテレメトリのベストプラクティスの詳細については、[データタイプ](../signals/logs)セクションを参照してください。
:::

## ソース

ソースは、シグナルの発信元と考えます。ソースには 2 つのタイプがあります。

1. 制御下にあるもの、つまりインストルメンテーションを介したアプリケーションソースコード。
1. 使用する可能性のあるその他すべてのもの、たとえばマネージドサービスなど、（直接的な）制御下にないもの。
   これらのタイプのソースは通常 AWS によって提供され、API を介してシグナルを公開します。

## エージェント

ソースから宛先にシグナルを転送するには、エージェントと呼ばれる仲介役が必要です。これらのエージェントは、ソースからシグナルを受信またはプルし、通常は設定を介して、シグナルの送信先を決定します。オプションでフィルタリングと集約をサポートします。

:::note
    「エージェント？ルーティング？シッピング？インジェスト？」
    ソースから宛先へシグナルを取得するプロセスを指すために、ルーティング、シッピング、アグリゲーション、インジェストなど、多くの用語が使用されています。これらは若干異なる意味を持つ場合がありますが、ここでは互換的に使用します。正式には、これらの中間トランスポートコンポーネントをエージェントと呼びます。
:::

## 送信先

シグナルが最終的に到達し、消費される場所です。後で消費するためにシグナルを保存したい場合でも、ダッシュボードに表示したい場合でも、特定の条件が真の場合にアラートを設定したい場合でも、シグナルを関連付けたい場合でも、エンドユーザーとしてのあなたに役立つすべてのコンポーネントが宛先です。
