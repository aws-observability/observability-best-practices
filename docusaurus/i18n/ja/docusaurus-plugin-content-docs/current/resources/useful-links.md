# 役立つリンク

AWS オブザーバビリティのベストプラクティスを学び、実装するのに役立つ、厳選されたリソースのコレクションです。

### 今後のイベントに登録する（AWS Connected Community にて）
開発者、DevOps エンジニア、ソリューションアーキテクト、システム管理者、およびクラウドオペレーションに関心のあるすべての方を対象としたライブウェビナーおよびワークショップへの登録は[こちら](https://aws-experience.com/amer/smb/events/series/Cloud-Operations-Enablement)から行えます。

### AWS re:Invent 2025 の Cloud Operations (COP) セッション (YouTube プレイリスト)

Cloud Operations (COP) セッションの AWS re:Invent 2025 ブレイクアウトセッションプレイリストは[こちら](https://www.youtube.com/playlist?list=PL2yQDdvlhXf8fUGVeljVNpQwugV7XBZRt)からご確認いただけます。

### インタラクティブデモ

[Storylane で表示](https://aws-cloudops.storylane.io/hub/9ajhvqeu78kb)


### AWS Cloud Operations Show

AWS クラウドオペレーションの興味深い機能とユースケースを深く掘り下げたライブストリーム録画のコレクションです。このビデオシリーズでは、クラウドオペレーションに関する実践的なデモンストレーションと専門家のインサイトを提供します。

[YouTube で視聴する](https://www.youtube.com/playlist?list=PLehXSATXjcQHj8bPSf0uZuQBoxJ7a7ag7)

### One Observability Workshop

このハンズオンワークショップでは、AWS がモニタリングとオブザーバビリティのために提供する多様なツールセットを実際に体験できます。このワークショップは高度にカスタマイズ可能で、興味や時間に応じて任意のモジュールを選択できます。

[ワークショップを開始する](https://catalog.workshops.aws/observability/)


### AWS オブザーバビリティベストプラクティスガイド

この Observability Best Practices ガイドのホームページです。この包括的なリソースでは、AWS 上でオブザーバビリティを実装するためのベストプラクティス、パターン、およびレシピを紹介しています。

[ガイドを見る](/observability-best-practices/ja/)

### AWS クラウドオペレーションのベストプラクティスガイド

Cloud Operations ベストプラクティスガイドのホームページです。この包括的なリソースでは、AWS 上で Cloud Operations ワークロードを設定および運用するための推奨アプローチを説明しています。

[ガイドを見る](https://aws-samples.github.io/cloud-operations-best-practices/)

### AWS Distro for OpenTelemetry (ADOT)

ADOT を使用してコードをインストルメント化する方法に関する包括的なガイドです。AWS がサポートする OpenTelemetry のディストリビューションを使用して、アプリケーションから分散トレースとメトリクスを収集する方法を学びます。

[ADOT ドキュメントを探索する](https://aws-otel.github.io/docs/introduction)

### AWS CloudTrail リソース

**AWS CloudTrail** のセキュリティプラクティスを強化し、インシデントレスポンスを改善し、モニタリング機能を向上させるための厳選されたリソースをご覧ください。

[ガイドを見る](https://aws-samples.github.io/resources/cloudtrail-resources)

### Application Signals デモ

AWS Application Signals の機能を紹介する完全なデモアプリケーションです。このデモでは、実際に動作するサンプル（Pet Clinic）を提供しており、デプロイして探索することで、Application Signals が実際にどのように機能するかを理解できます。

[GitHub で表示](https://github.com/aws-observability/application-signals-demo)
