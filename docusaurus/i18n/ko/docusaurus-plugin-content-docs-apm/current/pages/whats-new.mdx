# 새로운 소식

## 이 페이지의 내용

이 페이지는 주요 공개 마일스톤을 요약하고 상세 내용은 공식 AWS 소스를 참조합니다.

- **AWS 공지:** [AWS "What's New"](https://aws.amazon.com/about-aws/whats-new/)
- **CloudWatch 문서:** [Application Signals](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Application-Monitoring-Sections.html)

## 2024년 6월: Application Signals 정식 출시

CloudWatch Application Signals가 정식 출시(GA)되었습니다. 호출량, 가용성, 지연 시간, 장애, 오류와 같은 표준 애플리케이션 메트릭을 자동으로 수집하여 빠른 서비스 수준 분류 및 트러블슈팅에 중점을 둡니다.

AWS 공지 참조: [GA 발표](https://aws.amazon.com/about-aws/whats-new/2024/06/amazon-cloudwatch-application-signals-application-monitoring/)

## 2024년 9월: 요청 기반 SLO

AWS가 CloudWatch Application Signals에 대한 요청 기반 Service Level Objectives(SLO) 지원을 발표했습니다.

AWS 공지 참조: [요청 기반 SLO](https://aws.amazon.com/about-aws/whats-new/2024/09/cloudwatch-application-signals-service-level-objectives/)

## 진행 중: Application Map 및 분류 워크플로우

CloudWatch **Application Map**은 Application Signals의 기존 "Service Map" 용어를 대체합니다. 토폴로지(클라이언트, Canaries, 서비스, 종속성)를 시각화하고 SLO/SLI 상태를 표시하여 팀이 인시던트를 분류하는 데 도움을 줍니다.

문서 참조: [Application Map](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/ServiceMap.html)
