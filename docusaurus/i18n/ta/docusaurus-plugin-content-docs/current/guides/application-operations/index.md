# பயன்பாட்டு செயல்பாடுகள்

AWS வாடிக்கையாளர்கள் பெரும்பாலும் நூற்றுக்கணக்கான பயன்பாடுகளை இயக்குகிறார்கள் மற்றும் அவர்களின் பயன்பாடுகள் கிடைக்கின்றன, பாதுகாப்பானவை, செலவு-மேம்படுத்தப்பட்டவை மற்றும் சிறந்த முறையில் செயல்படுகின்றன என்பதை உறுதிசெய்ய தனிப்பட்ட வளங்களை கண்காணிக்கவும் நிர்வகிக்கவும் வேண்டும். பயன்பாடுகள் வாடிக்கையாளர் வணிகங்களுக்கு அத்தியாவசியமானவை. அவை இறுதி பயனருக்கு தேவையான குறிப்பிட்ட அம்சங்கள் அல்லது சேவைகளை வழங்க ஒன்றாக செயல்படும் வளங்களின் குழுக்கள். இன்றைய வேகமாக பரிணமிக்கும் டிஜிட்டல் நிலப்பரப்பில், திறமையான Cloud Operations க்கு உங்கள் AWS வளங்களை நன்கு வரையறுக்கப்பட்ட பயன்பாடுகளாக ஒழுங்கமைப்பது மிக முக்கியமானது. வள பரவல், செயல்பாட்டு திறனின்மைகள் மற்றும் பல AWS கணக்குகளில் வளங்களை நிர்வகிப்பதின் சிக்கலான தன்மை போன்ற பொதுவான சவால்களை தீர்க்க இந்த பயன்பாடு-மைய அணுகுமுறை தேவைப்படுகிறது.

AWS வள மேலாண்மையை எளிமைப்படுத்தவும், தெரிவுநிலையை மேம்படுத்தவும், ஒட்டுமொத்த செயல்பாட்டு திறனை மேம்படுத்தவும் உங்களை இயலச் செய்யும் பயன்பாடு-மைய cloud operations உத்தியை ஆதரிக்க வடிவமைக்கப்பட்ட விரிவான சேவைகளை AWS வழங்குகிறது.

பயன்பாட்டு செயல்பாடுகள் என்பது செலவு, நலம், பாதுகாப்பு நிலை மற்றும் உங்கள் பயன்பாடுகளின் செயல்திறன் போன்ற மெட்ரிக்குகளை குறைந்த முயற்சியுடனும் அளவிலும் கண்காணிக்க ஒரு நிலையான அணுகுமுறையை வழங்கும் AWS முழுவதும் உள்ள திறன்களின் தொகுப்பாகும். இந்த திறன்கள் பல AWS consoles முழுவதும் பயன்பாடு-மைய பார்வையை பின்னுகின்றன.

![Application Operations](/img/cloudops/guides/app-ops/BP-App-ops-1.png "Application Operations")

சிக்கலான கிளவுட் சூழல்களில், பயன்பாடுகளை நிர்வகிப்பது பல நிறுவனங்களுக்கு சவாலானதாகவும் நேரம் எடுக்கும் வகையிலும் இருக்கலாம். சவால் தனிப்பட்ட வளங்களை நிர்வகிப்பதில் மட்டுமல்ல, பயன்பாட்டு வாழ்க்கை சுழற்சியின் வெவ்வேறு நிலைகளில் பயன்பாட்டு பணிகளை செய்வதிலும் உள்ளது. இந்த துண்டு துண்டான அணுகுமுறை குறிப்பிட்ட பயன்பாடுகளுடன் தொடர்புடைய வளங்களை அடையாளம் காண்பதை கடினமாக்குகிறது, முக்கியமான நிகழ்வுகளின் போது பதில் நேரங்களை அதிகரிக்கிறது மற்றும் தொடர்புடைய செயல்பாட்டு தரவை அணுகுவதில் சிக்கல்களை ஏற்படுத்துகிறது.

இந்த சவால்களை தீர்க்க, வள மேலாண்மைக்கான உறுதியான அடித்தளத்தை நிறுவுவது அவசியம். இந்த அடித்தளம் வள நிலப்பரப்பின் விரிவான புரிதலை வளர்ப்பதிலும் பயன்பாடுகளை மையமாக வைத்து வலுவான tagging உத்தியை செயல்படுத்துவதிலும் தொடங்குகிறது.

### **அடித்தளத்தை நிறுவுதல்**

AWS வாடிக்கையாளர்கள் ஒரு கணக்கில் பல வளங்களை கையாள்கிறார்கள், மற்றும் அவர்களின் பயன்பாடுகளில் ஒருங்கிணைக்கப்பட்ட பார்வை இல்லாமை திறமையான நடவடிக்கை மற்றும் முடிவெடுப்பை கணிசமாக தடுக்கலாம். வணிக நோக்கங்களை அடையும் அதே நேரத்தில் செயல்பாடுகளை அளவிட உதவ, வள மேலாண்மை சேவைகள் AWS வளங்களை திறம்பட ஆராய்வதற்கும், ஒழுங்கமைப்பதற்கும் மற்றும் நிர்வகிப்பதற்கும் முக்கிய primitives, கருத்துக்கள் மற்றும் தொழில்நுட்பங்களை வழங்குகின்றன. அடித்தளம் பின்வரும் கூறுகளைக் கொண்டுள்ளது: Tagging, Tagging Policies, Resource Groups மற்றும் Resources Explorer.

### **பயன்பாட்டை வரையறுத்தல்**

Tags மற்றும் Resource Groups மீது கட்டமைத்து, AWS க்குள் ஒத்திசைவான அலகுகளாக பயன்பாடுகளை வரையறுப்பது Cloud Operations க்கு உண்மையான பயன்பாடு-மைய அணுகுமுறையை அனுமதிக்கிறது. பயன்பாடுகளை நிறுவ, AWS Service Catalog AppRegistry ஐ பயன்படுத்தி பயன்பாடுகளை வரையறுக்கவும் நிர்வகிக்கவும்.

### **பயன்பாடு-மைய பார்வைகள்**

பயன்பாட்டு செயல்பாடுகளுக்கு ஒரு நிலையான பயன்பாட்டு மாதிரி தேவை; [AWS Service Catalog AppRegistry](https://docs.aws.amazon.com/servicecatalog/latest/arguide/intro-app-registry.html) பயன்பாட்டு metadata ஐ சேமிக்கிறது, [AWS Resource Groups](https://docs.aws.amazon.com/ARG/latest/userguide/resource-groups.html) பயன்பாட்டு வளங்களை தர்க்கரீதியாக குழுவாக்குகிறது, மற்றும் resource tagging பயன்பாட்டின் வளங்களை தேடக்கூடிய resource groups ஆக ஒழுங்கமைக்கிறது.

AppRegistry application உருவாக்கப்படும்போது, AppRegistry ஒரு vended application tag ஐப் பயன்படுத்தி AWS வளங்களை resource group ஆக இணைக்கிறது. Tag key **awsApplication** ஆகவும் value பயன்பாட்டுக்கான தனிப்பட்ட identifier ஆகவும் இருக்கும். இந்த application tag AWS சேவைகள் தங்கள் consoles மற்றும் APIs க்குள் அந்த application tag ஐ குறிப்பிடுவதன் மூலம் பயன்பாட்டு செயல்பாடுகளை ஆதரிக்க அனுமதிக்கிறது.

myApplications dashboard application tag ஐப் பயன்படுத்தி செலவு மற்றும் பயன்பாடு, பாதுகாப்பு மற்றும் செயல்பாட்டு மெட்ரிக்குகள் உட்பட உங்கள் தேர்ந்தெடுக்கப்பட்ட பயன்பாட்டுக்கான மெட்ரிக்குகளின் ஒருங்கிணைக்கப்பட்ட பார்வையை வழங்குகிறது. myApplications dashboard உடன், நீங்கள் [Amazon CloudWatch](https://aws.amazon.com/cloudwatch) பயன்பாட்டு செயல்திறன், [AWS Cost Explorer](https://aws.amazon.com/aws-cost-management/aws-cost-explorer/) செலவு மற்றும் பயன்பாடு, மற்றும் [AWS Security Hub](https://aws.amazon.com/security-hub/) பாதுகாப்பு கண்டுபிடிப்புகள் போன்ற தொடர்புடைய சேவைகளில் குறிப்பிட்ட வளங்களில் நடவடிக்கை எடுக்க ஆழமாக செல்லலாம்.

### உத்தியிலிருந்து செயல்படுத்தல்

1. பயன்பாட்டு பெயர்கள், சூழல்கள், வணிக அலகுகள் மற்றும் செலவு மையங்களில் கவனம் செலுத்தி விரிவான tagging உத்தியை உருவாக்குவதன் மூலம் தொடங்கவும்.
2. இந்த tags ஐ அனைத்து தொடர்புடைய வளங்களில் முறையாகப் பயன்படுத்தவும், அதை வழங்கல் செயல்முறையின் பகுதியாக்கவும்.
3. இந்த tags அடிப்படையில் Resource Groups ஐ உருவாக்கவும்.
4. myApplications dashboard ஐ பயன்படுத்தி உங்கள் பயன்பாடுகளின் ஒருங்கிணைக்கப்பட்ட பார்வையைப் பெறவும்.

### முடிவுரை

வாடிக்கையாளர் வணிகங்கள் கிளவுட்டில் தொடர்ந்து வளர்ந்து பரிணமிக்கும்போது, வள மேலாண்மைக்கான இந்த சிறந்த நடைமுறைகளை பின்பற்றுவது அவசியம். அடித்தளத்தை அமைப்பதன் மூலம், நிறுவனங்கள் தங்கள் தற்போதைய தேவைகளை பூர்த்தி செய்வது மட்டுமல்லாமல் எதிர்கால வளர்ச்சி மற்றும் புதுமைகளுக்கும் தயாராக முடியும். AWS Application Operations மற்றும் myApplications இந்த அணுகுமுறையை மேலும் ஒரு படி மேலே எடுத்துச் செல்கிறது, பயன்பாட்டு வளங்கள் மற்றும் மெட்ரிக்குகளின் ஒருங்கிணைக்கப்பட்ட பார்வையை வழங்குகிறது. இது குழுக்களுக்கு தகவலறிந்த முடிவுகளை விரைவாக எடுக்கவும், சிக்கல்களுக்கு முன்கூட்டியே பதிலளிக்கவும், அளவில் வளங்களை மிகவும் திறம்பட நிர்வகிக்கவும் அதிகாரம் அளிக்கிறது.
