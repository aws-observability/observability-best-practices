# மையப்படுத்தப்பட்ட செயல்பாட்டு மேலாண்மை

## மையப்படுத்தப்பட்ட செயல்பாட்டு மேலாண்மை என்றால் என்ன?

AWS ஒரு [மையப்படுத்தப்பட்ட செயல்பாட்டு மேலாண்மை](https://aws.amazon.com/cloudops/centralized-operations-management/) தீர்வை வழங்குகிறது, இதை AWS இல், ஆன்-பிரமிசஸில், கலப்பு சூழல்களில் மற்றும் எட்ஜில் உங்கள் பயன்பாடுகளை நிர்வகிக்கவும் இயக்கவும் பயன்படுத்தலாம். தானியங்கு, ஒருங்கிணைப்புகள், உள்ளமைந்த சிறந்த நடைமுறைகள் மற்றும் கலப்பு திறன்களுடன் ஒரு மைய இடத்திலிருந்து உங்கள் பயன்பாடுகளை இயக்கவும். செயல்திறன் மற்றும் நிலைத்தன்மையை மேம்படுத்த உங்கள் IT சேவை மேலாண்மை (ITSM) கருவிகளை மேம்படுத்த விரும்பினால், உங்கள் தற்போதைய ஒருங்கிணைப்புகள் மற்றும் முதலீடுகளை தானியங்குபடுத்த AWS ஐ பயன்படுத்தலாம், அதே நேரத்தில் செயல்பாடுகளுக்கான ஒரே கருவியைப் பயன்படுத்தலாம்.

வாடிக்கையாளர்கள் ஆன்-பிரமிசஸ், கலப்பு மற்றும் AWS க்கான அளவில் தங்கள் வளங்களை நிர்வகிக்கவும் இயக்கவும் [AWS Systems Manager](https://aws.amazon.com/systems-manager/) ஐ பயன்படுத்துகிறார்கள். Systems Manager நோடுகளில் (எ.கா., Amazon EC2 நிகழ்வுகள், பிற கிளவுட்களில் உள்ள நோடுகள் மற்றும் ஆன்-பிரமிசஸ் நோடுகள்) பாதுகாப்பு தொடர்பான புதுப்பிப்புகளுடன் பேட்ச்சிங், SSH விசைகளை நிர்வகிக்காமல் அல்லது bastion hosts ஐ பராமரிக்காமல் நோடுகளுடன் இணைத்தல் மற்றும் அளவில் செயல்பாட்டு கட்டளைகளை தானியங்குபடுத்துதல் போன்ற செயல்பாட்டு பணிகளை எளிதாக்குகிறது. AWS இல், ஒரு நோடு ஆன்-பிரமிசஸ், கலப்பு மற்றும் AWS இல் முழுமையாக செயல்படும் SSM Agent இருக்கும்போது நிர்வகிக்கப்படுவதாக கருதப்படுகிறது.

Systems Manager இன் முக்கிய செயல்பாடு பயன்பாட்டு நிலைகளில் கவனம் செலுத்துகிறது. AWS Systems Manager அம்சங்களை பயன்படுத்துவதற்கான முக்கிய கூறு ஏஜென்ட் ஆகும். ஒரு நோடு Systems Manager ஆல் நிர்வகிக்கப்பட்டவுடன், Remote Management, Patch Management மற்றும் Session Management போன்ற பிற அம்சங்களை திறக்கவும் செயல்பாட்டு பணிகளை தானியங்குபடுத்தவும் முடியும்.
