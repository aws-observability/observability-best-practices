# ஏற்கனவே உள்ள EC2 பணிச்சுமைகளுக்கான OLA

## AWS OLA நிரல்

[AWS Optimization and Licensing Assessment (AWS OLA)](https://aws.amazon.com/optimization-and-licensing-assessment/) வாடிக்கையாளர்களுக்கு கிளவுட்டுக்கு பணிச்சுமைகளை இடம்பெயர்க்கவும் வளங்களை செலவு மேம்படுத்தவும் சிறந்த அணுகுமுறையை வழங்குகிறது. இது ஒரு இலவச நிரலாகும், இது வாடிக்கையாளர்கள் தங்கள் புதிய மற்றும் ஏற்கனவே உள்ள பணிச்சுமைகளை பகுப்பாய்வு செய்யவும், வள ஒதுக்கீடு, மூன்றாம் தரப்பு உரிமம் மற்றும் பயன்பாட்டு சார்புகளை மேம்படுத்த ஆன்-பிரமிசஸ் மற்றும் கிளவுட் சூழல்களை மதிப்பீடு செய்யவும் உதவுகிறது, இதன் மூலம் வள செயல்திறனை மேம்படுத்தவும் கணக்கீட்டு செலவினங்களில் சேமிக்கவும் முடியும்.

இந்த செயல்முறையில் சேகரிக்கப்பட்ட தரவின் மூலம், AWS OLA நிரல் வாடிக்கையாளர்கள் தங்கள் கிளவுட் பயணம் மற்றும் இடம்பெயர்வுக்கான தகவலறிந்த முடிவுகளை எடுக்க பயன்படுத்தக்கூடிய ஒரு விரிவான அறிக்கையை வழங்குகிறது. அறிக்கை உண்மையான வள பயன்பாடு, ஏற்கனவே உள்ள உரிம உரிமைகள் மற்றும் எங்கள் நெகிழ்வான உரிம விருப்பங்கள் மூலம் சாத்தியமான செலவு சேமிப்புகளை கண்டறிய வாடிக்கையாளர்களுக்கு உதவும் வகையில் நிலைநிறுத்தல் விருப்பங்களை மாதிரியாக்குகிறது.

AWS OLA நிரலை மேற்கொள்வதின் நன்மைகள்,

- ஒவ்வொரு பணிச்சுமைக்கும் சிறந்த Amazon Elastic Compute Cloud (Amazon EC2), Amazon Relational Database Service (Amazon RDS) அல்லது VMware Cloud on AWS instance அளவு மற்றும் வகையை கண்டறிய உதவும் கணக்கீட்டு வளங்கள் மற்றும் நுண்ணறிவுகளை வழங்கும் கருவி-அடிப்படையிலான கண்டுபிடிப்பு அணுகுமுறையுடன் உங்கள் பணிச்சுமைகளுக்கான **வள ஒதுக்கீட்டை சரியான அளவில் மாற்றவும்**.
- முக்கிய அம்சங்களில் ஒன்றான உங்கள் கிளவுட் உள்கட்டமைப்பை மேம்படுத்துவதன் மூலம் **செலவுகளை குறைக்கவும்**.
- பருவகால பணிச்சுமைகள் மற்றும் சுறுசுறுப்பான பரிசோதனைகளை நிர்வகிப்பதில் நெகிழ்வுத்தன்மைக்காக license-included அல்லது BYOL instances உட்பட உரிம சூழ்நிலைகளை மாதிரியாக்கி **மேம்படுத்தப்பட்ட உரிம விருப்பங்களை ஆராய்ந்து** தேவையற்ற உரிம செலவுகளை நீக்கவும்.

## ஏற்கனவே உள்ள EC2 பணிச்சுமைகளுக்கான AWS OLA

AWS OLA (Optimization and Licensing Assessment) ஏற்கனவே உள்ள EC2 பணிச்சுமைகளுக்கான செலவுகளை மேம்படுத்துவதில் கவனம் செலுத்துகிறது, இது '**AWS OLA for EEC2**' - **ஏற்கனவே உள்ள EC2 பணிச்சுமைகளுக்கான** AWS OLA (Optimization and Licensing Assessment) மதிப்பீடு என அழைக்கப்படுகிறது.

AWS OLA for EEC2 [AWS Enterprise Support](https://aws.amazon.com/premiumsupport/plans/enterprise/) திட்டத்தில் பதிவு செய்யப்பட்ட வாடிக்கையாளர்களுக்கு Amazon EC2 சரியான அளவிடல் பரிந்துரைகளை வழங்க [AWS Compute Optimizer](https://aws.amazon.com/compute-optimizer/) ஐ பயன்படுத்துகிறது. OLA for EEC2 என்பது ஒரு எளிமைப்படுத்தப்பட்ட செயல்முறை மூலம் ஒரு சுய-சேவை ஈடுபாடாகும், இதில் AWS OLA குழு பரிந்துரைகளை மதிப்பீட்டு அறிக்கையாக தயாரிக்கிறது மற்றும் அந்தந்த AWS கணக்கு குழு Amazon EC2 சரியான அளவிடல் மற்றும் செலவு மேம்படுத்தலுக்காக அந்த கண்டுபிடிப்புகளை வாடிக்கையாளருக்கு வழங்குகிறது. Amazon EC2 சரியான அளவிடல் பரிந்துரைகளுக்கு கூடுதலாக, AWS OLA BYOL (Bring Your Own License) மற்றும் License Included Microsoft SQL Server instances க்கான Microsoft SQL Server optimization strategies ஐயும் வழங்குகிறது.

மதிப்பீட்டை செய்ய, AWS OLA for EEC2 செயல்முறை வாடிக்கையாளரின் AWS கணக்குகளிலிருந்து நினைவகம் மற்றும் CPU பயன்பாடு (Amazon CloudWatch மற்றும் CloudWatch agent மூலம்) போன்ற மெட்ரிக்குகள் உட்பட சூழல் அளவுருக்களை சேகரிக்கிறது. தேவையான அளவுருக்கள் சேகரிக்கப்பட்டவுடன், AWS OLA குழு ஒருங்கிணைக்கப்பட்ட தரவைப் பயன்படுத்தி பரிந்துரைகளை தயாரித்து AWS TAMs மற்றும் கணக்கு குழுவுக்கு PPT deck மற்றும் Excel அறிக்கையை வழங்குகிறது, பின்னர் வாடிக்கையாளர்களுக்கு வழங்கலாம். அறிக்கை வழங்கும் நுண்ணறிவுகள் வாடிக்கையாளர்கள் தங்கள் ஏற்கனவே உள்ள Amazon EC2 செலவை மேம்படுத்தவும், தங்கள் பணிச்சுமைகளுக்கான உரிம மேம்படுத்தல் உத்திகளை ஆராயவும் உதவுகின்றன.

## AWS OLA for EEC2 மதிப்பீடு

Enterprise Support கொண்ட எந்த AWS வாடிக்கையாளரும் ஒரு இலவச Optimization and Licensing Assessment (OLA) for Existing EC2 workloads மூலம் தங்கள் ஏற்கனவே உள்ள Amazon EC2 Instances (Linux மற்றும் Windows) செலவுகளை மேம்படுத்தலாம். உங்கள் பணிச்சுமைகளுக்கான AWS OLA for EEC2 மதிப்பீட்டை இலவசமாக செய்ய, உங்கள் AWS கணக்கு குழுவை தொடர்பு கொள்ளவும்.

## துல்லியமான சரியான அளவிடலுக்கான Amazon CloudWatch நினைவக மெட்ரிக்குகள்

AWS OLA for EEC2 Amazon EC2 சரியான அளவிடலுக்கான மதிப்பீட்டு அறிக்கையை வழங்கும் அதே நேரத்தில், [Amazon CloudWatch](https://aws.amazon.com/cloudwatch/) வழங்கும் நுண்ணறிவுகள் வாடிக்கையாளர்களுக்கு மிகவும் துல்லியமான வள சரியான அளவிடலுக்கு நினைவக பயன்பாட்டு மெட்ரிக்குகளை இணைப்பதின் மதிப்பை புரிந்துகொள்கிறது. எனவே AWS OLA for EEC2 நிரலுடன் Amazon CloudWatch நினைவக மெட்ரிக்குகள் கண்காணிப்பை ஊக்குவிப்பதன் மூலமும் எளிதாக்குவதன் மூலமும், வாடிக்கையாளர்கள் தங்கள் AWS சூழல்களுக்கான மிகவும் தாக்கமான வள மேம்படுத்தல் பரிந்துரைகளைப் பெறுகிறார்கள் மற்றும் அவர்களின் பணிச்சுமை வள நுகர்வின் பரந்த கண்ணோட்டத்தையும் பெறுகிறார்கள். இது உங்கள் பணிச்சுமைகளின் செலவை குறைக்கவும் செயல்திறனை மேம்படுத்தவும் உதவுகிறது.

Amazon EC2 Instances இயல்பாகவே Amazon CloudWatch க்கு பல மெட்ரிக்குகளை வெளியிடுகின்றன. இருப்பினும், நினைவக மெட்ரிக்குகள் Amazon EC2 வழங்கும் இயல்புநிலை மெட்ரிக்குகளில் ஒன்றல்ல. Amazon EC2 இன் நினைவக மெட்ரிக்குகளை அறிந்துகொள்வது உங்கள் EC2 instances இன் தற்போதைய நினைவக பயன்பாட்டை புரிந்துகொள்ள உதவுகிறது, இதனால் instances குறைவாக வழங்கப்படவோ அதிகமாக வழங்கப்படவோ இல்லை. Amazon EC2 instances ஐ குறைவாக வழங்குவது பொதுவாக கணினி அல்லது பயன்பாட்டின் செயல்திறனை பாதிக்கிறது, அதே நேரத்தில் அதிகமாக வழங்குவது வீணான செலவினத்தை தருகிறது. Big Data Analytics, In-memory Databases, Real-time Streaming போன்ற நினைவக-கன பயன்பாடுகள் செயல்பாட்டு தெரிவுநிலைக்காக instances இல் நினைவக பயன்பாட்டை கண்காணிக்க வேண்டும்.

### Amazon EC2 Instances இலிருந்து நினைவக மெட்ரிக்குகள் சேகரிப்பு

[Amazon EC2 Instances](https://aws.amazon.com/ec2/) இலிருந்து நினைவக மெட்ரிக்குகளை சேகரிக்க, உயர் மட்ட படிகள் இங்கே.

- பின்வருவனவற்றுக்கான அனுமதிகளுடன் AWS Identity and Access Management (IAM) இல் ஒரு பாத்திரத்தை உருவாக்கவும்
  - Amazon EC2 Instance(கள்) Systems Manager ஆல் நிர்வகிக்கப்பட்டால் தேவைப்படும் Amazon EC2 instances ஐ நிர்வகிக்க [Amazon Systems Manager](https://aws.amazon.com/systems-manager/). [AWS Systems Manager Agent (SSM Agent)](https://docs.aws.amazon.com/systems-manager/latest/userguide/ssm-agent.html) Amazon EC2 instances இல் instance AWS Systems Manager உடன் தொடர்பு கொள்ளவும் EC2 Instances இல் Systems Manager Run Command இயக்குவது போன்ற instance க்கு எதிராக remote commands மற்றும் scripts ஐ செயல்படுத்தவும் தேவைப்படுகிறது. AWS Systems Manager Agent (SSM Agent) என்பது Amazon EC2 instances இல் நிறுவப்பட்டு இயக்கப்படும் Amazon மென்பொருள், இது Amazon Systems Manager சேவை EC2 Instances ஐ managed instances ஆக புதுப்பிக்கவும், நிர்வகிக்கவும், கட்டமைக்கவும் சாத்தியமாக்குகிறது. SSM agent Systems Manager சேவையிலிருந்து கோரிக்கைகளைப் பெற்று, அவற்றை செயலாக்கி, நிலை மற்றும் செயல்படுத்தல் தகவல்களை Systems Manager சேவைக்கு திருப்பி அனுப்புகிறது. AWS Systems Manager Agent (SSM Agent) இயல்பாகவே [சில Amazon Machine Images (AMIs) இல் முன்நிறுவப்பட்டுள்ளது](https://docs.aws.amazon.com/systems-manager/latest/userguide/ami-preinstalled-agent.html) என்பதை கவனிக்கவும்.
  - CloudWatch agent wizard CloudWatch agent configuration file ஐ உருவாக்க பயன்படுத்தப்பட்டால், Systems Manager Parameter Store configuration file ஐ மேலும் பெறுவதற்கான பாதுகாப்பான பொதுவான இடமாக பயன்படுத்தலாம். அப்போது CloudWatch Agent க்கு configuration file ஐ எழுத [Systems Manager Parameter Store](https://aws.amazon.com/systems-manager/features/#Parameter_Store) க்கு write-access மற்றும் configuration file ஐ படிக்க read-access தேவைப்படும்.
  - Amazon CloudWatch க்கு தரவை (மெட்ரிக்குகள் மற்றும் லாக்குகள்) எழுத [CloudWatch agent](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/Install-CloudWatch-Agent.html)
- Amazon EC2 Instance(களை) துவக்கி முந்தைய படியில் உருவாக்கப்பட்ட IAM role ஐ ஒதுக்கவும். இந்த IAM role க்கு, Trust Policy க்கான Appendix [1] மற்றும் Amazon Managed Policies - AmazonSSMManagedInstanceCore, CloudWatchAgentAdminPolicy மற்றும் CloudWatchAgentServerPolicy (JSON வடிவத்தில் அனுமதிகள் உட்பட) க்கான Appendix [2] ஐ கீழே பார்க்கவும்.
- தேவையான EC2 instance(களில்) (Windows அல்லது Linux) CloudWatch agent ஐ [கைமுறையாக](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/installing-cloudwatch-agent-commandline.html) அல்லது [Systems Manager Run Command](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/installing-cloudwatch-agent-ssm.html) ஐப் பயன்படுத்தி நிறுவவும்.
- நினைவக மெட்ரிக்குகளை சேகரித்து Amazon CloudWatch க்கு எழுத CloudWatch agent ஐ கட்டமைக்கவும்.

### அளவில் Amazon EC2 Instances இலிருந்து நினைவக மெட்ரிக்குகள் சேகரிப்பு

ஒன்று அல்லது அதற்கு மேற்பட்ட Amazon EC2 Instances இல் Amazon CloudWatch க்கு signal சேகரிப்பு (மெட்ரிக்குகள் மற்றும் லாக்குகள்) க்காக CloudWatch agent ஐ நிறுவி கட்டமைக்க கீழே உள்ள படிகளைப் பின்பற்றலாம்.

- CloudWatch Agent Configuration Wizard ஐ தயாரிக்க ஒரு முறை தேவைப்படும் Remote Desktop அல்லது SSH ஐப் பயன்படுத்தி Amazon EC2 Instance (Windows அல்லது Linux) உடன் இணைக்கவும்.
- மெட்ரிக்குகள் மற்றும் லாக்குகள் சேகரிப்பை அமைக்க CloudWatch Agent Configuration Wizard ஐ இயக்கவும்
  - CPU, நினைவகம், disks போன்ற host மெட்ரிக்குகளை கட்டமைக்கவும்
  - கண்காணிக்க தனிப்பயன் log files ஐ விரும்பினால் சேர்க்கவும் (எ.கா., IIS logs, Apache logs)
  - Windows Event logs ஐ விரும்பினால் கண்காணிக்கவும்
  - அதே கட்டமைப்பை மேலும் Amazon EC2 Instances க்கு பயன்படுத்த முடியுமானால், Systems Manager Parameter Store இல் configuration ஐ சேமிக்கவும்.
- Systems Manager Run Command ஐப் பயன்படுத்தி CloudWatch Agent configuration ஐ மற்ற EC2 Instances க்கு பயன்படுத்தவும். [AmazonCloudWatch-ManageAgent](https://docs.aws.amazon.com/prescriptive-guidance/latest/implementing-logging-monitoring-cloudwatch/create-store-cloudwatch-configurations.html#store-cloudwatch-configuration-s3) Systems Manager Command document ஐ ஒரே run இல் பல EC2 instances இல் CloudWatch configuration ஐ புதுப்பிக்க பயன்படுத்தலாம்.

### Amazon EC2 Instances இலிருந்து நினைவக மெட்ரிக்குகள் சேகரிப்பின் தானியங்குமயம்

Amazon CloudWatch க்கு signal சேகரிப்பை (மெட்ரிக்குகள் மற்றும் லாக்குகள்) தானியங்குபடுத்தவும், ஒருங்கிணைக்கவும் மற்றும் அளவில் நிர்வகிக்கவும் கீழே உள்ள படிகளைப் பின்பற்றலாம். பின்வரும் செயல்களை செய்ய [AWS CloudFormation](https://aws.amazon.com/cloudformation/) template ஐ பயன்படுத்தலாம்

- Amazon EC2 Instances இல் உங்கள் சார்பாக runbooks ஐ செயல்படுத்த Systems Manager automation ஐ அனுமதிக்கும் IAM execution role ஐ உருவாக்கவும்.
- Amazon CloudWatch க்கு தரவை (மெட்ரிக்குகள் மற்றும் லாக்குகள்) எழுத CloudWatch agent க்கான அனுமதிகளுடன் IAM role ஐ அமைக்கவும்
- Amazon EC2 Instances இல் CloudWatch agent ஐ நிறுவி கட்டமைக்க ஒரு [custom runbook](https://docs.aws.amazon.com/systems-manager/latest/userguide/automation-documents.html) ஐ உருவாக்கவும். CloudWatch Agent ஐ நிறுவி இயல்புநிலை மெட்ரிக்குகள் அல்லது Amazon Systems Manager Parameter Store இல் உள்ள parameter உடன் CloudWatch Agent ஐ கட்டமைக்க பயன்படுத்தக்கூடிய எடுத்துக்காட்டு custom runbook document க்கு கீழே உள்ள Appendix [3] ஐ பார்க்கவும்
- Systems manager parameter store க்கு CloudWatch agent configuration file ஐ பதிவேற்றவும்.

[3] CloudWatch Agent ஐ நிறுவி இயல்புநிலை மெட்ரிக்குகள் அல்லது Amazon Systems Manager Parameter Store இல் உள்ள parameter உடன் CloudWatch Agent ஐ கட்டமைக்க பயன்படுத்தக்கூடிய எடுத்துக்காட்டு custom runbook document

```
#-------------------------------------------------
# Composite document and State Manager association to install and configure the Amazon CloudWatch agent
#-------------------------------------------------
InstallAndConfigureCloudWatchAgent:
Type: AWS::SSM::Document
Properties:
    Content:
    schemaVersion: '2.2'
    description: The InstallAndManageCloudWatch command document installs the Amazon CloudWatch agent and manages the configuration of the agent for Amazon EC2 instances.
    parameters:
        action:
        description: The action CloudWatch Agent should take.
        type: String
        default: configure
        allowedValues:
        - configure
        - configure (append)
        - configure (remove)
        - start
        - status
        - stop
        mode:
        description: Controls platform-specific default behavior such as whether to include
            EC2 Metadata in metrics.
        type: String
        default: ec2
        allowedValues:
        - ec2
        - onPremise
        - auto
        optionalConfigurationSource:
        description: Only for 'configure' related actions. Use 'ssm' to apply a ssm parameter
            as config. Use 'default' to apply default config for amazon-cloudwatch-agent.
            Use 'all' with 'configure (remove)' to clean all configs for amazon-cloudwatch-agent.
        type: String
        allowedValues:
        - ssm
        - default
        - all
        default: ssm
        optionalConfigurationLocation:
        description: Only for 'configure' related actions. Only needed when Optional Configuration
            Source is set to 'ssm'. The value should be a ssm parameter name.
        type: String
        default: ''
        allowedPattern: '[a-zA-Z0-9-"~:_@./^(*)!<>?=+]*$'
        optionalRestart:
        description: Only for 'configure' related actions. If 'yes', restarts the agent
            to use the new configuration. Otherwise the new config will only apply on the
            next agent restart.
        type: String
        default: 'yes'
        allowedValues:
        - 'yes'
        - 'no'
    mainSteps:
    - inputs:
        documentParameters:
            name: AmazonCloudWatchAgent
            action: Install
        documentType: SSMDocument
        documentPath: AWS-ConfigureAWSPackage
        name: installCWAgent
        action: aws:runDocument
    - inputs:
        documentParameters:
            mode: '{{mode}}'
            optionalRestart: '{{optionalRestart}}'
            optionalConfigurationSource: '{{optionalConfigurationSource}}'
            optionalConfigurationLocation: '{{optionalConfigurationLocation}}'
            action: '{{action}}'
        documentType: SSMDocument
        documentPath: AmazonCloudWatch-ManageAgent
        name: manageCWAgent
        action: aws:runDocument
    DocumentFormat: YAML
    DocumentType: Command
    TargetType: /AWS::EC2::Instance

CloudWatchAgentAssociation:
Type: AWS::SSM::Association
Properties:
    AssociationName: InstallCloudWatchAgent
    Name: !Ref InstallAndConfigureCloudWatchAgent
    ScheduleExpression: rate(7 days)
    Targets:
    - Key: tag:Platform
    Values:
    - Linux
    WaitForSuccessTimeoutSeconds: 300
```
