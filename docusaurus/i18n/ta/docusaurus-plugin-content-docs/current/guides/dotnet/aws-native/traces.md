# ட்ரேஸ்கள்

ட்ரேஸ்கள் சிக்கலான விநியோகிக்கப்பட்ட அமைப்புகள் வழியாக கோரிக்கை செயலாக்கத்தை கண்காணிக்கின்றன, downstream AWS ஆதாரங்கள், மைக்ரோசர்வீஸ்கள், தரவுத்தளங்கள் மற்றும் API கள் உட்பட தனிப்பட்ட கூறுகள் வழியாக கோரிக்கை ஓட்டம் பற்றிய விரிவான தகவல்களை வழங்குகின்றன. இது தடைகள் மற்றும் தாமதச் சிக்கல்களை அடையாளம் காண்பதன் மூலம் செயல்திறன் மேம்படுத்தலுக்கு உதவும்.

இந்த பிரிவில், AWS X-Ray SDK for .NET ஐப் பயன்படுத்தி .NET பயன்பாடுகளை இன்ஸ்ட்ருமென்ட் செய்து X-Ray daemon வழியாக AWS X-Ray க்கு ட்ரேஸ் தகவலை உருவாக்கவும் அனுப்பவும் AWS ஆவணங்கள் மற்றும் ஓப்பன் சோர்ஸ் களஞ்சியங்களுக்கான இணைப்புகளைக் காண்பீர்கள்.

AWS X-Ray மற்றும் அதன் முக்கிய கருத்துக்கள் பற்றி அறிய [**AWS X-Ray என்றால் என்ன**](https://docs.aws.amazon.com/xray/latest/devguide/aws-xray.html) மற்றும் [**கருத்துக்கள்**](https://docs.aws.amazon.com/xray/latest/devguide/aws-xray.html) பிரிவுகளை AWS X-Ray Developer Guide இல் பார்வையிடுங்கள்.

.NET க்கான X-Ray SDK என்பது C# .NET வலை பயன்பாடுகள், .NET Core வலை பயன்பாடுகள் மற்றும் AWS Lambda இல் .NET Core செயல்பாடுகளை இன்ஸ்ட்ருமென்ட் செய்வதற்கான நூலகமாகும். X-Ray daemon க்கு ட்ரேஸ் தரவை உருவாக்கவும் அனுப்பவும் வகுப்புகள் மற்றும் முறைகளை வழங்குகிறது. பயன்பாடு வழங்கும் உள்வரும் கோரிக்கைகள், பயன்பாடு downstream AWS சேவைகள், HTTP web API கள் மற்றும் SQL தரவுத்தளங்களுக்கு செய்யும் அழைப்புகள் பற்றிய தகவல்களை இது உள்ளடக்கியது.

## Agent கள் மற்றும் SDK களுக்கான விருப்பங்கள்

Amazon EC2 இன்ஸ்டன்ஸ்கள் மற்றும் ஆன்-பிரிமைசஸ் சர்வர்களிலிருந்து ட்ரேஸ்களை சேகரித்து AWS X-Ray க்கு அனுப்ப AWS X-Ray daemon, CloudWatch agent மற்றும் AWS Distro for OpenTelemetry (ADOT) collector ஆகியவற்றுக்கு இடையே தேர்வு செய்யும் விருப்பம் உங்களுக்கு உள்ளது. நீங்கள் நிர்வகிக்க வேண்டிய agent களின் எண்ணிக்கையை குறைக்க உங்கள் பயன்பாட்டு வழக்கிற்கு சரியானதைத் தேர்ந்தெடுக்கவும்.

உங்கள் பயன்பாடு மற்றும் உள்கட்டமைப்பிலிருந்து ட்ரேஸ்களை சேகரித்து அனுப்ப X-Ray daemon ஐ கட்டமைப்பது பற்றி [**AWS X-Ray daemon**](https://docs.aws.amazon.com/xray/latest/devguide/xray-daemon.html) வழிகாட்டியைப் படிக்கவும். அதற்கு பதிலாக CloudWatch agent ஐப் பயன்படுத்த விரும்பினால், [**Amazon CloudWatch user guide**](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/Install-CloudWatch-Agent.html) CloudWatch agent ஐ அமைத்து கட்டமைக்க வழிமுறைகளை வழங்கும்.

ட்ரேஸ்களை உருவாக்க உங்கள் பயன்பாட்டை இன்ஸ்ட்ருமென்ட் செய்ய, OpenTelemetry மற்றும் X-Ray SDK for .NET க்கு இடையே தேர்வு செய்யும் விருப்பம் உங்களுக்கு உள்ளது. இந்த விருப்பங்களுக்கு இடையே தேர்வு செய்வதற்கான வழிகாட்டுதல் [**இங்கே**](https://docs.aws.amazon.com/xray/latest/devguide/xray-instrumenting-your-app.html#xray-instrumenting-choosing) கிடைக்கிறது.

## AWS X-Ray SDK for .NET
 
.NET க்கான X-Ray SDK ஒரு ஓப்பன் சோர்ஸ் திட்டமாகும். .NET க்கான X-Ray SDK .NET Framework 4.5 அல்லது அதற்குப் பிந்தைய பதிப்பை இலக்காகக் கொண்ட பயன்பாடுகளுக்கு ஆதரிக்கப்படுகிறது. .NET Core பயன்பாடுகளுக்கு, SDK க்கு .NET Core 2.0 அல்லது அதற்குப் பிந்தையது தேவை.

தொடங்குவதற்கான இணைப்புகள் இங்கே உள்ளன.

[**AWS X-Ray SDK for .NET developer guide**](https://docs.aws.amazon.com/xray/latest/devguide/xray-sdk-dotnet.html) - இந்த ஆவணம் NuGet வழியாக நிறுவல், கட்டமைப்பு விருப்பங்கள் மற்றும் தானியங்கி HTTP கோரிக்கை ட்ரேசிங் மற்றும் AWS சேவை அழைப்பு கண்காணிப்பு உட்பட பல்வேறு இன்ஸ்ட்ருமென்டேஷன் திறன்களை விளக்குகிறது. தனிப்பயன் segments ஐ உருவாக்குவது, annotations ஐ சேர்ப்பது மற்றும் தரவு சேகரிப்பை நிர்வகிக்க sampling rules ஐ பயன்படுத்துவது பற்றி இது விளக்குகிறது. ASP.NET பயன்பாடுகளில் X-Ray ட்ரேசிங்கை ஒருங்கிணைப்பதற்கான விரிவான தகவல்களை இந்த வழிகாட்டி வழங்குகிறது, பயன்பாட்டு செயல்திறன் பற்றிய தெரிவுநிலையைப் பெறவும் சிக்கல்களை திறம்பட சரிசெய்யவும் டெவலப்பர்களுக்கு உதவுகிறது.

[**SDK ஓப்பன் சோர்ஸ் திட்ட களஞ்சியம் - aws-xray-sdk-dotnet**](https://docs.aws.amazon.com/xray/latest/devguide/xray-sdk-dotnet.html) - aws-xray-sdk-dotnet களஞ்சியம் Amazon இன் X-Ray SDK for .NET க்கான ஓப்பன் சோர்ஸ் குறியீட்டை வைத்துள்ளது. .NET Core மற்றும் .NET Framework சூழல்களில் விநியோகிக்கப்பட்ட பயன்பாட்டு கண்காணிப்பை ஆதரிக்கும் இந்த ட்ரேசிங் கருவியின் செயல்படுத்தலை டெவலப்பர்கள் பார்க்கலாம். HTTP கோரிக்கைகள், AWS சேவை அழைப்புகள் மற்றும் தனிப்பயன் இன்ஸ்ட்ருமென்டிங் திறன்களின் தானியங்கி இன்ஸ்ட்ருமென்டேஷனுக்கான மூல குறியீட்டை களஞ்சியம் உள்ளடக்கியது. SDK ASP.NET கட்டமைப்புகளுடன் எவ்வாறு ஒருங்கிணைகிறது மற்றும் sampling rules ஐ செயல்படுத்துகிறது என்பதை நீங்கள் மதிப்பாய்வு செய்யலாம். இந்த GitHub திட்டம் SDK இன் செயல்பாட்டில் வெளிப்படைத்தன்மையை வழங்குகிறது, அதே நேரத்தில் டெவலப்பர்கள் சிக்கல்களைப் புகாரளிக்கவும் அல்லது குறியீட்டு அடிப்படையில் மேம்பாடுகளை பங்களிக்கவும் அனுமதிக்கிறது.
