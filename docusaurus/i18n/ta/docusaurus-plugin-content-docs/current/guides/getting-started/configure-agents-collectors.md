# ஏஜென்ட்கள்/கலெக்டர்களை கட்டமைத்தல்

உங்கள் கண்காணிப்பு கணக்கு கட்டமைப்பு இருந்தவுடன், உங்கள் பயன்பாடுகள், சேவைகள் மற்றும் பிற உள்கட்டமைப்பு கூறுகளை CloudWatch க்கு டெலிமெட்ரி அனுப்ப கட்டமைக்க வேண்டும்.

இது உங்கள் ஏஜென்ட்கள் மற்றும் கலெக்டர்களை கட்டமைப்பதற்கான உயர் மட்ட வழிகாட்டி. ஆழமான வழிகாட்டுதலுக்கு, இந்த சிறந்த நடைமுறைகள் வழிகாட்டியின் பல்வேறு பிரிவுகளைப் பார்க்கவும்.

## Amazon EKS

EKS க்கு, observability ஐ கட்டமைப்பதற்கான எளிமையான வழி Amazon EKS add-on ஐ பயன்படுத்துவதாகும். இது Amazon EKS க்கான மேம்படுத்தப்பட்ட observability உடன் Container Insights ஐ நிறுவும். Add-on கிளஸ்டரிலிருந்து உள்கட்டமைப்பு மெட்ரிக்குகளை அனுப்ப CloudWatch ஏஜென்ட்டை நிறுவுகிறது, கண்டெய்னர் லாக்குகளை அனுப்ப Fluent Bit ஐ நிறுவுகிறது, மற்றும் பயன்பாட்டு செயல்திறன் டெலிமெட்ரியை அனுப்ப CloudWatch Application Signals ஐயும் இயக்குகிறது. (Application Signals, Container Insights போன்றவை தேவையில்லை என்றால் கட்டமைக்கக்கூடியது.)

பொதுவாக, Amazon CloudWatch Observability EKS add-on ஒரு DaemonSet ஆக நிறுவப்படுகிறது.

EKS க்கான சில விருப்பங்கள்:

### EKS க்கான CloudWatch Agent

- Amazon CloudWatch Observability EKS add-on
- Amazon CloudWatch Observability Helm Chart

### EKS க்கான OTEL Collector

மாற்றாக, நீங்கள் OTEL collector ஐ பயன்படுத்த விரும்பினால்:
- AWS Exporters ஐ கட்டமைக்கலாம்
- உங்கள் OTLP exporter ஐ log மற்றும் traces OTLP எண்ட்பாயிண்ட்களுக்கு சுட்டிக்காட்டலாம்
- உங்கள் செயலாக்க பைப்லைன்களை வரையறுக்கலாம்
- OTEL நூலகங்களைப் பயன்படுத்தி உங்கள் பயன்பாட்டை கருவிப்படுத்தலாம் (தேவைப்பட்டால்)

## Amazon ECS

ECS க்கு, உங்கள் கிளஸ்டர்களுக்கான உள்கட்டமைப்பு மெட்ரிக்குகளை சேகரிக்க Container Insights ஐ இயக்கலாம். பயன்பாட்டு செயல்திறன் டெலிமெட்ரி மற்றும் தொடர்புடைய ட்ரேஸ்களை சேகரிக்க Application Signals ஐயும் நிலைநிறுத்தலாம். லாக்குகளுக்கு, உங்கள் லாக் தரவை CloudWatch க்கு அனுப்ப awslogs driver ஐ பயன்படுத்தலாம், அல்லது தரவை அனுப்ப OpenTelemetry collectors ஐ பயன்படுத்தலாம்.

ECS க்கான சில விருப்பங்கள்:

### ECS க்கான CloudWatch Agent

- Container Insights ஐ இயக்குதல்
- Application Signals ஐ நிலைநிறுத்துதல் (விரும்பினால்)
- awslogs log driver ஐ பயன்படுத்துதல்

### ECS க்கான OTEL Collector

மாற்றாக:
- Sidecar ஆக இயக்கலாம்
- AWS Exporters ஐ கட்டமைக்கலாம்
- OTLP எண்ட்பாயிண்ட்களை அமைக்கலாம்
- செயலாக்க பைப்லைன்களை வரையறுக்கலாம்
- பயன்பாடுகளை கருவிப்படுத்தலாம் (தேவைப்பட்டால்)

## Amazon EC2 மற்றும் ஆன்-பிரமிசஸ்

EC2 நிகழ்வுகள், பிற விர்ச்சுவல் மெஷின்கள் மற்றும் ஆன்-பிரமிசஸ் சர்வர்களிலிருந்து CloudWatch க்கு டெலிமெட்ரி தரவை அனுப்ப CloudWatch agent ஐ பயன்படுத்தலாம்.

### நிலைநிறுத்தல் விருப்பங்கள்

- **EC2 க்கான Workload Detection** – ஏஜென்ட்டை நிலைநிறுத்த ஒரு தானியங்கு வழியை வழங்குகிறது

![EC2 Workload Detection](../../images/GettingStarted/ec2workloaddetection.png)

- **Systems Manager** – AWS Systems Manager ஐப் பயன்படுத்தி ஏஜென்ட்டை நிலைநிறுத்தி கட்டமைக்கலாம்
- **தனிப்பயன் தானியங்கு** – உங்கள் சொந்த தானியங்கு கருவிகளைப் பயன்படுத்தலாம்
- **கைமுறை நிறுவல்** – குறிப்பிட்ட பயன்பாட்டு நிலைகளுக்கு கைமுறையாக நிறுவலாம்

config file மூலம் (தானாக அல்லது கைமுறையாக) டெலிமெட்ரியை கட்டமைக்கலாம்/தனிப்பயனாக்கலாம், மற்றும் உங்கள் அமைப்புகளை நுணுக்கமாக சரிசெய்ய ஒரு wizard கிடைக்கிறது.

### EC2 க்கான OTEL Collector

OTEL collector ஐயும் பயன்படுத்தலாம்:

**OTLP Exporters:**

![OTLP Configuration](../../images/GettingStarted/otlp.png)

trace மற்றும் log OTLP எண்ட்பாயிண்ட்களுக்கு OTLP exporters ஐ பயன்படுத்தவும்.

**AWS-குறிப்பிட்ட Exporters:**

![ADOT Configuration](../../images/GettingStarted/adot.png)

AWS-குறிப்பிட்ட exporters மற்றும் செயலாக்க பைப்லைன்களைப் பயன்படுத்தவும்.

## சுருக்கம்

சுருக்கமாக:
1. உங்கள் கணக்கீட்டு தளத்திற்கு (EKS, ECS, EC2) பொருத்தமான ஏஜென்ட்/கலெக்டரைத் தேர்வு செய்யவும்
2. தானியங்கு முறைகள் (add-ons, Helm charts, Systems Manager) அல்லது கைமுறை நிறுவலைப் பயன்படுத்தி நிலைநிறுத்தவும்
3. உங்கள் தேவைகளின் அடிப்படையில் டெலிமெட்ரி சேகரிப்பை கட்டமைக்கவும்
4. விற்பனையாளர்-நடுநிலை கருவிப்படுத்தலுக்கு விரும்பினால் OpenTelemetry ஐ பயன்படுத்தவும்

விரிவான கட்டமைப்பு வழிகாட்டிகளுக்கு, உங்கள் கணக்கீட்டு தளம் மற்றும் observability கருவிகளுக்கான இந்த சிறந்த நடைமுறைகள் வழிகாட்டியின் குறிப்பிட்ட பிரிவுகளைப் பார்க்கவும்.
