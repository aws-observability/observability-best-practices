# டாஷ்போர்டுகள் மற்றும் எச்சரிக்கைகள்

உங்கள் டெலிமெட்ரி பாய்ந்தவுடன், உங்கள் பயன்பாட்டு நிலைக்கு ஏற்ற டாஷ்போர்டுகள் மற்றும் எச்சரிக்கைகளை அமைக்கலாம்.

## தேர்ந்தெடுக்கப்பட்ட டாஷ்போர்டுகள்

CloudWatch கன்சோலின் பல்வேறு பகுதிகளில் காணக்கூடிய தேர்ந்தெடுக்கப்பட்ட டாஷ்போர்டுகளை பயன்படுத்திக் கொள்ளுங்கள்.

எடுத்துக்காட்டாக, Dashboards பிரிவின் கீழ் பல சேவைகளுக்கு (Lambda, EC2, API Gateway மற்றும் பல) தானியங்கு டாஷ்போர்டுகளை காணலாம்.

நீங்கள் Application Signals ஐ பயன்படுத்தினால், Application Signals (APM) பிரிவின் கீழ் பயன்பாட்டு வரைபடங்கள் மற்றும் டாஷ்போர்டுகளை காணலாம். கூடுதலாக, Observability இல் உள்ள இடைவெளிகளை எடுத்துக்காட்டும் கருவி-பொருத்தப்படாத சேவைகளையும் காணலாம்.

## தனிப்பயன் டாஷ்போர்டுகள்

உங்கள் சொந்த வணிக-குறிப்பிட்ட டாஷ்போர்டுகளையும் வடிவமைக்க வேண்டும். செயல்பாட்டு சிறப்புக்கான டாஷ்போர்டுகளை எவ்வாறு வடிவமைப்பது என்பது குறித்த இந்த வழிகாட்டியைப் பார்க்கவும்: [Building Dashboards for Operational Visibility](https://aws.amazon.com/builders-library/building-dashboards-for-operational-visibility/)

## CloudWatch Alarms

உங்கள் சேவைகள் மற்றும் உள்கட்டமைப்பில் ஏதேனும் சிக்கல்களை சமிக்ஞை செய்ய எச்சரிக்கைகளை (CloudWatch இல் Alarms) உருவாக்குவீர்கள். மையப்படுத்தப்பட்ட எச்சரிக்கை தெரிவுநிலைக்கு உங்கள் கண்காணிப்பு கணக்கில் அல்லது/மற்றும் உள்ளூர் கணக்குகளில் தனிப்பட்ட எச்சரிக்கைகளை உருவாக்கலாம்.

### எச்சரிக்கை பரிந்துரைகள்

எவ்வாறு தொடங்குவது என்று தெரியவில்லை என்றால், Alarm Recommendations உதவும். எச்சரிக்கை பரிந்துரைகள் கண்காணிப்பு சிறந்த நடைமுறைகளின் அடிப்படையில் அமைந்துள்ளன. எச்சரிக்கையை உருவாக்குவதற்கு முன் பரிந்துரைக்கப்பட்ட எச்சரிக்கை கட்டமைப்புகளை மதிப்பாய்வு செய்யுங்கள்.

மேலும் விவரங்களுக்கு, [AWS சேவைகளுக்கான எச்சரிக்கை பரிந்துரைகள்](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/Best_Practice_Recommended_Alarms_AWS_Services.html) ஐ பார்க்கவும்.

## சேவை நிலை நோக்கங்கள் (SLOs)

முக்கியமான KPI களை கண்காணிக்க SLO கள் மற்றும் தொடர்புடைய எச்சரிக்கைகளையும் உருவாக்கலாம்.

மேலும் தகவலுக்கு, [CloudWatch SLOs](../../tools/slos.md) ஐ பார்க்கவும்.

## சுருக்கம்

இது CloudWatch தொடங்குதல் வழிகாட்டியை முடிக்கிறது. நாம் உள்ளடக்கிய படிகள் இங்கே:

1. **கண்காணிப்பு மற்றும் மூல கணக்குகளை அமைத்தல்** – பல AWS கணக்குகள் மற்றும் பகுதிகளிலிருந்து டெலிமெட்ரி தரவை மையப்படுத்த குறுக்கு-கணக்கு observability ஐ கட்டமைத்தல்
2. **ஒருங்கிணைக்கப்பட்ட தரவு சேமிப்பகத்தை அமைத்தல்** – ஒருங்கிணைக்கப்பட்ட வினவல் மற்றும் பகுப்பாய்விற்காக லாக் தரவை ஒரு கணக்கு மற்றும் பகுதியில் மையப்படுத்துதல்
3. **ஏஜென்ட்கள்/கலெக்டர்களை கட்டமைத்தல்** – உங்கள் பயன்பாடுகள் மற்றும் உள்கட்டமைப்பிலிருந்து டெலிமெட்ரி அனுப்ப CloudWatch ஏஜென்ட்கள் மற்றும்/அல்லது OpenTelemetry கலெக்டர்களை நிலைநிறுத்துதல்
4. **டாஷ்போர்டுகள் மற்றும் எச்சரிக்கைகள்** – தெரிவுநிலைக்கான டாஷ்போர்டுகள் மற்றும் உங்கள் சேவைகளின் நலத்தை கண்காணிக்க எச்சரிக்கைகளை உருவாக்குதல்
