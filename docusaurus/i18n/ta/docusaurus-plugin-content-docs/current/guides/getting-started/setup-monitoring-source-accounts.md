# கண்காணிப்பு மற்றும் மூல கணக்குகளை அமைத்தல்

பெரும்பாலான சந்தர்ப்பங்களில், வாடிக்கையாளர்கள் பல AWS கணக்குகளிலிருந்து டெலிமெட்ரி தரவை காட்சிப்படுத்தி தொடர்புபடுத்த வேண்டும், ஏனெனில் அவர்களின் சேவைகள் பல கணக்குகளிலும் சில நேரங்களில் பல பகுதிகளிலும் இயங்குகின்றன.

ஒரே கணக்கில் மட்டும் உங்கள் observability மற்றும் சேவைகளை இயக்க திட்டமிட்டால், இந்த படியை தவிர்க்கலாம்.

முதல் படி உங்கள் கண்காணிப்பு மற்றும் மூல கணக்குகளை அமைத்து நீங்கள் பகிர விரும்பும் டெலிமெட்ரியை சரியாக குறிப்பிடுவதாகும். இதைச் செய்ய குறுக்கு-கணக்கு observability ஐ பயன்படுத்துவீர்கள். இது பகுதி அடிப்படையில் செயல்படுகிறது என்பதை கவனிக்கவும்.

குறுக்கு-கணக்கு observability ஐ எவ்வாறு அமைப்பது என்பது பற்றிய விரிவான வழிமுறைகளுக்கு, [CloudWatch Cross-Account Observability](../cloudwatch_cross_account_observability.md) வழிகாட்டியைப் பார்க்கவும்.

## கண்காணிப்பு கணக்கு

டெலிமெட்ரி தரவை மையப்படுத்தப்பட்ட முறையில் பார்க்க விரும்பும் ஒரு கண்காணிப்பு கணக்கை நியமிக்கவும்.

பின்னர் எந்த கணக்குகள் உங்கள் கண்காணிப்பு கணக்குடன் தரவைப் பகிரும் என்பதை வரையறுக்கவும். உங்கள் AWS நிறுவனத்தில் உள்ள அனைத்து கணக்குகளையும் தேர்வு செய்யலாம் அல்லது தனிப்பட்ட மூல கணக்குகளை தேர்வு செய்யலாம். கண்காணிப்பு கணக்குடன் பகிர விரும்பும் டெலிமெட்ரி தரவையும் குறிப்பிடுவீர்கள் (எ.கா. லாக்குகள், மெட்ரிக்குகள், ட்ரேஸ்கள், application signals போன்றவை).

பின்னர் அமைப்பை முடிக்க [மூல கணக்குகளை இணைப்பீர்கள்](../cloudwatch_cross_account_observability.md#step-2-link-the-source-accounts).

உங்கள் வழக்கமான கண்காணிப்பு கணக்கு கட்டமைப்பு இது போல் இருக்கும்:

![Monitoring Account Structure](../../images/GettingStarted/monitoring-acct-struct.png)

CloudWatch அமைப்புகளில் பகுதி அடிப்படையில் இதை [கட்டமைப்பீர்கள்](../cloudwatch_cross_account_observability.md#step-1-set-up-a-monitoring-account).

:::info
குறுக்கு-கணக்கு observability உடன், லாக்குகள் மற்றும் மெட்ரிக்குகள் மூல கணக்குகளிலிருந்து நகலெடுக்கப்படுவதில்லை, ஆனால் trace தரவு கண்காணிப்பு கணக்கிற்கு நகலெடுக்கப்படுகிறது (முதல் கண்காணிப்பு கணக்கிற்கான trace நகல் கூடுதல் செலவில்லாமல் சேர்க்கப்படுகிறது). நீங்கள் லாக்குகள், மெட்ரிக்குகள், ட்ரேஸ்கள் மற்றும் பிற டெலிமெட்ரியை மையமாக பார்க்கிறீர்கள்.
:::

## பல கண்காணிப்பு கணக்குகள்

ஒவ்வொரு கண்காணிப்பு கணக்கும் 100,000 மூல கணக்குகள் வரை இணைக்கப்படலாம்.

இருப்பினும், பல கண்காணிப்பு கணக்குகள் தேவைப்படும் செயல்பாட்டு சூழ்நிலைகள் இருக்கலாம். உங்கள் சொந்த தேவைகளின் அடிப்படையில் பல கண்காணிப்பு கணக்குகளை வைத்திருக்கலாம். இந்த அமைப்பு இது போல் இருக்கலாம்:

![Multiple Monitoring Accounts](../../images/GettingStarted/multiple-mon-accts.png)

:::info
ஒரு மூல கணக்கிலிருந்து பல கண்காணிப்பு கணக்குகளுடன் தரவைப் பகிர வேண்டுமென்றால், ஒவ்வொரு மூல கணக்கும் 5 கண்காணிப்பு கணக்குகள் வரை தரவைப் பகிரலாம் என்பதால் அதுவும் கட்டமைக்கக்கூடியது.
:::

## டெலிமெட்ரி கட்டுப்பாடு

மெட்ரிக் மற்றும் log வடிகட்டிகளை குறிப்பிடும் திறனுடன் நீங்கள் பகிரும் டெலிமெட்ரி தரவின் மீது கட்டுப்பாட்டையும் கொண்டுள்ளீர்கள், இது கூடுதல் நுணுக்கத்தை வழங்குகிறது.

![Telemetry Configuration](../../images/GettingStarted/telemetry-config.png)

இப்போது ஒரு கண்காணிப்பு கணக்கிலிருந்து (பகுதி அடிப்படையில்) பல கணக்குகளிலிருந்து [குறுக்கு-கணக்கு தரவை காட்சிப்படுத்தி வினவ](../cloudwatch_cross_account_observability.md#querying-cross-account-telemetry-data) முடியும்.

## சுருக்கம்

சுருக்கமாக:
1. கண்காணிப்பு கணக்கு(களை) நியமித்து கட்டமைக்கவும்
2. மூல கணக்குகளை கட்டமைக்கவும்
3. எந்த டெலிமெட்ரியை பகிர விரும்புகிறீர்கள் என்பதை நுணுக்கமாக சரிசெய்யவும்
4. கண்காணிப்பு கணக்கிலிருந்து அனைத்து மூல கணக்கு தரவையும் காட்சிப்படுத்தி வினவவும்
