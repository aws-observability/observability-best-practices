# Observability என்றால் என்ன

## இது என்ன

Observability என்பது கண்காணிக்கப்படும் அமைப்பிலிருந்து வரும் சிக்னல்களின் அடிப்படையில் செயல்படக்கூடிய நுண்ணறிவுகளை தொடர்ச்சியாக உருவாக்கி கண்டறியும் திறன் ஆகும். வேறு வார்த்தைகளில் கூறினால், observability பயனர்களுக்கு ஒரு அமைப்பின் வெளிப்புற வெளியீட்டிலிருந்து அதன் நிலையைப் புரிந்துகொள்ளவும் (திருத்தும்) நடவடிக்கை எடுக்கவும் அனுமதிக்கிறது.

## இது தீர்க்கும் சிக்கல்

கணினி அமைப்புகள் CPU நேரம், நினைவகம், வட்டு இடம் போன்ற குறைந்த நிலை சிக்னல்கள் மற்றும் API மறுமொழி நேரங்கள், பிழைகள், வினாடிக்கு பரிவர்த்தனைகள் போன்ற உயர் நிலை மற்றும் வணிக சிக்னல்களைக் கண்காணிப்பதன் மூலம் அளவிடப்படுகின்றன.

ஒரு அமைப்பின் observability அதன் இயக்க மற்றும் மேம்பாட்டு செலவுகளில் குறிப்பிடத்தக்க தாக்கத்தை ஏற்படுத்துகிறது. கவனிக்கத்தக்க அமைப்புகள் தங்கள் ஆபரேட்டர்களுக்கு அர்த்தமுள்ள, செயல்படக்கூடிய தரவை வழங்குகின்றன, இது சாதகமான முடிவுகளை (விரைவான சம்பவ மறுமொழி, அதிகரித்த டெவலப்பர் உற்பத்தித்திறன்) மற்றும் குறைவான உழைப்பு மற்றும் டவுன்டைமை அடைய அனுமதிக்கிறது.

## இது எவ்வாறு உதவுகிறது

அதிக தகவல் என்பது அதிக கவனிக்கத்தக்க அமைப்பாக மாற்றப்படுவதில்லை என்பதைப் புரிந்துகொள்வது முக்கியமானது. உண்மையில், சில நேரங்களில், ஒரு அமைப்பால் உருவாக்கப்படும் தகவலின் அளவு, பயன்பாட்டால் உருவாக்கப்படும் இரைச்சலிலிருந்து மதிப்புமிக்க ஆரோக்கிய சிக்னல்களை அடையாளம் காண்பதை கடினமாக்கலாம். Observability க்கு சரியான முடிவுகளை எடுக்க சரியான நுகர்வோருக்கு (மனிதர் அல்லது மென்பொருள்) சரியான நேரத்தில் சரியான தரவு தேவைப்படுகிறது.

## இங்கே நீங்கள் என்ன காண்பீர்கள்

இந்த தளம் observability க்கான எங்கள் சிறந்த நடைமுறைகளைக் கொண்டுள்ளது: என்ன செய்ய வேண்டும், என்ன *செய்யக்கூடாது*, மற்றும் அவற்றை எவ்வாறு செய்வது என்பதற்கான செய்முறைகளின் தொகுப்பு. இங்குள்ள பெரும்பாலான உள்ளடக்கம் விற்பனையாளர் அறியாதது மற்றும் எந்தவொரு நல்ல observability தீர்வும் வழங்குவதைக் குறிக்கிறது.

Observability ஐ ஒரு *தீர்வாக* கருதுவது முக்கியம், *தயாரிப்பாக* அல்ல. Observability உங்கள் நடைமுறைகளிலிருந்து வருகிறது, மேலும் இது வலுவான மேம்பாடு மற்றும் DevOps தலைமைத்துவத்தின் ஒருங்கிணைந்த பகுதியாகும். நன்கு கவனிக்கப்படும் பயன்பாடு என்பது, பாதுகாப்பை ஒரு திட்டத்தை எவ்வாறு ஒழுங்கமைக்கிறீர்கள் என்பதன் முன்னணியில் வைக்க வேண்டும் என்பதைப் போலவே, observability ஐ செயல்பாடுகளின் முதன்மை கோட்பாடாக வைப்பதாகும். உண்மைக்குப் பிறகு observability ஐ "போல்ட்-ஆன்" செய்ய முயற்சிப்பது ஒரு எதிர்-வடிவமாகும் மற்றும் குறைவான வெற்றியைப் பெறுகிறது.
