# CloudWatch Container Insights

## அறிமுகம்

Amazon CloudWatch Container Insights கண்டெய்னரைஸ்டு அப்ளிகேஷன்கள் மற்றும் மைக்ரோசர்வீஸ்களிலிருந்து மெட்ரிக்குகள் மற்றும் லாக்குகளைச் சேகரிக்க, ஒருங்கிணைக்க மற்றும் சுருக்கமாகத் தருவதற்கான சக்திவாய்ந்த கருவியாகும். இந்த ஆவணம் EKS Fargate பணிச்சுமைகளுக்கான ADOT மற்றும் CloudWatch Container Insights ஒருங்கிணைப்பின் வடிவமைப்பு, டிப்ளாய்மென்ட் செயல்முறை மற்றும் நன்மைகளின் கண்ணோட்டத்தை வழங்குகிறது.

## EKS Fargate-க்கான ADOT Collector வடிவமைப்பு

ADOT Collector மூன்று முக்கிய கூறுகளைக் கொண்ட பைப்லைன் கட்டமைப்பைப் பயன்படுத்துகிறது:

1. Receiver: குறிப்பிட்ட வடிவத்தில் தரவை ஏற்றுக்கொண்டு உள் வடிவத்திற்கு மொழிபெயர்க்கிறது.
2. Processor: தொகுத்தல், வடிகட்டுதல் மற்றும் தரவின் மாற்றங்கள் போன்ற பணிகளைச் செய்கிறது.
3. Exporter: மெட்ரிக்குகள், லாக்குகள் அல்லது ட்ரேஸ்களை அனுப்ப வேண்டிய இலக்கை தீர்மானிக்கிறது.

EKS Fargate-க்கு, ADOT Collector Kubernetes API சேவையகத்திலிருந்து மெட்ரிக்குகளை ஸ்கிரேப் செய்ய Prometheus Receiver ஐப் பயன்படுத்துகிறது, இது தொழிலாளர் நோடுகளில் kubelet-க்கான ப்ராக்ஸியாகச் செயல்படுகிறது. EKS Fargate-ல் kubelet-க்கு நேரடி அணுகலைத் தடுக்கும் நெட்வொர்க்கிங் வரம்புகளால் இந்த அணுகுமுறை அவசியமாகும். சேகரிக்கப்பட்ட மெட்ரிக்குகள் வடிகட்டுதல், மறுபெயரிடுதல், தரவு ஒருங்கிணைப்பு மற்றும் மாற்றத்திற்கான தொடர் செயலாக்கிகள் வழியாகச் செல்கின்றன. இறுதியாக, AWS CloudWatch EMF Exporter மெட்ரிக்குகளை embedded metric format (EMF) ஆக மாற்றி CloudWatch Logs-க்கு அனுப்புகிறது.

## டிப்ளாய்மென்ட் செயல்முறை

EKS Fargate கிளஸ்டரில் ADOT Collector ஐ டிப்ளாய் செய்ய:

1. Kubernetes உடன் ஒரு EKS கிளஸ்டரை உருவாக்கவும்
2. Fargate pod execution role ஐ அமைக்கவும்.
3. தேவையான நேம்ஸ்பேஸ்களுக்கு Fargate profiles ஐ வரையறுக்கவும்.
4. தேவையான அனுமதிகளுடன் ADOT Collector-க்கான IAM role ஐ உருவாக்கவும்.
5. வழங்கப்பட்ட manifest ஐப் பயன்படுத்தி ADOT Collector ஐ Kubernetes StatefulSet ஆக டிப்ளாய் செய்யவும்.
6. மெட்ரிக்குகள் சேகரிப்பைச் சோதிக்க மாதிரி பணிச்சுமைகளை டிப்ளாய் செய்யவும்.


## நன்மைகள் மற்றும் குறைபாடுகள்

### நன்மைகள்:

1. ஒருங்கிணைந்த கண்காணிப்பு: EKS EC2 மற்றும் Fargate பணிச்சுமைகளில் நிலையான கண்காணிப்பு அனுபவத்தை வழங்குகிறது.
2. அளவிடக்கூடியது: ஒரே ADOT Collector நிகழ்வு EKS கிளஸ்டரில் உள்ள அனைத்து தொழிலாளர் நோடுகளிலிருந்தும் மெட்ரிக்குகளைக் கண்டறிந்து சேகரிக்க முடியும்.
3. வளமான மெட்ரிக்குகள்: CPU, நினைவகம், வட்டு மற்றும் நெட்வொர்க் பயன்பாடு உள்ளிட்ட விரிவான அமைப்பு மெட்ரிக்குகளைச் சேகரிக்கிறது.
4. எளிதான ஒருங்கிணைப்பு: ஏற்கனவே உள்ள CloudWatch டாஷ்போர்டுகள் மற்றும் அலாரங்களுடன் தடையின்றி ஒருங்கிணைகிறது.
5. செலவு-குறைந்தது: கூடுதல் கண்காணிப்பு உள்கட்டமைப்பு தேவையில்லாமல் Fargate பணிச்சுமைகளைக் கண்காணிக்க உதவுகிறது.

### குறைபாடுகள்:

1. உள்ளமைவு சிக்கலான தன்மை: ADOT Collector ஐ அமைக்க IAM roles, Fargate profiles மற்றும் Kubernetes resources-ன் கவனமான உள்ளமைவு தேவை.
2. வள மேல்நிலை: ADOT Collector தானே Fargate கிளஸ்டரில் வளங்களை நுகர்கிறது, இதை திறன் திட்டமிடலில் கணக்கில் எடுத்துக்கொள்ள வேண்டும்.


EKS Fargate பணிச்சுமைகளுக்கான CloudWatch Container Insights உடன் AWS Distro for OpenTelemetry-ன் ஒருங்கிணைப்பு கண்டெய்னரைஸ்டு அப்ளிகேஷன்களைக் கண்காணிப்பதற்கான சக்திவாய்ந்த தீர்வை வழங்குகிறது. வெவ்வேறு EKS டிப்ளாய்மென்ட் விருப்பங்களில் ஒருங்கிணைந்த கண்காணிப்பு அனுபவத்தை வழங்குகிறது மற்றும் OpenTelemetry ஃபிரேம்வொர்க்கின் அளவிடக்கூடியது மற்றும் நெகிழ்வுத்தன்மையைப் பயன்படுத்துகிறது. Fargate பணிச்சுமைகளிலிருந்து அமைப்பு மெட்ரிக்குகளின் சேகரிப்பை இயக்குவதன் மூலம், இந்த ஒருங்கிணைப்பு வாடிக்கையாளர்கள் தங்கள் அப்ளிகேஷன் செயல்திறனில் ஆழமான நுண்ணறிவுகளைப் பெறவும், தகவலறிந்த அளவிடுதல் முடிவுகளை எடுக்கவும், வள பயன்பாட்டை உகந்ததாக்கவும் உதவுகிறது.
