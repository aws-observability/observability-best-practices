# Application Signals மூலம் APM

நவீன அப்ளிகேஷன் மேம்பாட்டின் தொடர்ச்சியாக மாறிவரும் உலகில், உகந்த செயல்திறனை உறுதி செய்வதும் சேவை நிலை இலக்குகளை (SLOs) நிறைவேற்றுவதும் தடையற்ற பயனர் அனுபவத்தையும் வணிக தொடர்ச்சியையும் பராமரிக்க மிக முக்கியமானது. Amazon CloudWatch Application Signals, OpenTelemetry (OTel) இணக்கமான அப்ளிகேஷன் செயல்திறன் கண்காணிப்பு (APM) அம்சமாகும், இது AWS-ல் இயங்கும் அப்ளிகேஷன்களைக் கண்காணிக்கும் மற்றும் சிக்கல் தீர்க்கும் முறையை புரட்சிகரமாக மாற்றுகிறது.

CloudWatch Application Signals மெட்ரிக்குகள், ட்ரேஸ்கள், லாக்குகள், உண்மையான பயனர் கண்காணிப்பு மற்றும் செயற்கை கண்காணிப்பு உள்ளிட்ட பல மூலங்களிலிருந்து டெலிமெட்ரி தரவை தடையின்றி தொடர்புபடுத்தி அப்ளிகேஷன் செயல்திறன் கண்காணிப்புக்கு ஒரு முழுமையான அணுகுமுறையை எடுக்கிறது. இந்த ஒருங்கிணைந்த அணுகுமுறை நிறுவனங்கள் தங்கள் அப்ளிகேஷன்களின் செயல்திறன் பற்றிய விரிவான நுண்ணறிவுகளைப் பெறவும், சிக்கல்களின் மூல காரணங்களைக் கண்டறியவும், சாத்தியமான இடையூறுகளை முன்கூட்டியே சரிசெய்யவும் உதவுகிறது.

CloudWatch Application Signals-ன் முக்கிய நன்மைகளில் ஒன்று அதன் தானியங்கி கருவியமைப்பு மற்றும் கண்காணிப்பு திறன்கள் ஆகும். கைமுறை முயற்சி அல்லது தனிப்பயன் குறியீடு தேவையில்லாமல், Application Signals அப்ளிகேஷன் செயல்திறனுக்கான மிக முக்கியமான மெட்ரிக்குகளைக் காட்டும் முன்கூட்டியே கட்டமைக்கப்பட்ட, தரப்படுத்தப்பட்ட டாஷ்போர்டை வழங்குகிறது - அளவு, கிடைக்கும்தன்மை, தாமதம், தவறுகள் மற்றும் பிழைகள் - AWS-ல் இயங்கும் ஒவ்வொரு அப்ளிகேஷனுக்கும். இந்த எளிமைப்படுத்தப்பட்ட அணுகுமுறை தனிப்பயன் டாஷ்போர்டுகளின் தேவையை நீக்குகிறது, சேவை ஆபரேட்டர்கள் தங்கள் வரையறுக்கப்பட்ட SLO-களுக்கு எதிராக அப்ளிகேஷன் ஆரோக்கியத்தையும் செயல்திறனையும் விரைவாக மதிப்பீடு செய்ய உதவுகிறது.

*படம் 1: CloudWatch Application Signals மெட்ரிக்குகள், லாக்குகள் மற்றும் ட்ரேஸ்கள் அனுப்புதல்*

CloudWatch Application Signals நிறுவனங்களுக்கு பின்வரும் திறன்களை வழங்குகிறது:

1. **விரிவான அப்ளிகேஷன் செயல்திறன் கண்காணிப்பு**: Application Signals மெட்ரிக்குகள், ட்ரேஸ்கள், லாக்குகள், உண்மையான பயனர் கண்காணிப்பு மற்றும் செயற்கை கண்காணிப்பு ஆகியவற்றின் நுண்ணறிவுகளை ஒருங்கிணைத்து அப்ளிகேஷன் செயல்திறனின் ஒருங்கிணைந்த காட்சியை வழங்குகிறது. இந்த முழுமையான அணுகுமுறை நிறுவனங்கள் செயல்திறன் தடைகளை அடையாளம் காணவும், மூல காரணங்களைக் கண்டறியவும், உகந்த அப்ளிகேஷன் செயல்திறனை உறுதி செய்ய முன்னெச்சரிக்கை நடவடிக்கைகள் எடுக்கவும் உதவுகிறது.

2. **தானியங்கி கருவியமைப்பு மற்றும் கண்காணிப்பு**: கைமுறை முயற்சி அல்லது தனிப்பயன் குறியீடு தேவையில்லாமல், Application Signals வரையறுக்கப்பட்ட SLO-களுக்கு எதிராக அப்ளிகேஷன் செயல்திறனை தானாகவே கருவியமைத்துக் கண்காணிக்கிறது. இந்த எளிமைப்படுத்தப்பட்ட அணுகுமுறை கைமுறை கருவியமைப்பு மற்றும் உள்ளமைவுடன் தொடர்புடைய மேல்நிலையைக் குறைக்கிறது, நிறுவனங்கள் அப்ளிகேஷன் மேம்பாடு மற்றும் உகந்தமாக்கலில் கவனம் செலுத்த உதவுகிறது.

3. **தரப்படுத்தப்பட்ட டாஷ்போர்டு மற்றும் காட்சிப்படுத்தல்**: Application Signals அளவு, கிடைக்கும்தன்மை, தாமதம், தவறுகள் மற்றும் பிழைகள் உள்ளிட்ட அப்ளிகேஷன் செயல்திறனுக்கான மிக முக்கியமான மெட்ரிக்குகளைக் காட்டும் முன்கூட்டியே கட்டமைக்கப்பட்ட, தரப்படுத்தப்பட்ட டாஷ்போர்டை வழங்குகிறது. இந்த தரப்படுத்தப்பட்ட காட்சி சேவை ஆபரேட்டர்கள் அப்ளிகேஷன் ஆரோக்கியத்தையும் செயல்திறனையும் விரைவாக மதிப்பீடு செய்ய உதவுகிறது, தகவலறிந்த முடிவெடுத்தல் மற்றும் முன்னெச்சரிக்கை சிக்கல் தீர்வை எளிதாக்குகிறது.

4. **தடையற்ற தொடர்புபடுத்தல் மற்றும் சிக்கல் தீர்வு**: பல மூலங்களிலிருந்து டெலிமெட்ரி தரவை தொடர்புபடுத்துவதன் மூலம், Application Signals சிக்கல் தீர்வு செயல்முறையை எளிமைப்படுத்துகிறது. சேவை ஆபரேட்டர்கள் செயல்திறன் சிக்கல்கள் அல்லது முரண்பாடுகளின் மூல காரணத்தை அடையாளம் காண தொடர்புடைய ட்ரேஸ்கள், லாக்குகள் மற்றும் மெட்ரிக்குகளில் தடையின்றி ஆழமாக ஆராயலாம், சராசரி தீர்வு நேரத்தை (MTTR) குறைத்து அப்ளிகேஷன் இடையூறுகளைக் குறைக்கிறது.

5. **Container Insights உடன் ஒருங்கிணைப்பு**: கண்டெய்னரைஸ்டு சூழல்களில் இயங்கும் அப்ளிகேஷன்களுக்கு, CloudWatch Application Signals Container Insights உடன் தடையின்றி ஒருங்கிணைகிறது, கண்டெய்னர் பாட்களில் நினைவக பற்றாக்குறை அல்லது அதிக CPU பயன்பாடு போன்ற அப்ளிகேஷன் செயல்திறனைப் பாதிக்கக்கூடிய உள்கட்டமைப்பு தொடர்பான சிக்கல்களை அடையாளம் காண நிறுவனங்களுக்கு உதவுகிறது.

அப்ளிகேஷன் செயல்திறன் கண்காணிப்புக்கு CloudWatch Application Signals ஐப் பயன்படுத்த, நிறுவனங்கள் இந்த பொதுவான படிகளைப் பின்பற்றலாம்:

1. **Application Signals ஐ இயக்கவும்**: AWS Management Console, AWS Command Line Interface (CLI) அல்லது AWS SDK-களைப் பயன்படுத்தி நிரல்ரீதியாக AWS-ல் இயங்கும் உங்கள் அப்ளிகேஷன்களுக்கு CloudWatch Application Signals ஐ இயக்கவும்.

2. **சேவை நிலை இலக்குகளை (SLOs) வரையறுக்கவும்**: வணிகத் தேவைகள் மற்றும் வாடிக்கையாளர் எதிர்பார்ப்புகளுக்கு ஏற்ப இலக்கு கிடைக்கும்தன்மை, அதிகபட்ச தாமதம் அல்லது பிழை வரம்புகள் போன்ற உங்கள் அப்ளிகேஷன்களுக்கான விரும்பிய SLO-களை நிறுவி உள்ளமைக்கவும்.

3. **செயல்திறனைக் கண்காணித்து பகுப்பாய்வு செய்யவும்**: வரையறுக்கப்பட்ட SLO-களுக்கு எதிராக அப்ளிகேஷன் செயல்திறனைக் கண்காணிக்க Application Signals வழங்கும் முன்கூட்டியே கட்டமைக்கப்பட்ட, தரப்படுத்தப்பட்ட டாஷ்போர்டைப் பயன்படுத்தவும். செயல்திறன் சிக்கல்கள் அல்லது முரண்பாடுகளை அடையாளம் காண மெட்ரிக்குகள், ட்ரேஸ்கள், லாக்குகள், உண்மையான பயனர் கண்காணிப்பு மற்றும் செயற்கை கண்காணிப்பு தரவை பகுப்பாய்வு செய்யவும்.

4. **சிக்கல்களைத் தீர்த்து சரிசெய்யவும்**: செயல்திறன் சிக்கல்கள் அல்லது மூல காரணங்களை விரைவாக அடையாளம் கண்டு தீர்க்க Application Signals-ன் தடையற்ற தொடர்புபடுத்தல் திறன்களைப் பயன்படுத்தி தொடர்புடைய ட்ரேஸ்கள், லாக்குகள் மற்றும் மெட்ரிக்குகளில் ஆழமாக ஆராயவும்.

5. **Container Insights உடன் ஒருங்கிணைக்கவும் (பொருந்தினால்)**: கண்டெய்னரைஸ்டு அப்ளிகேஷன்களுக்கு, அப்ளிகேஷன் செயல்திறனைப் பாதிக்கக்கூடிய உள்கட்டமைப்பு தொடர்பான சிக்கல்களை அடையாளம் காண CloudWatch Application Signals ஐ Container Insights உடன் ஒருங்கிணைக்கவும்.

CloudWatch Application Signals சக்திவாய்ந்த அப்ளிகேஷன் செயல்திறன் கண்காணிப்பு திறன்களை வழங்கும் அதே நேரத்தில், தரவு அளவு மற்றும் செலவு மேலாண்மை போன்ற சாத்தியமான சவால்களைக் கருத்தில் கொள்வது முக்கியம். அப்ளிகேஷன் சிக்கலான தன்மை மற்றும் அளவு அதிகரிக்கும்போது, உருவாக்கப்படும் டெலிமெட்ரி தரவின் அளவு குறிப்பிடத்தக்க அளவில் வளரலாம், செயல்திறனைப் பாதிக்கலாம் மற்றும் கூடுதல் செலவுகளை ஏற்படுத்தலாம். திறமையான மற்றும் செலவு-குறைந்த கண்காணிப்பு தீர்வை உறுதி செய்ய தரவு மாதிரி உத்திகள், தக்கவைப்பு கொள்கைகள் மற்றும் செலவு உகந்தமாக்கல் நுட்பங்களை செயல்படுத்துவது அவசியமாக இருக்கலாம்.

கூடுதலாக, உங்கள் அப்ளிகேஷன் செயல்திறன் தரவுக்கு சரியான அணுகல் கட்டுப்பாடு மற்றும் தரவு பாதுகாப்பை உறுதி செய்வது மிக முக்கியமானது. CloudWatch Application Signals நுணுக்கமான அணுகல் கட்டுப்பாட்டிற்கு AWS Identity and Access Management (IAM) ஐப் பயன்படுத்துகிறது, மேலும் உங்கள் அப்ளிகேஷன் செயல்திறன் தரவின் ரகசியத்தன்மை மற்றும் ஒருமைப்பாட்டைப் பாதுகாக்க ஓய்வில் மற்றும் போக்குவரத்தில் டெலிமெட்ரி தரவுக்கு தரவு குறியாக்கம் பயன்படுத்தப்படுகிறது.

முடிவாக, CloudWatch Application Signals AWS-ல் இயங்கும் அப்ளிகேஷன்களுக்கு அப்ளிகேஷன் செயல்திறன் கண்காணிப்பை புரட்சிகரமாக மாற்றுகிறது. தானியங்கி கருவியமைப்பு, தரப்படுத்தப்பட்ட டாஷ்போர்டுகள் மற்றும் டெலிமெட்ரி தரவின் தடையற்ற தொடர்புபடுத்தல் வழங்குவதன் மூலம், Application Signals நிறுவனங்களை அப்ளிகேஷன் செயல்திறனை முன்னெச்சரிக்கையாகக் கண்காணிக்கவும், SLO இணக்கத்தை உறுதி செய்யவும், செயல்திறன் சிக்கல்களை விரைவாகத் தீர்க்கவும் சரிசெய்யவும் அதிகாரம் அளிக்கிறது. அதன் ஒருங்கிணைப்பு திறன்கள் மற்றும் OpenTelemetry இணக்கத்துடன், CloudWatch Application Signals கிளவுட்டில் அப்ளிகேஷன் செயல்திறன் கண்காணிப்புக்கான விரிவான மற்றும் எதிர்காலத்திற்குத் தயாரான தீர்வை வழங்குகிறது.
