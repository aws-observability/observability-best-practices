# AWS ஓப்பன் சோர்ஸ் சேவையுடன் குறுக்கு-கணக்கு கண்காணிப்பு

## அறிமுகம்

நவீன கிளவுட் சூழல்கள் பெரும்பாலும் பல கணக்குகளில் பரவியிருக்கும் மற்றும் ஆன்-பிரமைஸ் உள்கட்டமைப்பை உள்ளடக்கி, சிக்கலான கண்காணிப்பு சவால்களை உருவாக்குகின்றன. இந்த சவால்களை சமாளிக்க, AWS சேவைகள் மற்றும் தொழில்துறை-தரமான கருவிகளைப் பயன்படுத்தி அதிநவீன கண்காணிப்பு கட்டமைப்பை செயல்படுத்தலாம். இந்த கட்டமைப்பு பல்வேறு சூழல்களில் விரிவான தெரிவுநிலையை வழங்குகிறது, திறமையான மேலாண்மை மற்றும் விரைவான சிக்கல் தீர்வை எளிதாக்குகிறது.

## முக்கிய கூறுகள்

இந்த கண்காணிப்பு தீர்வின் மையத்தில் AWS Distro for OpenTelemetry (ADOT) உள்ளது, இது பல்வேறு மூலங்களிலிருந்து மெட்ரிக்குகளுக்கான மையப்படுத்தப்பட்ட சேகரிப்பு புள்ளியாகச் செயல்படுகிறது. ADOT ஒரு அர்ப்பணிக்கப்பட்ட மைய AWS கணக்கில் டிப்ளாய் செய்யப்பட்டு, கண்காணிப்பு உள்கட்டமைப்பின் மையமாக அமைகிறது. இந்த மைய டிப்ளாய்மென்ட் ஒழுங்கமைக்கப்பட்ட தரவு ஒருங்கிணைப்பு மற்றும் செயலாக்கத்தை அனுமதிக்கிறது.

Amazon Managed Service for Prometheus மற்றொரு முக்கிய கூறாகும், சேகரிக்கப்பட்ட மெட்ரிக்குகளைச் சேமிப்பதற்கான அளவிடக்கூடிய மற்றும் நிர்வகிக்கப்படும் நேர-தொடர் தரவுத்தளத்தை வழங்குகிறது. இந்த சேவை சுய-நிர்வகிக்கப்படும் Prometheus நிகழ்வுகளின் தேவையை நீக்குகிறது, செயல்பாட்டு மேல்நிலையைக் குறைத்து மெட்ரிக் தரவின் உயர் கிடைக்கும்தன்மையை உறுதி செய்கிறது.

காட்சிப்படுத்தல் மற்றும் பகுப்பாய்வுக்காக, Grafana கட்டமைப்பில் ஒருங்கிணைக்கப்பட்டுள்ளது. Grafana Amazon Managed Service for Prometheus-உடன் இணைகிறது, சக்திவாய்ந்த வினவல் திறன்கள் மற்றும் தனிப்பயனாக்கக்கூடிய டாஷ்போர்டுகளை வழங்குகிறது. இது குழுக்கள் நுண்ணறிவுள்ள காட்சிப்படுத்தல்களை உருவாக்கவும் சேகரிக்கப்பட்ட மெட்ரிக்குகளின் அடிப்படையில் எச்சரிக்கைகளை அமைக்கவும் அனுமதிக்கிறது.

## தரவு சேகரிப்பு மற்றும் ஓட்டம்

கட்டமைப்பு கண்காணிக்கப்படும் கணக்குகள் என குறிப்பிடப்படும் பல AWS கணக்குகளிலிருந்து தரவு சேகரிப்பை ஆதரிக்கிறது. இந்த கணக்குகள் தங்கள் மெட்ரிக்குகளை மைய ADOT நிகழ்வுக்கு ஏற்றுமதி செய்ய OpenTelemetry Protocol (OTLP) ஐப் பயன்படுத்துகின்றன. இந்த தரப்படுத்தப்பட்ட அணுகுமுறை தரவு வடிவத்தில் நிலைத்தன்மையை உறுதி செய்கிறது மற்றும் புதிய கணக்குகளை கண்காணிப்பு அமைப்பில் எளிதாக ஒருங்கிணைக்க உதவுகிறது.

ஆன்-பிரமைஸ் உள்கட்டமைப்பும் இந்த கண்காணிப்பு தீர்வில் இணைக்கப்பட்டுள்ளது. இந்த அமைப்புகள் பாதுகாப்பான HTTPS POST கோரிக்கைகளைப் பயன்படுத்தி மைய ADOT நிகழ்வுக்கு தங்கள் மெட்ரிக்குகள் தரவை அனுப்புகின்றன. இந்த முறை பழைய அல்லது கிளவுட் அல்லாத அமைப்புகளை ஒட்டுமொத்த கண்காணிப்பு உத்தியில் சேர்க்க அனுமதிக்கிறது, முழு IT சூழலின் உண்மையிலேயே விரிவான காட்சியை வழங்குகிறது.

தரவு மைய ADOT நிகழ்வை அடைந்தவுடன், Prometheus remote write நெறிமுறையைப் பயன்படுத்தி செயலாக்கப்பட்டு Amazon Managed Service for Prometheus-க்கு அனுப்பப்படுகிறது. இந்த படி சேகரிக்கப்பட்ட அனைத்து மெட்ரிக்குகளும் நேர-தொடர் தரவுக்கு உகந்த வடிவத்தில் சேமிக்கப்படுவதை உறுதி செய்கிறது, திறமையான வினவல் மற்றும் பகுப்பாய்வை இயக்குகிறது.

## நன்மைகள் மற்றும் கருத்தில் கொள்ள வேண்டியவை

இந்த கட்டமைப்பு பல முக்கிய நன்மைகளை வழங்குகிறது. பல்வேறு மூலங்களிலிருந்து மெட்ரிக்குகளின் மையப்படுத்தப்பட்ட காட்சியை வழங்குகிறது, சிக்கலான சூழல்களின் முழுமையான கண்காணிப்பை இயக்குகிறது. நிர்வகிக்கப்படும் சேவைகளின் பயன்பாடு குழுக்களின் செயல்பாட்டு சுமையைக் குறைக்கிறது, உள்கட்டமைப்பு பராமரிப்பை விட பகுப்பாய்வில் கவனம் செலுத்த உதவுகிறது. கூடுதலாக, கட்டமைப்பு அதிக அளவிடக்கூடியது, கண்காணிக்கப்படும் அமைப்புகளின் எண்ணிக்கை மற்றும் சேகரிக்கப்படும் மெட்ரிக்குகளின் அளவு இரண்டிலும் வளர்ச்சியைத் தாங்க முடியும்.

இருப்பினும், இந்த கட்டமைப்பை செயல்படுத்துவதில் கருத்தில் கொள்ள வேண்டியவையும் உள்ளன. தீர்வின் மையப்படுத்தப்பட்ட தன்மை மைய கணக்கில் உள்ள கண்காணிப்பு உள்கட்டமைப்பு முக்கியமானதாக மாறுகிறது, உயர் கிடைக்கும்தன்மை மற்றும் பேரழிவு மீட்புக்கான கவனமான திட்டமிடல் தேவை. கணக்குகளுக்கு இடையேயான தரவு பரிமாற்றம் மற்றும் நிர்வகிக்கப்படும் சேவைகளின் பயன்பாடு தொடர்பான செலவு தாக்கங்களும் இருக்கலாம், இவை பட்ஜெட் முடிவுகளில் கருத்தில் கொள்ளப்பட வேண்டும்.

பாதுகாப்பு மற்றொரு முக்கிய அம்சமாகும். பாதுகாப்பான குறுக்கு-கணக்கு மெட்ரிக் சேகரிப்பை அனுமதிக்க சரியான IAM பாத்திரங்கள் மற்றும் அனுமதிகள் அமைக்கப்பட வேண்டும். ஆன்-பிரமைஸ் அமைப்புகளுக்கு, கண்காணிப்பு தரவின் ஒருமைப்பாடு மற்றும் ரகசியத்தன்மையைப் பராமரிக்க பாதுகாப்பான மற்றும் அங்கீகரிக்கப்பட்ட HTTPS இணைப்புகளை உறுதி செய்வது மிக முக்கியமானது.

## முடிவு

இந்த மேம்பட்ட AWS கிளவுட் கண்காணிப்பு கட்டமைப்பு சிக்கலான, பல-கணக்கு மற்றும் கலப்பு உள்கட்டமைப்பு சூழல்களைக் கொண்ட நிறுவனங்களுக்கு வலுவான தீர்வை வழங்குகிறது. AWS நிர்வகிக்கப்படும் சேவைகள் மற்றும் OpenTelemetry மற்றும் Grafana போன்ற தொழில்துறை-தரமான கருவிகளைப் பயன்படுத்துவதன் மூலம், இது அளவிடக்கூடிய மற்றும் சக்திவாய்ந்த கண்காணிப்பு தீர்வை வழங்குகிறது. திறம்பட செயல்படுத்த கவனமான திட்டமிடல் மற்றும் மேலாண்மை தேவைப்பட்டாலும், விரிவான தெரிவுநிலை மற்றும் மையப்படுத்தப்பட்ட கண்காணிப்பின் நன்மைகள் நவீன கிளவுட்-நேட்டிவ் மற்றும் கலப்பு சூழல்களுக்கு மதிப்புள்ள அணுகுமுறையாக இதை மாற்றுகிறது.

கட்டமைப்பின் நெகிழ்வுத்தன்மை பல்வேறு நிறுவன தேவைகளுக்கு ஏற்ப மாற்றிக்கொள்ள அனுமதிக்கிறது மற்றும் கண்காணிப்பு தேவைகள் மாறும்போது வளர்ச்சியடையலாம். கிளவுட் சூழல்கள் சிக்கலான தன்மையில் தொடர்ந்து வளரும்போது, அனைத்து உள்கட்டமைப்பு கூறுகளிலும் செயல்பாட்டு சிறப்பைப் பராமரிக்கவும் உகந்த செயல்திறனை உறுதி செய்யவும் இத்தகைய மையப்படுத்தப்பட்ட மற்றும் விரிவான கண்காணிப்பு தீர்வு பெருகிய முறையில் முக்கியமானதாகிறது.
