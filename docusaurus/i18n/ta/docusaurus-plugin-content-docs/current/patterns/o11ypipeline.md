# ADOT Observability பைப்லைன்

Observability பைப்லைன் பல்வேறு மூலங்களிலிருந்து Observability தரவைச் சேகரிக்க, நிர்வகிக்க மற்றும் பகுப்பாய்வு செய்ய இணைந்து செயல்படும் பல கூறுகளைக் கொண்டுள்ளது.

## EKS கிளஸ்டர்

EKS (Elastic Kubernetes Service) கிளஸ்டர் Observability பைப்லைனின் முக்கிய கூறுகளை ஹோஸ்ட் செய்கிறது.

### ADOT Operator Helm Chart நிறுவுதல்

ADOT (AWS Distro for OpenTelemetry) Operator ஒரு Helm chart ஐப் பயன்படுத்தி நிறுவப்படுகிறது. இது Observability பைப்லைன் கூறுகளின் டிப்ளாய்மென்ட் மற்றும் உள்ளமைவை நிர்வகிக்கிறது.

### பயனர் உள்ளமைக்கப்பட்ட Collector

ஆபரேட்டரால் நிர்வகிக்கப்படும் பயனர் உள்ளமைக்கப்பட்ட கலெக்டர் பின்வரும் கூறுகளைக் கொண்டுள்ளது:

- Deployment ஆக Collector: கலெக்டர் ஒரு Kubernetes deployment ஆக டிப்ளாய் செய்யப்படுகிறது, இது உயர் கிடைக்கும்தன்மை மற்றும் அளவிடக்கூடியதை உறுதி செய்கிறது.
- Collector-0, Collector-1, Collector-2: உள்வரும் Observability தரவைக் கையாள பல கலெக்டர் நிகழ்வுகள் டிப்ளாய் செய்யப்படுகின்றன. பணிச்சுமையை விநியோகித்து நம்பகமான தரவு சேகரிப்பை உறுதி செய்ய அவை இணைந்து செயல்படுகின்றன.

### நிலையான தொகுதி

சேகரிக்கப்பட்ட Observability தரவைச் சேமிக்க நிலையான தொகுதி பயன்படுத்தப்படுகிறது. இது தரவு நீடித்த தன்மையை உறுதி செய்கிறது மற்றும் நீண்ட கால சேமிப்பு மற்றும் பகுப்பாய்வை அனுமதிக்கிறது.

### Kubernetes நோடு

Kubernetes நோடு அப்ளிகேஷன் பாட்கள் மற்றும் sidecar ஆக கலெக்டரை ஹோஸ்ட் செய்கிறது.

- அப்ளிகேஷன் கண்டெய்னர்: அப்ளிகேஷன் கண்டெய்னர் உண்மையான அப்ளிகேஷன் குறியீட்டை இயக்கி Observability தரவை உருவாக்குகிறது.
- Sidecar ஆக Collector: கலெக்டர் அப்ளிகேஷன் கண்டெய்னருடன் sidecar கண்டெய்னராக இயங்குகிறது. அப்ளிகேஷனால் உருவாக்கப்படும் Observability தரவைச் சேகரிக்கிறது.

## ஸ்கிரேப் இலக்குகள்

Observability பைப்லைன் பல்வேறு ஸ்கிரேப் இலக்குகளிலிருந்து தரவைச் சேகரிக்கிறது:

- ட்ரேஸ்கள்/மெட்ரிக்குகள் ஸ்கிரேப் செய்தல்: பைப்லைன் அப்ளிகேஷன் மற்றும் உள்கட்டமைப்பு கூறுகளிலிருந்து ட்ரேஸ்கள் மற்றும் மெட்ரிக்குகளை ஸ்கிரேப் செய்கிறது.

## AWS Prometheus Remote Write Exporter

சேகரிக்கப்பட்ட Observability தரவை AWS சேவைகளுக்கு ஏற்றுமதி செய்ய AWS Prometheus Remote Write Exporter பயன்படுத்தப்படுகிறது.

## AWS CloudWatch EMF Exporter

மெட்ரிக்குகளை AWS CloudWatch-க்கு ஏற்றுமதி செய்ய AWS CloudWatch EMF (Embedded Metric Format) Exporter பயன்படுத்தப்படுகிறது.

## AWS X-Ray Tracing Exporter

விநியோகிக்கப்பட்ட ட்ரேசிங் மற்றும் செயல்திறன் பகுப்பாய்வுக்காக ட்ரேசிங் தரவை AWS X-Ray-க்கு ஏற்றுமதி செய்ய AWS X-Ray Tracing Exporter பயன்படுத்தப்படுகிறது.

Observability பைப்லைன் ஸ்கிரேப் இலக்குகளிலிருந்து தரவைச் சேகரித்து, கலெக்டர்களைப் பயன்படுத்தி செயலாக்கி, மேலும் பகுப்பாய்வு மற்றும் காட்சிப்படுத்தலுக்காக பல்வேறு AWS சேவைகளுக்கு ஏற்றுமதி செய்கிறது.


## ADOT மூலம் மெட்ரிக்குகள் மற்றும் நுண்ணறிவுகளைச் சேகரித்தல்

1. **கருவியமைப்பு**: OpenTelemetry போலவே, ADOT உங்கள் அப்ளிகேஷன்கள் மற்றும் சேவைகளைக் கருவியமைக்க நூலகங்கள் மற்றும் SDK-களை வழங்குகிறது, மெட்ரிக்குகள், ட்ரேஸ்கள் மற்றும் லாக்குகள் போன்ற டெலிமெட்ரி தரவைக் கைப்பற்றுகிறது.

2. **மெட்ரிக்குகள் சேகரிப்பு**: ADOT AWS சேவை மெட்ரிக்குகள் உட்பட அமைப்பு மற்றும் அப்ளிகேஷன்-நிலை மெட்ரிக்குகளைச் சேகரிக்கவும் ஏற்றுமதி செய்யவும் ஆதரிக்கிறது, வள பயன்பாடு மற்றும் அப்ளிகேஷன் செயல்திறன் பற்றிய நுண்ணறிவுகளை வழங்குகிறது.

3. **விநியோகிக்கப்பட்ட ட்ரேசிங்**: ADOT AWS சேவைகள், கண்டெய்னர்கள் மற்றும் ஆன்-பிரமைஸ் சூழல்கள் முழுவதும் விநியோகிக்கப்பட்ட ட்ரேசிங்கை இயக்குகிறது, கோரிக்கைகள் மற்றும் செயல்பாடுகளை முழுமையாக ட்ரேஸ் செய்ய உங்களை அனுமதிக்கிறது.

4. **லாக்கிங்**: ADOT கட்டமைக்கப்பட்ட லாக்கிங்கிற்கான ஆதரவை உள்ளடக்கியது, விரிவான Observability-க்காக மற்ற டெலிமெட்ரி சிக்னல்களுடன் லாக் தரவைத் தொடர்புபடுத்துகிறது.

5. **AWS சேவை ஒருங்கிணைப்புகள்**: ADOT AWS X-Ray, AWS CloudWatch, Amazon Managed Service for Prometheus மற்றும் AWS Distro for OpenTelemetry Operator போன்ற AWS சேவைகளுடன் நெருக்கமான ஒருங்கிணைப்புகளை வழங்குகிறது, AWS சூழலுக்குள் தடையற்ற டெலிமெட்ரி சேகரிப்பு மற்றும் பகுப்பாய்வை இயக்குகிறது.

6. **தானியங்கி கருவியமைப்பு**: ADOT பிரபலமான ஃபிரேம்வொர்க்குகள் மற்றும் நூலகங்களுக்கான தானியங்கி கருவியமைப்பு திறன்களை வழங்குகிறது, ஏற்கனவே உள்ள அப்ளிகேஷன்களைக் கருவியமைக்கும் செயல்முறையை எளிமைப்படுத்துகிறது.

7. **தரவு செயலாக்கம் மற்றும் பகுப்பாய்வு**: ADOT ஆல் சேகரிக்கப்பட்ட டெலிமெட்ரி தரவை AWS X-Ray, Amazon Managed Service for Prometheus மற்றும் AWS CloudWatch போன்ற AWS Observability சேவைகளுக்கு ஏற்றுமதி செய்யலாம், AWS-நேட்டிவ் பகுப்பாய்வு மற்றும் காட்சிப்படுத்தல் கருவிகளைப் பயன்படுத்தலாம்.

## ADOT பயன்படுத்துவதன் நன்மைகள்

1. **AWS-நேட்டிவ் ஒருங்கிணைப்பு**: ADOT AWS சேவைகள் மற்றும் உள்கட்டமைப்புடன் தடையின்றி ஒருங்கிணைக்க வடிவமைக்கப்பட்டுள்ளது, AWS சூழலுக்குள் ஒருங்கிணைந்த Observability அனுபவத்தை வழங்குகிறது.

2. **செயல்திறன் மற்றும் அளவிடக்கூடியது**: ADOT பெரிய அளவிலான AWS சூழல்களில் திறமையான டெலிமெட்ரி சேகரிப்பு மற்றும் பகுப்பாய்வை இயக்க செயல்திறன் மற்றும் அளவிடக்கூடியதற்கு உகந்ததாக்கப்பட்டுள்ளது.

3. **பாதுகாப்பு மற்றும் இணக்கம்**: ADOT AWS பாதுகாப்பு சிறந்த நடைமுறைகளைப் பின்பற்றுகிறது மற்றும் பல்வேறு தொழில் தரங்களுடன் இணக்கமானது, பாதுகாப்பான மற்றும் இணக்கமான Observability நடைமுறைகளை உறுதி செய்கிறது.

4. **AWS ஆதரவு**: AWS-ஆதரிக்கப்படும் விநியோகமாக, ADOT AWS-ன் விரிவான ஆவணம், ஆதரவு சேனல்கள் மற்றும் OpenTelemetry திட்டத்திற்கான நீண்ட கால அர்ப்பணிப்பு ஆகியவற்றின் நன்மையைப் பெறுகிறது.

## OpenTelemetry மற்றும் ADOT இடையேயான வேறுபாடு

ADOT மற்றும் OpenTelemetry பல முக்கிய திறன்களைப் பகிர்ந்து கொண்டாலும், சில முக்கிய வேறுபாடுகள் உள்ளன:

1. **AWS ஒருங்கிணைப்பு**: ADOT குறிப்பாக AWS சூழல்களுக்காக வடிவமைக்கப்பட்டு AWS சேவைகளுடன் நெருக்கமான ஒருங்கிணைப்புகளை வழங்குகிறது, அதே நேரத்தில் OpenTelemetry ஒரு விற்பனையாளர்-நடுநிலை திட்டமாகும்.

2. **AWS உகந்தமாக்கல்**: ADOT AWS சூழல்களுக்குள் செயல்திறன், அளவிடக்கூடியது மற்றும் பாதுகாப்புக்கு உகந்ததாக்கப்பட்டுள்ளது, AWS-நேட்டிவ் சேவைகள் மற்றும் சிறந்த நடைமுறைகளைப் பயன்படுத்துகிறது.

3. **AWS ஆதரவு**: ADOT அதிகாரப்பூர்வ AWS ஆதரவு, ஆவணம் மற்றும் நீண்ட கால அர்ப்பணிப்பின் நன்மையைப் பெறுகிறது, அதே நேரத்தில் OpenTelemetry சமூக ஆதரவை நம்பியுள்ளது.

4. **AWS-குறிப்பிட்ட அம்சங்கள்**: ADOT AWS சேவைகளுக்கான AWS-குறிப்பிட்ட அம்சங்கள் மற்றும் தானியங்கி கருவியமைப்பை உள்ளடக்கியது, அதே நேரத்தில் OpenTelemetry பொது-நோக்க Observability-ல் கவனம் செலுத்துகிறது.

5. **விற்பனையாளர் நடுநிலை**: OpenTelemetry ஒரு விற்பனையாளர்-நடுநிலை திட்டமாகும், பல்வேறு Observability தளங்களுடன் ஒருங்கிணைப்பை அனுமதிக்கிறது, அதே நேரத்தில் ADOT முதன்மையாக AWS Observability சேவைகளில் கவனம் செலுத்துகிறது.

ADOT ஐப் பயன்படுத்துவதன் மூலம், நிறுவனங்கள் AWS சூழலுக்குள் விரிவான Observability-ஐ அடையலாம், AWS-நேட்டிவ் ஒருங்கிணைப்புகள், உகந்த செயல்திறன் மற்றும் AWS ஆதரவின் நன்மையைப் பெறலாம், அதே நேரத்தில் OpenTelemetry திறன்கள் மற்றும் சமூக பங்களிப்புகளைப் பயன்படுத்தும் நெகிழ்வுத்தன்மையைப் பராமரிக்கலாம்.
