# கிளவுட் பொறியாளர்

சிக்கலான AWS உள்கட்டமைப்பை நிர்வகிக்கும் கிளவுட் பொறியாளர்களாக, நம்பகமான மற்றும் திறமையான செயல்பாடுகளை பராமரிப்பதற்கு Observability இன்றியமையாதது. இன்றைய மைக்ரோசர்வீஸ்கள், கண்டெய்னர்கள் மற்றும் சர்வர்லெஸ் கட்டமைப்புகளின் உலகில், நமது அமைப்புகளில் தெளிவான தெரிவுநிலை வெற்றிக்கு முக்கியமானது.

இந்த வழிகாட்டி கிளவுட் பொறியாளர்களுக்கான முக்கிய Observability சிறந்த நடைமுறைகளை ஆராய்கிறது, AWS சூழல்களை பெரிய அளவில் கண்காணிக்க, சரிசெய்ய மற்றும் மேம்படுத்துவதற்கான நடைமுறை உத்திகளில் கவனம் செலுத்துகிறது.

- **டேக்குகளை திறம்பட பயன்படுத்துங்கள்**: செலவுகளை கண்காணிக்கவும் ஒதுக்கவும் டேக்கிங் மிகவும் சக்திவாய்ந்த வழிகளில் ஒன்றாகும். ஆரம்பத்திலேயே செயல்படுத்துவது எதிர்காலத்தில் குறிப்பிடத்தக்க நேரத்தை மிச்சப்படுத்தும்.

| **மேம்பட்ட** | [மாற்ற கண்காணிப்பு](https://aws-observability.github.io/observability-best-practices/recipes/anomaly-detection) | உள் மற்றும் வெளி மூலங்களிலிருந்து உங்கள் பணிச்சுமைகளில் திடீர் மாற்றங்களை கண்காணிக்கவும் | **விரைவான வெற்றி:** எதிர்பாராத உள்நுழைவு வடிவங்கள் அல்லது பயனர் செயல்பாடுகளுக்கான எச்சரிக்கைகளை அமைக்கவும் | **பொதுவான தவறு:** நிலையான அலாரம் வரம்பில் மட்டும் சார்ந்திருத்தல் |

### பரிந்துரைகள்:
- **பாதுகாப்புக்கு முன்னுரிமை அளிக்கவும்**: பாதுகாப்பு ஒருபோதும் பிந்தைய சிந்தனையாக இருக்கக்கூடாது. அடிப்படை கண்காணிப்புடன் தொடங்கி, மேலும் அதிநவீன கட்டமைப்புகளுக்கு முன்னேறுங்கள்.
- **எச்சரிக்கைகளை தானியக்கமாக்குங்கள்**: அசாதாரண நடவடிக்கைகளுக்கான தானியங்கி எச்சரிக்கைகளை அமைப்பது, சாத்தியமான அச்சுறுத்தல்கள் தீவிரமடைவதற்கு முன் கண்டறிய உதவுகிறது.

## சர்வர்லெஸ் பணிச்சுமை கண்காணிப்பு ⚡

**இலக்கு:** நம்பகத்தன்மை மற்றும் செலவு திறனை உறுதிசெய்ய சர்வர்லெஸ் பயன்பாடுகளை திறம்பட கண்காணித்து மேம்படுத்தவும்.

| நிலை | கூறு | விளக்கம் | குறிப்புகள் & எடுத்துக்காட்டுகள் | கூடுதல் குறிப்புகள் |
|-------|-----------|-------------|-----------------|------------------|
| **அடிப்படை** | [Lambda Function சிறந்த நடைமுறைகள்](https://aws-observability.github.io/observability-best-practices/guides/serverless/aws-native/lambda-based-observability) | முக்கிய Lambda மெட்ரிக்குகள் மற்றும் இயக்க புள்ளிவிவரங்களை கண்காணிக்கவும் | **எடுத்துக்காட்டு:** invocations, duration மற்றும் error rates-ஐ கண்காணிக்கவும்  
**விரைவான வெற்றி:** Lambda insights-க்கான CloudWatch டாஷ்போர்டுகளை அமைக்கவும் | **நிபுணர் குறிப்பு:** செலவுகளை மேம்படுத்த cold starts மற்றும் memory utilization-ஐ கண்காணிக்கவும் |
| **இடைநிலை** | [Event Source கண்காணிப்பு](https://docs.aws.amazon.com/lambda/latest/dg/monitoring-metrics.html) | நிகழ்வு மூலங்கள் மற்றும் ஒருங்கிணைப்புகளின் செயல்திறனை கண்காணிக்கவும் | **எடுத்துக்காட்டு:** SQS queue depth, API Gateway latency-ஐ கண்காணிக்கவும்  
**விரைவான வெற்றி:** தோல்வியடைந்த நிகழ்வுகளுக்கு dead-letter queues-ஐ அமைக்கவும் | **தெரியுமா?** சரியான event source கண்காணிப்பு அடுக்கு தோல்விகளை தடுக்கும் |
| **மேம்பட்ட** | [சுருக்கமான நுண்ணறிவுகளை வழங்குதல்](https://docs.aws.amazon.com/xray/latest/devguide/xray-services-lambda.html) | உங்கள் சர்வர்லெஸ் மற்றும் கண்டெய்னரைஸ் செய்யப்பட்ட பயன்பாடுகளின் பணிச்சுமை செயல்திறன், வள பயன்பாடு மற்றும் செயல்பாட்டு வடிவங்கள் பற்றிய தானியங்கி, விரிவான பகுப்பாய்வுகளை பெற CloudWatch-ன் சிறப்பு insight கருவிகளை பயன்படுத்துங்கள். | **எடுத்துக்காட்டு:** [Lambda Insights](https://aws-observability.github.io/observability-best-practices/guides/serverless/aws-native/lambda-based-observability#use-cloudwatch-lambda-insights-to-monitor-system-level-metrics)

### பரிந்துரைகள்:
- **கட்டமைக்கப்பட்ட லாகிங்கை செயல்படுத்துங்கள்**: சிறந்த தேடல் திறனுக்கு நிலையான JSON லாகிங் வடிவமைப்பைப் பயன்படுத்துங்கள்
- **ஒரே நேர இயக்க வரம்புகளை கண்காணிக்கவும்**: throttling-ஐ தடுக்க function concurrency-ஐ கண்காணிக்கவும்
- **செலவு மேம்படுத்தல்**: செலவு ஒதுக்கீடு டேக்குகளை அமைத்து ஒவ்வொரு function-க்கான செலவுகளை கண்காணிக்கவும்
