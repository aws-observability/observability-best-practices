# Internet Monitor

:::warning
	இந்த ஆவணம் எழுதப்படும் நேரத்தில், [Internet Monitor](https://aws.amazon.com/blogs/aws/cloudwatch-internet-monitor-end-to-end-visibility-into-internet-performance-for-your-applications/) CloudWatch கன்சோலில் **முன்னோட்டமாக** கிடைக்கிறது. பொது வெளியீட்டின் அம்ச வரம்பு இன்று நீங்கள் அனுபவிப்பதிலிருந்து மாறலாம்.
:::
[உங்கள் பணிச்சுமையின் அனைத்து அடுக்குகளிலிருந்தும் டெலிமெட்ரி சேகரிப்பது](../guides/index.md#collect-telemetry-from-all-tiers-of-your-workload) சிறந்த நடைமுறையாகும், இது சில நேரங்களில் சவாலாக இருக்கலாம். ஆனால் உங்கள் பணிச்சுமையின் அடுக்குகள் என்ன? சிலருக்கு இது வலை, பயன்பாடு மற்றும் தரவுத்தள சர்வர்களாக இருக்கலாம். மற்றவர்கள் தங்கள் பணிச்சுமையை முன்பக்கம் மற்றும் பின்பக்கம் என்று பார்க்கலாம். வலை பயன்பாடுகளை இயக்குபவர்கள் இறுதிப் பயனர்கள் அனுபவிக்கும் இந்த ஆப்களின் ஆரோக்கியத்தைக் கவனிக்க [Real User Monitoring](./rum.md)(RUM) பயன்படுத்தலாம்.

ஆனால் கிளையன்ட் மற்றும் தரவு மையம் அல்லது கிளவுட் சேவை வழங்குநருக்கு இடையிலான போக்குவரத்து பற்றி என்ன? வலைப் பக்கங்களாக வழங்கப்படாத, எனவே RUM பயன்படுத்த முடியாத பயன்பாடுகளுக்கு என்ன?

![இணையத்தைக் கடக்கும் பயன்பாடுகளிலிருந்து நெட்வொர்க் டெலிமெட்ரி](../images/internet_monitor.png)

Internet Monitor நெட்வொர்க்கிங் நிலையில் செயல்படுகிறது மற்றும் கவனிக்கப்பட்ட போக்குவரத்தின் ஆரோக்கியத்தை மதிப்பீடு செய்கிறது, அறியப்பட்ட இணைய சிக்கல்கள் பற்றிய [AWS-ன் ஏற்கனவே உள்ள அறிவுடன்](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-IM-inside-internet-monitor.html) தொடர்புபடுத்துகிறது. சுருக்கமாக, ஒரு இணைய சேவை வழங்குநருக்கு (ISP) செயல்திறன் அல்லது கிடைக்கும் தன்மை சிக்கல் இருந்தால் **மற்றும்** உங்கள் பயன்பாட்டின் போக்குவரத்து கிளையன்ட்/சர்வர் தொடர்பாடலுக்கு இந்த ISP-ஐ பயன்படுத்தினால், Internet Monitor உங்கள் பணிச்சுமைக்கான இந்த தாக்கத்தை முன்கூட்டியே தெரிவிக்கலாம். கூடுதலாக, நீங்கள் தேர்ந்தெடுத்த ஹோஸ்டிங் ரீஜன் மற்றும் [CloudFront](https://aws.amazon.com/cloudfront/)-ஐ Content Delivery Network-ஆகப் பயன்படுத்துவதன் அடிப்படையில் பரிந்துரைகளை வழங்கலாம்[^1].

:::tip
	Internet Monitor உங்கள் பணிச்சுமைகள் கடக்கும் நெட்வொர்க்குகளின் போக்குவரத்தை மட்டுமே மதிப்பீடு செய்கிறது. எடுத்துக்காட்டாக, வேறொரு நாட்டின் ISP பாதிக்கப்பட்டாலும், உங்கள் பயனர்கள் அந்த carrier-ஐப் பயன்படுத்தவில்லை என்றால், அந்த சிக்கலில் உங்களுக்கு தெரிவுநிலை இருக்காது.
:::

## இணையத்தைக் கடக்கும் பயன்பாடுகளுக்கு monitor-களை உருவாக்குதல்

Internet Monitor செயல்படும் விதம் பாதிக்கப்பட்ட ISP-களிலிருந்து உங்கள் CloudFront விநியோகங்களுக்கு அல்லது உங்கள் VPC-களுக்கு வரும் போக்குவரத்தைக் கவனிப்பதாகும். இது உங்கள் கட்டுப்பாட்டிற்கு வெளியே உள்ள நெட்வொர்க் சிக்கல்களின் விளைவாக எழும் வணிக சிக்கல்களை ஈடுசெய்ய உதவும் பயன்பாட்டு நடத்தை, ரூட்டிங் அல்லது பயனர் அறிவிப்பு பற்றிய முடிவுகளை எடுக்க உதவுகிறது.

![உங்கள் பணிச்சுமை மற்றும் ISP சிக்கல்களின் குறுக்குவெட்டு](../images/internet_monitor_2.png)

:::info
	இணையத்தைக் கடக்கும் போக்குவரத்தைக் கவனிக்கும் monitor-களை மட்டுமே உருவாக்கவும். தனிப்பட்ட நெட்வொர்க்கில் ([RFC1918](https://www.arin.net/reference/research/statistics/address_filters/)) இரண்டு ஹோஸ்ட்களுக்கு இடையிலான தனிப்பட்ட போக்குவரத்து Internet Monitor பயன்படுத்தி கண்காணிக்க முடியாது.
:::
:::info
	பொருந்தும் இடங்களில் மொபைல் பயன்பாடுகளின் போக்குவரத்திற்கு முன்னுரிமை அளிக்கவும். வழங்குநர்களுக்கு இடையே ரோமிங் செய்யும் அல்லது தொலைதூர புவியியல் இடங்களில் உள்ள வாடிக்கையாளர்கள் நீங்கள் அறிந்திருக்க வேண்டிய வேறுபட்ட அல்லது எதிர்பாராத அனுபவங்களைப் பெறலாம்.
:::
## EventBridge மற்றும் CloudWatch மூலம் செயல்களை இயக்குதல்

கவனிக்கப்பட்ட சிக்கல்கள் `aws.internetmonitor` என அடையாளம் காணப்பட்ட source-ஐக் கொண்ட [ஸ்கீமா](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-IM-EventBridge-integration.html) பயன்படுத்தி [EventBridge](https://aws.amazon.com/eventbridge/) மூலம் வெளியிடப்படும். EventBridge-ஐ உங்கள் டிக்கெட் மேலாண்மை அமைப்பில் தானாக சிக்கல்களை உருவாக்கவும், உங்கள் ஆதரவு குழுக்களுக்கு பேஜ் செய்யவும், அல்லது சில சூழ்நிலைகளைத் தணிக்க உங்கள் பணிச்சுமையை மாற்றும் தானியக்கத்தைத் தூண்டவும் பயன்படுத்தலாம்.

```json
{
  "source": ["aws.internetmonitor"]
}
```

அதேபோல், கவனிக்கப்பட்ட நகரங்கள், நாடுகள், பெருநகரங்கள் மற்றும் துணைப்பிரிவுகளுக்கான போக்குவரத்தின் விரிவான விவரங்கள் [CloudWatch Logs](./logs/index.md)-ல் கிடைக்கின்றன. இது பாதிக்கப்பட்ட வாடிக்கையாளர்களுக்கு அவர்களுக்கு உள்ளூர் சிக்கல்கள் பற்றி முன்கூட்டியே அறிவிக்கக்கூடிய மிகவும் இலக்கு நிர்ணயிக்கப்பட்ட செயல்களை உருவாக்க உதவுகிறது. ஒரு தனி வழங்குநர் பற்றிய நாடு-நிலை கவனிப்பின் எடுத்துக்காட்டு இங்கே:

```json
{
    "version": 1,
    "timestamp": 1669659900,
    "clientLocation": {
        "latitude": 0,
        "longitude": 0,
        "country": "United States",
        "subdivision": "",
        "metro": "",
        "city": "",
        "countryCode": "US",
        "subdivisionCode": "",
        "asn": 00000,
        "networkName": "MY-AWESOME-ASN"
    },
    "serviceLocation": "us-east-1",
    "percentageOfTotalTraffic": 0.36,
    "bytesIn": 23,
    "bytesOut": 0,
    "clientConnectionCount": 0,
    "internetHealth": {
        "availability": {
            "experienceScore": 100,
            "percentageOfTotalTrafficImpacted": 0,
            "percentageOfClientLocationImpacted": 0
        },
        "performance": {
            "experienceScore": 100,
            "percentageOfTotalTrafficImpacted": 0,
            "percentageOfClientLocationImpacted": 0,
            "roundTripTime": {
                "p50": 71,
                "p90": 72,
                "p95": 73
            }
        }
    },
    "trafficInsights": {
        "timeToFirstByte": {
            "currentExperience": {
                "serviceName": "VPC",
                "serviceLocation": "us-east-1",
                "value": 48
            },
            "ec2": {
                "serviceName": "EC2",
                "serviceLocation": "us-east-1",
                "value": 48
            }
        }
    }
}
```

:::info
	`percentageOfTotalTraffic` போன்ற மதிப்புகள் வாடிக்கையாளர்கள் உங்கள் பணிச்சுமைகளை எங்கிருந்து அணுகுகிறார்கள் என்பதைப் பற்றிய சக்திவாய்ந்த நுண்ணறிவுகளை வெளிப்படுத்தலாம், மேலும் மேம்பட்ட பகுப்பாய்வுக்குப் பயன்படுத்தலாம்.
:::

:::warning
	Internet Monitor உருவாக்கிய லாக் குழுக்கள் இயல்பு தக்கவைப்பு காலத்தை *காலாவதியாகாது* என்று அமைக்கும். AWS உங்கள் சம்மதமின்றி தரவை நீக்காது, எனவே உங்கள் தேவைகளுக்கு ஏற்ற தக்கவைப்பு காலத்தை அமைக்க மறக்காதீர்கள்.
:::
:::info
	ஒவ்வொரு monitor-ம் குறைந்தது 10 தனித்தனி CloudWatch மெட்ரிக்குகளை உருவாக்கும். இவை வேறு எந்த செயல்பாட்டு மெட்ரிக்கிலும் செய்வது போல் [அலாரங்களை](./alarms.md) உருவாக்கப் பயன்படுத்தப்பட வேண்டும்.
:::
## போக்குவரத்து மேம்படுத்தல் பரிந்துரைகளைப் பயன்படுத்துதல்

Internet Monitor-ல் போக்குவரத்து மேம்படுத்தல் பரிந்துரைகள் அம்சம் உள்ளது, இது சிறந்த வாடிக்கையாளர் அனுபவங்களைப் பெற உங்கள் பணிச்சுமைகளை எங்கு வைப்பது சிறந்தது என்று ஆலோசனை வழங்கலாம். உலகளாவிய பணிச்சுமைகள் அல்லது உலகளாவிய வாடிக்கையாளர்கள் கொண்ட பணிச்சுமைகளுக்கு இந்த அம்சம் குறிப்பாக மதிப்புமிக்கது.

![Internet Monitor கன்சோல்](../images/internet_monitor_3.png)

:::info
	போக்குவரத்து மேம்படுத்தல் பரிந்துரைகள் பார்வையில் தற்போதைய, கணிக்கப்பட்ட மற்றும் குறைந்த time-to-first-byte (TTFB) மதிப்புகளில் கவனம் செலுத்துங்கள், ஏனெனில் இவை மற்றபடி கவனிக்கக் கடினமான, சாத்தியமான மோசமான இறுதிப் பயனர் அனுபவங்களைக் குறிக்கலாம்.
:::
[^1]: இந்த புதிய அம்சம் பற்றிய எங்கள் வெளியீட்டு வலைப்பதிவுக்கு [https://aws.amazon.com/blogs/aws/cloudwatch-internet-monitor-end-to-end-visibility-into-internet-performance-for-your-applications/](https://aws.amazon.com/blogs/aws/cloudwatch-internet-monitor-end-to-end-visibility-into-internet-performance-for-your-applications/) பாருங்கள்.
