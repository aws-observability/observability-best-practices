# Service Level Objectives (SLOs)

அதிக கிடைக்கும் தன்மை மற்றும் மீள்தன்மை கொண்ட பயன்பாடுகள் உங்கள் நிறுவனத்திற்கு செயலூக்கமான வணிக இயக்கியா**?**  
பதில் '**ஆம்**' எனில், தொடர்ந்து படியுங்கள்.

தோல்விகள் தவிர்க்க முடியாதவை, காலப்போக்கில் எல்லாமே இறுதியில் தோல்வியடையும்! அளவிடக்கூடிய பயன்பாடுகளை உருவாக்கும்போது இது மிகவும் முக்கியமான பாடமாகிறது. இங்குதான் SLO-களின் முக்கியத்துவம் வெளிப்படுகிறது.

SLO-கள் முக்கியமான இறுதிப் பயனர் பயணங்களின் அடிப்படையில் சேவை கிடைக்கும் தன்மைக்கான ஒப்புக்கொள்ளப்பட்ட இலக்கை அளவிடுகின்றன. ஒப்புக்கொள்ளப்பட்ட இலக்கு உங்கள் வாடிக்கையாளர் / இறுதிப் பயனருக்கு முக்கியமானவற்றைச் சுற்றி வடிவமைக்கப்பட வேண்டும். இத்தகைய மீள்தன்மையான சுற்றுச்சூழல் அமைப்பை உருவாக்க, செயல்திறனை புறநிலையாக அளவிடவும், அர்த்தமுள்ள, யதார்த்தமான மற்றும் செயல்படக்கூடிய SLO-களைப் பயன்படுத்தி நம்பகத்தன்மையை துல்லியமாக அறிக்கையிடவும் வேண்டும். இப்போது, முக்கிய சேவை நிலை சொற்களை அறிந்துகொள்வோம்.

## சேவை நிலை சொற்கள்

- SLI என்பது சேவை நிலை குறிகாட்டி (Service Level Indicator): வழங்கப்படும் சேவை நிலையின் குறிப்பிட்ட அம்சத்தின் கவனமாக வரையறுக்கப்பட்ட அளவு அளவீடு.

- SLO என்பது சேவை நிலை குறிக்கோள் (Service Level Objective): SLI-ஆல் அளவிடப்படும் சேவை நிலைக்கான இலக்கு மதிப்பு அல்லது மதிப்பு வரம்பு, ஒரு குறிப்பிட்ட காலத்தில் அளவிடப்படுகிறது.

- SLA என்பது சேவை நிலை ஒப்பந்தம் (Service Level Agreement): SLO-களை நிறைவேற்றாததன் விளைவுகளை உள்ளடக்கிய உங்கள் வாடிக்கையாளர்களுடனான ஒப்பந்தம்.

கீழ்க்கண்ட வரைபடம் SLA என்பது ஒரு 'வாக்குறுதி/ஒப்பந்தம்', SLO என்பது ஒரு 'குறிக்கோள்/இலக்கு மதிப்பு', மற்றும் SLI என்பது 'சேவை எவ்வாறு செயல்பட்டது?' என்பதன் அளவீடு என்பதை விளக்குகிறது.

### இவை அனைத்தையும் கண்காணிக்க ஒரு AWS கருவி உள்ளதா?

பதில் '**ஆம்**'!

[Amazon CloudWatch Application Signals](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Application-Monitoring-Sections.html) என்பது AWS-ல் பயன்பாடுகளை எளிதாக தானாக கருவியமைத்து இயக்க உதவும் புதிய திறன் ஆகும். Application Signals உங்கள் AWS பயன்பாடுகளை கருவியமைத்து, பயன்பாட்டின் ஆரோக்கியத்தைக் கண்காணிக்கவும், வணிக குறிக்கோள்களுக்கு எதிரான செயல்திறனைக் கண்காணிக்கவும் உதவுகிறது. Application Signals உங்கள் பயன்பாடுகள், சேவைகள் மற்றும் சார்புகளின் ஒருங்கிணைந்த, பயன்பாடு-மையமான பார்வையை வழங்குகிறது, மேலும் பயன்பாட்டு ஆரோக்கியத்தைக் கண்காணிக்கவும் வகைப்படுத்தவும் உதவுகிறது. Application Signals Amazon EKS, Amazon ECS மற்றும் Amazon EC2-ல் ஆதரிக்கப்பட்டு சோதிக்கப்பட்டுள்ளது, இதை எழுதும் நேரத்தில் Java பயன்பாடுகளை மட்டுமே ஆதரிக்கிறது!

Application Signals உங்கள் முக்கிய செயல்திறன் மெட்ரிக்குகளில் SLO-களை அமைக்க உதவுகிறது. உங்கள் முக்கிய வணிக செயல்பாடுகளுக்கான சேவைகளுக்கு சேவை நிலை குறிக்கோள்களை உருவாக்க Application Signals-ஐப் பயன்படுத்தலாம். இந்த சேவைகளில் SLO-களை உருவாக்குவதன் மூலம், SLO டாஷ்போர்டில் அவற்றைக் கண்காணிக்கலாம், உங்கள் மிக முக்கியமான செயல்பாடுகளின் ஒரு பார்வையில் புரிந்துகொள்ளக்கூடிய காட்சியை வழங்குகிறது. மூல காரணத்தை விரைவாகக் கண்டறிய, Application Signals பயன்பாட்டு செயல்திறனின் விரிவான பார்வையை வழங்குகிறது, முக்கிய API-கள் மற்றும் பயனர் தொடர்புகளைக் கண்காணிக்கும் CloudWatch Synthetics மற்றும் உண்மையான பயனர் செயல்திறனைக் கண்காணிக்கும் CloudWatch RUM ஆகியவற்றின் கூடுதல் செயல்திறன் சிக்னல்களை ஒருங்கிணைக்கிறது.

Application Signals கண்டறியும் ஒவ்வொரு சேவை மற்றும் செயல்பாட்டிற்கும் தாமதம் மற்றும் கிடைக்கும் தன்மை மெட்ரிக்குகளை தானாகச் சேகரிக்கிறது, இந்த மெட்ரிக்குகள் SLI-களாகப் பயன்படுத்த பெரும்பாலும் சிறந்தவை. அதே நேரத்தில், Application Signals எந்த CloudWatch மெட்ரிக் அல்லது மெட்ரிக் வெளிப்பாட்டையும் SLI-ஆகப் பயன்படுத்தும் நெகிழ்வுத்தன்மையை வழங்குகிறது!

Application Signals பயன்பாட்டு செயல்திறனுக்கான சிறந்த நடைமுறைகளின் அடிப்படையில் பயன்பாடுகளை தானாக கருவியமைக்கிறது மற்றும் Amazon EKS-ல் இயங்கும் பயன்பாடுகளுக்கான மெட்ரிக்குகள், ட்ரேஸ்கள், லாக்குகள், Real User Monitoring மற்றும் Synthetic Monitoring முழுவதும் டெலிமெட்ரியை தொடர்புபடுத்துகிறது. மேலும் விவரங்களுக்கு இந்த [வலைப்பதிவைப்](https://aws.amazon.com/blogs/aws/amazon-cloudwatch-application-signals-for-automatic-instrumentation-of-your-applications-preview/) படிக்கவும்.

ஒரு சேவையின் நம்பகத்தன்மையைக் கண்காணிக்க CloudWatch Application Signals-ல் SLO-ஐ எவ்வாறு அமைப்பது என்பதை அறிய இந்த [வலைப்பதிவைப்](https://aws.amazon.com/blogs/mt/how-to-monitor-application-health-using-slos-with-amazon-cloudwatch-application-signals/) பாருங்கள்.

Observability என்பது நம்பகமான சேவையை நிறுவுவதற்கான அடிப்படை உறுப்பு ஆகும், இதன் மூலம் உங்கள் நிறுவனம் பெரிய அளவில் திறம்பட செயல்படும் பாதையில் நன்றாக வைக்கப்படும். [Amazon CloudWatch Application Signals](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Application-Monitoring-Sections.html) அந்த குறிக்கோளை அடைய உங்களுக்கு உதவும் சிறந்த கருவியாக இருக்கும் என்று நாங்கள் நம்புகிறோம்.
