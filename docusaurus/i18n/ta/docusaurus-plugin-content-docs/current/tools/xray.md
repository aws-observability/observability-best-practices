# AWS X-Ray
 
## மாதிரியெடுப்பு விதிகள்

X-Ray பயன்படுத்தி மாதிரியெடுப்பு விதிகளை AWS Console, உள்ளூர் உள்ளமைவு கோப்பு அல்லது இரண்டிலும் உள்ளமைக்கலாம். உள்ளூர் உள்ளமைவு கன்சோலில் அமைக்கப்பட்டவற்றை மேலெழுதும்.

:::info
	முடிந்தவரை X-Ray கன்சோல், API அல்லது CloudFormation-ஐப் பயன்படுத்தவும். இது இயக்க நேரத்தில் பயன்பாட்டின் மாதிரியெடுப்பு நடத்தையை மாற்ற உதவுகிறது.
:::
இந்த ஒவ்வொரு அளவுகோல்களுக்கும் தனித்தனியாக மாதிரி விகிதங்களை அமைக்கலாம்:

* சேவை பெயர் (எ.கா., billing, payments)
* சேவை வகை (எ.கா., EC2, Container)
* HTTP முறை
* URL பாதை
* Resource ARN
* ஹோஸ்ட் (எ.கா., www.example.com)

சிறந்த நடைமுறை என்னவென்றால், சிக்கல்களைக் கண்டறிந்து செயல்திறன் சுயவிவரங்களைப் புரிந்துகொள்ள போதுமான தரவைச் சேகரிக்கும் அதே நேரத்தில், நிர்வகிக்க முடியாத அளவுக்கு அதிக தரவைச் சேகரிக்காத மாதிரி விகிதத்தை அமைப்பதாகும். எடுத்துக்காட்டாக, இறங்கு பக்கத்திற்கான போக்குவரத்தின் 1%-ஐ மாதிரியாக எடுப்பதும், ஆனால் கட்டணப் பக்க கோரிக்கைகளின் 10%-ஐ மாதிரியாக எடுப்பதும், வலுவான Observability நடைமுறையுடன் நன்றாக ஒத்திருக்கும்.

சில பரிவர்த்தனைகளின் 100%-ஐ கைப்பற்ற விரும்பலாம். ஆனால் ட்ரேஸ்கள் உங்கள் பணிச்சுமைக்கான அணுகலின் தடயவியல் தணிக்கைக்காக நோக்கமானவை அல்ல என்பதால் கவனமாக இருங்கள்!

:::warning
	ட்ரேஸ்கள் தணிக்கை அல்லது தடயவியல் பகுப்பாய்வுக்காகப் பயன்படுத்தப்பட நோக்கமானவை அல்ல என்பதால், 100% மாதிரி விகிதங்களைத் தவிர்க்கவும். இது X-Ray (இயல்பாக UDP எமிட்டர் பயன்படுத்தும்) ஒரு பரிவர்த்தனை ட்ரேஸையும் இழக்காது என்ற தவறான எதிர்பார்ப்பை உருவாக்கலாம்.
:::
ஒரு விதியாக, பரிவர்த்தனை ட்ரேஸ்களைக் கைப்பற்றுவது உங்கள் ஊழியர்கள் அல்லது உங்கள் AWS பில்லில் பெரும் சுமையை ஏற்படுத்தக்கூடாது. உங்கள் பணிச்சுமை வெளியிடும் தரவின் அளவைக் கற்றுக்கொள்ளும்போது உங்கள் சூழலில் ட்ரேஸ்களை மெதுவாகச் சேர்க்கவும்.

:::info
	இயல்பாக, X-Ray SDK ஒவ்வொரு வினாடியும் முதல் கோரிக்கையையும், கூடுதல் கோரிக்கைகளின் ஐந்து சதவிகிதத்தையும் பதிவு செய்கிறது.
	எப்போதும் நீங்கள் தாங்கக்கூடிய reservoir அளவை அமைக்கவும். reservoir அளவு நீங்கள் கைப்பற்றும் வினாடிக்கான அதிகபட்ச கோரிக்கைகளின் எண்ணிக்கையைத் தீர்மானிக்கிறது. இது தீங்கிழைக்கும் தாக்குதல், வேண்டாத கட்டணங்கள் மற்றும் உள்ளமைவு பிழைகளிலிருந்து உங்களைப் பாதுகாக்கிறது.
:::
## Daemon உள்ளமைவு

X-Ray daemon பகுப்பாய்வுக்காக X-Ray dataplane-க்கு டெலிமெட்ரி அனுப்பும் முயற்சியை இறக்கி வைக்க நோக்கமானது. எனவே, மூலப் பயன்பாடு இயங்கும் சர்வர், கண்டெய்னர் அல்லது நிகழ்வில் அதிக வளங்களை உபயோகிக்கக்கூடாது.

:::info
	சிறந்த நடைமுறை என்னவென்றால் X-Ray daemon-ஐ மற்றொரு நிகழ்வு அல்லது கண்டெய்னரில் இயக்குவது, இதன் மூலம் கவலைகளின் பிரிவை நடைமுறைப்படுத்தி உங்கள் மூல அமைப்பு தடையின்றி இயங்க அனுமதிக்கிறது.
:::

:::info
	Kubernetes போன்ற கண்டெய்னர் ஆர்கெஸ்ட்ரேஷன் முறையில், உங்கள் X-Ray daemon-ஐ sidecar-ஆக இயக்குவது பொதுவான நடைமுறை.
:::
daemon பாதுகாப்பான இயல்பு அமைப்புகளைக் கொண்டுள்ளது மற்றும் பெரும்பாலான சந்தர்ப்பங்களில் கூடுதல் உள்ளமைவு இல்லாமல் EC2, ECS, EKS அல்லது Fargate சூழல்களில் இயங்கலாம். இருப்பினும், கலப்பு மற்றும் பிற கிளவுட் சூழல்களில், உங்கள் தொலைதூர சூழல்களை ஒருங்கிணைக்க Direct Connect அல்லது VPN பயன்படுத்தினால், `Endpoint`-ஐ [VPC எண்ட்பாயிண்ட்](https://docs.aws.amazon.com/vpc/latest/privatelink/concepts.html)-க்கு ஏற்ப சரிசெய்ய விரும்பலாம்.

:::tip
	X-Ray daemon-ஐ மூலப் பயன்பாட்டுடன் ஒரே நிகழ்வு அல்லது மெய்நிகர் இயந்திரத்தில் இயக்க வேண்டுமென்றால், X-Ray நீங்கள் தாங்கக்கூடியதை விட அதிக கணினி வளங்களை உபயோகிக்காமல் இருக்க `TotalBufferSizeMB` அமைப்பை உள்ளமைக்கவும்.
:::
## குறிப்புரைகள்

AWS X-Ray உங்கள் ட்ரேஸ்களுடன் தன்னிச்சையான மெட்டாடேட்டாவை அனுப்புவதை ஆதரிக்கிறது. இவை *குறிப்புரைகள்* என அழைக்கப்படுகின்றன. இவை உங்கள் ட்ரேஸ்களை தருக்கரீதியாக குழுவாக்க உதவும் சக்தி வாய்ந்த அம்சமாகும். குறிப்புரைகள் குறியீட்டு முறையிலும் வைக்கப்படுகின்றன, ஒரு தனி நிறுவனத்துடன் தொடர்புடைய ட்ரேஸ்களை எளிதாகக் கண்டறிய உதவுகிறது.

X-Ray-க்கான தானியங்கி கருவியமைப்பு SDK-களைப் பயன்படுத்தும்போது, குறிப்புரைகள் தானாக தோன்றாமல் இருக்கலாம். அவற்றை உங்கள் குறியீட்டில் சேர்க்க வேண்டும், இது உங்கள் ட்ரேஸ்களை பெரிதும் செறிவூட்டி, X-Ray Insights உருவாக்குவதற்கும், உங்கள் குறிப்புரைகளின் அடிப்படையிலான மெட்ரிக்குகள், உங்கள் கணினி நடத்தையிலிருந்து அலாரங்கள் மற்றும் அசாதாரண கண்டறிதல் மாதிரிகள் உருவாக்குவதற்கும், உங்கள் பயனர்களை பாதிக்கும் ஒரு கூறு கண்டறியப்படும்போது டிக்கெட் உருவாக்கம் மற்றும் தானியங்கி தீர்வுக்கான வழிகளை உருவாக்குகிறது.

:::info
	உங்கள் சூழலில் தரவு ஓட்டத்தைப் புரிந்துகொள்ள குறிப்புரைகளைப் பயன்படுத்தவும்.
:::

:::info
	உங்கள் குறிப்புரைகள் இடப்பட்ட ட்ரேஸ்களின் செயல்திறன் மற்றும் முடிவுகளின் அடிப்படையில் அலாரங்களை உருவாக்கவும்.
:::
