# AWS Organization లో SSM Agent Manage చేయడం

ఈ section AWS Organization లో SSM agent installation manage చేయడం ఎలాగో వివరిస్తుంది మరియు CloudFormation, Terraform, Shell, PowerShell, Ansible, Chef, మరియు Puppet sample code అందిస్తుంది.

దయచేసి గుర్తుంచుకోండి ఇది sample code మరియు production environment లో ఏదైనా ఉపయోగానికి ముందు development environment లో సమగ్రంగా test మరియు validate చేయాలి.

## SSM Agent అర్థం చేసుకోవడం

మునుపటి sections లో చర్చించినట్లు, Systems Manager ఉపయోగించి మీ environment లోని nodes ను manage చేయడం ప్రారంభించడానికి, nodes [managed](https://docs.aws.amazon.com/systems-manager/latest/userguide/operating-systems-and-machine-types.html#supported-machine-types) అయి ఉండాలి, అంటే SSM Agent machine మీద install చేయబడి ఉండాలి మరియు agent అవసరమైన permissions తో Systems Manager service endpoints తో communicate చేయగలగాలి.

AWS Systems Manager Agent (SSM Agent) అనేది Amazon Elastic Compute Cloud (Amazon EC2) instances, edge devices, on-premises servers, మరియు virtual machines (VMs) మీద run అయ్యే Amazon software. SSM Agent కోసం source code [GitHub](https://github.com/aws/amazon-ssm-agent) లో అందుబాటులో ఉంది. SSM Agent Systems Manager ఈ resources ను update, manage, మరియు configure చేయడం సాధ్యం చేస్తుంది. Agent AWS Cloud లోని Systems Manager service నుండి requests ను process చేస్తుంది, తర్వాత request లో specify చేయబడినట్లు వాటిని run చేస్తుంది.

అందువల్ల, nodes మీద SSM Agent install చేయబడిందని నిర్ధారించడం అవసరం మరియు AWS Systems Manager service ఉపయోగించి node management operations ప్రారంభించేటప్పుడు address చేయాల్సిన మొదటి step.

## Operating system support

SSM Agent విస్తృత శ్రేణి operating systems మరియు machine types మీద supported. మీ operating system (OS), OS version, మరియు machine type supported అయ్యాయో verify చేయడానికి User guide లో [Supported operating systems and machine types](https://docs.aws.amazon.com/systems-manager/latest/userguide/operating-systems-and-machine-types.html) చూడండి.

## SSM Agent pre-installed ఉన్న AMIs

Systems Manager తో ప్రారంభించడానికి, AWS Systems Manager Agent (SSM Agent) AWS మరియు trusted third-parties అందించిన కొన్ని Amazon Machine Images (AMIs) మీద preinstall చేయబడి ఉంటుంది. ఇది preinstall చేయబడిన operating systems జాబితా పొందడానికి [Find AMIs with the SSM Agent preinstalled](https://docs.aws.amazon.com/systems-manager/latest/userguide/ami-preinstalled-agent.html) చూడండి.

## SSM Agent installation methods

ఇప్పుడు ఏ operating systems SSM Agent కు support ఇస్తాయో అర్థం చేసుకున్నాము, SSM Agent preinstall చేయబడని instances మీద agent install చేయడానికి వివిధ methods ను explore చేద్దాం. Installation approach రెండు main scenarios గా categorize చేయవచ్చు:

* **కొత్త instance deployment**: కొత్త instances launch చేసేటప్పుడు SSM Agent install చేయడం.
* **ఇప్పటికే ఉన్న instance management**: ఇప్పటికే running instances మీద SSM Agent install చేయడం.

### కొత్త instances మీద SSM Agent install చేయడం

కొత్త instances launch చేసేటప్పుడు, మీ initial setup process లో SSM Agent installation చేర్చడం ప్రారంభం నుండి consistent management capabilities నిర్ధారిస్తుంది. ఈ approach post-deployment installation అవసరాన్ని eliminate చేస్తుంది. Golden AMIs, user data scripts, లేదా infrastructure as code tools వంటి వివిధ methods ద్వారా మీరు దీనిని సాధించవచ్చు. Method ఎంపిక మీ organization యొక్క deployment practices మరియు scaling requirements మీద ఆధారపడి ఉంటుంది.

#### Golden AMI Approach

Golden AMI అనేది EC2 instances launch చేయడానికి standardized template గా పనిచేసే customized Amazon Machine Image. Systems Manager integration కోసం, organizations SSM Agent pre-installed తో Golden AMIs create చేయవచ్చు, ఈ AMIs నుండి launch చేయబడిన అన్ని instances Systems Manager ద్వారా వెంటనే manageable అయ్యేలా నిర్ధారిస్తుంది. ఈ approach post-launch installation scripts అవసరాన్ని eliminate చేస్తుంది మరియు deployment failures risk ను తగ్గిస్తుంది.

Golden AMI Create చేయడం

1. Manually: Instance launch చేసి, SSM Agent install మరియు configure చేసి, configuration test చేసి, తర్వాత ఆ instance నుండి AMI create చేయడం ద్వారా Golden AMI manually create చేయవచ్చు.
1. EC2 Image Builder: Golden images create చేయడానికి EC2 Image Builder ఉపయోగిస్తుంటే, [image recipe](https://docs.aws.amazon.com/imagebuilder/latest/userguide/start-build-image-pipeline.html#start-build-image-step2) configure చేసి output image లో SSM agent ఉంచే option select చేయండి.
1. Hashicorp Packer: Hashicorp Packer ఉపయోగిస్తుంటే, SSM agent install చేయడానికి shell commands run చేయడానికి shell Packer provisioner ఉపయోగించండి.

```json
{
  "provisioners": [
    {
      "type": "shell",
      "inline": [
        "if [ -f /etc/system-release ]; then",
        "  # Amazon Linux",
        "  sudo yum install -y amazon-ssm-agent",
        "  sudo systemctl enable amazon-ssm-agent",
        "  sudo systemctl start amazon-ssm-agent",
        "elif [ -f /etc/lsb-release ]; then",
        "  # Ubuntu",
        "  sudo snap install amazon-ssm-agent —classic",
        "  sudo systemctl enable snap.amazon-ssm-agent.amazon-ssm-agent.service",
        "  sudo systemctl start snap.amazon-ssm-agent.amazon-ssm-agent.service",
        "fi"
      ]
    }
  ]
}
```

ప్రత్యామ్నాయంగా, Agent installation shell commands ఉన్న script కు path అందించండి.

```json
{
  "provisioners": [{
    "type": "shell",
    "script": "scripts/install_ssm.sh"
  }]
}
```

Packer తో script provisioner ఉపయోగించేటప్పుడు, `install_ssm.sh` మీ Packer configuration file (`.json` లేదా `.pkr` `.hcl`) కు relative గా store చేయాలి.

మీ environment లో ఇప్పటికే running ఉన్న instances కోసం, వివిధ remote access మరియు automation methods ఉపయోగించి SSM Agent install చేయవచ్చు.
```yaml
- name: Install SSM Agent across fleet
  hosts: all
  become: yes

  tasks:
    - name: Detect OS family
      ansible.builtin.set_fact:
        os_family: "{{ ansible_facts['os_family'] }}"

    - name: Install SSM Agent on RedHat family
      block:
        - name: Download SSM Agent RPM
          get_url:
            url: "https://s3.{{ aws_region }}.amazonaws.com/amazon-ssm-{{ aws_region }}/latest/linux_amd64/amazon-ssm-agent.rpm"
            dest: /tmp/amazon-ssm-agent.rpm
            mode: '0644'

        - name: Install SSM Agent
          yum:
            name: /tmp/amazon-ssm-agent.rpm
            state: present
      when: os_family == "RedHat"

    - name: Install SSM Agent on Debian family
      block:
        - name: Install SSM Agent via Snap
          community.general.snap:
            name: amazon-ssm-agent
            classic: yes
      when: os_family == "Debian"

    - name: Enable and start SSM Agent
      systemd:
        name: amazon-ssm-agent
        enabled: yes
        state: started

    - name: Verify installation
      command: systemctl status amazon-ssm-agent
      register: ssm_status
      ignore_errors: yes

    - name: Report status
      debug:
        var: ssm_status.stdout_lines
```

#### Group Policy (Windows కోసం)

Group Policy సాంప్రదాయకంగా .msi packages కు support ఇస్తుంది, కానీ AWS Systems Manager (SSM) Agent .exe file గా వస్తుంది. ఈ limitation ను overcome చేయడానికి, మీ Windows environment అంతటా SSM Agent deploy చేయడానికి Group Policy మరియు Scheduled Tasks కలయికను ఉపయోగిస్తాము.

1. Active Directory లో appropriate hierarchy/OU లో GPO create చేయండి (లేదా existing GPO modify చేయండి). Computer scheduled task script configuration ఉపయోగించేలా policy edit చేయండి.

2. Option Computer Configuration | Preferences | Control Panel Settings | Scheduled Tasks | Right-click చేసి New→Immediate Task select చేయండి అనే path లో ఉంటుంది.

3. Action కోసం **Create** select చేయండి, task కు name ఇవ్వండి, మరియు task run చేయడానికి user account గా `NT AUTHORITY\System` specify చేయండి. **Run whether user is logged on or not** radio button click చేసి **Triggers** tab click చేసి next section కు proceed అవ్వండి.

![Task Manager screenshot for SSMAgent properties](/img/cloudops/guides/centralized-operations-management/node-management/task-manager-ssm-agent.png "Task Manager screenshot for SSMAgent properties")

4. Triggers కోసం, one time ఎంచుకోండి.

5. **Start a program** కు new action create చేసి `powershell.exe` ఎంచుకోండి మరియు **Argument** కోసం కింది వాడండి:

```powershell
-ExecutionPolicy Bypass -Command "$dir = $env:TEMP + '\ssm'; New-Item -ItemType directory -Path $dir -Force; Set-Location $dir; (New-Object System.Net.WebClient).DownloadFile('https://s3.amazonaws.com/ec2-downloads-windows/SSMAgent/latest/windows_amd64/AmazonSSMAgentSetup.exe', $dir + '\AmazonSSMAgentSetup.exe'); Start-Process .\AmazonSSMAgentSetup.exe -ArgumentList @('/q', '/log', 'install.log') -Wait"
```

6. New Action window మీద **OK** click చేసి Properties window మీద **OK** click చేసి ఆ dialogues close చేయండి.

7. ఇది complete అయిన తర్వాత, GP policy update అయినప్పుడు లేదా command prompt నుండి `gpudate /force` ఉపయోగించి ఆ action force చేసినప్పుడు ఏదైనా linked computers కు installer push చేయబడుతుంది.

మరింత సమాచారం కోసం, [Automate Registering Windows Managed Nodes in AWS Systems Manager](https://github.com/aws-samples/sample-ssm-automate-windows-hybrid-install) చూడండి.

#### SSH (Linux కోసం)

SSH ద్వారా accessible అయిన Linux instances కోసం, కింది sample script ఉపయోగించి SSM Agent installation automate చేయవచ్చు. ఈ script OS type detect చేసి appropriate package install చేస్తుంది.

```bash
#!/bin/bash
USERNAME=ec2-user
HOSTS=("192.168.1.121" "192.168.1.122" "192.168.1.123")
KEY_PATH="/path/to/your/key.pem"

for HOSTNAME in ${HOSTS}; do
    # Detect OS type
    OS_TYPE=$(ssh -i "${KEY_PATH}" -l ${USERNAME} ${HOSTNAME} "cat /etc/os-release | grep '^ID=' | cut -d= -f2 | tr -d '\"'")

    case ${OS_TYPE} in
        "amzn"|"rhel"|"centos")
            SCRIPT="sudo yum install -y s3.amazonaws.com/ec2-downloads-windows/SSMAgent/latest/linux_amd64/amazon-ssm-agent.rpm && sudo systemctl start amazon-ssm-agent"
            ;;
        "ubuntu"|"debian")
            SCRIPT="wget s3.amazonaws.com/ec2-downloads-windows/SSMAgent/latest/debian_amd64/amazon-ssm-agent.deb && sudo dpkg -i amazon-ssm-agent.deb && sudo systemctl start amazon-ssm-agent"
            ;;
        *)
            echo "Unsupported OS on ${HOSTNAME}"
            continue
            ;;
    esac

    ssh -i "${KEY_PATH}" -l ${USERNAME} ${HOSTNAME} "${SCRIPT}"
done
```

## SSM Agent కోసం IAM permissions manage చేయడం

SSM Agent functionality కోసం సరైన IAM permissions చాలా ముఖ్యం. AWS ఈ permissions manage చేయడానికి manual instance profile attachment నుండి automated organization-wide solutions వరకు బహుళ approaches అందిస్తుంది.

### Default Host Management Configuration (DHMC)

DHMC మీ organization అంతటా SSM Agent permissions manage చేయడానికి AWS recommended approach. ఇది manually AWS Identity and Access Management (IAM) instance profile create చేయకుండానే EC2 instances manage చేయడం సాధ్యం చేస్తుంది. బదులుగా, Default Host Management Configuration ఇది activate చేయబడిన AWS account మరియు AWS Region లోని అన్ని instances manage చేయడానికి Systems Manager కు permissions నిర్ధారించే default IAM role create మరియు apply చేస్తుంది.

#### DHMC enable చేయడానికి methods:
* **Unified Console Experience ద్వారా**: [Unified console experience కు onboarding](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-setting-up-organizations.html) చేసేటప్పుడు, DHMC automatically enable అవుతుంది. ఇది onboarding సమయంలో create చేయబడిన State Manager association ఉపయోగించి enable చేయబడుతుంది మరియు ప్రతి target account మరియు region లో DHMC enable చేస్తుంది.
* **Fleet Manager ఉపయోగించి**: Fleet Manager console నుండి Default Host Management Configuration turn on చేయవచ్చు. మీ Amazon EC2 instances managed చేయాలనుకునే ప్రతి Region లో ఒక్కొక్కటిగా Default Host Management Configuration turn on చేయాలి. [ఇక్కడ](https://docs.aws.amazon.com/systems-manager/latest/userguide/setup-instance-permissions.html#default-host-management) steps చూడండి.
* **CLI/SDK ద్వారా**: Required role create చేసి AWS CLI/SDK ఉపయోగించి service setting update చేయడం ద్వారా ఇది చేయబడుతుంది. మీ Amazon EC2 instances managed చేయాలనుకునే ప్రతి Region లో ఒక్కొక్కటిగా Default Host Management Configuration turn on చేయాలి. [ఇక్కడ](https://docs.aws.amazon.com/systems-manager/latest/userguide/fleet-manager-default-host-management-configuration.html#dhmc-activate) steps చూడండి.
* **Quick Setup ద్వారా**: Quick Setup తో, AWS Organizations లో add చేయబడిన అన్ని accounts మరియు Regions కోసం Default Host Management Configuration activate చేయవచ్చు. [ఇక్కడ](https://docs.aws.amazon.com/systems-manager/latest/userguide/fleet-manager-default-host-management-configuration.html#dhmc-activate) steps చూడండి.
* **CloudFormation ఉపయోగించి**: StackSets తో multi accounts మరియు region లో DHMC enable చేయడానికి CloudFormation కూడా ఉపయోగించవచ్చు. ఈ [blog](https://aws.amazon.com/blogs/mt/enable-management-of-your-amazon-ec2-instances-in-aws-systems-manager-using-default-host-management-configuration/) చూడండి.

### Individual instance permissions

DHMC ఉపయోగించకపోతే, SSM agent కోసం permission ప్రతి instance level వద్ద IAM Instance Profiles ఉపయోగించి grant చేయాలి. SSM agent కోసం recommended Amazon Managed Policy, AmazonSSMManagedInstanceCore తో Instance Profile create చేయడం recommend చేస్తాము. Use case ను బట్టి ఈ instance profile కు additional permissions అవసరం అవుతాయి. SSM agent use కోసం IAM instance profile create చేయడం మరియు additional policy considerations గురించి చర్చించే User guide లో [Additional policy considerations for managed instances](https://docs.aws.amazon.com/systems-manager/latest/userguide/setup-instance-permissions.html#instance-profile-policies-overview) చూడండి.

```yaml
      IamInstanceProfile: !Ref SSMInstanceProfile
```

### ఇప్పటికే running EC2 instances కు instance profiles attach చేయడం

* Quick Setup ఉపయోగించి (Recommended): [Amazon EC2 host management using Quick Setup](https://docs.aws.amazon.com/systems-manager/latest/userguide/quick-setup-host-management.html) ఉపయోగించి, మీ instances కు attached existing instance profiles కు required IAM policies add చేయడానికి, లేదా మీరు ఎంచుకున్న configuration కోసం అవసరమైన permissions తో IAM policies మరియు instance profiles create చేయడానికి Quick Setup ను allow చేయడానికి configure చేయవచ్చు. ఈ option organization లో multiple accounts లేదా specific OUs లోని instances ను target చేయవచ్చు.

* Systems Manager Automation ఉపయోగించి: Systems Manager access కోసం SSMRoleForManagedInstance managed IAM role తో instance configure చేయడానికి automation runbook [AWS-SetupManagedRoleOnEc2Instance](https://docs.aws.amazon.com/systems-manager-automation-runbooks/latest/userguide/automation-aws-setupmanagedroleonec2instance.html) ఉపయోగించవచ్చు. Specified role exist కాకపోతే, automation ద్వారా create చేయబడుతుంది. Multiple account మరియు regions లోని instances target చేయడానికి, [multi-account/multi-Region](https://docs.aws.amazon.com/systems-manager/latest/userguide/running-automations-multiple-accounts-regions.html) mode లో automation run చేయండి.
