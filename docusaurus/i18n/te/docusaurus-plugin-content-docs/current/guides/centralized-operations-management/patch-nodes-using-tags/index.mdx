# AWS Systems Manager మరియు tagging ఉపయోగించి managed nodes ను patching చేయడం

## పరిచయం

మీ node infrastructure అంతటా security మరియు performance maintain చేయడానికి robust patch management strategy అవసరం. ఈ guided solution [AWS Systems Manager](https://aws.amazon.com/systems-manager/) ఉపయోగించి automated, tag-based patch management system implement చేయడంలో మిమ్మల్ని walk through చేస్తుంది. ఈ guided solution follow చేయడం ద్వారా, manual intervention తగ్గించి security compliance improve చేసే scalable మరియు efficient patching process establish చేస్తారు. ఈ method centralized approach కోసం [AWS Organizations](https://aws.amazon.com/organizations/) లో utilize చేయవచ్చు, మరియు single accounts కోసం కూడా implement చేయవచ్చు.

ఇలాంటి tagging scheduling method application owners తమ nodes updates receive చేసే సమయం manage చేయడానికి అనుమతిస్తుంది. Approved schedules, వాటికి corresponding [patch policy](https://docs.aws.amazon.com/systems-manager/latest/userguide/quick-setup-patch-manager.html#create-patch-policy) ఉంటుంది, [AWS CLI](https://aws.amazon.com/cli/) ఉపయోగించి query చేయవచ్చు, node schedule మార్చడానికి self-service method అనుమతిస్తుంది.

## Solution ను అర్థం చేసుకోవడం

మీ patch management solution లో కొన్ని parts implement చేయడం ప్రారంభించే ముందు, ఈ solution ఎలా work చేస్తుందో అర్థం చేసుకోవడం ముఖ్యం. ఈ approach AWS Systems Manager patch management capabilities ను standardized tagging strategy తో combine చేస్తుంది. ఈ integration update schedules పై granular control maintain చేస్తూ మీ managed nodes అంతటా patching schedules automate చేయడానికి అనుమతిస్తుంది.

ప్రతి managed node కు predefined tag assign చేయబడుతుంది, ఈ case లో, `maintenance:patching`. ఈ tag value cron expression కు similar entry contain చేస్తుంది, ఇది node evaluate చేయబడి updates apply చేయబడే schedule indicate చేస్తుంది.

Managed node updates receive చేయవలసిన ప్రతి authorized schedule కు match అయ్యే patch policies create చేయబడతాయి.

Custom patch baselines create చేసి patch policies కు associate చేయవచ్చు, apply చేయబడే updates పై additional control అనుమతిస్తుంది లేదా default patch baselines utilize చేయవచ్చు.

Patch policies మరియు tags establish చేసి assign చేసిన తర్వాత, managed nodes కు patch policies మరియు patch baselines లోని specifications ప్రకారం patch policies apply అవుతాయి.

## Foundation build చేయడం

### Tag-based patching implementation కోసం Prerequisites

#### AWS Systems Manager configuration

Tag-based patching implement చేయడానికి ముందు AWS Systems Manager account లేదా organizational level వద్ద [properly configured](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-quick-setup.html) ఉండాలి. Organizational context లో, AWS Organizations ద్వారా అన్ని relevant accounts అంతటా Systems Manager enabled అని ensure చేయండి. Systems Manager resources manage చేయడానికి allow చేసే appropriate IAM roles మరియు policies configure చేయండి, required service-linked roles మరియు instance profiles include చేస్తూ. Multiple regions లేదా accounts అంతటా centralized visibility అవసరమైతే resource data sync establish చేయండి.

#### SSM Agent management

అన్ని managed nodes properly configured [SSM Agent](https://docs.aws.amazon.com/systems-manager/latest/userguide/patch-manager-prerequisites.html) run చేయాలి. చాలా AWS AMIs agent default గా include చేస్తాయి, కానీ ఇది present అని మరియు latest version కు updated అని verify చేయండి. Non-AWS managed nodes కోసం, instance provisioning సమయంలో agent install మరియు configure చేయడానికి process implement చేయండి. Agent updates maintain చేయడానికి automated process establish చేయండి, appropriate ఉన్న చోట Systems Manager agent auto-update capability leverage చేస్తూ.

#### Network connectivity requirements

Systems Manager nodes ను effectively manage చేయడానికి [consistent network connectivity](https://docs.aws.amazon.com/systems-manager/latest/userguide/patch-manager-prerequisites.html) require చేస్తుంది. Managed nodes internet ద్వారా లేదా VPC endpoints ద్వారా Systems Manager endpoints reach చేయగలని ensure చేయండి. Private subnets లోని nodes కోసం, Systems Manager కోసం అవసరమైన VPC endpoints (ssm, ssmmessages, మరియు ec2messages) configure చేయండి. Security groups మరియు network ACLs required traffic permit చేస్తున్నట్లు verify చేయండి. Proxy use చేస్తుంటే, SSM Agent ను appropriate proxy settings తో configure చేయండి.

#### Patch tagging strategy

Organizational needs తో align అయ్యే comprehensive tagging strategy develop మరియు document చేయండి. Define చేయండి:

* Patch groups identify చేయడానికి Tag key-value pairs (ఉదా., `maintenance:patching`)
* AWS Config rules ద్వారా Tag enforcement mechanisms
* Tag governance మరియు ownership బాధ్యతలు
* Tag updates మరియు modifications కోసం procedures

Different environments లేదా application types కోసం అవసరమైన ఏదైనా variations తో సహా complete tagging schema document చేయండి. Tag compliance validate చేయడానికి మరియు improperly tagged resources remediate చేయడానికి processes establish చేయండి.

:::note
Patch source లేదా repositories కు Network connectivity కూడా required అవుతుంది. మరింత information కోసం [Connectivity to the patch source](https://docs.aws.amazon.com/systems-manager/latest/userguide/patch-manager-prerequisites.html#source-connectivity) చూడండి.

## Patching process implement చేయడం

మీ [managed nodes](https://docs.aws.amazon.com/systems-manager/latest/userguide/operating-systems-and-machine-types.html#supported-machine-types) up-to-date మరియు secure గా ఉంచడం మీ AWS, on-premises, మరియు multi-cloud infrastructure maintain చేయడంలో ముఖ్యమైన భాగం. AWS Systems Manager తో, మీ managed node fleet అంతటా చాలా security మరియు ఇతర types software updates (patches) automate చేయవచ్చు. అయితే, patching schedules మరియు cadences manage చేయడం ఇప్పటికీ complex గా ఉంటుంది, ముఖ్యంగా మీ infrastructure scale అయ్యేకొద్దీ. ఈ guided solution flexibility తో patching schedules manage చేయడంలో assist చేయడానికి employ చేయగల ఒక method యొక్క example.

### Step 1: మీ tagging strategy establish చేయడం

`maintenance:patching` tagging convention implement చేయడం ద్వారా ప్రారంభించండి. ఈ tag ప్రతి node కు patches ఎప్పుడు apply చేయాలో define చేస్తుంది. Tag value schedule specify చేయడానికి cron-like expression use చేస్తుంది. Organizations off hour (non production) times updates apply చేయడానికి select చేయడం common. ఇలాంటి cases లో, select times మరియు days of the month ప్రతి node align అవ్వవలసిన standards గా choose చేయవచ్చు.

ఉదాహరణకు:

```language=yaml
Key: maintenance:patching
Value: 2SATX4    # Runs at 2 AM every 4th Saturday
```

:::note
Value లోని 'X' '#' symbol ను replace చేయడానికి use చేయబడుతుంది, ఇది tag values కోసం allowed characters లో లేదు
:::

Values కోసం ఇతర common patterns:

* `0SUN` ప్రతి Sunday, midnight maintenance కోసం
* `4TUEX2` ప్రతి 2nd Tuesday, 4 AM maintenance కోసం
* `22MONX3` ప్రతి third Monday, 10 PM maintenance కోసం

కింది diagram managed nodes పై tags, వివిధ sources నుండి, patch policy తో ఎలా align అవ్వవచ్చో demonstrate చేస్తుంది.

![Example diagram that demonstrates how tags on managed nodes can align with a patch policy](/img/cloudops/solutions/centralized-operations-management/patch-nodes-using-tags/PatchTagging.drawio.png "Example diagram that demonstrates how tags on managed nodes can align with a patch policy")

ఈ solution యొక్క ఒక implementation Friday evenings 10 PM న ప్రారంభమయ్యే implementation times provide చేసింది. 10 PM Friday తర్వాత ప్రతి నాలుగు గంటలకు 10 PM Sunday వరకు subsequent time offerings create చేయబడ్డాయి. ఇది weekly updates apply చేయడానికి 13 options allow చేసింది, ఇవి environment అంతటా అవసరమైన విధంగా spread చేయవచ్చు, development, test, మరియు production వంటి environments ను separate schedules లో update చేయగల ability తో సహా.

### Step 2: మీ tag strategy deploy చేయడం

మీ tagging strategy decide చేసిన తర్వాత, మీ existing nodes కొత్త required tag receive చేస్తున్నట్లు ensure చేయాలి మరియు అన్ని new nodes పై tag require చేసే standards enact చేయాలి.

#### Compliance enforcement

మీ environment అంతటా tag governance maintain చేయడానికి AWS Config rules implement చేయండి. అన్ని managed nodes పై required patching tags presence verify చేయడానికి మరియు tag values approved standards కు conform అవుతున్నట్లు validate చేయడానికి rules configure చేయండి. ఈ automated oversight మీ infrastructure అంతటా consistent tagging ensure చేస్తుంది మరియు non-compliant resources త్వరగా identify చేయడంలో help చేస్తుంది.

#### Automated remediation

Non-compliant resources address చేయడానికి AWS Systems Manager Automation use చేసి automated remediation processes establish చేయండి. మీ remediation workflow resource characteristics ఆధారంగా appropriate default values తో missing tags add చేయాలి. Standard cases కోసం automatic మరియు human review require చేసే exceptions కోసం manual అయ్యేలా మీ remediation design చేయండి.

#### Audit మరియు review process

`maintenance:patching` tag కోసం managed nodes ను automatically monitor చేయడానికి AWS Config rules deploy చేయండి, tag presence మరియు valid patching schedule values (ఉదా., `2SATX4`, `4TUEX2`) రెండూ ensure చేస్తూ. Non-compliant resources identify చేయడానికి మరియు auto-remediation properly functioning అని validate చేయడానికి Config compliance reports weekly review చేయండి. Approved exceptions documentation centralized location లో create మరియు maintain చేయండి.

#### Governance model

Tag management కోసం clear ownership మరియు responsibilities establish చేయండి, Cloud Operations consistent reference కోసం AWS Systems Manager [Parameter Store](https://docs.aws.amazon.com/systems-manager/latest/userguide/sysman-paramstore-su-create.html) లో approved patching schedules maintain చేస్తూ [organization అంతటా](https://docs.aws.amazon.com/systems-manager/latest/userguide/parameter-store-shared-parameters.html#share). Tag standards మరియు patching schedules కు changes request చేయడానికి processes define చేయండి, updates Parameter Store మరియు associated Config rules రెండింటిలోనూ reflect అయ్యేలా ensure చేస్తూ. Changes కోసం documented approval workflows create చేయండి, Cloud Operations centralized parameters update చేయడానికి ముందు Security team validation require చేస్తూ. Tagging strategy మరియు maintenance windows organizational needs continue to meet చేస్తున్నట్లు ensure చేయడానికి operations, security, మరియు application teams మధ్య feedback loops maintain చేయండి, approved values కోసం source of truth గా Parameter Store use చేస్తూ.

### Step 3: Systems Manager Configure చేయడం

మీ tagging strategy in place ఉన్నప్పుడు మీ Cloud Operations Team మీ patching strategy execute చేయడానికి Systems Manager configure చేయవచ్చు:

#### Patch baseline guidance

[Custom patch baseline](https://docs.aws.amazon.com/systems-manager/latest/userguide/patch-manager-predefined-and-custom-patch-baselines.html) create చేయడానికి ముందు, AWS predefined baselines మీ patching requirements meet చేస్తాయో evaluate చేయండి. మీ operating systems కోసం default AWS-managed baselines review చేయండి, ఇవి సాధారణంగా critical updates మరియు security patches కోసం approved patches include చేస్తాయి. మీ organization కు specific compliance requirements, patch exclusions, delayed patch approvals, లేదా particular application dependencies manage చేయవలసిన అవసరం ఉంటే, మీకు likely custom baseline అవసరం. అయితే, మీరు simply regular schedule లో critical మరియు security updates apply చేయవలసి ఉంటే, AWS-managed baselines often తక్కువ maintenance overhead require చేస్తూ sufficient coverage provide చేస్తాయి. మీ patching requirements document చేసి informed decision తీసుకోవడానికి predefined baseline configurations against compare చేయండి.

#### Patch policy configuration guidance

Parameter Store లో define చేయబడిన ప్రతి approved patching schedule కోసం, AWS Systems Manager లో corresponding [patch policy](https://docs.aws.amazon.com/systems-manager/latest/userguide/quick-setup-patch-manager.html#create-patch-policy) create చేయండి. Patching schedules తో easy correlation ensure చేస్తూ CRON schedule clearly reflect చేయడానికి మీ policy name చేయండి (ఉదా., `Patch-2SATX4`). Policy ను scan మరియు install operations రెండింటికీ configure చేయండి, patching schedule తో align అయ్యే మీ pre-approved CRON expression specify చేయడానికి Custom Scan schedule option utilize చేస్తూ.

Policy scope define చేసేటప్పుడు, మీ designated patch baseline select చేసి Custom Targets option use చేసి targeting configure చేయండి. ఈ policy apply అవ్వాల్సిన appropriate organizational units (OUs) మరియు AWS regions specify చేయండి. Most critically, `maintenance:patching` tag use చేసి target nodes set చేయండి, ఈ patching schedule కు correspond చేసే specific tag value enter చేస్తూ (ఉదా., `2SATX4`). ఇది ఈ patching schedule కోసం explicitly tagged nodes మాత్రమే patching operation లో include అవుతాయని ensure చేస్తుంది.

ఈ configuration approach మీ tagging strategy, patching schedules, మరియు patch policy execution మధ్య consistency maintain చేస్తుంది, ప్రతి scheduled period లో ఏ nodes patch అవుతాయనే clear visibility provide చేస్తూ.

#### Automation workflow guidance (optional)

AWS Systems Manager లో supplemental automation workflows implement చేయడం ద్వారా మీ patching strategy enhance చేయండి. System readiness verify చేయడానికి pre-patch validation checks design చేయండి, available disk space confirm చేయడం, running process verification, మరియు critical configurations backup చేయడం వంటివి. System functionality validate చేయడానికి post-patch testing automation create చేయండి, service status checks, application health monitoring, మరియు basic connectivity tests include చేస్తూ. Patch execution status, test failures, లేదా validation issues గురించి appropriate teams alert చేయడానికి SNS topics మరియు notification pathways configure చేయండి. ఈ automated workflows patch cycle అంతటా consistent validation మరియు clear communication channels provide చేయడం ద్వారా మీ patching process strengthen చేస్తాయి.
