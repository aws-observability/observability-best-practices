# CloudTrail Lake event enrichment కోసం ఉత్తమ పద్ధతులు

CloudTrail event enrichment అనేది event data store సృష్టించేటప్పుడు లేదా అప్‌డేట్ చేసేటప్పుడు resource tag keys మరియు IAM global condition keys (principal tag keys తో సహా) జోడించడం ద్వారా management మరియు data events ను మెరుగుపరచడానికి మిమ్మల్ని అనుమతిస్తుంది. ఈ enrichment cost allocation, financial management, operations, మరియు data security requirements వంటి వ్యాపార సందర్భం ఆధారంగా CloudTrail events యొక్క మెరుగైన వర్గీకరణ, శోధన మరియు విశ్లేషణను అనుమతిస్తుంది. enriched events లో API చర్యలకు సందర్భ సమాచారాన్ని అందించే eventContext field ఉంటుంది, దీనిని మీరు CloudTrail Lake లో queries రన్ చేయడం ద్వారా లేదా Amazon Athena తో federation ద్వారా విశ్లేషించవచ్చు. ఈ ఫీచర్ resources, principals మరియు authorization conditions గురించి అదనపు metadata ను వారి event logs లో చేర్చడం ద్వారా సంస్థలు తమ AWS resource వినియోగం మరియు security posture ను మెరుగ్గా అర్థం చేసుకోవడానికి మరియు ట్రాక్ చేయడానికి సహాయపడుతుంది.

### Resource Tag Enrichment

* వ్యాపార సందర్భం ఆధారంగా events ను వర్గీకరించడానికి మరియు విశ్లేషించడానికి event data store సృష్టన లేదా అప్‌డేట్ సమయంలో resource tag keys ను కాన్ఫిగర్ చేయండి
* resource సృష్టన తర్వాత జోడించిన resource tags CloudTrail events లో కనిపించడానికి ముందు ఆలస్యం అనుభవించవచ్చని గమనించండి
* తొలగించబడిన resources కోసం resource tags లో tag సమాచారం ఉండకపోవచ్చని తెలుసుకోండి
* resource సృష్టన సమయంలో వర్తించే tags లో తక్కువ లేదా ఎటువంటి ఆలస్యం ఉండదని అర్థం చేసుకోండి

### IAM Global Condition Keys Enrichment

* authorization processes గురించి అదనపు సందర్భాన్ని చేర్చడానికి సంబంధిత IAM global condition keys ను ఎంచుకోండి
* కాన్ఫిగర్ చేసిన condition keys IAM policy evaluation కు సంబంధించనప్పుడు ప్రతి event లో కనిపించకపోవచ్చని గుర్తుంచుకోండి
* ఈ క్రింది వంటి [supported condition keys](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#condition-keys-supported-services) వాడటం పరిగణించండి:
    * aws:FederatedProvider
    * aws:MultiFactorAuthPresent
    * aws:PrincipalAccount
    * aws:PrincipalType
    * aws:ViaAWSService
    * aws:SecureTransport
    * మరియు అనేక ఇతరాలు ([Supported IAM global condition keys for enriched events](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#condition-keys-supported-services))

### Service పరిగణనలు

* resource tag updates లో ఆలస్యం అనుభవించే [services](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#resource-tags-supported-services) గురించి తెలుసుకోండి, వీటిలో:
    * Amazon S3
    * AWS CloudTrail
    * AWS Lambda
    * Amazon DynamoDB
    * AWS Organizations
    * AWS KMS
    * Amazon SQS
    * మరియు అనేక ఇతరాలు ([AWS documentation](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-context-events.html#resource-tags-supported-services) లో అదనపు services జాబితా చేయబడ్డాయి)

### Event Context Management

* enriched సమాచారం కోసం CloudTrail events లోని eventContext field ను పర్యవేక్షించండి
* enrichment తో కాన్ఫిగర్ చేసిన event data stores కోసం events లో మాత్రమే eventContext field ఉంటుందని గమనించండి
* ఆలస్యమైన events లో eventContext data ఉండదని తెలుసుకోండి. ఆలస్యంగా డెలివరీ అయిన events లో event ఎందుకు ఆలస్యమైందో సమాచారాన్ని చూపే "addendum" field ఉంటుంది. [CloudTrail events records](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-event-reference-record-contents.html) గురించి మరింత చదవండి.
* సరైన tag population నిర్ధారించడానికి AWSServiceRoleForCloudTrailEventContext service-linked role ను నిర్వహించండి
* event context జోడించడం వల్ల event యొక్క మొత్తం పరిమాణం పెరుగుతుంది, ఇది CloudTrail Lake కోసం అదనపు ingestion cost కు దారితీయవచ్చు

### Operational పరిగణనలు

* resource tag updates లో ఆలస్యానికి కారణమయ్యే సంభావ్య service outages కోసం ప్లాన్ చేయండి
* service outages తర్వాత resource tag changes గురించి సమాచారాన్ని కలిగి ఉన్న CloudTrail events లోని addendum fields ను పర్యవేక్షించండి
* Event history, EventBridge మరియు trails లోని events లో eventContext field ఉండదని అర్థం చేసుకోండి
* enrichment కోసం tag keys మరియు condition keys ఎంపిక చేసేటప్పుడు cost allocation, operations మరియు security కోసం వ్యాపార అవసరాలను పరిగణించండి

### CloudTrail Lake కోసం Sample SQL Queries

#### **Event Enrichment కోసం Resource Tags**

```
SELECT eventtime, eventName, substr(userIdentity.arn, strpos(userIdentity.arn, '/') +1) as IAM,
eventContext.tagContext.resourceTags[1].tags as tags,
eventContext.tagContext.resourceTags[1].arn as resourceArn, 
element_at(requestParameters, 'key') as S3Object
FROM $EDS_ID
WHERE eventContext IS NOT NULL 
and eventSource = 's3.amazonaws.com'
and eventname in ('DeleteObject', 'PutObject')
AND eventtime >= '2025-06-05 00:00:00'
AND eventtime <= '2025-06-05 23:59:59'
```

#### **Data Classification Tag పై Event Enrichment కోసం Resource Tags**

```
SELECT eventtime, eventName, substr(userIdentity.arn, strpos(userIdentity.arn, '/') +1) as IAM,
element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') as DataClassification,
eventContext.tagContext.resourceTags[1].arn as resourceArn, 
element_at(requestParameters, 'key') as S3Object
FROM $EDS_ID
WHERE eventContext IS NOT NULL
and element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') = 'sensitive'
and eventSource = 's3.amazonaws.com'
and eventname in ('DeleteObject', 'PutObject')
AND eventtime >= '2025-06-05 00:00:00'
AND eventtime <= '2025-06-05 23:59:59'
```

#### **Event Enrichment కోసం Visualization Chart సృష్టించడానికి Sample Query**

```
SELECT count(*) as count, eventName, substr(userIdentity.arn, 
strpos(userIdentity.arn, '/') +1) as IAM,
element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') as DataClassification,
eventContext.tagContext.resourceTags[1].arn as resourceArn, 
element_at(requestParameters, 'key') as S3Object
FROM $EDS_ID
WHERE eventContext IS NOT NULL
and element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification') = 'sensitive'
and eventSource = 's3.amazonaws.com'
and eventname in ('DeleteObject', 'PutObject')
AND eventtime >= '2025-06-05 00:00:00'
AND eventtime <= '2025-06-05 23:59:59'
Group BY eventName,
substr(userIdentity.arn, strpos(userIdentity.arn, '/') +1),
element_at(eventContext.tagContext.resourceTags[1].tags, 'DataClassification'),
eventContext.tagContext.resourceTags[1].arn,
element_at(requestParameters, 'key')
```
