# CloudWatch Container Insights

## పరిచయం

Amazon CloudWatch Container Insights కంటైనరైజ్డ్ అప్లికేషన్‌లు మరియు మైక్రోసర్వీసెస్ నుండి మెట్రిక్స్ మరియు లాగ్‌లను సేకరించడానికి, అగ్రిగేట్ చేయడానికి మరియు సంక్షిప్తం చేయడానికి శక్తివంతమైన సాధనం. ఈ డాక్యుమెంట్ EKS Fargate వర్క్‌లోడ్‌ల కోసం ADOT మరియు CloudWatch Container Insights మధ్య ఏకీకరణ, దాని డిజైన్, డిప్లాయ్‌మెంట్ ప్రక్రియ మరియు ప్రయోజనాలతో సహా అవలోకనాన్ని అందిస్తుంది.

## EKS Fargate కోసం ADOT కలెక్టర్ డిజైన్

ADOT కలెక్టర్ మూడు ప్రధాన కాంపోనెంట్‌లతో కూడిన పైప్‌లైన్ ఆర్కిటెక్చర్ ఉపయోగిస్తుంది:

1. రిసీవర్: నిర్దేశించిన ఫార్మాట్‌లో డేటాను అంగీకరించి అంతర్గత ఫార్మాట్‌లోకి అనువదిస్తుంది.
2. ప్రాసెసర్: డేటాపై బ్యాచింగ్, ఫిల్టరింగ్ మరియు ట్రాన్స్‌ఫర్మేషన్‌ల వంటి పనులను నిర్వహిస్తుంది.
3. ఎక్స్‌పోర్టర్: మెట్రిక్స్, లాగ్‌లు లేదా ట్రేసెస్ పంపడానికి డెస్టినేషన్‌ను నిర్ణయిస్తుంది.

EKS Fargate కోసం, ADOT కలెక్టర్ Kubernetes API సర్వర్ నుండి మెట్రిక్స్‌ను స్క్రేప్ చేయడానికి Prometheus Receiver ఉపయోగిస్తుంది, ఇది వర్కర్ నోడ్‌లలో kubelet కోసం ప్రాక్సీగా పనిచేస్తుంది. EKS Fargate లో kubelet కు నేరుగా యాక్సెస్ నిరోధించే నెట్‌వర్కింగ్ పరిమితుల కారణంగా ఈ విధానం అవసరం. సేకరించిన మెట్రిక్స్ ఫిల్టరింగ్, రీనేమింగ్, డేటా అగ్రిగేషన్ మరియు కన్వర్షన్ కోసం ప్రాసెసర్‌ల శ్రేణి ద్వారా వెళ్తాయి. చివరగా, AWS CloudWatch EMF Exporter మెట్రిక్స్‌ను embedded metric format (EMF) కు మార్చి CloudWatch Logs కు పంపుతుంది.

## డిప్లాయ్‌మెంట్ ప్రక్రియ

EKS Fargate క్లస్టర్‌లో ADOT కలెక్టర్‌ను డిప్లాయ్ చేయడానికి:

1. Kubernetes తో EKS క్లస్టర్ సృష్టించండి
2. Fargate పాడ్ ఎగ్జిక్యూషన్ రోల్ సెటప్ చేయండి.
3. అవసరమైన నేమ్‌స్పేస్‌ల కోసం Fargate ప్రొఫైల్‌లను నిర్వచించండి.
4. అవసరమైన అనుమతులతో ADOT కలెక్టర్ కోసం IAM రోల్ సృష్టించండి.
5. అందించిన manifest ఉపయోగించి ADOT కలెక్టర్‌ను Kubernetes StatefulSet గా డిప్లాయ్ చేయండి.
6. మెట్రిక్స్ సేకరణను టెస్ట్ చేయడానికి నమూనా వర్క్‌లోడ్‌లను డిప్లాయ్ చేయండి.


## ప్రయోజనాలు మరియు అప్రయోజనాలు

### ప్రయోజనాలు:

1. ఏకీకృత మానిటరింగ్: EKS EC2 మరియు Fargate వర్క్‌లోడ్‌ల మధ్య స్థిరమైన మానిటరింగ్ అనుభవాన్ని అందిస్తుంది.
2. స్కేలబిలిటీ: ఒకే ADOT కలెక్టర్ ఇన్‌స్టాన్స్ EKS క్లస్టర్‌లోని అన్ని వర్కర్ నోడ్‌ల నుండి మెట్రిక్స్‌ను కనుగొని సేకరించగలదు.
3. సమృద్ధ మెట్రిక్స్: CPU, మెమరీ, డిస్క్ మరియు నెట్‌వర్క్ వినియోగంతో సహా సిస్టమ్ మెట్రిక్స్ యొక్క సమగ్ర సెట్‌ను సేకరిస్తుంది.
4. సులభ ఏకీకరణ: ప్రస్తుత CloudWatch డాష్‌బోర్డ్‌లు మరియు అలారంలతో అతుకులు లేకుండా ఏకీకృతమవుతుంది.
5. ఖర్చు-ప్రభావ: అదనపు మానిటరింగ్ మౌలిక సదుపాయాల అవసరం లేకుండా Fargate వర్క్‌లోడ్‌ల మానిటరింగ్‌ను అనుమతిస్తుంది.

### అప్రయోజనాలు:

1. కాన్ఫిగరేషన్ సంక్లిష్టత: ADOT కలెక్టర్‌ను సెటప్ చేయడానికి IAM రోల్‌లు, Fargate ప్రొఫైల్‌లు మరియు Kubernetes వనరుల జాగ్రత్తగా కాన్ఫిగరేషన్ అవసరం.
2. వనరుల ఓవర్‌హెడ్: ADOT కలెక్టర్ కూడా Fargate క్లస్టర్‌లో వనరులను వినియోగిస్తుంది, కెపాసిటీ ప్లానింగ్‌లో దీన్ని పరిగణనలోకి తీసుకోవాలి.


EKS Fargate వర్క్‌లోడ్‌ల కోసం CloudWatch Container Insights తో AWS Distro for OpenTelemetry ఏకీకరణ కంటైనరైజ్డ్ అప్లికేషన్‌లను మానిటర్ చేయడానికి శక్తివంతమైన సొల్యూషన్‌ను అందిస్తుంది. ఇది వివిధ EKS డిప్లాయ్‌మెంట్ ఆప్షన్‌ల మధ్య ఏకీకృత మానిటరింగ్ అనుభవాన్ని అందిస్తుంది మరియు OpenTelemetry ఫ్రేమ్‌వర్క్ యొక్క స్కేలబిలిటీ మరియు సౌకర్యాన్ని ఉపయోగిస్తుంది. Fargate వర్క్‌లోడ్‌ల నుండి సిస్టమ్ మెట్రిక్స్ సేకరణను అనుమతించడం ద్వారా, ఈ ఏకీకరణ కస్టమర్‌లు తమ అప్లికేషన్ పనితీరుపై లోతైన అంతర్దృష్టులు పొందడానికి, సమాచారబద్ధ స్కేలింగ్ నిర్ణయాలు తీసుకోవడానికి మరియు వనరుల వినియోగాన్ని ఆప్టిమైజ్ చేయడానికి అనుమతిస్తుంది.
