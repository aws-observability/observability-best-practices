# ADOT Observability పైప్‌లైన్

Observability పైప్‌లైన్ వివిధ సోర్స్‌ల నుండి observability డేటాను సేకరించడానికి, నిర్వహించడానికి మరియు విశ్లేషించడానికి కలిసి పనిచేసే అనేక కాంపోనెంట్‌లను కలిగి ఉంటుంది.

## EKS క్లస్టర్

EKS (Elastic Kubernetes Service) క్లస్టర్ observability పైప్‌లైన్ యొక్క ప్రధాన కాంపోనెంట్‌లను హోస్ట్ చేస్తుంది.

### ADOT Operator Helm Chart ఇన్‌స్టాల్ చేయండి

ADOT (AWS Distro for OpenTelemetry) Operator Helm chart ఉపయోగించి ఇన్‌స్టాల్ చేయబడుతుంది. ఇది observability పైప్‌లైన్ కాంపోనెంట్‌ల డిప్లాయ్‌మెంట్ మరియు కాన్ఫిగరేషన్‌ను నిర్వహిస్తుంది.

### వినియోగదారు కాన్ఫిగర్ చేసిన కలెక్టర్

వినియోగదారు-కాన్ఫిగర్ చేసిన కలెక్టర్ ఆపరేటర్ ద్వారా నిర్వహించబడుతుంది మరియు ఈ క్రింది కాంపోనెంట్‌లను కలిగి ఉంటుంది:

- Deployment గా కలెక్టర్: కలెక్టర్ Kubernetes deployment గా డిప్లాయ్ చేయబడుతుంది, ఇది అధిక అందుబాటు మరియు స్కేలబిలిటీని నిర్ధారిస్తుంది.
- Collector-0, Collector-1, Collector-2: ఇన్‌కమింగ్ observability డేటాను నిర్వహించడానికి బహుళ కలెక్టర్ ఇన్‌స్టాన్స్‌లు డిప్లాయ్ చేయబడతాయి. అవి వర్క్‌లోడ్‌ను పంపిణీ చేయడానికి మరియు విశ్వసనీయ డేటా సేకరణను నిర్ధారించడానికి కలిసి పనిచేస్తాయి.

### పర్సిస్టెంట్ వాల్యూమ్

సేకరించిన observability డేటాను నిల్వ చేయడానికి పర్సిస్టెంట్ వాల్యూమ్ ఉపయోగించబడుతుంది. ఇది డేటా మన్నిక మరియు దీర్ఘకాలిక నిల్వ మరియు విశ్లేషణను అనుమతిస్తుంది.

### Kubernetes నోడ్

Kubernetes నోడ్ అప్లికేషన్ పాడ్‌లు మరియు సైడ్‌కార్‌గా కలెక్టర్‌ను హోస్ట్ చేస్తుంది.

- అప్లికేషన్ కంటైనర్: అప్లికేషన్ కంటైనర్ వాస్తవ అప్లికేషన్ కోడ్‌ను రన్ చేస్తుంది మరియు observability డేటాను ఉత్పత్తి చేస్తుంది.
- సైడ్‌కార్‌గా కలెక్టర్: కలెక్టర్ అప్లికేషన్ కంటైనర్ పక్కన సైడ్‌కార్ కంటైనర్‌గా నడుస్తుంది. ఇది అప్లికేషన్ ఉత్పత్తి చేసిన observability డేటాను సేకరిస్తుంది.

## స్క్రేప్ టార్గెట్‌లు

Observability పైప్‌లైన్ వివిధ స్క్రేప్ టార్గెట్‌ల నుండి డేటాను సేకరిస్తుంది, అవి:

- ట్రేసెస్/మెట్రిక్స్ స్క్రేప్ చేయడం: పైప్‌లైన్ అప్లికేషన్ మరియు మౌలిక సదుపాయాల కాంపోనెంట్‌ల నుండి ట్రేసెస్ మరియు మెట్రిక్స్‌ను స్క్రేప్ చేస్తుంది.

## AWS Prometheus Remote Write Exporter

సేకరించిన observability డేటాను AWS సేవలకు ఎక్స్‌పోర్ట్ చేయడానికి AWS Prometheus Remote Write Exporter ఉపయోగించబడుతుంది.

## AWS CloudWatch EMF Exporter

AWS CloudWatch కు మెట్రిక్స్‌ను ఎక్స్‌పోర్ట్ చేయడానికి AWS CloudWatch EMF (Embedded Metric Format) Exporter ఉపయోగించబడుతుంది.

## AWS X-Ray Tracing Exporter

డిస్ట్రిబ్యూటెడ్ ట్రేసింగ్ మరియు పనితీరు విశ్లేషణ కోసం AWS X-Ray కు ట్రేసింగ్ డేటాను ఎక్స్‌పోర్ట్ చేయడానికి AWS X-Ray Tracing Exporter ఉపయోగించబడుతుంది.

Observability పైప్‌లైన్ స్క్రేప్ టార్గెట్‌ల నుండి డేటాను సేకరిస్తుంది, కలెక్టర్‌లను ఉపయోగించి ప్రాసెస్ చేస్తుంది, మరియు మరింత విశ్లేషణ మరియు విజ్యువలైజేషన్ కోసం వివిధ AWS సేవలకు ఎక్స్‌పోర్ట్ చేస్తుంది.


## ADOT తో మెట్రిక్స్ మరియు అంతర్దృష్టులు సేకరించడం

1. **ఇన్‌స్ట్రుమెంటేషన్**: OpenTelemetry మాదిరిగానే, ADOT మీ అప్లికేషన్‌లు మరియు సేవలను ఇన్‌స్ట్రుమెంట్ చేయడానికి లైబ్రరీలు మరియు SDKలను అందిస్తుంది, మెట్రిక్స్, ట్రేసెస్ మరియు లాగ్‌లు వంటి టెలిమెట్రీ డేటాను క్యాప్చర్ చేస్తుంది.

2. **మెట్రిక్స్ సేకరణ**: ADOT AWS సర్వీస్ మెట్రిక్స్‌తో సహా సిస్టమ్ మరియు అప్లికేషన్-స్థాయి మెట్రిక్స్‌ను సేకరించడానికి మరియు ఎక్స్‌పోర్ట్ చేయడానికి సపోర్ట్ చేస్తుంది, వనరుల వినియోగం మరియు అప్లికేషన్ పనితీరుపై అంతర్దృష్టులను అందిస్తుంది.

3. **డిస్ట్రిబ్యూటెడ్ ట్రేసింగ్**: ADOT AWS సేవలు, కంటైనర్‌లు మరియు ఆన్-ప్రెమిసెస్ పరిసరాల మధ్య డిస్ట్రిబ్యూటెడ్ ట్రేసింగ్‌ను అనుమతిస్తుంది, రిక్వెస్ట్‌లు మరియు ఆపరేషన్‌లను ఎండ్-టు-ఎండ్ ట్రేస్ చేయడానికి మిమ్మల్ని అనుమతిస్తుంది.

4. **లాగింగ్**: ADOT స్ట్రక్చర్డ్ లాగింగ్ సపోర్ట్‌ను కలిగి ఉంది, సమగ్ర observability కోసం లాగ్ డేటాను ఇతర టెలిమెట్రీ సిగ్నల్స్‌తో సహసంబంధం చేస్తుంది.

5. **AWS సర్వీస్ ఇంటిగ్రేషన్‌లు**: ADOT AWS X-Ray, AWS CloudWatch, Amazon Managed Service for Prometheus మరియు AWS Distro for OpenTelemetry Operator వంటి AWS సేవలతో గట్టి ఇంటిగ్రేషన్‌లను అందిస్తుంది, AWS ఎకోసిస్టమ్‌లో అతుకులు లేని టెలిమెట్రీ సేకరణ మరియు విశ్లేషణను అనుమతిస్తుంది.

6. **ఆటోమేటిక్ ఇన్‌స్ట్రుమెంటేషన్**: ADOT ప్రముఖ ఫ్రేమ్‌వర్క్‌లు మరియు లైబ్రరీల కోసం ఆటోమేటిక్ ఇన్‌స్ట్రుమెంటేషన్ సామర్థ్యాలను అందిస్తుంది, ప్రస్తుత అప్లికేషన్‌లను ఇన్‌స్ట్రుమెంట్ చేయే ప్రక్రియను సరళీకృతం చేస్తుంది.

7. **డేటా ప్రాసెసింగ్ మరియు విశ్లేషణ**: ADOT సేకరించిన టెలిమెట్రీ డేటాను AWS X-Ray, Amazon Managed Service for Prometheus మరియు AWS CloudWatch వంటి AWS observability సేవలకు ఎక్స్‌పోర్ట్ చేయవచ్చు, AWS-నేటివ్ విశ్లేషణ మరియు విజ్యువలైజేషన్ సాధనాలను ఉపయోగిస్తుంది.

## ADOT ఉపయోగించడం వల్ల ప్రయోజనాలు

1. **AWS-నేటివ్ ఇంటిగ్రేషన్**: ADOT AWS సేవలు మరియు మౌలిక సదుపాయాలతో అతుకులు లేకుండా ఏకీకరించడానికి రూపొందించబడింది, AWS ఎకోసిస్టమ్‌లో సమన్వయ observability అనుభవాన్ని అందిస్తుంది.

2. **పనితీరు మరియు స్కేలబిలిటీ**: ADOT పెద్ద-స్కేల్ AWS పరిసరాలలో సమర్థవంతమైన టెలిమెట్రీ సేకరణ మరియు విశ్లేషణను అనుమతిస్తూ పనితీరు మరియు స్కేలబిలిటీ కోసం ఆప్టిమైజ్ చేయబడింది.

3. **భద్రత మరియు సమ్మతి**: ADOT AWS భద్రతా ఉత్తమ పద్ధతులకు కట్టుబడి ఉంటుంది మరియు వివిధ పరిశ్రమ ప్రమాణాలకు అనుగుణంగా ఉంటుంది, సురక్షిత మరియు సమ్మతి observability పద్ధతులను నిర్ధారిస్తుంది.

4. **AWS సపోర్ట్**: AWS-సపోర్టెడ్ డిస్ట్రిబ్యూషన్‌గా, ADOT AWS యొక్క విస్తృత డాక్యుమెంటేషన్, సపోర్ట్ చానల్‌లు మరియు OpenTelemetry ప్రాజెక్ట్‌కు దీర్ఘకాలిక నిబద్ధత నుండి ప్రయోజనం పొందుతుంది.

## OpenTelemetry మరియు ADOT మధ్య తేడా

ADOT మరియు OpenTelemetry అనేక ప్రధాన సామర్థ్యాలను పంచుకున్నప్పటికీ, కొన్ని కీలక తేడాలు ఉన్నాయి:

1. **AWS ఇంటిగ్రేషన్**: ADOT ప్రత్యేకంగా AWS పరిసరాలకు రూపొందించబడింది మరియు AWS సేవలతో గట్టి ఇంటిగ్రేషన్‌లను అందిస్తుంది, OpenTelemetry వెండర్-న్యూట్రల్ ప్రాజెక్ట్.

2. **AWS ఆప్టిమైజేషన్**: ADOT AWS పరిసరాలలో పనితీరు, స్కేలబిలిటీ మరియు భద్రత కోసం ఆప్టిమైజ్ చేయబడింది, AWS-నేటివ్ సేవలు మరియు ఉత్తమ పద్ధతులను ఉపయోగిస్తుంది.

3. **AWS సపోర్ట్**: ADOT అధికారిక AWS సపోర్ట్, డాక్యుమెంటేషన్ మరియు దీర్ఘకాలిక నిబద్ధత నుండి ప్రయోజనం పొందుతుంది, OpenTelemetry కమ్యూనిటీ సపోర్ట్‌పై ఆధారపడుతుంది.

4. **AWS-నిర్దిష్ట ఫీచర్‌లు**: ADOT AWS-నిర్దిష్ట ఫీచర్‌లు మరియు AWS సేవల కోసం ఆటోమేటిక్ ఇన్‌స్ట్రుమెంటేషన్‌ను కలిగి ఉంది, OpenTelemetry సాధారణ-ప్రయోజన observability పై దృష్టి పెడుతుంది.

5. **వెండర్ న్యూట్రాలిటీ**: OpenTelemetry వెండర్-న్యూట్రల్ ప్రాజెక్ట్, వివిధ observability ప్లాట్‌ఫారమ్‌లతో ఇంటిగ్రేషన్‌ను అనుమతిస్తుంది, ADOT ప్రాథమికంగా AWS observability సేవలపై దృష్టి పెడుతుంది.

ADOT ను ఉపయోగించడం ద్వారా, సంస్థలు AWS ఎకోసిస్టమ్‌లో సమగ్ర observability సాధించగలవు, AWS-నేటివ్ ఇంటిగ్రేషన్‌లు, ఆప్టిమైజ్ చేసిన పనితీరు మరియు AWS సపోర్ట్ నుండి ప్రయోజనం పొందుతూ, OpenTelemetry సామర్థ్యాలు మరియు కమ్యూనిటీ సహకారాలను ఉపయోగించే సౌకర్యాన్ని నిర్వహిస్తాయి.
