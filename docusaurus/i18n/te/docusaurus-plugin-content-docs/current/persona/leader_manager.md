#### Customer experience కొలవడం

మొదట, customer experience measure చేయడం traditional system metrics దాటి move అవ్వాలి. మీ primary measurement framework గా Service Level Objectives (SLOs) implement చేయడాన్ని మేము recommend చేస్తాము. SLOs system metrics కాకుండా critical end-user journeys ఆధారంగా service availability కోసం agreed-upon targets ను provide చేస్తాయి. ఈ customer-centric approach మీ observability strategy most important విషయంతో directly align అవుతుందని ensure చేస్తుంది - end-user experience, ఇది అన్ని technical decisions కోసం మీ North Star కావాలి. ఇప్పుడు, మీ customers కు మీరు చేసే assurance మరియు మీ services ఎంత healthy గా ఉన్నాయో చెప్పే trackable measurements ను represent చేసే terminology తో familiarize అవ్వండి.

- SLI (Service Level Indicator) provide చేయబడిన service level యొక్క some aspect యొక్క carefully defined quantitative measure.
- SLO (Service Level Objective) ఒక time period లో SLI ద్వారా measured అయిన service level కోసం target value లేదా range of values.
- SLA (Service Level Agreement) మీరు deliver చేయాలని promise చేసే service level ను outline చేసే మీ customer తో agreement. Requirements meet కానప్పుడు additional support లేదా pricing discounts వంటి course of action ను SLA detail చేస్తుంది.

Amazon CloudWatch [Application Signals](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Application-Monitoring-Sections.html) introduction తో మీరు ఇప్పుడు AWS లో natively SLOs create మరియు monitor చేయవచ్చు. Application Signals CloudWatch లో comprehensive application performance monitoring solution ను provide చేస్తుంది ఇది SLOs ను మీ APM experience కు connect చేయడానికి enable చేస్తుంది. CloudWatch లో మీకు available ఏ metric ఉపయోగించైనా SLOs తో get started చేయవచ్చు. ఇది ఈరోజు CloudWatch లో available metrics తో get started చేయడం easy చేస్తుంది. తదుపరి learning కోసం, [Improve application reliability with effective SLOs](https://aws.amazon.com/blogs/mt/improve-application-reliability-with-effective-slos) blog refer చేయండి. Customer satisfaction paramount అయినప్పటికీ, ఇది మీ applications performance మరియు reliability తో directly tied. ఈ crucial aspects ను monitor మరియు improve చేయడం ఎలాగో explore చేద్దాం.

#### Application performance మరియు reliability improve చేయడం

[Amazon Managed Grafana](https://docs.aws.amazon.com/grafana/latest/userguide/what-is-Amazon-Managed-Service-Grafana.html) తో, మీ operational data ను scale లో visualize మరియు analyze చేయవచ్చు. AWS data sources తో seamless integration మరియు unified dashboards ద్వారా cross-team collaboration enable చేయడం ద్వారా, multiple sources నుండి observability data ను - metrics, logs, మరియు traces include చేసి - customizable visualizations లో consolidate చేయడానికి allow చేస్తుంది ఇది operational issues ను quickly identify మరియు resolve చేయడానికి help చేస్తుంది.
