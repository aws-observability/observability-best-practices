# AWS CloudTrail రిసోర్స్‌లు

మీ **AWS CloudTrail** భద్రతా పద్ధతులను బలపరచడానికి, incident response ను మెరుగుపరచడానికి, మరియు మానిటరింగ్ సామర్థ్యాలను పెంచడానికి క్యూరేటెడ్ రిసోర్స్‌లను అన్వేషించండి. ఈ పేజీ రెండు విభాగాలుగా విభజించబడింది: ఆచరణాత్మక మార్గదర్శకత్వం కోసం **AWS బ్లాగ్ పోస్ట్‌లు** మరియు సాధారణ భద్రతా దృశ్యాల hands-on సిమ్యులేషన్‌ల కోసం **AWS CloudTrail వర్క్‌షాప్‌లు**.

## AWS బ్లాగ్ పోస్ట్‌లు

ఈ బ్లాగ్ పోస్ట్‌లు మీ AWS ఎన్విరాన్‌మెంట్‌ను సురక్షితం చేయడానికి, సంఘటనలను గుర్తించడానికి, మరియు ప్రతిస్పందనలను ఆటోమేట్ చేయడానికి AWS CloudTrail ను ఉపయోగించడం కోసం చర్య తీసుకోగల అంతర్దృష్టులను అందిస్తాయి. సులభ నావిగేషన్ కోసం రిసోర్స్‌లు దృష్టి ప్రాంతం ప్రకారం సమూహం చేయబడ్డాయి.


### సంఘటన ఇన్వెస్టిగేషన్ మరియు ఫోరెన్సిక్స్
- **[CloudTrail Lake తో భద్రతా ఈవెంట్‌లను ఇన్వెస్టిగేట్ చేయండి](https://aws.amazon.com/blogs/security/investigate-security-events-by-using-aws-cloudtrail-lake-advanced-queries/)**  
  రాజీపడిన యాక్సెస్ కీల వంటి భద్రతా సంఘటనలను విశ్లేషించడానికి CloudTrail Lake క్వెరీలను ఉపయోగించండి.
- **[CloudTrail Insights: అసాధారణ API కార్యాచరణను గుర్తించండి](https://aws.amazon.com/blogs/aws/announcing-cloudtrail-insights-identify-and-respond-to-unusual-api-activity/)**  
  CloudTrail Insights తో అనామలస్ API కాల్‌లను గుర్తించడం మరియు ప్రతిస్పందించడం నేర్చుకోండి.
- **[CloudTrail Lake ప్రకటన – Managed Audit and Security Lake](https://aws.amazon.com/blogs/mt/announcing-aws-cloudtrail-lake-a-managed-audit-and-security-lake/)**  
  కేంద్రీకృత audit మరియు భద్రతా లాగ్ నిర్వహణ కోసం CloudTrail Lake ను కనుగొనండి.

### మానిటరింగ్ మరియు అలర్ట్‌లు
- **[Root User కార్యాచరణను మానిటర్ చేయండి](https://aws.amazon.com/blogs/mt/monitor-and-notify-on-aws-account-root-user-activity/)**  
  మెరుగైన భద్రత కోసం AWS అకౌంట్ root user కార్యాచరణను ట్రాక్ చేయడానికి నోటిఫికేషన్‌లను సెట్ చేయండి.
- **[IAM Access Key వినియోగం కోసం అలర్ట్‌లు](https://repost.aws/knowledge-center/track-access-key-credential)**  
  IAM access keys ఉపయోగించినప్పుడు గుర్తించడానికి అలర్ట్‌లను కాన్ఫిగర్ చేయండి.
- **[IAM కాన్ఫిగరేషన్ మార్పుల కోసం నోటిఫికేషన్‌లు](https://aws.amazon.com/blogs/security/how-to-receive-alerts-when-your-iam-configuration-changes/)**  
  IAM కాన్ఫిగరేషన్‌లలో మార్పుల కోసం అలర్ట్‌లను అమలు చేయండి.
- **[IAM User సృష్టిని మానిటర్ చేయండి](https://docs.aws.amazon.com/prescriptive-guidance/latest/patterns/send-a-notification-when-an-iam-user-is-created.html)**  
  అనధికార మార్పులను గుర్తించడానికి కొత్త IAM user సృష్టి కోసం నోటిఫికేషన్‌లను సెట్ చేయండి.
- **[Account మరియు EC2 Security Group మార్పులను ట్రాక్ చేయండి](https://aws.amazon.com/blogs/security/how-to-monitor-aws-account-configuration-changes-and-api-calls-to-amazon-ec2-security-groups/)**  
  కాన్ఫిగరేషన్ మార్పులు మరియు EC2 security groups కు API కాల్‌లను మానిటర్ చేయండి.
- **[CloudWatch లో CloudTrail ను విశ్లేషించండి](https://aws.amazon.com/blogs/mt/analyzing-cloudtrail-in-cloudwatch/)**  
  రియల్-టైమ్ లాగ్ విశ్లేషణ కోసం CloudTrail ను CloudWatch తో ఇంటిగ్రేట్ చేయండి.

## AWS CloudTrail వర్క్‌షాప్‌లు

ఈ hands-on వర్క్‌షాప్‌లు సాధారణ భద్రతా ఈవెంట్‌లను సిమ్యులేట్ చేస్తాయి మరియు AWS CloudTrail మరియు సంబంధిత సేవలను ఉపయోగించి గుర్తింపు మరియు ప్రతిస్పందనను ప్రదర్శిస్తాయి. ప్రతి వర్క్‌షాప్‌లో వాస్తవ-ప్రపంచ దృశ్యాలను పునరావృతం చేయడానికి CloudFormation టెంప్లేట్ లేదా స్క్రిప్ట్ ఉంటుంది.

### వర్క్‌షాప్ జాబితా
- **[అనధికార IAM Credential వినియోగం](https://catalog.us-east-1.prod.workshops.aws/workshops/6a8ad836-10a6-4694-9a3b-f53f193041de/en-US/setup/cloudformation)**  
  **వివరణ**: reconnaissance మరియు privilege escalation ను అనుకరిస్తూ CloudShell స్క్రిప్ట్ ద్వారా అనధికార IAM credential వినియోగాన్ని సిమ్యులేట్ చేయండి.  
  **ఇది ఎందుకు ముఖ్యం**: AWS ఎన్విరాన్‌మెంట్‌లలో సాధారణ దాడి వెక్టర్ అయిన అనధికార ప్రాప్యతను గుర్తించడం మరియు తగ్గించడం నేర్చుకోండి.

- **[S3 పై Ransomware](https://catalog.workshops.aws/aws-cirt-ransomware-simulation-and-detection/en-US)**  
  **వివరణ**: S3 బకెట్‌లను సృష్టించడానికి CloudFormation టెంప్లేట్ ఉపయోగించి CloudShell లో డేటా exfiltration మరియు deletion ద్వారా ransomware ను సిమ్యులేట్ చేయండి.  
  **ఇది ఎందుకు ముఖ్యం**: అనధికార S3 సవరణలను గుర్తించడం ద్వారా డేటా నష్టం లేదా బహిర్గతం నుండి రక్షించుకోండి.

- **[Cryptominer భద్రతా ఈవెంట్‌లు](https://catalog.workshops.aws/aws-cirt-cryptominer-simulation-and-detection/en-US)**  
  **వివరణ**: తెలిసిన domains కు DNS అభ్యర్థనల ద్వారా cryptomining ను సిమ్యులేట్ చేయడానికి CloudFormation ద్వారా EC2 instances ను డిప్లాయ్ చేయండి.  
  **ఇది ఎందుకు ముఖ్యం**: ఖర్చులను తగ్గించడానికి మరియు EC2 instances ను సురక్షితం చేయడానికి రిసోర్స్-ఇంటెన్సివ్ cryptomining ను గుర్తించండి.

- **[IMDSv1 పై SSRF](https://catalog.us-east-1.prod.workshops.aws/workshops/6a8ad836-10a6-4694-9a3b-f53f193041de/en-US/setup/ssrf)**  
  **వివరణ**: IMDSv1 vulnerabilities ద్వారా EC2 instance credentials పై server-side request forgery (SSRF) దాడిని సిమ్యులేట్ చేయండి.  
  **ఇది ఎందుకు ముఖ్యం**: SSRF mitigation ద్వారా instance credentials కు అనధికార ప్రాప్యతను నిరోధించండి.
