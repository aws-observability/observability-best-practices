# AWS X-Ray
 
## శాంప్లింగ్ రూల్స్

X-Ray ఉపయోగించి శాంప్లింగ్ రూల్స్‌ను AWS Console లో, లోకల్ కాన్ఫిగరేషన్ ఫైల్ ద్వారా, లేదా రెండింటి ద్వారా కాన్ఫిగర్ చేయవచ్చు. లోకల్ కాన్ఫిగరేషన్ కన్సోల్‌లో సెట్ చేసిన వాటిని ఓవర్‌రైడ్ చేస్తుంది.

:::info
	సాధ్యమైనప్పుడల్లా X-Ray console, API లేదా CloudFormation ను ఉపయోగించండి. ఇది రన్‌టైమ్‌లో అప్లికేషన్ యొక్క శాంప్లింగ్ ప్రవర్తనను మార్చడానికి మిమ్మల్ని అనుమతిస్తుంది.
:::
మీరు ఈ ప్రతి ప్రమాణం కోసం శాంపిల్ రేట్‌లను విడిగా సెట్ చేయవచ్చు:

* సేవ పేరు (ఉదా. billing, payments)
* సేవ రకం (ఉదా. EC2, Container)
* HTTP method
* URL path
* Resource ARN
* Host (ఉదా. www.example.com)

సమస్యలను నిర్ధారించడానికి మరియు పనితీరు ప్రొఫైల్‌లను అర్థం చేసుకోవడానికి తగినంత డేటాను సేకరించే శాంపిల్ రేట్‌ను సెట్ చేయడం ఉత్తమ పద్ధతి, అయితే అది నిర్వహించలేనంత ఎక్కువ డేటాను సేకరించకూడదు. ఉదాహరణకు, ల్యాండింగ్ పేజీకి ట్రాఫిక్‌లో 1% ను శాంపిల్ చేయడం, కానీ పేమెంట్ పేజీకి అభ్యర్థనల్లో 10% ను శాంపిల్ చేయడం, బలమైన observability ప్రాక్టీస్‌తో బాగా సరిపోతుంది.

కొన్ని లావాదేవీలను మీరు 100% సంగ్రహించాలనుకోవచ్చు. అయితే జాగ్రత్తగా ఉండండి ఎందుకంటే ట్రేసెస్ మీ వర్క్‌లోడ్‌కు ప్రాప్యత యొక్క ఫోరెన్సిక్ ఆడిట్‌ల కోసం ఉద్దేశించబడలేదు!

:::warning
	ట్రేసెస్ ఆడిటింగ్ లేదా ఫోరెన్సిక్ విశ్లేషణ కోసం ఉపయోగించబడటానికి ఉద్దేశించబడలేదు కాబట్టి, 100% శాంపిల్ రేట్‌లను నివారించండి. ఇది X-Ray (డిఫాల్ట్‌గా UDP ఎమిటర్ ఉపయోగించి) ఎప్పుడూ ట్రాన్సాక్షన్ ట్రేస్‌ను కోల్పోదనే తప్పుడు అంచనాను సెట్ చేయవచ్చు.
:::
నియమంగా, ట్రాన్సాక్షన్ ట్రేసెస్‌ను సంగ్రహించడం మీ సిబ్బందిపై లేదా మీ AWS బిల్‌పై భారీ భారాన్ని ఎప్పుడూ సృష్టించకూడదు. మీ వర్క్‌లోడ్ ఎమిట్ చేసే డేటా వాల్యూమ్‌ను మీరు నేర్చుకుంటున్నప్పుడు మీ పరిసరానికి ట్రేసెస్‌ను నెమ్మదిగా జోడించండి.

:::info
	డిఫాల్ట్‌గా, X-Ray SDK ప్రతి సెకను మొదటి అభ్యర్థనను మరియు ఏదైనా అదనపు అభ్యర్థనల్లో ఐదు శాతం రికార్డ్ చేస్తుంది.
	మీరు భరించగల reservoir size ను ఎల్లప్పుడూ సెట్ చేయండి. Reservoir size మీరు సంగ్రహించే ప్రతి సెకనుకు గరిష్ట అభ్యర్థనల సంఖ్యను నిర్ధారిస్తుంది. ఇది దుర్మార్గపు దాడి, అవాంఛిత ఛార్జీలు మరియు కాన్ఫిగరేషన్ లోపాల నుండి మిమ్మల్ని రక్షిస్తుంది.
:::
## Daemon కాన్ఫిగరేషన్

X-Ray daemon విశ్లేషణ కోసం X-Ray dataplane కు టెలిమెట్రీని పంపే ప్రయత్నాన్ని ఆఫ్‌లోడ్ చేయడానికి ఉద్దేశించబడింది. అందువల్ల, సోర్స్ అప్లికేషన్ నడుస్తున్న సర్వర్, కంటైనర్ లేదా instance పై ఎక్కువ resources ను ఉపయోగించకూడదు.

:::info
	X-Ray daemon ను మరొక instance లేదా కంటైనర్‌పై నడపడం ఉత్తమ పద్ధతి, తద్వారా concerns యొక్క వేరుచేయడాన్ని అమలు చేయడం మరియు మీ సోర్స్ సిస్టమ్‌ను అడ్డంకి లేకుండా ఉండనివ్వడం.
:::

:::info
	Kubernetes వంటి కంటైనర్ ఆర్కెస్ట్రేషన్ ప్యాటర్న్‌లో, మీ X-Ray daemon ను sidecar గా ఆపరేట్ చేయడం ఒక సాధారణ పద్ధతి.
:::
Daemon సురక్షిత డిఫాల్ట్ సెట్టింగ్‌లను కలిగి ఉంది మరియు చాలా సందర్భాలలో తదుపరి కాన్ఫిగరేషన్ లేకుండా EC2, ECS, EKS లేదా Fargate పరిసరాలలో ఆపరేట్ చేయగలదు. హైబ్రిడ్ మరియు ఇతర క్లౌడ్ పరిసరాల కోసం, మీరు మీ రిమోట్ పరిసరాలను ఇంటిగ్రేట్ చేయడానికి Direct Connect లేదా VPN ఉపయోగిస్తుంటే [VPC endpoint](https://docs.aws.amazon.com/vpc/latest/privatelink/concepts.html) ను ప్రతిబింబించేలా `Endpoint` ను సర్దుబాటు చేయవచ్చు.

:::tip
	మీరు X-Ray daemon ను సోర్స్ అప్లికేషన్‌తో అదే instance లేదా virtual machine పై నడపవలసి వస్తే, X-Ray మీరు భరించగలిగిన దానికంటే ఎక్కువ system resources ను ఉపయోగించదని నిర్ధారించడానికి `TotalBufferSizeMB` సెట్టింగ్‌ను సెట్ చేయడాన్ని పరిగణించండి.
:::
## Annotations

AWS X-Ray మీ ట్రేసెస్‌తో పాటు arbitrary metadata పంపడానికి మద్దతు ఇస్తుంది. వీటిని *annotations* అని పిలుస్తారు. అవి మీ ట్రేసెస్‌ను తార్కికంగా గ్రూప్ చేయడానికి అనుమతించే శక్తివంతమైన ఫీచర్. Annotations ఇండెక్స్ చేయబడతాయి, ఒకే ఎంటిటీకి సంబంధించిన ట్రేసెస్‌ను కనుగొనడానికి సులభమైన మార్గాన్ని అందిస్తాయి.

మీరు X-Ray కోసం auto-instrumentation SDK లను ఉపయోగించినప్పుడు, annotations స్వయంచాలకంగా కనిపించకపోవచ్చు. మీ ట్రేసెస్‌ను చాలా సుసంపన్నం చేసే మరియు X-Ray Insights, మీ annotations ఆధారంగా మెట్రిక్స్, మీ system ప్రవర్తన నుండి అలారాలు మరియు anomaly detection models ను జనరేట్ చేయడానికి మరియు మీ వినియోగదారులను ప్రభావితం చేసే కాంపోనెంట్ గమనించినప్పుడు టికెటింగ్ మరియు రిమీడియేషన్‌ను ఆటోమేట్ చేయడానికి మార్గాలను సృష్టించే వాటిని మీ కోడ్‌కు జోడించాలి.

:::info
	మీ పరిసరంలో డేటా ప్రవాహాన్ని అర్థం చేసుకోవడానికి annotations ను ఉపయోగించండి.
:::

:::info
	మీ annotated traces యొక్క పనితీరు మరియు ఫలితాల ఆధారంగా అలారాలను సృష్టించండి.
:::
