# 遥测

遥测关注的是如何从各种来源收集信号——包括您自己的应用程序和基础设施——并将其路由到消费它们的目的地：

## 来源

我们将来源视为信号的产生之处。来源有两种类型：

1. 您可以控制的事物，即通过埋点获取的应用程序源代码。
1. 您可能使用的其他所有事物，例如不在您（直接）控制下的托管服务。这些类型的来源通常由 AWS 提供，通过 API 暴露信号。

## 代理

为了将信号从来源传输到目的地，您需要某种中间组件，我们称之为代理。这些代理接收或拉取来自来源的信号，并通常通过配置确定信号应该去往何处，还可选择性地支持过滤和聚合。

:::note
    "代理？路由？传输？摄入？"
    人们使用许多术语来指代将信号从来源传输到目的地的过程，包括路由、传输、聚合、摄入等。虽然它们可能含义略有不同，但我们在这里将互换使用它们。我们将规范地把这些中间传输组件称为代理。
:::

## 目的地

信号最终到达的地方，用于消费。无论您是想存储信号以供日后消费、将其展示在 dashboard 上、在特定条件为真时设置告警，还是关联信号。所有这些为您（最终用户）服务的组件都是目的地。
