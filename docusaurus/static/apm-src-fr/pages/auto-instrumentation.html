Documentation CloudWatch APM

Instrumentation automatique

Ce que signifie "instrumentation automatique"

L'instrumentation automatique ajoute l'instrumentation OpenTelemetry au moment de l'execution (sans modifier le code de votre application) afin que le service emette la telemetrie qu'Application Signals utilise pour la decouverte et la surveillance de la sante.

Composants de configuration AWS courants

  • AWS Distro for OpenTelemetry (ADOT) SDK et/ou collecteur
  • Agent CloudWatch pour collecter/exporter la telemetrie dans les configurations prises en charge
  • Activation de la plateforme (module complementaire EKS / sidecar ECS / configuration de l'agent EC2 / couches Lambda)

Commencez ici : Presentation d'Application Signals.

Langages et systemes pris en charge

Application Signals prend en charge les applications Java, Python, Node.js et .NET et est pris en charge/teste sur Amazon EKS, Kubernetes, Amazon ECS et Amazon EC2.

Voir : Systemes pris en charge.

Notes et pieges

  • Node.js ESM : la prise en charge ESM est limitee ; CommonJS est le chemin recommande a moins de suivre les instructions specifiques ESM dans la documentation.
  • Python dans les conteneurs : vous devrez peut-etre configurer PYTHONPATH pour l'auto-instrumentation OTel.
