Concepts et fondamentaux

Concepts fondamentaux

Comprendre les concepts fondamentaux de l'Application Performance Monitoring (APM) est essentiel pour utiliser efficacement CloudWatch Application Signals. Cette section couvre les elements de base qui constituent le fondement de l'observabilite moderne.

Tracage distribue

Le tracage distribue est la pratique du suivi des requetes a travers les systemes distribues. Dans les architectures de microservices, une seule requete utilisateur peut traverser plusieurs services, bases de donnees et API externes. Le tracage distribue offre une visibilite sur l'ensemble de ce parcours.

Composants cles :

  • Trace : Un parcours complet de bout en bout d'une requete a travers votre systeme
  • Span : Une operation unique au sein d'une trace (par ex., une requete de base de donnees ou un appel API)
  • Contexte de trace : Les metadonnees qui connectent les spans a travers les frontieres de services

Traces, spans et segments

Ce sont les structures de donnees fondamentales utilisees dans le tracage distribue :

Traces

Une trace represente le chemin complet d'une requete a travers votre application. Elle contient des informations de chronometrage, des metadonnees et tous les spans qui composent le parcours de la requete.

Spans

Un span represente une unite de travail unique au sein d'une trace. Chaque span possede :

  • Un nom et un type d'operation
  • Des horodatages de debut et de fin
  • Des tags et attributs pour un contexte supplementaire
  • Des references aux spans parents

Segments

Dans la terminologie AWS X-Ray, les segments sont les unites de base des donnees de trace. Chaque segment correspond a un span et contient des informations sur le travail effectue pour cette partie de la trace.

Cartes de services

Les cartes de services fournissent une representation visuelle de l'architecture et des dependances de votre application. Elles montrent :

  • Comment les services se connectent entre eux
  • Le flux de trafic entre les services
  • Les metriques de performance pour chaque service
  • Les taux d'erreur et les schemas de latence

Les cartes de services sont generees automatiquement a partir des donnees de trace et vous aident a comprendre la topologie de votre systeme en un coup d'oeil.

Instrumentation

L'instrumentation est le processus d'ajout de code d'observabilite a vos applications. Il existe plusieurs approches :

Auto-instrumentation

L'instrumentation automatique ne necessite aucune modification du code. Les bibliotheques et agents instrumentent automatiquement les frameworks et bibliotheques courants.

Instrumentation manuelle

L'instrumentation manuelle implique l'ajout de code de tracage a votre application. Cela offre un controle precis mais necessite plus d'effort de developpement.

OpenTelemetry

OpenTelemetry est un framework d'observabilite open source qui fournit des API neutres vis-a-vis des fournisseurs pour l'instrumentation. Il prend en charge plusieurs langages de programmation et peut exporter des donnees vers differents backends.

Modele de donnees

Application Signals utilise un modele de donnees structure pour organiser les donnees de telemetrie :

Metriques

  • Latence : Mesures du temps de reponse (P50, P90, P99)
  • Debit : Taux de requetes (requetes par seconde)
  • Taux d'erreur : Pourcentage de requetes echouees
  • Saturation : Utilisation des ressources (CPU, memoire, etc.)

Traces

Donnees de trace structurees avec des spans, des tags et des informations de chronometrage.

Logs

Logs applicatifs correles avec les traces et les metriques pour un contexte complet.

Strategies d'echantillonnage

L'echantillonnage controle quelles requetes sont tracees pour equilibrer l'observabilite avec le cout et la performance :

Echantillonnage en tete

Decide s'il faut tracer une requete au debut de son cycle de vie.

Echantillonnage en queue

Prend des decisions d'echantillonnage apres la collecte des donnees de trace, permettant un echantillonnage plus intelligent base sur les caracteristiques de la requete.

Echantillonnage base sur le taux

Trace un pourcentage fixe des requetes (par ex., 10% de toutes les requetes).
