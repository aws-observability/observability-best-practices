Vues intégrées d'Application Signals

Application Signals fournit un flux de travail centré sur les applications dans la console CloudWatch :

  • Services : lister les services, repérer les pannes/latences élevées, voir l'état des SLI lorsque des SLO existent.
  • Détails du service : examiner en détail les opérations, les dépendances et les traces corrélées.
  • Carte des applications : visualiser la topologie et propager l'état de santé à travers les dépendances.

Voir : Surveillance de la santé opérationnelle avec Application Signals.

Tableaux de bord CloudWatch (personnalisés)

Vous pouvez également créer des tableaux de bord CloudWatch en utilisant les métriques émises par Application Signals (espace de noms ApplicationSignals) et les corréler avec des alarmes/journaux selon vos besoins.

  • Métriques de service : latence, pannes, erreurs selon les dimensions Service/Opération/Environnement.
  • Métriques de dépendance : dimensions de service/opération distants pour les appels en aval.
  • Métriques d'exécution (lorsque pris en charge) : corréler GC/CPU/mémoire avec les pics de latence.

Voir : Métriques collectées par Application Signals.
