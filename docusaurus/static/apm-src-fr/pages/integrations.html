Documentation CloudWatch APM

Integrations

Integrations de services AWS

CloudWatch Application Signals s'integre de maniere transparente avec les services AWS pour fournir une observabilite complete sur l'ensemble de votre infrastructure cloud.

API Gateway

Surveillez la performance des API et suivez les requetes depuis API Gateway vers vos services backend.

Fonctionnalites d'integration :

  • Tracage automatique des appels API
  • Surveillance de la latence pour les points de terminaison API
  • Suivi du taux d'erreur
  • Correlation requete/reponse

Configuration :

Activez le tracage X-Ray dans la configuration de votre etape API Gateway.

Application Load Balancer (ALB)

Suivez le flux de trafic a travers les repartiteurs de charge et correlez avec la performance applicative.

Avantages de l'integration :

  • Visibilite du routage des requetes
  • Metriques de performance des groupes cibles
  • Surveillance des controles de sante
  • Analyse de la repartition de charge

DynamoDB

Surveillez les operations de base de donnees et identifiez les goulots d'etranglement de performance dans les schemas d'acces aux donnees.

Capacites de tracage :

  • Chronometrage des operations de requete
  • Suivi des limitations et des erreurs
  • Performance des tables et des index
  • Surveillance de la replication inter-regions

Amazon RDS

Obtenez des informations sur la performance de la base de donnees et le pool de connexions.

Surveillance de base de donnees :

  • Temps d'execution des requetes
  • Utilisation du pool de connexions
  • Metriques de l'instance de base de donnees
  • Identification des requetes lentes

Amazon S3

Suivez les operations de stockage d'objets et la performance du transfert de donnees.

Operations de stockage :

  • Tracage Put/Get d'objets
  • Surveillance de la latence de transfert
  • Analyse du taux d'erreur
  • Suivi des transferts inter-regions

SNS/SQS

Surveillez le traitement des messages et la performance des files d'attente dans les architectures evenementielles.

Informations sur la messagerie :

  • Tracage de la publication/abonnement des messages
  • Surveillance de la profondeur de file d'attente
  • Latence de traitement des messages
  • Analyse des files de lettres mortes

AWS Step Functions

Visualisez l'execution des flux de travail et identifiez les goulots d'etranglement dans le traitement des machines a etats.

Surveillance des flux de travail :

  • Tracage des transitions d'etat
  • Analyse du temps d'execution
  • Identification des etats d'erreur
  • Surveillance de l'execution parallele

Amazon EventBridge

Suivez le routage et le traitement des evenements dans les systemes evenementiels.

Traitement des evenements :

  • Visibilite du routage des evenements
  • Chronometrage de l'evaluation des regles
  • Tracage de l'invocation des cibles
  • Surveillance de la transformation des evenements

AWS Lambda

Surveillez la performance des fonctions serverless et l'impact des demarrages a froid.

Informations serverless :

  • Tracage de l'invocation des fonctions
  • Duree de demarrage a froid
  • Utilisation de la memoire et du CPU
  • Suivi des erreurs et des delais d'expiration

Amazon ECS/EKS

Integration de la plateforme d'orchestration de conteneurs pour la surveillance des microservices.

Surveillance des conteneurs :

  • Tracage du maillage de services
  • Communication pod a pod
  • Utilisation des ressources
  • Suivi des deploiements

Integrations tierces

Etendez Application Signals avec des outils et plateformes tiers.

OpenTelemetry

Tirez parti de l'ecosysteme OpenTelemetry pour une observabilite neutre vis-a-vis des fournisseurs.

Options d'integration :

  • Bibliotheques d'instrumentation personnalisees
  • Prise en charge d'agents tiers
  • Export de donnees multi-backend
  • Contributions de la communaute au collecteur

Fournisseurs APM

Migrez depuis ou integrez avec d'autres solutions APM.

Integrations prises en charge :

  • Export de donnees vers des outils APM tiers
  • Observabilite cloud hybride
  • Correlation multi-fournisseurs
  • Utilitaires de migration de donnees

Surveillance et alertes

Connectez Application Signals avec votre pile de surveillance existante.

Partenaires d'integration :

  • PagerDuty : Reponse aux incidents et alertes
  • Slack : Notifications d'equipe
  • Datadog : Plateforme de surveillance unifiee
  • New Relic : Surveillance de la performance applicative
  • Splunk : Analyse et correlation des logs

Integration CI/CD

Integrez l'observabilite dans votre pipeline de developpement.

Outils DevOps :

  • GitHub Actions : Tests et deploiements automatises
  • Jenkins : Integration au pipeline de build
  • GitLab CI : Flux DevSecOps
  • Terraform : Infrastructure en tant que code
  • AWS CDK : Provisionnement d'infrastructure cloud

Integrations personnalisees

Construisez vos propres integrations a l'aide d'API et de SDK.

API CloudWatch

Utilisez les API CloudWatch pour creer des tableaux de bord personnalises, des alarmes et des flux de travail pilotes par les metriques autour de la telemetrie qu'Application Signals publie vers CloudWatch.

API disponibles :

  • PutMetricData - Envoyer des metriques personnalisees
  • GetMetricData - Recuperer des donnees de metriques
  • PutDashboard / GetDashboard - Gerer les tableaux de bord
  • PutMetricAlarm / DescribeAlarms - Gerer les alarmes

API AWS X-Ray

Application Signals utilise AWS X-Ray comme magasin de traces. Si vous avez besoin d'une integration directe d'ingestion/requete de traces, utilisez les API X-Ray.

  • PutTraceSegments - Ingerer des segments de trace
  • GetTraceSummaries / BatchGetTraces - Interroger les traces

Webhooks et callbacks

Recevez des notifications en temps reel et integrez avec des flux de travail personnalises.

Schemas d'integration :

  • Webhooks d'alerte pour les systemes de notification personnalises
  • Traitement evenementiel avec Lambda
  • Mises a jour de tableaux de bord personnalises
  • Flux de remediation automatises

SDK et bibliotheques

L'instrumentation se fait generalement via OpenTelemetry et/ou AWS Distro for OpenTelemetry (ADOT). Pour les appels d'API AWS, utilisez les SDK AWS avec les instrumentations OpenTelemetry prises en charge.

Langages pris en charge :

  • JavaScript/Node.js : AWS SDK pour JavaScript
  • Python : SDK AWS Boto3
  • Java : AWS SDK pour Java
  • .NET : AWS SDK pour .NET
  • Go : AWS SDK pour Go
