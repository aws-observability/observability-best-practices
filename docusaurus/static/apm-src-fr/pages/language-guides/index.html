Metriques personnalisees (a partir des exemples apm-center)

Certains services d'exemple dans apm-center incluent un point de terminaison utilise pour exercer le comportement des "metriques personnalisees" pendant les tests. Par exemple, l'exemple EKS Python fournit GET /api/metrics :

Ouvrir le guide d'activation EKS Python

@app.get("/api/metrics")
async def metrics():
    """Custom metrics endpoint"""
    # Simulate some business logic
    processing_time = random.uniform(0.1, 0.5)
    time.sleep(processing_time)

    # Simulate different response times based on some logic
    if processing_time > 0.3:
        status = "slow"
    else:
        status = "fast"

    return {
        "message": "Metrics endpoint",
        "processing_time": processing_time,
        "status": status,
        "timestamp": time.time(),
        "service": "python-eks-service"
    }
