Sécurité et conformité

Sécurité

CloudWatch Application Signals met en œuvre plusieurs couches de sécurité pour protéger vos données d'observabilité et assurer la conformité aux normes du secteur.

Chiffrement des données

Chiffrement au repos :

  • Toutes les données de traces sont chiffrées avec le chiffrement AES-256
  • Les données RUM sont chiffrées avec AWS Key Management Service (KMS)
  • Les données de surveillance synthétique sont stockées de manière chiffrée
  • Vous pouvez utiliser des clés gérées par le client pour un contrôle supplémentaire

Chiffrement en transit :

  • Toutes les transmissions de données utilisent TLS 1.2 ou supérieur
  • Les appels API aux services CloudWatch sont chiffrés
  • La collecte de données RUM utilise HTTPS
  • La communication entre l'agent et le collecteur est sécurisée

Gestion des identités et des accès

Permissions IAM :

Utilisez le principe du moindre privilège lors de l'octroi d'accès à Application Signals :

Accès en lecture seule :
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
Accès complet :
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

Confidentialité des données et protection des informations personnelles

Assainissement des données :

  • Mettre en œuvre le masquage des données pour les informations sensibles
  • Utiliser la tokenisation pour les informations personnelles dans les attributs de traces
  • Configurer des politiques de rétention des données
  • Audit régulier des données collectées

Bonnes pratiques :

  • Ne jamais inclure de mots de passe, clés API ou jetons dans les traces
  • Hacher ou masquer les adresses e-mail et les numéros de téléphone
  • Supprimer les paramètres de requête des URL contenant des données sensibles
  • Utiliser des identifiants génériques au lieu d'informations personnelles

Sécurité réseau

Configuration VPC :

  • Déployer les agents dans votre VPC pour une sécurité renforcée
  • Utiliser des points de terminaison VPC pour les appels API CloudWatch
  • Configurer les groupes de sécurité pour restreindre le trafic
  • Implémenter des ACL réseau pour un contrôle supplémentaire

Sécurité de l'agent :

  • Exécuter les agents avec les permissions minimales requises
  • Maintenir les agents à jour avec les correctifs de sécurité
  • Surveiller les journaux des agents pour les événements de sécurité
  • Utiliser un accès en lecture seule lorsque possible

Confidentialité des données

Comprendre comment Application Signals gère vos exigences en matière de confidentialité des données.

Collecte et utilisation des données

Ce que nous collectons :

  • Métriques de performance des applications et traces
  • Informations sur les dépendances de services
  • Informations sur les erreurs et traces de pile
  • Métadonnées système et d'infrastructure

Ce que nous ne collectons pas :

  • Données métier des applications ou contenu utilisateur
  • Contenu des bases de données ou données du système de fichiers
  • Identifiants utilisateur ou jetons d'authentification
  • Informations personnellement identifiables (sauf si explicitement incluses)

Rétention des données

Périodes de rétention par défaut :

  • Données de traces : 30 jours (configurable jusqu'à 1 an)
  • Métriques : 15 mois (rétention standard CloudWatch)
  • Données RUM : 30 jours (configurable jusqu'à 1 an)
  • Résultats Synthetics : 90 jours (configurable)

Suppression des données :

  • Utiliser les politiques de rétention des données CloudWatch
  • Implémenter une gestion automatisée du cycle de vie des données
  • Demander la suppression des données via AWS Support si nécessaire

Cadres de conformité

Normes prises en charge :

  • SOC 2 : Contrôles de sécurité, disponibilité et confidentialité
  • PCI DSS : Normes de sécurité des données de l'industrie des cartes de paiement
  • HIPAA : Loi sur la portabilité et la responsabilité de l'assurance maladie
  • RGPD : Règlement général sur la protection des données
  • FedRAMP : Programme fédéral de gestion des risques et des autorisations

Guide de conformité

Conseils spécifiques pour les scénarios de conformité courants.

Conformité HIPAA

Exigences :

  • Chiffrer les informations de santé protégées (PHI) dans les traces
  • Implémenter des contrôles d'accès et une journalisation d'audit
  • Utiliser des accords d'association commerciale (BAA) avec AWS
  • Configurer des politiques de rétention des données appropriées

Mise en œuvre :

  • Masquer les PHI dans les attributs de traces et les noms de spans
  • Utiliser des clés KMS gérées par le client
  • Activer CloudTrail pour la journalisation d'audit
  • Configurer des points de terminaison VPC pour l'isolation des données

Conformité PCI DSS

Considérations clés :

  • Ne jamais inclure de données de titulaires de carte dans les traces ou les journaux
  • Implémenter la segmentation réseau
  • Utiliser le chiffrement pour les données en transit et au repos
  • Évaluations de sécurité régulières et tests de pénétration

Bonnes pratiques :

  • Tokeniser les données sensibles avant l'instrumentation
  • Utiliser des sous-réseaux privés pour les agents Application Signals
  • Implémenter des politiques IAM de moindre privilège
  • Revue régulière des données collectées

Conformité RGPD

Droits des personnes concernées :

  • Droit d'accès aux données personnelles
  • Droit de rectification
  • Droit à l'effacement (« droit à l'oubli »)
  • Droit à la portabilité des données

Mise en œuvre :

  • Minimiser la collecte d'informations personnelles dans les données de télémétrie
  • Implémenter des techniques d'anonymisation des données
  • Configurer la rétention des données en fonction des besoins métier
  • Documenter les activités de traitement des données

Surveillance de la sécurité

Surveiller votre déploiement Application Signals pour les événements de sécurité.

Bonnes pratiques de sécurité

  • Activer CloudTrail pour la journalisation de l'activité API
  • Surveiller les modifications de politiques IAM et les modèles d'accès
  • Configurer des alertes pour les modèles d'accès inhabituels aux données
  • Évaluations de sécurité régulières et analyses de vulnérabilités
  • Implémenter l'authentification multi-facteurs (MFA)

Réponse aux incidents

  • Disposer d'un plan de réponse aux incidents pour les événements de sécurité
  • Savoir comment désactiver Application Signals en cas de compromission
  • Documenter les procédures d'escalade
  • Tests réguliers de la réponse aux incidents
