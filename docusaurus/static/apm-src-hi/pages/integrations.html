CloudWatch APM दस्तावेज़ीकरण

एकीकरण

AWS सर्विस एकीकरण

CloudWatch Application Signals आपके संपूर्ण क्लाउड इंफ्रास्ट्रक्चर में व्यापक observability प्रदान करने के लिए AWS सर्विसेज़ के साथ निर्बाध रूप से एकीकृत होता है।

API Gateway

API प्रदर्शन की निगरानी करें और API Gateway से आपकी बैकएंड सर्विसेज़ तक अनुरोधों को ट्रैक करें।

एकीकरण सुविधाएं:

  • API कॉल की स्वचालित ट्रेसिंग
  • API एंडपॉइंट के लिए लेटेंसी मॉनिटरिंग
  • एरर रेट ट्रैकिंग
  • रिक्वेस्ट/रिस्पॉन्स सहसंबंध

सेटअप:

अपने API Gateway स्टेज कॉन्फ़िगरेशन में X-Ray ट्रेसिंग सक्षम करें।

Application Load Balancer (ALB)

लोड बैलेंसर के माध्यम से ट्रैफ़िक प्रवाह को ट्रैक करें और एप्लिकेशन प्रदर्शन के साथ सहसंबंधित करें।

एकीकरण लाभ:

  • रिक्वेस्ट रूटिंग विज़िबिलिटी
  • टारगेट ग्रुप प्रदर्शन मेट्रिक्स
  • हेल्थ चेक मॉनिटरिंग
  • लोड वितरण विश्लेषण

DynamoDB

डेटाबेस ऑपरेशन की निगरानी करें और डेटा एक्सेस पैटर्न में प्रदर्शन बाधाओं की पहचान करें।

ट्रेसिंग क्षमताएं:

  • क्वेरी ऑपरेशन टाइमिंग
  • थ्रॉटल और एरर ट्रैकिंग
  • टेबल और इंडेक्स प्रदर्शन
  • क्रॉस-रीजन रेप्लिकेशन मॉनिटरिंग

Amazon RDS

डेटाबेस प्रदर्शन और कनेक्शन पूलिंग में अंतर्दृष्टि प्राप्त करें।

डेटाबेस मॉनिटरिंग:

  • क्वेरी निष्पादन समय
  • कनेक्शन पूल उपयोग
  • डेटाबेस इंस्टेंस मेट्रिक्स
  • धीमी क्वेरी पहचान

Amazon S3

ऑब्जेक्ट स्टोरेज ऑपरेशन और डेटा ट्रांसफर प्रदर्शन को ट्रैक करें।

स्टोरेज ऑपरेशन:

  • Put/Get ऑब्जेक्ट ट्रेसिंग
  • ट्रांसफर लेटेंसी मॉनिटरिंग
  • एरर रेट विश्लेषण
  • क्रॉस-रीजन ट्रांसफर ट्रैकिंग

SNS/SQS

इवेंट-ड्रिवन आर्किटेक्चर में मैसेज प्रोसेसिंग और क्यू प्रदर्शन की निगरानी करें।

मैसेजिंग अंतर्दृष्टि:

  • मैसेज पब्लिश/सब्सक्राइब ट्रेसिंग
  • क्यू डेप्थ मॉनिटरिंग
  • मैसेज प्रोसेसिंग लेटेंसी
  • डेड लेटर क्यू विश्लेषण

AWS Step Functions

वर्कफ़्लो निष्पादन को विज़ुअलाइज़ करें और स्टेट मशीन प्रोसेसिंग में बाधाओं की पहचान करें।

वर्कफ़्लो मॉनिटरिंग:

  • स्टेट ट्रांज़िशन ट्रेसिंग
  • निष्पादन समय विश्लेषण
  • एरर स्टेट पहचान
  • पैरेलल निष्पादन मॉनिटरिंग

Amazon EventBridge

इवेंट-ड्रिवन सिस्टम में इवेंट रूटिंग और प्रोसेसिंग को ट्रैक करें।

इवेंट प्रोसेसिंग:

  • इवेंट रूटिंग विज़िबिलिटी
  • रूल इवैल्यूएशन टाइमिंग
  • टारगेट इन्वोकेशन ट्रेसिंग
  • इवेंट ट्रांसफ़ॉर्मेशन मॉनिटरिंग

AWS Lambda

सर्वरलेस फ़ंक्शन प्रदर्शन और कोल्ड स्टार्ट प्रभावों की निगरानी करें।

सर्वरलेस अंतर्दृष्टि:

  • फ़ंक्शन इन्वोकेशन ट्रेसिंग
  • कोल्ड स्टार्ट अवधि
  • मेमोरी और CPU उपयोग
  • एरर और टाइमआउट ट्रैकिंग

Amazon ECS/EKS

माइक्रोसर्विसेज़ मॉनिटरिंग के लिए कंटेनर ऑर्केस्ट्रेशन प्लेटफ़ॉर्म एकीकरण।

कंटेनर मॉनिटरिंग:

  • सर्विस मेश ट्रेसिंग
  • पॉड-टू-पॉड संचार
  • रिसोर्स उपयोग
  • डिप्लॉयमेंट ट्रैकिंग

थर्ड-पार्टी एकीकरण

थर्ड-पार्टी टूल और प्लेटफ़ॉर्म के साथ Application Signals का विस्तार करें।

OpenTelemetry

वेंडर-न्यूट्रल observability के लिए OpenTelemetry इकोसिस्टम का लाभ उठाएं।

एकीकरण विकल्प:

  • कस्टम इंस्ट्रूमेंटेशन लाइब्रेरी
  • थर्ड-पार्टी एजेंट सपोर्ट
  • मल्टी-बैकएंड डेटा एक्सपोर्ट
  • कम्युनिटी कलेक्टर योगदान

APM वेंडर

अन्य APM समाधानों से माइग्रेट करें या उनके साथ एकीकृत करें।

समर्थित एकीकरण:

  • थर्ड-पार्टी APM टूल में डेटा एक्सपोर्ट
  • हाइब्रिड क्लाउड observability
  • मल्टी-वेंडर सहसंबंध
  • डेटा माइग्रेशन यूटिलिटी

मॉनिटरिंग और अलर्टिंग

Application Signals को अपने मौजूदा मॉनिटरिंग स्टैक के साथ कनेक्ट करें।

एकीकरण भागीदार:

  • PagerDuty: इंसिडेंट रिस्पॉन्स और अलर्टिंग
  • Slack: टीम नोटिफ़िकेशन
  • Datadog: यूनिफ़ाइड मॉनिटरिंग प्लेटफ़ॉर्म
  • New Relic: एप्लिकेशन परफ़ॉर्मेंस मॉनिटरिंग
  • Splunk: लॉग विश्लेषण और सहसंबंध

CI/CD एकीकरण

अपनी डेवलपमेंट पाइपलाइन में observability शामिल करें।

DevOps टूल:

  • GitHub Actions: स्वचालित परीक्षण और डिप्लॉयमेंट
  • Jenkins: बिल्ड पाइपलाइन एकीकरण
  • GitLab CI: DevSecOps वर्कफ़्लो
  • Terraform: इंफ्रास्ट्रक्चर एज़ कोड
  • AWS CDK: क्लाउड इंफ्रास्ट्रक्चर प्रोविज़निंग

कस्टम एकीकरण

API और SDK का उपयोग करके अपने स्वयं के एकीकरण बनाएं।

CloudWatch API

Application Signals द्वारा CloudWatch में प्रकाशित टेलीमेट्री के आधार पर कस्टम डैशबोर्ड, अलार्म और मेट्रिक-ड्रिवन वर्कफ़्लो बनाने के लिए CloudWatch API का उपयोग करें।

उपलब्ध API:

  • PutMetricData - कस्टम मेट्रिक्स भेजें
  • GetMetricData - मेट्रिक डेटा प्राप्त करें
  • PutDashboard / GetDashboard - डैशबोर्ड प्रबंधित करें
  • PutMetricAlarm / DescribeAlarms - अलार्म प्रबंधित करें

AWS X-Ray API

Application Signals ट्रेस स्टोर के रूप में AWS X-Ray का उपयोग करता है। यदि आपको सीधे ट्रेस इनजेशन/क्वेरी एकीकरण की आवश्यकता है, तो X-Ray API का उपयोग करें।

  • PutTraceSegments - ट्रेस सेगमेंट इनजेस्ट करें
  • GetTraceSummaries / BatchGetTraces - ट्रेसेज़ क्वेरी करें

वेबहुक और कॉलबैक

रियल-टाइम नोटिफ़िकेशन प्राप्त करें और कस्टम वर्कफ़्लो के साथ एकीकृत करें।

एकीकरण पैटर्न:

  • कस्टम नोटिफ़िकेशन सिस्टम के लिए अलर्ट वेबहुक
  • Lambda के साथ इवेंट-ड्रिवन प्रोसेसिंग
  • कस्टम डैशबोर्ड अपडेट
  • स्वचालित रेमेडिएशन वर्कफ़्लो

SDK और लाइब्रेरी

इंस्ट्रूमेंटेशन आमतौर पर OpenTelemetry और/या AWS Distro for OpenTelemetry (ADOT) के माध्यम से किया जाता है। AWS API कॉल के लिए, समर्थित OpenTelemetry इंस्ट्रूमेंटेशन के साथ AWS SDK का उपयोग करें।

समर्थित भाषाएं:

  • JavaScript/Node.js: AWS SDK for JavaScript
  • Python: Boto3 AWS SDK
  • Java: AWS SDK for Java
  • .NET: AWS SDK for .NET
  • Go: AWS SDK for Go
