CloudWatch APM दस्तावेज़ीकरण

सुरक्षा और अनुपालन

सुरक्षा

CloudWatch Application Signals आपके observability डेटा की सुरक्षा और उद्योग मानकों के अनुपालन को सुनिश्चित करने के लिए सुरक्षा की कई परतें लागू करता है।

डेटा एन्क्रिप्शन

विश्राम में एन्क्रिप्शन:

  • सभी ट्रेस डेटा AES-256 एन्क्रिप्शन का उपयोग करके एन्क्रिप्ट किया जाता है
  • RUM डेटा AWS Key Management Service (KMS) के साथ एन्क्रिप्ट किया जाता है
  • सिंथेटिक मॉनिटरिंग डेटा एन्क्रिप्टेड संग्रहीत किया जाता है
  • अतिरिक्त नियंत्रण के लिए आप कस्टमर-मैनेज्ड कीज़ का उपयोग कर सकते हैं

पारगमन में एन्क्रिप्शन:

  • सभी डेटा ट्रांसमिशन TLS 1.2 या उच्चतर का उपयोग करता है
  • CloudWatch सर्विसेज़ के API कॉल एन्क्रिप्टेड हैं
  • RUM डेटा संग्रह HTTPS का उपयोग करता है
  • एजेंट-टू-कलेक्टर संचार सुरक्षित है

पहचान और एक्सेस प्रबंधन

IAM अनुमतियां:

Application Signals तक पहुंच प्रदान करते समय न्यूनतम विशेषाधिकार के सिद्धांत का उपयोग करें:

रीड-ओनली एक्सेस:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
पूर्ण एक्सेस:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

डेटा गोपनीयता और PII सुरक्षा

डेटा सैनिटाइज़ेशन:

  • संवेदनशील जानकारी के लिए डेटा मास्किंग लागू करें
  • ट्रेस एट्रिब्यूट में PII के लिए टोकनाइज़ेशन का उपयोग करें
  • डेटा रिटेंशन नीतियां कॉन्फ़िगर करें
  • एकत्रित डेटा का नियमित ऑडिट

सर्वोत्तम प्रथाएं:

  • ट्रेसेज़ में कभी भी पासवर्ड, API कीज़, या टोकन शामिल न करें
  • ईमेल एड्रेस और फ़ोन नंबर को हैश या मास्क करें
  • संवेदनशील डेटा वाले URL से क्वेरी पैरामीटर हटाएं
  • व्यक्तिगत जानकारी के बजाय जेनेरिक आइडेंटिफ़ायर का उपयोग करें

नेटवर्क सुरक्षा

VPC कॉन्फ़िगरेशन:

  • बढ़ी हुई सुरक्षा के लिए अपने VPC के भीतर एजेंट डिप्लॉय करें
  • CloudWatch API कॉल के लिए VPC एंडपॉइंट का उपयोग करें
  • ट्रैफ़िक को प्रतिबंधित करने के लिए सिक्योरिटी ग्रुप कॉन्फ़िगर करें
  • अतिरिक्त नियंत्रण के लिए नेटवर्क ACL लागू करें

एजेंट सुरक्षा:

  • न्यूनतम आवश्यक अनुमतियों के साथ एजेंट चलाएं
  • सुरक्षा पैच के साथ एजेंट अपडेट रखें
  • सुरक्षा इवेंट के लिए एजेंट लॉग की निगरानी करें
  • जहां संभव हो रीड-ओनली एक्सेस का उपयोग करें

डेटा गोपनीयता

Application Signals आपकी डेटा गोपनीयता आवश्यकताओं को कैसे संभालता है यह समझना।

डेटा संग्रह और उपयोग

हम क्या एकत्र करते हैं:

  • एप्लिकेशन प्रदर्शन मेट्रिक्स और ट्रेसेज़
  • सर्विस डिपेंडेंसी जानकारी
  • एरर जानकारी और स्टैक ट्रेस
  • सिस्टम और इंफ्रास्ट्रक्चर मेटाडेटा

हम क्या एकत्र नहीं करते:

  • एप्लिकेशन बिज़नेस डेटा या उपयोगकर्ता सामग्री
  • डेटाबेस सामग्री या फ़ाइल सिस्टम डेटा
  • उपयोगकर्ता क्रेडेंशियल या ऑथेंटिकेशन टोकन
  • व्यक्तिगत रूप से पहचान योग्य जानकारी (जब तक स्पष्ट रूप से शामिल न की गई हो)

डेटा रिटेंशन

डिफ़ॉल्ट रिटेंशन अवधि:

  • ट्रेस डेटा: 30 दिन (1 वर्ष तक कॉन्फ़िगर करने योग्य)
  • मेट्रिक्स: 15 महीने (मानक CloudWatch रिटेंशन)
  • RUM डेटा: 30 दिन (1 वर्ष तक कॉन्फ़िगर करने योग्य)
  • Synthetics परिणाम: 90 दिन (कॉन्फ़िगर करने योग्य)

डेटा हटाना:

  • CloudWatch डेटा रिटेंशन नीतियों का उपयोग करें
  • स्वचालित डेटा लाइफ़साइकल प्रबंधन लागू करें
  • यदि आवश्यक हो तो AWS सपोर्ट के माध्यम से डेटा हटाने का अनुरोध करें

अनुपालन फ्रेमवर्क

समर्थित मानक:

  • SOC 2: सुरक्षा, उपलब्धता, और गोपनीयता नियंत्रण
  • PCI DSS: पेमेंट कार्ड इंडस्ट्री डेटा सिक्योरिटी मानक
  • HIPAA: हेल्थ इंश्योरेंस पोर्टेबिलिटी एंड अकाउंटेबिलिटी एक्ट
  • GDPR: जनरल डेटा प्रोटेक्शन रेगुलेशन
  • FedRAMP: फ़ेडरल रिस्क एंड ऑथराइज़ेशन मैनेजमेंट प्रोग्राम

अनुपालन मार्गदर्शन

सामान्य अनुपालन परिदृश्यों के लिए विशिष्ट मार्गदर्शन।

HIPAA अनुपालन

आवश्यकताएं:

  • ट्रेसेज़ में संरक्षित स्वास्थ्य जानकारी (PHI) एन्क्रिप्ट करें
  • एक्सेस कंट्रोल और ऑडिट लॉगिंग लागू करें
  • AWS के साथ बिज़नेस एसोसिएट एग्रीमेंट (BAA) का उपयोग करें
  • उचित डेटा रिटेंशन नीतियां कॉन्फ़िगर करें

कार्यान्वयन:

  • ट्रेस एट्रिब्यूट और स्पैन नामों में PHI मास्क करें
  • कस्टमर-मैनेज्ड KMS कीज़ का उपयोग करें
  • ऑडिट लॉगिंग के लिए CloudTrail सक्षम करें
  • डेटा आइसोलेशन के लिए VPC एंडपॉइंट कॉन्फ़िगर करें

PCI DSS अनुपालन

प्रमुख विचार:

  • ट्रेसेज़ या लॉग्स में कभी भी कार्डहोल्डर डेटा शामिल न करें
  • नेटवर्क सेगमेंटेशन लागू करें
  • पारगमन और विश्राम में डेटा के लिए एन्क्रिप्शन का उपयोग करें
  • नियमित सुरक्षा मूल्यांकन और पेनेट्रेशन टेस्टिंग

सर्वोत्तम प्रथाएं:

  • इंस्ट्रूमेंटेशन से पहले संवेदनशील डेटा को टोकनाइज़ करें
  • Application Signals एजेंट के लिए प्राइवेट सबनेट का उपयोग करें
  • न्यूनतम-विशेषाधिकार IAM नीतियां लागू करें
  • एकत्रित डेटा की नियमित समीक्षा

GDPR अनुपालन

डेटा सब्जेक्ट अधिकार:

  • व्यक्तिगत डेटा तक पहुंच का अधिकार
  • सुधार का अधिकार
  • मिटाने का अधिकार ("भूल जाने का अधिकार")
  • डेटा पोर्टेबिलिटी का अधिकार

कार्यान्वयन:

  • टेलीमेट्री डेटा में PII संग्रह को न्यूनतम करें
  • डेटा अनामीकरण तकनीकें लागू करें
  • बिज़नेस आवश्यकताओं के आधार पर डेटा रिटेंशन कॉन्फ़िगर करें
  • डेटा प्रोसेसिंग गतिविधियों का दस्तावेज़ीकरण करें

सुरक्षा मॉनिटरिंग

सुरक्षा इवेंट के लिए अपने Application Signals डिप्लॉयमेंट की निगरानी करें।

सुरक्षा सर्वोत्तम प्रथाएं

  • API गतिविधि लॉगिंग के लिए CloudTrail सक्षम करें
  • IAM नीति परिवर्तन और एक्सेस पैटर्न की निगरानी करें
  • असामान्य डेटा एक्सेस पैटर्न के लिए अलर्ट सेटअप करें
  • नियमित सुरक्षा मूल्यांकन और भेद्यता स्कैन
  • मल्टी-फ़ैक्टर ऑथेंटिकेशन (MFA) लागू करें

इंसिडेंट रिस्पॉन्स

  • सुरक्षा इवेंट के लिए इंसिडेंट रिस्पॉन्स प्लान रखें
  • यदि समझौता हो तो Application Signals को कैसे अक्षम करें यह जानें
  • एस्केलेशन प्रक्रियाओं का दस्तावेज़ीकरण करें
  • नियमित इंसिडेंट रिस्पॉन्स टेस्टिंग
