CloudWatch APM दस्तावेज़ीकरण

समस्या निवारण

सामान्य समस्याएं

यह अनुभाग CloudWatch Application Signals को लागू करते समय सबसे अधिक सामना की जाने वाली समस्याओं और उन्हें कैसे हल करें को कवर करता है।

CloudWatch में ट्रेसेज़ दिखाई नहीं दे रहे

संभावित कारण:

  • सर्विस के लिए Application Signals सक्षम नहीं है
  • गलत IAM अनुमतियां
  • एजेंट ठीक से कॉन्फ़िगर या चल नहीं रहा
  • सैंपलिंग रेट बहुत कम सेट है
  • CloudWatch से नेटवर्क कनेक्टिविटी समस्याएं

समाधान:

  1. सत्यापित करें कि AWS कंसोल में Application Signals सक्षम है
  2. जांचें कि IAM अनुमतियों में cloudwatch:PutMetricData और xray:PutTraceSegments शामिल हैं
  3. पुष्टि करें कि एजेंट चल रहा है और सही ढंग से कॉन्फ़िगर है
  4. परीक्षण के लिए सैंपलिंग रेट बढ़ाएं (अस्थायी रूप से 100% सेट करें)
  5. जांचें कि VPC सिक्योरिटी ग्रुप और NACL CloudWatch को आउटबाउंड ट्रैफ़िक की अनुमति देते हैं

अनुपलब्ध या अपूर्ण ट्रेसेज़

सामान्य समस्याएं:

  • सर्विसेज़ के बीच कॉन्टेक्स्ट प्रोपेगेट नहीं हो रहा
  • कुछ सर्विसेज़ इंस्ट्रूमेंट नहीं हैं
  • सैंपलिंग स्पैन ड्रॉप कर रही है
  • एजेंट कॉन्फ़िगरेशन समस्याएं

डिबगिंग चरण:

  1. अनइंस्ट्रूमेंटेड सर्विसेज़ की पहचान के लिए सर्विस मैप जांचें
  2. सत्यापित करें कि ट्रेस कॉन्टेक्स्ट हेडर पास हो रहे हैं (जैसे, x-amzn-trace-id)
  3. कॉन्फ़िगरेशन एरर के लिए एजेंट लॉग की समीक्षा करें
  4. सैंपलिंग समस्याओं को खारिज करने के लिए 100% सैंपलिंग के साथ टेस्ट करें
  5. आंशिक ट्रेसेज़ खोजने के लिए ट्रेस सर्च का उपयोग करें

उच्च इंस्ट्रूमेंटेशन ओवरहेड

लक्षण:

  • बढ़ा हुआ CPU उपयोग
  • अधिक मेमोरी खपत
  • बढ़ी हुई लेटेंसी
  • एप्लिकेशन प्रदर्शन गिरावट

अनुकूलन रणनीतियां:

  • सैंपलिंग रेट कम करें (प्रोडक्शन के लिए 10% से शुरू करें)
  • टेल-बेस्ड के बजाय हेड-बेस्ड सैंपलिंग का उपयोग करें
  • उचित स्पैन सीमाएं कॉन्फ़िगर करें
  • बैच स्पैन एक्सपोर्ट
  • इंस्ट्रूमेंटेशन मेट्रिक्स की निगरानी करें

RUM डेटा दिखाई नहीं दे रहा

चेकलिस्ट:

  • RUM वेब क्लाइंट ठीक से इंस्टॉल और लोड है
  • एप्लिकेशन ID और कॉन्फ़िगरेशन सही है
  • RUM डोमेन के लिए CORS हेडर कॉन्फ़िगर हैं
  • Content Security Policy RUM स्क्रिप्ट की अनुमति देती है
  • ब्राउज़र कंसोल में कोई JavaScript एरर नहीं दिखता

सत्यापन:

// Check if RUM is loaded
console.log(window.cwr);

// Verify configuration
console.log(cwr('getConfig'));

सिंथेटिक कैनरी विफल हो रहे हैं

सामान्य कारण:

  • कैनरी VPC से नेटवर्क कनेक्टिविटी समस्याएं
  • ऑथेंटिकेशन या ऑथराइज़ेशन विफलताएं
  • एप्लिकेशन परिवर्तन टेस्ट स्क्रिप्ट तोड़ रहे हैं
  • कैनरी रनटाइम या डिपेंडेंसीज़ पुरानी हैं

डिबगिंग:

  1. CloudWatch Logs में कैनरी लॉग की समीक्षा करें
  2. कैनरी स्क्रिप्ट को स्थानीय रूप से टेस्ट करें
  3. VPC रूटिंग और सिक्योरिटी ग्रुप जांचें
  4. यदि आवश्यक हो तो कैनरी रनटाइम अपडेट करें
  5. सत्यापित करें कि एप्लिकेशन एंडपॉइंट एक्सेसिबल हैं

डिबगिंग गाइड

जटिल समस्याओं के निदान के लिए चरण-दर-चरण गाइड।

ट्रेस कॉन्टेक्स्ट प्रोपेगेशन डिबगिंग

चरण 1: इंस्ट्रूमेंटेशन सत्यापित करें

जांचें कि आपकी कॉल चेन में सभी सर्विसेज़ ठीक से इंस्ट्रूमेंट हैं:

# Check for trace headers in requests
curl -H "x-amzn-trace-id: Root=1-5759e988-bd862e3fe1be46a994272793;Parent=53995c3f42cd8ad8;Sampled=1" \
     http://your-service/endpoint

चरण 2: एजेंट कॉन्फ़िगरेशन जांचें

एजेंट कॉन्फ़िगरेशन फ़ाइलों और एनवायरनमेंट वेरिएबल को सत्यापित करें।

चरण 3: एंड-टू-एंड टेस्ट करें

एक टेस्ट रिक्वेस्ट बनाएं और सभी सर्विसेज़ के माध्यम से इसे ट्रेस करें।

प्रदर्शन समस्या निदान

बाधा की पहचान करें:

  1. उच्च-लेटेंसी सर्विसेज़ के लिए सर्विस मैप जांचें
  2. धीमे स्पैन के लिए ट्रेस टाइमलाइन की जांच करें
  3. रिसोर्स मेट्रिक्स (CPU, मेमोरी, डिस्क I/O) की समीक्षा करें
  4. लॉग्स में एरर पैटर्न देखें
  5. डेटाबेस क्वेरी प्रदर्शन जांचें

सामान्य बाधाएं:

  • डेटाबेस क्वेरी अनुकूलन की आवश्यकता
  • बाहरी API कॉल टाइमआउट हो रहे हैं
  • रिसोर्स बाधाएं (CPU/मेमोरी)
  • थ्रेड पूल एग्ज़ॉशन
  • सर्विसेज़ के बीच नेटवर्क लेटेंसी

एजेंट समस्या निवारण

एजेंट स्थिति जांचें:

# For Java applications
jps -l | grep opentelemetry

# For Python applications
ps aux | grep opentelemetry

# Check agent logs
tail -f /var/log/aws-opentelemetry-agent.log

सामान्य एजेंट समस्याएं:

  • JVM के लिए गलत Java एजेंट वर्ज़न
  • अनुपलब्ध डिपेंडेंसीज़ या लाइब्रेरी
  • कॉन्फ़िगरेशन फ़ाइल सिंटैक्स एरर
  • लॉग फ़ाइलों में लिखने की अनुमति समस्याएं

टूल और कमांड

समस्या निवारण के लिए उपयोगी टूल और कमांड।

CloudWatch CLI कमांड

# List Application Signals services
aws application-signals list-services

# Get service metrics
aws cloudwatch get-metric-data --metric-data-queries '[
  {
    "Id": "m1",
    "MetricStat": {
      "Metric": {
        "Namespace": "AWS/ApplicationSignals",
        "MetricName": "Latency"
      },
      "Period": 300,
      "Stat": "Average"
    }
  }
]'

# Query traces
aws xray get-trace-summaries --start-time 2024-01-01T00:00:00Z --end-time 2024-01-02T00:00:00Z

लॉग विश्लेषण

टेलीमेट्री डेटा का विश्लेषण करने के लिए CloudWatch Logs Insights का उपयोग करें:

# Find errors in application logs
fields @timestamp, @message
| filter @message like /ERROR/
| sort @timestamp desc
| limit 100

# Correlate traces with logs
fields @timestamp, traceId, @message
| filter ispresent(traceId)
| sort @timestamp desc

सहायता प्राप्त करना

जब आपको अतिरिक्त सहायता की आवश्यकता हो:

दस्तावेज़ीकरण संसाधन:

  • Application Signals उपयोगकर्ता गाइड
  • Synthetics उपयोगकर्ता गाइड
  • RUM उपयोगकर्ता गाइड

सपोर्ट विकल्प:

  • AWS सपोर्ट सेंटर
  • AWS re:Post कम्युनिटी
  • OpenTelemetry प्रोजेक्ट के लिए GitHub issues
