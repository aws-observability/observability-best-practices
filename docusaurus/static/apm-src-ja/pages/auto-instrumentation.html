CloudWatch APM ドキュメント

Auto Instrumentation

「auto instrumentation」の意味

Auto instrumentation は、OpenTelemetry のインストルメンテーションをランタイム時に追加します（アプリケーションコードを変更することなく）。これにより、サービスは Application Signals がディスカバリーおよびヘルスモニタリングに使用するテレメトリを出力します。

AWS セットアップの共通コンポーネント

  • AWS Distro for OpenTelemetry (ADOT) SDK および/またはコレクター
  • CloudWatch Agent サポートされているセットアップでテレメトリを収集・エクスポートするため
  • プラットフォームの有効化 （EKS アドオン / ECS サイドカー / EC2 エージェントセットアップ / Lambda レイヤー）

まずはこちら： Application Signals の概要.

サポートされている言語とシステム

Application Signals は Java、Python、Node.js、および .NET アプリケーションをサポートしており、Amazon EKS、Kubernetes、Amazon ECS、および Amazon EC2 でサポート・テストされています。

参照： サポートされているシステム.

注意事項と注意点

  • Node.js ESM： ESM のサポートは限定的です。ドキュメントの ESM 固有のガイダンスに従わない限り、CommonJS が推奨される方法です。
  • コンテナ内の Python： 設定が必要な場合があります PYTHONPATH OTel の auto instrumentation 用に設定されています。
