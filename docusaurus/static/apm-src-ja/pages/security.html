CloudWatch APM Documentation

Security & Compliance

セキュリティ

CloudWatch Application Signals は、オブザーバビリティデータを保護し、業界標準へのコンプライアンスを確保するために、複数のセキュリティレイヤーを実装しています。

データの暗号化

保存時の暗号化：

  • すべてのトレースデータは AES-256 暗号化を使用して暗号化されます
  • RUM データは AWS Key Management Service (KMS) で暗号化されます
  • Synthetic モニタリングデータは暗号化されて保存されます
  • 追加の制御のためにカスタマーマネージドキーを使用できます

転送時の暗号化：

  • すべてのデータ転送には TLS 1.2 以上を使用します
  • CloudWatch サービスへの API 呼び出しは暗号化されます
  • RUM データ収集には HTTPS を使用します
  • エージェントとコレクター間の通信はセキュリティで保護されます

アイデンティティとアクセス管理

IAM 権限：

Application Signals へのアクセスを付与する際は最小権限の原則を使用してください：

読み取り専用アクセス：
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
フルアクセス：
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

データプライバシーと PII 保護

データのサニタイズ：

  • 機密情報にはデータマスキングを実装してください
  • トレース属性内の PII にはトークン化を使用してください
  • データ保持ポリシーを設定してください
  • 収集したデータの定期的な監査

ベストプラクティス：

  • パスワード、API キー、またはトークンをトレースに含めないでください
  • メールアドレスや電話番号はハッシュ化またはマスクしてください
  • 機密データを含む URL からクエリパラメータを削除してください
  • 個人情報の代わりに汎用識別子を使用してください

ネットワークセキュリティ

VPC の設定：

  • セキュリティ強化のため、VPC 内にエージェントをデプロイしてください
  • CloudWatch API 呼び出しには VPC エンドポイントを使用してください
  • トラフィックを制限するセキュリティグループを設定してください
  • 追加の制御のためにネットワーク ACL を実装してください

エージェントのセキュリティ：

  • 必要最小限の権限でエージェントを実行してください
  • エージェントをセキュリティパッチで最新の状態に保つ
  • セキュリティイベントのエージェントログを監視する
  • 可能な限り読み取り専用アクセスを使用する

データプライバシー

Application Signals がデータプライバシー要件をどのように処理するかを理解する。

データの収集と使用

収集する情報：

  • アプリケーションのパフォーマンスメトリクスとトレース
  • サービス依存関係情報
  • エラー情報とスタックトレース
  • システムおよびインフラストラクチャのメタデータ

収集しない情報：

  • アプリケーションのビジネスデータまたはユーザーコンテンツ
  • データベースの内容またはファイルシステムデータ
  • ユーザー認証情報または認証トークン
  • 個人を特定できる情報（明示的に含まれている場合を除く）

データ保持

デフォルトの保持期間：

  • トレースデータ： 30日間（最大1年まで設定可能）
  • メトリクス： 15か月（標準的な CloudWatch の保持期間）
  • RUM データ： 30日間（最大1年まで設定可能）
  • Synthetics の結果： 90日間（設定可能）

データの削除：

  • CloudWatch データ保持ポリシーを使用する
  • 自動化されたデータライフサイクル管理を実装する
  • 必要に応じて AWS Support を通じてデータ削除をリクエストする

コンプライアンスフレームワーク

サポートされている標準：

  • SOC 2： セキュリティ、可用性、および機密性の管理
  • PCI DSS： ペイメントカード業界のデータセキュリティ標準
  • HIPAA： 医療保険の携行性と責任に関する法律
  • GDPR： 一般データ保護規則
  • FedRAMP: 連邦リスク認可管理プログラム

コンプライアンスガイダンス

一般的なコンプライアンスシナリオに対する具体的なガイダンス。

HIPAA コンプライアンス

要件:

  • トレース内の保護医療情報 (PHI) を暗号化する
  • アクセス制御と監査ログを実装する
  • AWS との業務提携契約 (BAA) を使用する
  • 適切なデータ保持ポリシーを設定する

実装:

  • トレース属性とスパン名の PHI をマスクする
  • カスタマーマネージド KMS キーを使用する
  • 監査ログ記録のために CloudTrail を有効にする
  • データ分離のために VPC エンドポイントを設定する

PCI DSS コンプライアンス

主な考慮事項:

  • カード会員データをトレースやログに含めない
  • ネットワークセグメンテーションを実装する
  • 転送中および保存中のデータに暗号化を使用する
  • 定期的なセキュリティ評価とペネトレーションテスト

ベストプラクティス:

  • インストルメンテーション前に機密データをトークン化する
  • Application Signals エージェントにプライベートサブネットを使用する
  • 最小権限の IAM ポリシーを実装する
  • 収集データの定期的なレビュー

GDPR コンプライアンス

データ主体の権利:

  • 個人データへのアクセス権
  • 訂正の権利
  • 消去の権利 (「忘れられる権利」)
  • データポータビリティの権利

実装:

  • テレメトリデータにおける PII 収集を最小化する
  • データ匿名化技術を実装する
  • ビジネスニーズに基づいてデータ保持を設定する
  • データ処理活動を文書化する

セキュリティモニタリング

Application Signals のデプロイメントをセキュリティイベントについて監視する。

セキュリティのベストプラクティス

  • API アクティビティログ記録のために CloudTrail を有効化する
  • IAM ポリシーの変更とアクセスパターンを監視する
  • 異常なデータアクセスパターンに対するアラートを設定する
  • 定期的なセキュリティ評価と脆弱性スキャン
  • 多要素認証 (MFA) を実装する

インシデント対応

  • セキュリティイベントに対するインシデント対応計画を策定する
  • 侵害された場合に Application Signals を無効化する方法を把握する
  • エスカレーション手順を文書化する
  • 定期的なインシデント対応テスト
