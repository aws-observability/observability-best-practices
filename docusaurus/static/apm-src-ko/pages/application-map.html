CloudWatch APM 문서

애플리케이션 맵

개요

CloudWatch 애플리케이션 맵은 애플리케이션 토폴로지와 운영 상태를 시각화합니다. Application Signals에서 기존 "서비스 맵" 용어를 대체합니다.

클라이언트, 카나리, 서비스 및 종속성 간의 연결을 보여주어 팀이 실행 흐름을 이해하고 장애 및 지연 시간이 발생하는 위치를 빠르게 찾을 수 있도록 합니다.

주요 기능

  • SLO/SLI 상태: 어떤 서비스가 목표를 충족하거나 위반하고 있는지, 그리고 영향이 다운스트림으로 어떻게 전파되는지 확인합니다.
  • 드릴다운: 노드/엣지를 선택하여 메트릭을 보고 서비스 세부 정보로 이동합니다.
  • 그룹화 및 필터링: 기본 그룹화(관련 서비스, 환경)를 사용하거나 태그 또는 OTEL 리소스 속성을 사용하여 사용자 정의 그룹을 정의합니다.
  • 비계측 서비스: Application Signals는 아직 계측되지 않은 서비스를 표시하여 롤아웃 우선순위 결정에 도움을 줄 수 있습니다.
  • 교차 계정 가시성: 교차 계정 Observability가 구성된 경우 여러 계정의 서비스를 시각화합니다.
