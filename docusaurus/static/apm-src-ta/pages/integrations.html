CloudWatch APM ஆவணங்கள்

ஒருங்கிணைப்புகள்

AWS சேவை ஒருங்கிணைப்புகள்

CloudWatch Application Signals உங்கள் முழு cloud infrastructure முழுவதும் விரிவான observability வழங்க AWS சேவைகளுடன் தடையின்றி ஒருங்கிணைகிறது.

API Gateway

API செயல்திறனைக் கண்காணித்து API Gateway-லிருந்து உங்கள் backend சேவைகளுக்கான கோரிக்கைகளைக் கண்காணிக்கவும்.

ஒருங்கிணைப்பு அம்சங்கள்:

  • API அழைப்புகளின் தானியங்கி tracing
  • API endpoints-க்கான தாமத கண்காணிப்பு
  • பிழை விகித கண்காணிப்பு
  • Request/response correlation

அமைப்பு:

உங்கள் API Gateway stage configuration-ல் X-Ray tracing-ஐ இயக்குங்கள்.

Application Load Balancer (ALB)

Load balancers வழியாக traffic flow-ஐ கண்காணித்து application செயல்திறனுடன் correlate செய்யுங்கள்.

ஒருங்கிணைப்பு நன்மைகள்:

  • Request routing visibility
  • Target group செயல்திறன் மெட்ரிக்ஸ்
  • Health check கண்காணிப்பு
  • Load distribution analysis

DynamoDB

Database செயல்பாடுகளைக் கண்காணித்து data access patterns-ல் செயல்திறன் bottlenecks-ஐ அடையாளம் காணுங்கள்.

Tracing திறன்கள்:

  • Query செயல்பாட்டு நேரம்
  • Throttle மற்றும் error கண்காணிப்பு
  • Table மற்றும் index செயல்திறன்
  • Cross-region replication கண்காணிப்பு

Amazon RDS

Database செயல்திறன் மற்றும் connection pooling பற்றிய insights பெறுங்கள்.

Database கண்காணிப்பு:

  • Query execution நேரங்கள்
  • Connection pool பயன்பாடு
  • Database instance மெட்ரிக்ஸ்
  • மெதுவான query அடையாளம்

Amazon S3

Object storage செயல்பாடுகள் மற்றும் data transfer செயல்திறனைக் கண்காணிக்கவும்.

Storage செயல்பாடுகள்:

  • Put/Get object tracing
  • Transfer தாமத கண்காணிப்பு
  • பிழை விகித analysis
  • Cross-region transfer கண்காணிப்பு

SNS/SQS

Event-driven architectures-ல் message processing மற்றும் queue செயல்திறனைக் கண்காணிக்கவும்.

Messaging Insights:

  • Message publish/subscribe tracing
  • Queue depth கண்காணிப்பு
  • Message processing தாமதம்
  • Dead letter queue analysis

AWS Step Functions

Workflow execution-ஐ காட்சிப்படுத்தி state machine processing-ல் bottlenecks-ஐ அடையாளம் காணுங்கள்.

Workflow கண்காணிப்பு:

  • State transition tracing
  • Execution நேர analysis
  • Error state அடையாளம்
  • Parallel execution கண்காணிப்பு

Amazon EventBridge

Event-driven systems வழியாக event routing மற்றும் processing-ஐ கண்காணிக்கவும்.

Event Processing:

  • Event routing visibility
  • Rule evaluation timing
  • Target invocation tracing
  • Event transformation கண்காணிப்பு

AWS Lambda

Serverless function செயல்திறன் மற்றும் cold start தாக்கங்களைக் கண்காணிக்கவும்.

Serverless Insights:

  • Function invocation tracing
  • Cold start duration
  • Memory மற்றும் CPU பயன்பாடு
  • Error மற்றும் timeout கண்காணிப்பு

Amazon ECS/EKS

Microservices கண்காணிப்புக்கான container orchestration platform ஒருங்கிணைப்பு.

Container கண்காணிப்பு:

  • Service mesh tracing
  • Pod-to-pod communication
  • Resource பயன்பாடு
  • Deployment கண்காணிப்பு

Third-Party ஒருங்கிணைப்புகள்

Third-party tools மற்றும் platforms-உடன் Application Signals-ஐ விரிவுபடுத்துங்கள்.

OpenTelemetry

Vendor-neutral observability-க்கு OpenTelemetry ecosystem-ஐ பயன்படுத்துங்கள்.

ஒருங்கிணைப்பு விருப்பங்கள்:

  • தனிப்பயன் instrumentation libraries
  • Third-party agent support
  • Multi-backend data export
  • Community collector contributions

APM Vendors

பிற APM solutions-லிருந்து migrate செய்யுங்கள் அல்லது ஒருங்கிணையுங்கள்.

ஆதரிக்கப்படும் ஒருங்கிணைப்புகள்:

  • Third-party APM tools-க்கு data export
  • Hybrid cloud observability
  • Multi-vendor correlation
  • Data migration utilities

கண்காணிப்பு மற்றும் எச்சரிக்கை

உங்கள் தற்போதைய monitoring stack-உடன் Application Signals-ஐ இணைக்கவும்.

ஒருங்கிணைப்பு Partners:

  • PagerDuty: சம்பவ பதில் மற்றும் எச்சரிக்கை
  • Slack: குழு அறிவிப்புகள்
  • Datadog: ஒருங்கிணைந்த கண்காணிப்பு platform
  • New Relic: Application performance monitoring
  • Splunk: Log analysis மற்றும் correlation

CI/CD ஒருங்கிணைப்பு

உங்கள் development pipeline-ல் observability-ஐ உள்ளடக்குங்கள்.

DevOps Tools:

  • GitHub Actions: தானியங்கி testing மற்றும் deployment
  • Jenkins: Build pipeline ஒருங்கிணைப்பு
  • GitLab CI: DevSecOps workflows
  • Terraform: Infrastructure as code
  • AWS CDK: Cloud infrastructure provisioning

தனிப்பயன் ஒருங்கிணைப்புகள்

APIs மற்றும் SDKs பயன்படுத்தி உங்கள் சொந்த ஒருங்கிணைப்புகளை உருவாக்குங்கள்.

CloudWatch APIs

Application Signals CloudWatch-க்கு publish செய்யும் telemetry-ஐ சுற்றி தனிப்பயன் டாஷ்போர்டுகள், alarms மற்றும் metric-driven workflows-ஐ உருவாக்க CloudWatch APIs-ஐ பயன்படுத்துங்கள்.

கிடைக்கும் APIs:

  • PutMetricData - தனிப்பயன் metrics அனுப்புதல்
  • GetMetricData - Metric data எடுத்தல்
  • PutDashboard / GetDashboard - டாஷ்போர்டுகளை நிர்வகித்தல்
  • PutMetricAlarm / DescribeAlarms - Alarms நிர்வகித்தல்

AWS X-Ray APIs

Application Signals trace store ஆக AWS X-Ray-ஐ பயன்படுத்துகிறது. நேரடி trace ingestion/query ஒருங்கிணைப்பு தேவைப்பட்டால், X-Ray APIs-ஐ பயன்படுத்துங்கள்.

  • PutTraceSegments - Trace segments ingest செய்தல்
  • GetTraceSummaries / BatchGetTraces - Traces query செய்தல்

Webhooks மற்றும் Callbacks

நிகழ்நேர அறிவிப்புகளைப் பெற்று தனிப்பயன் workflows-உடன் ஒருங்கிணையுங்கள்.

ஒருங்கிணைப்பு Patterns:

  • தனிப்பயன் notification systems-க்கான alert webhooks
  • Lambda-உடன் event-driven processing
  • தனிப்பயன் டாஷ்போர்ட் updates
  • தானியங்கி remediation workflows

SDKs மற்றும் Libraries

Instrumentation பொதுவாக OpenTelemetry மற்றும்/அல்லது AWS Distro for OpenTelemetry (ADOT) வழியாக செய்யப்படுகிறது. AWS API அழைப்புகளுக்கு, ஆதரிக்கப்படும் OpenTelemetry instrumentations-உடன் AWS SDKs-ஐ பயன்படுத்துங்கள்.

ஆதரிக்கப்படும் மொழிகள்:

  • JavaScript/Node.js: AWS SDK for JavaScript
  • Python: Boto3 AWS SDK
  • Java: AWS SDK for Java
  • .NET: AWS SDK for .NET
  • Go: AWS SDK for Go
