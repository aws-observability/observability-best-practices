CloudWatch APM ஆவணங்கள்

Java Spring + OTLP + CloudWatch (நடைமுறை பயிற்சி)

இந்த வழிகாட்டி Java Spring Boot சேவையை OpenTelemetry-உடன் instrument செய்து, AWS-க்கு telemetry ஏற்றுமதி செய்து, ECS/Fargate-க்கு deploy செய்து, traces/logs/Application Signals நடத்தையை சரிபார்க்கும் முழு பயிற்சியை ஆவணப்படுத்துகிறது.

இது ஒரு implementation example பாணியில் எழுதப்பட்டுள்ளது மற்றும் runbook அல்லது workshop handout ஆக பயன்படுத்தப்படலாம்.

1) இந்த பயிற்சி என்ன நிரூபிக்கிறது

  • Java app-லிருந்து End-to-end telemetry:
    • Traces (X-Ray / Transaction Search)
    • Logs (CloudWatch Logs OTLP endpoint அல்லது ECS-ல் awslogs)
    • Metrics/Application Signals (CloudWatch Agent sidecar path)
  • இவற்றுக்கிடையேயான வேறுபாடு:
    • Micrometer tracing bridge (micrometer-tracing-bridge-otel) tracing API bridging-க்கு
    • Micrometer registries (micrometer-registry-cloudwatch2, micrometer-registry-otlp) இவை metrics-only sinks
  • OTLP endpoint wiring env vars (OTEL_*) வழியாக, hardcoded Java constants அல்ல.

4) OTLP endpoint mapping

Signal Endpoint pattern பொதுவான env vars
Traces https://xray.<region>.amazonaws.com/v1/traces OTEL_EXPORTER_OTLP_TRACES_ENDPOINT, OTEL_EXPORTER_OTLP_TRACES_PROTOCOL=http/protobuf
Metrics https://monitoring.<region>.amazonaws.com/v1/metrics OTEL_EXPORTER_OTLP_METRICS_ENDPOINT (அல்லது sidecar வழியாக App Signals path)
Logs https://logs.<region>.amazonaws.com/v1/logs OTEL_EXPORTER_OTLP_LOGS_ENDPOINT, OTEL_EXPORTER_OTLP_LOGS_HEADERS

ECS sidecar mode-ல், app container traces/metrics-ஐ http://localhost:4316/...-க்கு point செய்கிறது மற்றும் CloudWatch Agent forwarding-ஐ கையாளுகிறது.

5) இந்த repo-வில் பயன்படுத்தப்படும் Instrumentation model

5.1 Dependencies

  • micrometer-tracing-bridge-otel pom.xml-ல் உள்ளது.
  • Tracing-க்கு micrometer-registry-cloudwatch2 அல்லது micrometer-registry-otlp மீது dependency இல்லை.

5.2 Auto + manual tracing

  • ADOT Java agent-லிருந்து Auto instrumentation framework/client spans-ஐ capture செய்கிறது.
  • DemoService.greet()-க்குள் Manual enrichment சேர்க்கப்பட்டுள்ளது:
    • attributes: demo.operation, demo.request.name, demo.downstream.status, demo.downstream.elapsed_ms, demo.custom.instrumented
    • events: demo.greet.start, demo.httpbin.success, demo.httpbin.failure
    • recordException(...) மற்றும் setStatus(ERROR, ...)-உடன் error recording

7) AWS Console-ல் telemetry-ஐ எங்கே பார்ப்பது

Traces (முதன்மை)

  1. X-Ray → Traces
  2. Region: us-east-1
  3. நேர வரம்பு: கடந்த 15-60 நிமிடங்கள்
  4. Service: java-spring-otel-poc

தனிப்பயன் fields-க்கு:

  • ஒரு trace திறக்கவும் → segment java-spring-otel-poc → Metadata
  • இந்த keys-ஐ பாருங்கள்:
    • demo.operation
    • demo.request.name
    • demo.downstream.status
    • demo.downstream.elapsed_ms
    • demo.custom.instrumented

Logs

  • ECS app logs: /ecs/java-spring-otel
  • CW agent sidecar logs: /ecs/ecs-cwagent-java-spring-otel
  • Local OTLP log group (locally இயக்கினால்): /aws/otel/java-spring-otel-poc

Application Signals

  • CloudWatch → Application Signals → Services
  • Sustained traffic மற்றும் sidecar path வழியாக வெற்றிகரமான metrics forwarding-க்குப் பிறகு java-spring-otel-poc-ஐ பாருங்கள்.

8) முக்கிய takeaway

OpenTelemetry auto instrumentation spans-ஐ உருவாக்க முடியும், ஆனால் வெற்றிகரமான AWS observability outcomes இன்னும் இவற்றைச் சார்ந்துள்ளன:

  1. சரியான OTLP endpoint wiring
  2. சரியான auth/forwarding path (metrics/App Signals-க்கு பெரும்பாலும் sidecar/collector வழியாக)
  3. Tracing bridge vs metrics registries இடையே சரியான வேறுபாடு
  4. Sustained traffic மற்றும் சரியான console view (traces-க்கு X-Ray)
