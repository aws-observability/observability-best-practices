CloudWatch APM ஆவணங்கள்

பாதுகாப்பு & இணக்கம்

பாதுகாப்பு

CloudWatch Application Signals உங்கள் observability தரவை பாதுகாக்கவும் தொழில் தரநிலைகளுடன் இணக்கத்தை உறுதிப்படுத்தவும் பல அடுக்கு பாதுகாப்பை செயல்படுத்துகிறது.

தரவு குறியாக்கம்

Rest-ல் குறியாக்கம்:

  • அனைத்து trace data AES-256 குறியாக்கத்தைப் பயன்படுத்தி encrypt செய்யப்படுகிறது
  • RUM data AWS Key Management Service (KMS) மூலம் encrypt செய்யப்படுகிறது
  • Synthetic monitoring data encrypted ஆக சேமிக்கப்படுகிறது
  • கூடுதல் கட்டுப்பாட்டிற்கு customer-managed keys பயன்படுத்தலாம்

Transit-ல் குறியாக்கம்:

  • அனைத்து data transmission TLS 1.2 அல்லது அதற்கு மேல் பயன்படுத்துகிறது
  • CloudWatch services-க்கான API calls encrypt செய்யப்படுகின்றன
  • RUM data collection HTTPS பயன்படுத்துகிறது
  • Agent-to-collector communication பாதுகாக்கப்படுகிறது

Identity மற்றும் Access Management

IAM Permissions:

Application Signals-க்கு access வழங்கும்போது least privilege கொள்கையைப் பயன்படுத்துங்கள்:

Read-Only Access:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
Full Access:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

தரவு தனியுரிமை மற்றும் PII பாதுகாப்பு

தரவு Sanitization:

  • முக்கிய தகவல்களுக்கு data masking செயல்படுத்துங்கள்
  • Trace attributes-ல் PII-க்கு tokenization பயன்படுத்துங்கள்
  • Data retention policies கட்டமைக்கவும்
  • சேகரிக்கப்பட்ட தரவின் வழக்கமான audit

சிறந்த நடைமுறைகள்:

  • Traces-ல் passwords, API keys அல்லது tokens-ஐ ஒருபோதும் சேர்க்காதீர்கள்
  • Email addresses மற்றும் phone numbers-ஐ hash அல்லது mask செய்யுங்கள்
  • முக்கிய தரவு கொண்ட URLs-லிருந்து query parameters-ஐ அகற்றுங்கள்
  • தனிப்பட்ட தகவலுக்கு பதிலாக generic identifiers பயன்படுத்துங்கள்

Network பாதுகாப்பு

VPC கட்டமைப்பு:

  • மேம்பட்ட பாதுகாப்புக்கு உங்கள் VPC-க்குள் agents deploy செய்யுங்கள்
  • CloudWatch API calls-க்கு VPC endpoints பயன்படுத்துங்கள்
  • Traffic-ஐ கட்டுப்படுத்த security groups கட்டமைக்கவும்
  • கூடுதல் கட்டுப்பாட்டிற்கு network ACLs செயல்படுத்துங்கள்

Agent பாதுகாப்பு:

  • குறைந்தபட்ச தேவையான permissions-உடன் agents இயக்குங்கள்
  • Agents-ஐ security patches-உடன் updated ஆக வைத்திருங்கள்
  • Security events-க்கு agent logs-ஐ கண்காணிக்கவும்
  • சாத்தியமான இடங்களில் read-only access பயன்படுத்துங்கள்

தரவு தனியுரிமை

Application Signals உங்கள் தரவு தனியுரிமை தேவைகளை எவ்வாறு கையாளுகிறது என்பதைப் புரிந்துகொள்ளுதல்.

தரவு சேகரிப்பு மற்றும் பயன்பாடு

நாங்கள் சேகரிப்பது:

  • Application performance metrics மற்றும் traces
  • சேவை சார்பு தகவல்
  • Error தகவல் மற்றும் stack traces
  • System மற்றும் infrastructure metadata

நாங்கள் சேகரிக்காதது:

  • Application business data அல்லது user content
  • Database contents அல்லது file system data
  • User credentials அல்லது authentication tokens
  • தனிப்பட்ட அடையாளத் தகவல் (வெளிப்படையாக சேர்க்கப்படாவிட்டால்)

தரவு Retention

இயல்புநிலை Retention காலங்கள்:

  • Trace data: 30 நாட்கள் (1 வருடம் வரை configurable)
  • Metrics: 15 மாதங்கள் (standard CloudWatch retention)
  • RUM data: 30 நாட்கள் (1 வருடம் வரை configurable)
  • Synthetics results: 90 நாட்கள் (configurable)

தரவு நீக்குதல்:

  • CloudWatch data retention policies பயன்படுத்துங்கள்
  • தானியங்கி data lifecycle management செயல்படுத்துங்கள்
  • தேவைப்பட்டால் AWS Support வழியாக data deletion கோருங்கள்

Compliance Frameworks

ஆதரிக்கப்படும் தரநிலைகள்:

  • SOC 2: பாதுகாப்பு, கிடைக்கும் தன்மை மற்றும் இரகசியத்தன்மை கட்டுப்பாடுகள்
  • PCI DSS: Payment card industry data security standards
  • HIPAA: Health Insurance Portability and Accountability Act
  • GDPR: General Data Protection Regulation
  • FedRAMP: Federal Risk and Authorization Management Program

Compliance வழிகாட்டுதல்

பொதுவான compliance scenarios-க்கான குறிப்பிட்ட வழிகாட்டுதல்.

HIPAA Compliance

தேவைகள்:

  • Traces-ல் protected health information (PHI)-ஐ encrypt செய்யுங்கள்
  • Access controls மற்றும் audit logging செயல்படுத்துங்கள்
  • AWS-உடன் Business Associate Agreements (BAA) பயன்படுத்துங்கள்
  • பொருத்தமான data retention policies கட்டமைக்கவும்

செயல்படுத்தல்:

  • Trace attributes மற்றும் span names-ல் PHI-ஐ mask செய்யுங்கள்
  • Customer-managed KMS keys பயன்படுத்துங்கள்
  • Audit logging-க்கு CloudTrail-ஐ இயக்குங்கள்
  • Data isolation-க்கு VPC endpoints கட்டமைக்கவும்

PCI DSS Compliance

முக்கிய considerations:

  • Traces அல்லது logs-ல் cardholder data-ஐ ஒருபோதும் சேர்க்காதீர்கள்
  • Network segmentation செயல்படுத்துங்கள்
  • Transit மற்றும் rest-ல் data-க்கு encryption பயன்படுத்துங்கள்
  • வழக்கமான security assessments மற்றும் penetration testing

சிறந்த நடைமுறைகள்:

  • Instrumentation-க்கு முன் sensitive data-ஐ tokenize செய்யுங்கள்
  • Application Signals agents-க்கு private subnets பயன்படுத்துங்கள்
  • Least-privilege IAM policies செயல்படுத்துங்கள்
  • சேகரிக்கப்பட்ட தரவின் வழக்கமான review

GDPR Compliance

Data Subject உரிமைகள்:

  • தனிப்பட்ட தரவை அணுகும் உரிமை
  • திருத்தம் செய்யும் உரிமை
  • நீக்குதல் உரிமை ("மறக்கப்படும் உரிமை")
  • தரவு portability உரிமை

செயல்படுத்தல்:

  • Telemetry data-ல் PII சேகரிப்பை குறைக்கவும்
  • Data anonymization நுட்பங்களை செயல்படுத்துங்கள்
  • வணிக தேவைகளின் அடிப்படையில் data retention கட்டமைக்கவும்
  • Data processing நடவடிக்கைகளை ஆவணப்படுத்துங்கள்

பாதுகாப்பு கண்காணிப்பு

Security events-க்கு உங்கள் Application Signals deployment-ஐ கண்காணிக்கவும்.

பாதுகாப்பு சிறந்த நடைமுறைகள்

  • API activity logging-க்கு CloudTrail-ஐ இயக்குங்கள்
  • IAM policy changes மற்றும் access patterns-ஐ கண்காணிக்கவும்
  • அசாதாரண data access patterns-க்கு alerts அமைக்கவும்
  • வழக்கமான security assessments மற்றும் vulnerability scans
  • Multi-factor authentication (MFA) செயல்படுத்துங்கள்

சம்பவ பதில்

  • Security events-க்கான incident response plan வைத்திருங்கள்
  • Compromise ஆனால் Application Signals-ஐ எவ்வாறு disable செய்வது என்று தெரிந்திருங்கள்
  • Escalation procedures-ஐ ஆவணப்படுத்துங்கள்
  • வழக்கமான incident response testing
