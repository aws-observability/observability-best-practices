Overview

This guide provides step-by-step instructions to deploy a Node.js Express application on Amazon EC2 with AWS Application Signals enabled. This is the EC2 equivalent of the Lambda Node.js function, reusing the same business logic and endpoints.

What you'll accomplish:
  • Deploy a Node.js Express app on EC2
  • Enable AWS Application Signals auto-instrumentation
  • View your service in CloudWatch Application Signals console
  • Monitor service metrics, traces, and operations
Time to complete: 30-45 minutes

Prerequisites

Required Tools

  • AWS CLI v2 (latest version)
  • Docker Desktop
  • Node.js 18+ (for local development)
  • npm (for local development)
  • AWS Account with appropriate permissions
  • Text editor

AWS Permissions Required

  • EC2: Full access (create instances, security groups, key pairs)
  • IAM: Create roles and policies
  • CloudWatch: Logs and Application Signals access
  • VPC: Create and manage VPC resources
  • SSM: Parameter Store write access

Knowledge Assumed

  • Basic AWS CLI usage
  • Basic Docker commands
  • Basic understanding of EC2 concepts
  • Basic Node.js/Express knowledge

Architecture

Your EC2 instance will run:

  1. Node.js Express Application: Your web API with Application Signals instrumentation
  2. OpenTelemetry Auto-instrumentation: Automatic tracing and metrics collection
  3. CloudWatch Agent: Collects traces and metrics (optional, can use direct OTLP)

Flow:

EC2 Instance
  → Node.js Express App (auto-instrumented)
  → OpenTelemetry Collector (optional)
  → CloudWatch Application Signals

1.3 Create Main Application

Create index.js:

/**
 * Node.js Express application for AWS Application Signals testing on EC2
 * 
 * This application demonstrates auto-instrumentation with AWS Application Signals
 * on EC2. OpenTelemetry handles all instrumentation automatically when configured.
 */

const express = require('express');
const cors = require('cors');
const AWS = require('aws-sdk');
const axios = require('axios');

// Initialize Express app
const app = express();
const port = process.env.PORT || 8080;

// Initialize AWS services
const s3 = new AWS.S3();

// Middleware
app.use(cors());
app.use(express.json());

/**
 * Root endpoint handler
 */
app.get('/', (req, res) => {
    res.json({
        message: 'AWS Application Signals EC2 Node.js Demo',
        timestamp: Date.now(),
        service: 'ec2-nodejs',
        runtime: 'nodejs18.x',
        host: process.env.HOSTNAME || 'unknown',
        environment: 'ec2'
    });
});

/**
 * Health check endpoint handler
 */
app.get('/health', (req, res) => {
    res.json({
        status: 'healthy',
        timestamp: Date.now(),
        service: 'ec2-nodejs',
        runtime: 'nodejs18.x',
        host: process.env.HOSTNAME || 'unknown'
    });
});

/**
 * Personalized greeting handler with simulated processing time
 */
app.get('/api/hello/:name', async (req, res) => {
    // Simulate some processing time
    await new Promise(resolve => setTimeout(resolve, Math.random() * 100 + 10));

    res.json({
        message: `Hello, ${req.params.name}!`,
        timestamp: Date.now(),
        service: 'ec2-nodejs',
        runtime: 'nodejs18.x',
        greeting_id: Math.floor(Math.random() * 10000)
    });
});

/**
 * Intentionally slow endpoint for latency testing
 */
app.get('/api/slow', async (req, res) => {
    // Simulate slow operation (500-1500ms)
    const delay = Math.random() * 1000 + 500;
    await new Promise(resolve => setTimeout(resolve, delay));

    res.json({
        message: 'Slow operation completed',
        timestamp: Date.now(),
        service: 'ec2-nodejs',
        duration_ms: Math.floor(delay)
    });
});

/**
 * Endpoint that randomly raises errors for fault testing
 */
app.get('/api/error', (req, res) => {
    if (Math.random() > 0.5) {
        throw new Error('Simulated error for Application Signals testing');
    }

    res.json({
        message: 'No error occurred',
        timestamp: Date.now(),
        service: 'ec2-nodejs'
    });
});

/**
 * Makes external HTTP call for distributed tracing
 */
app.get('/api/external', async (req, res) => {
    try {
        const response = await axios.get('https://httpbin.org/delay/1', {
            timeout: 10000
        });

        res.json({
            message: 'External call completed',
            external_status_code: response.status,
            timestamp: Date.now(),
            service: 'ec2-nodejs'
        });

    } catch (error) {
        throw new Error(`External call failed: ${error.message}`);
    }
});

/**
 * AWS S3 call for testing AWS SDK instrumentation
 */
app.get('/api/s3', async (req, res) => {
    try {
        // List buckets (this is a safe operation that doesn't require specific permissions)
        const response = await s3.listBuckets().promise();

        const result = {
            message: 'S3 call completed successfully',
            bucket_count: response.Buckets.length,
            timestamp: Date.now(),
            service: 'ec2-nodejs'
        };

        // Add first few bucket names
        if (response.Buckets.length > 0) {
            result.first_bucket = response.Buckets[0].Name;
        }

        res.json(result);

    } catch (error) {
        res.status(500).json({
            error: 'S3 call failed',
            message: error.message,
            timestamp: Date.now(),
            service: 'ec2-nodejs'
        });
    }
});

/**
 * Database simulation endpoint
 */
app.get('/api/database', async (req, res) => {
    // Simulate database query time
    await new Promise(resolve => setTimeout(resolve, Math.random() * 200 + 50));

    res.json({
        message: 'Database query completed',
        timestamp: Date.now(),
        service: 'ec2-nodejs',
        records_returned: Math.floor(Math.random() * 100) + 1
    });
});

/**
 * Complex operation with multiple steps
 */
app.get('/api/complex', async (req, res) => {
    // Step 1: Simulate authentication
    await new Promise(resolve => setTimeout(resolve, Math.random() * 50 + 10));

    // Step 2: Simulate business logic
    await new Promise(resolve => setTimeout(resolve, Math.random() * 100 + 20));

    // Step 3: Simulate database operations
    await new Promise(resolve => setTimeout(resolve, Math.random() * 150 + 30));

    res.json({
        message: 'Complex operation completed successfully',
        timestamp: Date.now(),
        service: 'ec2-nodejs',
        steps_completed: 3,
        total_processing_time_ms: Math.floor(Math.random() * 300) + 60

/**
 * Error handling middleware
 */

    console.error('Error processing request:', err);

    res.status(500).json({
        error: 'Internal Server Error',
        message: err.message,
        timestamp: Date.now(),

    console.log(`EC2 Node.js Application Signals Demo running on port ${port}`);

1.4 Create OpenTelemetry Configuration

Create otel-config.js:

/**
 * OpenTelemetry configuration for Node.js EC2 Application Signals
 * This file sets up automatic instrumentation for the Express application
 */

const { NodeSDK } = require('@opentelemetry/sdk-node');
const { getNodeAutoInstrumentations } = require('@opentelemetry/auto-instrumentations-node');
const { OTLPTraceExporter } = require('@opentelemetry/exporter-otlp-http');
const { OTLPMetricExporter } = require('@opentelemetry/exporter-otlp-http');
const { Resource } = require('@opentelemetry/resources');
const { SemanticResourceAttributes } = require('@opentelemetry/semantic-conventions');

// Configure OpenTelemetry
const sdk = new NodeSDK({
    resource: new Resource({
        [SemanticResourceAttributes.SERVICE_NAME]: 'ec2-nodejs',
        [SemanticResourceAttributes.SERVICE_NAMESPACE]: 'aws-observability',
        [SemanticResourceAttributes.SERVICE_VERSION]: '1.0.0',
        [SemanticResourceAttributes.DEPLOYMENT_ENVIRONMENT]: 'ec2',
    }),
    traceExporter: new OTLPTraceExporter({
        url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT || 'https://otlp.nr-data.net/v1/traces',
        headers: {
            'api-key': process.env.OTEL_EXPORTER_OTLP_HEADERS?.split('=')[1] || 'your-api-key',
        },
    }),
    metricExporter: new OTLPMetricExporter({
        url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT || 'https://otlp.nr-data.net/v1/metrics',
        headers: {
            'api-key': process.env.OTEL_EXPORTER_OTLP_HEADERS?.split('=')[1] || 'your-api-key',
        },
    }),
    instrumentations: [
        getNodeAutoInstrumentations({
            // Disable some instrumentations that might cause issues
            '@opentelemetry/instrumentation-fs': {
                enabled: false,
            },
        }),
    ],
});

// Initialize the SDK and register with the OpenTelemetry API
sdk.start();

console.log('OpenTelemetry SDK initialized for ec2-nodejs');

// Gracefully shut down the SDK on process exit
process.on('SIGTERM', () => {
    sdk.shutdown()
        .then(() => console.log('OpenTelemetry SDK terminated'))
        .catch((error) => console.log('Error terminating OpenTelemetry SDK', error))
        .finally(() => process.exit(0));
});

Step 3: Deploy to EC2

3.1 Create EC2 Instance

# Create a key pair
aws ec2 create-key-pair --key-name ec2-nodejs-key --query 'KeyMaterial' --output text > ec2-nodejs-key.pem
chmod 400 ec2-nodejs-key.pem

# Create security group
aws ec2 create-security-group \
    --group-name ec2-nodejs-sg \
    --description "Security group for Node.js EC2 Application Signals demo"

# Get security group ID
SG_ID=$(aws ec2 describe-security-groups --group-names ec2-nodejs-sg --query 'SecurityGroups[0].GroupId' --output text)

# Allow HTTP traffic
aws ec2 authorize-security-group-ingress \
    --group-id $SG_ID \
    --protocol tcp \
    --port 8080 \
    --cidr 0.0.0.0/0

# Allow SSH
aws ec2 authorize-security-group-ingress \
    --group-id $SG_ID \
    --protocol tcp \
    --port 22 \
    --cidr 0.0.0.0/0

# Launch EC2 instance
aws ec2 run-instances \
    --image-id ami-0c02fb55956c7d316 \
    --count 1 \
    --instance-type t3.medium \
    --key-name ec2-nodejs-key \
    --security-group-ids $SG_ID \
    --user-data '#!/bin/bash
yum update -y
yum install -y docker
systemctl start docker
systemctl enable docker
usermod -a -G docker ec2-user
yum install -y git wget
# Install Node.js 18
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs'

3.2 Get Instance Information

# Get instance ID
INSTANCE_ID=$(aws ec2 describe-instances --filters "Name=instance-state-name,Values=running" "Name=key-name,Values=ec2-nodejs-key" --query 'Reservations[0].Instances[0].InstanceId' --output text)

# Get public IP
PUBLIC_IP=$(aws ec2 describe-instances --instance-ids $INSTANCE_ID --query 'Reservations[0].Instances[0].PublicIpAddress' --output text)

echo "Instance ID: $INSTANCE_ID"
echo "Public IP: $PUBLIC_IP"

BASE_URL="http://$PUBLIC_IP:8080"

Troubleshooting

Common Issues

  1. Application won't start

    • Check Node.js 18 is installed: node --version
    • Verify port 8080 is open in security group
    • Check application logs

  2. No data in Application Signals

    • Verify environment variables are set
    • Check OpenTelemetry configuration
    • Ensure IAM role has proper permissions

  3. S3 calls failing

    • Attach IAM role with S3 read permissions
    • Verify AWS credentials are configured
