Overview

This guide provides step-by-step instructions to deploy a Python FastAPI application on Amazon ECS with AWS Application Signals enabled. No CDK required - pure AWS CLI and Docker.

What you'll accomplish:
  • Deploy a Python FastAPI app on ECS Fargate
  • Enable AWS Application Signals auto-instrumentation
  • View your service in CloudWatch Application Signals console
  • Monitor service metrics, traces, and operations
Time to complete: 30-45 minutes

Prerequisites

Required Tools

  • AWS CLI v2 (latest version)
  • Docker Desktop
  • Python 3.11+
  • AWS Account with appropriate permissions
  • Text editor

AWS Permissions Required

  • ECS: Full access (create clusters, services, task definitions)
  • ECR: Push images
  • IAM: Create roles and policies
  • CloudWatch: Logs and Application Signals access
  • EC2: VPC, subnet, and security group access
  • SSM: Parameter Store write access

Knowledge Assumed

  • Basic AWS CLI usage
  • Basic Docker commands
  • Basic understanding of ECS concepts

Architecture

Your ECS task will contain 3 containers:

  1. Init Container (init): Copies ADOT Python auto-instrumentation files
  2. CloudWatch Agent (ecs-cwagent): Collects traces and metrics
  3. Application Container (your-app): Your FastAPI application

Flow:

Init Container (runs once, copies files) 
  → Application starts (auto-instrumented via PYTHONPATH)
  → Sends traces/metrics to CloudWatch Agent (localhost:4316)
  → CloudWatch Agent sends to Application Signals

Step 1: Enable Application Signals in Your Account

Important: Application Signals must be enabled once per AWS account per region.

Option 1: Enable via AWS Console (Recommended)

  1. Go to AWS Console → CloudWatch → Application Signals
  2. Select your region (e.g., us-east-1)
  3. Click "Get Started" or "Enable Application Signals"
  4. Wait 2-3 minutes for activation

Option 2: Verify if Already Enabled

# Check if service-linked role exists
aws iam get-role \
  --role-name AWSServiceRoleForCloudWatchApplicationSignals \
  --region us-east-1

# If role exists, Application Signals is enabled

Output should show: Role ARN if enabled, error if not.

Step 2: Prepare Your Python FastAPI Application

2.1 Create Application Directory

mkdir -p ~/my-appsignals-app
cd ~/my-appsignals-app

2.2 Create app.py

"""
Simple FastAPI application for AWS Application Signals
"""
import time
import random
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import requests

app = FastAPI(
    title="My Application Signals App",
    description="FastAPI app with Application Signals integration",
    version="1.0.0"
)


class HealthResponse(BaseModel):
    status: str
    timestamp: float


@app.get("/", )
async def root():
    return {
        "message": "Application Signals Demo",
        "timestamp": time.time()
    }


@app.get("/health", response_model=HealthResponse)
async def health_check():
    """Health check endpoint"""
    return HealthResponse(
        status="healthy",
        timestamp=time.time()
    )


@app.get("/api/hello/{name}")
async def hello(name: str):
    """Personalized greeting with simulated processing"""
    time.sleep(random.uniform(0.01, 0.1))

    return {
        "message": f"Hello, {name}!",
        "timestamp": time.time()
    }


@app.get("/api/slow")
async def slow_endpoint():
    """Intentionally slow endpoint for latency testing"""
    time.sleep(random.uniform(0.5, 1.5))
    return {"message": "Slow operation completed"}


@app.get("/api/error")
async def error_endpoint():
    """Endpoint that randomly raises errors"""
    if random.random() > 0.5:
        raise HTTPException(status_code=500, detail="Simulated error")
    return {"message": "No error"}


@app.get("/api/external")
async def external_call():
    """Makes external HTTP call for distributed tracing"""
    try:
        response = requests.get("https://httpbin.org/delay/1", timeout=5)
        return {
            "message": "External call completed",
            "status_code": response.status_code
        }
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))


if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=80)

2.3 Create requirements.txt

# Core dependencies only - OpenTelemetry provided by ADOT init container
fastapi==0.115.0
uvicorn==0.32.1
pydantic==2.10.3
requests==2.32.3

IMPORTANT: Do NOT install opentelemetry-* packages. They're provided by the ADOT init container.

2.4 Create start.sh

#!/bin/bash
# Startup script for Application Signals

echo "Starting Application with Application Signals..."
echo "PYTHONPATH: ${PYTHONPATH:-not set}"
echo "Application Signals: ${OTEL_AWS_APPLICATION_SIGNALS_ENABLED:-not set}"

# Run uvicorn directly - NO opentelemetry-instrument wrapper needed
# NO --workers flag - auto-instrumentation requires single worker
exec uvicorn app:app --host 0.0.0.0 --port 80

Make it executable:

chmod +x start.sh

2.5 Create Dockerfile

FROM public.ecr.aws/docker/library/python:3.11-slim as builder

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y gcc && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY . .

# Production stage
FROM public.ecr.aws/docker/library/python:3.11-slim

WORKDIR /app

# Install runtime dependencies
RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/*

# Copy from builder
COPY --from=builder /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY --from=builder /usr/local/bin /usr/local/bin
COPY --from=builder /app .

# Make start script executable
RUN chmod +x start.sh

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:80/health || exit 1

EXPOSE 80

CMD ["./start.sh"]

Step 3: Build and Push Docker Image

3.1 Set Variables

# Replace with your values
export AWS_REGION="us-east-1"

aws ecr create-repository \
  --repository-name ${APP_NAME} \
  --region ${AWS_REGION}

Output: Repository URI (save this)

3.3 Build Docker Image

cd ~/my-appsignals-app
docker build -t ${APP_NAME} .

3.4 Push to ECR

# Login to ECR
aws ecr get-login-password --region ${AWS_REGION} | \
  docker login --username AWS --password-stdin \
  ${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com

# Tag image
docker tag ${APP_NAME}:latest \
  ${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com/${APP_NAME}:latest

# Push image
docker push ${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com/${APP_NAME}:latest

Step 4: Create CloudWatch Agent Configuration

4.1 Create Configuration File

cat > /tmp/ecs-cwagent.json << 'EOF'
{
  "traces": {
    "traces_collected": {
      "application_signals": {}
    }
  },
  "logs": {
    "metrics_collected": {
      "application_signals": {}
    }
  }
}
EOF

4.2 Upload to SSM Parameter Store

aws ssm put-parameter \
  --name "ecs-cwagent-${APP_NAME}" \
  --type "String" \
  --value "$(cat /tmp/ecs-cwagent.json)" \
  --region "${AWS_REGION}" \
  --overwrite

Output: Version number (1 if new, incremented if updating)

Step 5: Create IAM Roles

5.1 Create Trust Policy

cat > /tmp/task-trust-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "ecs-tasks.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
EOF

5.2 Create Task Role

aws iam create-role \
  --role-name ${APP_NAME}-ecs-task-role \
  --assume-role-policy-document file:///tmp/task-trust-policy.json

# Attach required policies
aws iam attach-role-policy \
  --role-name ${APP_NAME}-ecs-task-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchAgentServerPolicy

aws iam attach-role-policy \
  --role-name ${APP_NAME}-ecs-task-role \
  --policy-arn arn:aws:iam::aws:policy/AWSXRayDaemonWriteAccess

5.3 Create Task Execution Role

aws iam create-role \
  --role-name ${APP_NAME}-ecs-execution-role \
  --assume-role-policy-document file:///tmp/task-trust-policy.json

# Attach policies
aws iam attach-role-policy \
  --role-name ${APP_NAME}-ecs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AmazonECSTaskExecutionRolePolicy

aws iam attach-role-policy \
  --role-name ${APP_NAME}-ecs-execution-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonSSMReadOnlyAccess

5.4 Add CloudWatch Logs Permissions

cat > /tmp/logs-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "logs:CreateLogGroup",
        "logs:CreateLogStream",
        "logs:PutLogEvents"
      ],
      "Resource": "arn:aws:logs:*:*:log-group:/ecs/*"
    }
  ]
}
EOF

aws iam put-role-policy \
  --role-name ${APP_NAME}-ecs-execution-role \
  --policy-name CloudWatchLogsPolicy \
  --policy-document file:///tmp/logs-policy.json

Step 7: Setup Networking

7.1 Get VPC and Subnets

If you already have a VPC:

# List VPCs
aws ec2 describe-vpcs --region ${AWS_REGION} \
  --query 'Vpcs[*].{VpcId:VpcId,CIDR:CidrBlock,Name:Tags[?Key==`Name`].Value|[0]}'

# Choose your VPC
export VPC_ID="vpc-xxxxx"

# Get private subnets
aws ec2 describe-subnets --region ${AWS_REGION} \
  --filters "Name=vpc-id,Values=${VPC_ID}" \
  --query 'Subnets[?MapPublicIpOnLaunch==`false`].[SubnetId,AvailabilityZone,CidrBlock]'

# Save subnet IDs
export SUBNET_1="subnet-xxxxx"
export SUBNET_2="subnet-yyyyy"

7.2 Create Security Group

export SG_NAME="${APP_NAME}-ecs-sg"

# Create security group
SG_ID=$(aws ec2 create-security-group \
  --group-name ${SG_NAME} \
  --description "Security group for ${APP_NAME} ECS tasks" \
  --vpc-id ${VPC_ID} \
  --region ${AWS_REGION} \
  --query 'GroupId' \
  --output text)

echo "Security Group ID: $SG_ID"
export SG_ID

# Add ingress rule (allow HTTP from anywhere for testing)
aws ec2 authorize-security-group-ingress \
  --group-id ${SG_ID} \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0 \
  --region ${AWS_REGION}

For production: Replace 0.0.0.0/0 with specific IP ranges or load balancer security group.

Step 9: Deploy ECS Service

aws ecs create-service \
  --cluster ${CLUSTER_NAME} \
  --service-name ${APP_NAME}-service \
  --task-definition ${APP_NAME}-task:1 \
  --desired-count 1 \
  --launch-type FARGATE \
  --network-configuration "awsvpcConfiguration={subnets=[${SUBNET_1},${SUBNET_2}],securityGroups=[${SG_ID}],assignPublicIp=ENABLED}" \
  --region ${AWS_REGION}

Note: assignPublicIp=ENABLED is for testing. For production, use a load balancer.

Troubleshooting

Issue: Service Not Appearing in Application Signals

Check:

# 1. Verify metrics are being published
aws cloudwatch list-metrics \
  --namespace ApplicationSignals \
  --region ${AWS_REGION} \
  --dimensions Name=Service,Value=${APP_NAME}

# 2. Check if Application Signals is enabled
aws iam get-role \
  --role-name AWSServiceRoleForCloudWatchApplicationSignals

# 3. Check container logs for errors
aws logs tail /ecs/${APP_NAME} --region ${AWS_REGION} --since 10m

Issue: Init Container Fails

Check:

# Describe task to see init container exit code
aws ecs describe-tasks \
  --cluster ${CLUSTER_NAME} \
  --tasks ${TASK_ARN} \
  --region ${AWS_REGION} \
  --query 'tasks[0].containers[?name==`init`].{name:name,exitCode:exitCode,reason:reason}'

Expected: exitCode: 0

If non-zero: Image might not be available. Verify tag v0.12.1 exists:

docker pull public.ecr.aws/aws-observability/adot-autoinstrumentation-python:v0.12.1

Issue: Application Can't Connect to CloudWatch Agent

Symptoms:

ConnectionRefusedError: [Errno 111] Connection refused

Check:

  1. Verify CloudWatch agent container is RUNNING
  2. Check container dependency - app should depend on init with condition SUCCESS
  3. Verify environment variables (especially endpoints: http://localhost:4316 and http://localhost:2000)

Issue: No Traces in X-Ray

Common causes:

  1. Missing OTEL_PYTHON_DISTRO=aws_distro - Critical for Application Signals
  2. Missing OTEL_TRACES_SAMPLER_ARG - X-Ray sampler endpoint required
  3. Using multiple workers - Must run single worker (uvicorn app:app without --workers)
  4. Using opentelemetry-instrument wrapper - Don't use it, let PYTHONPATH handle instrumentation

Issue: IAM Permission Errors

Symptoms:

AccessDeniedException: ... logs:CreateLogGroup

Fix: Add CloudWatch Logs permissions to execution role (see Step 5.4)

Init Container Details

Image: public.ecr.aws/aws-observability/adot-autoinstrumentation-python:v0.12.1

Why versioned tag, not :latest?

  • The :latest tag doesn't exist for Python ADOT images
  • Use specific version tags (check https://github.com/aws-observability/aws-otel-python-instrumentation/releases)
  • Current stable: v0.12.1

What it does:

  1. Runs once before app container starts
  2. Copies ADOT Python instrumentation files from /autoinstrumentation/ to shared volume
  3. Exits with code 0
  4. App container mounts the volume and uses PYTHONPATH to load instrumentation

CloudWatch Agent Configuration

Must include BOTH sections:

{
  "traces": {
    "traces_collected": {
      "application_signals": {}   // Not just "otlp"!
    }
  },
  "logs": {
    "metrics_collected": {
      "application_signals": {}
    }
  }
}

Application Code Requirements

DO:

  • Write normal FastAPI code
  • Use standard libraries (requests, psycopg2, etc.)
  • Keep code clean - no OpenTelemetry imports needed

DON'T:

Cleanup

To remove all resources:

# Delete ECS service
aws ecs update-service \
  --cluster ${CLUSTER_NAME} \
  --service ${APP_NAME}-service \
  --desired-count 0 \
  --region ${AWS_REGION}

aws ecs delete-service \
  --cluster ${CLUSTER_NAME} \
  --service ${APP_NAME}-service \
  --region ${AWS_REGION}

# Delete ECS cluster
aws ecs delete-cluster \
  --cluster ${CLUSTER_NAME} \
  --region ${AWS_REGION}

# Delete ECR repository
aws ecr delete-repository \
  --repository-name ${APP_NAME} \
  --force \
  --region ${AWS_REGION}

# Delete security group
aws ec2 delete-security-group \
  --group-id ${SG_ID} \
  --region ${AWS_REGION}

# Delete IAM roles
aws iam delete-role \
  --role-name ${APP_NAME}-ecs-task-role

aws iam delete-role \
  --role-name ${APP_NAME}-ecs-execution-role

# Delete SSM parameter
aws ssm delete-parameter \
  --name ecs-cwagent-${APP_NAME} \
  --region ${AWS_REGION}
