మీరు CloudWatch Application Signals కోసం ఎనేబుల్ చేస్తున్న ప్లాట్‌ఫార్మ్ మరియు రన్‌టైమ్‌ను ఎంచుకోండి. ప్రతి మార్గదర్శకం apm-center లో నిర్వహించబడే ఎనేబుల్‌మెంట్ కంటెంట్ ఆధారంగా రూపొందించబడింది మరియు అదే డాక్స్ UI లో ఇక్కడ ప్రదర్శించబడుతుంది.

ఎనేబుల్‌మెంట్ మార్గదర్శకాలు (16 కాంబినేషన్లు)

మార్గదర్శకాలు లోడ్ అవుతున్నాయి...

కస్టమ్ మెట్రిక్స్ (apm-center నమూనాల నుండి)

apm-center లోని కొన్ని నమూనా సేవలు పరీక్ష సమయంలో "కస్టమ్ మెట్రిక్స్" ప్రవర్తనను ఉపయోగించడానికి ఒక ఎండ్‌పాయింట్‌ను కలిగి ఉంటాయి. ఉదాహరణకు, EKS Python నమూనా GET /api/metrics అందిస్తుంది:

EKS Python ఎనేబుల్‌మెంట్ మార్గదర్శకాన్ని తెరవండి

@app.get("/api/metrics")
async def metrics():
    """Custom metrics endpoint"""
    # Simulate some business logic
    processing_time = random.uniform(0.1, 0.5)
    time.sleep(processing_time)

    # Simulate different response times based on some logic
    if processing_time > 0.3:
        status = "slow"
    else:
        status = "fast"

    return {
        "message": "Metrics endpoint",
        "processing_time": processing_time,
        "status": status,
        "timestamp": time.time(),
        "service": "python-eks-service"
    }
