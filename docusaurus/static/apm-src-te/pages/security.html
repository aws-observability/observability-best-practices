CloudWatch APM డాక్యుమెంటేషన్

భద్రత & సమ్మతి

భద్రత

CloudWatch Application Signals మీ observability డేటాను రక్షించడానికి మరియు పరిశ్రమ ప్రమాణాలతో సమ్మతిని నిర్ధారించడానికి బహుళ భద్రత పొరలను అమలు చేస్తుంది.

డేటా ఎన్‌క్రిప్షన్

విశ్రాంతిలో ఎన్‌క్రిప్షన్:

  • అన్ని ట్రేస్ డేటా AES-256 ఎన్‌క్రిప్షన్ ఉపయోగించి ఎన్‌క్రిప్ట్ చేయబడుతుంది
  • RUM డేటా AWS Key Management Service (KMS) తో ఎన్‌క్రిప్ట్ చేయబడుతుంది
  • Synthetic monitoring డేటా ఎన్‌క్రిప్ట్ చేసి నిల్వ చేయబడుతుంది
  • అదనపు నియంత్రణ కోసం మీరు కస్టమర్-నిర్వహిత కీలను ఉపయోగించవచ్చు

రవాణాలో ఎన్‌క్రిప్షన్:

  • అన్ని డేటా ప్రసారం TLS 1.2 లేదా అంతకంటే ఎక్కువ ఉపయోగిస్తుంది
  • CloudWatch సేవలకు API కాల్‌లు ఎన్‌క్రిప్ట్ చేయబడతాయి
  • RUM డేటా సేకరణ HTTPS ఉపయోగిస్తుంది
  • ఏజెంట్-టు-కలెక్టర్ కమ్యూనికేషన్ భద్రపరచబడుతుంది

గుర్తింపు మరియు యాక్సెస్ నిర్వహణ

IAM అనుమతులు:

Application Signals కు యాక్సెస్ ఇచ్చేటప్పుడు కనీస అధికార సూత్రాన్ని ఉపయోగించండి:

చదవడానికి మాత్రమే యాక్సెస్:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
పూర్తి యాక్సెస్:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

డేటా గోప్యత మరియు PII రక్షణ

డేటా శుద్ధీకరణ:

  • సున్నితమైన సమాచారం కోసం డేటా మాస్కింగ్ అమలు చేయండి
  • ట్రేస్ అట్రిబ్యూట్‌లలో PII కోసం టోకెనైజేషన్ ఉపయోగించండి
  • డేటా నిలుపుదల విధానాలను కాన్ఫిగర్ చేయండి
  • సేకరించిన డేటా యొక్క సాధారణ ఆడిట్

ఉత్తమ పద్ధతులు:

  • ట్రేస్‌లలో పాస్‌వర్డ్‌లు, API కీలు లేదా టోకెన్‌లను ఎప్పుడూ చేర్చకండి
  • ఇమెయిల్ చిరునామాలు మరియు ఫోన్ నంబర్‌లను హ్యాష్ లేదా మాస్క్ చేయండి
  • సున్నితమైన డేటా కలిగి ఉన్న URL ల నుండి క్వెరీ పారామీటర్‌లను తొలగించండి
  • వ్యక్తిగత సమాచారం బదులు సాధారణ ఐడెంటిఫైయర్‌లను ఉపయోగించండి

నెట్‌వర్క్ భద్రత

VPC కాన్ఫిగరేషన్:

  • మెరుగైన భద్రత కోసం మీ VPC లో ఏజెంట్‌లను డిప్లాయ్ చేయండి
  • CloudWatch API కాల్‌ల కోసం VPC ఎండ్‌పాయింట్‌లను ఉపయోగించండి
  • ట్రాఫిక్‌ను పరిమితం చేయడానికి సెక్యూరిటీ గ్రూప్‌లను కాన్ఫిగర్ చేయండి
  • అదనపు నియంత్రణ కోసం నెట్‌వర్క్ ACL లను అమలు చేయండి

ఏజెంట్ భద్రత:

  • కనీస అవసరమైన అనుమతులతో ఏజెంట్‌లను రన్ చేయండి
  • భద్రతా ప్యాచ్‌లతో ఏజెంట్‌లను అప్‌డేట్ చేసి ఉంచండి
  • భద్రతా సంఘటనల కోసం ఏజెంట్ లాగ్‌లను మానిటర్ చేయండి
  • సాధ్యమైన చోట చదవడానికి మాత్రమే యాక్సెస్ ఉపయోగించండి

డేటా గోప్యత

Application Signals మీ డేటా గోప్యత అవసరాలను ఎలా నిర్వహిస్తుందో అర్థం చేసుకోవడం.

డేటా సేకరణ మరియు ఉపయోగం

మేము ఏమి సేకరిస్తాము:

  • అప్లికేషన్ పనితీరు మెట్రిక్స్ మరియు ట్రేస్‌లు
  • సేవా డిపెండెన్సీ సమాచారం
  • ఎర్రర్ సమాచారం మరియు స్టాక్ ట్రేస్‌లు
  • సిస్టమ్ మరియు ఇన్‌ఫ్రాస్ట్రక్చర్ మెటాడేటా

మేము ఏమి సేకరించము:

  • అప్లికేషన్ వ్యాపార డేటా లేదా వినియోగదారు కంటెంట్
  • డేటాబేస్ కంటెంట్‌లు లేదా ఫైల్ సిస్టమ్ డేటా
  • వినియోగదారు ఆధారాలు లేదా ప్రమాణీకరణ టోకెన్‌లు
  • వ్యక్తిగతంగా గుర్తించగల సమాచారం (స్పష్టంగా చేర్చబడితే తప్ప)

డేటా నిలుపుదల

డిఫాల్ట్ నిలుపుదల కాలాలు:

  • ట్రేస్ డేటా: 30 రోజులు (1 సంవత్సరం వరకు కాన్ఫిగర్ చేయగలరు)
  • మెట్రిక్స్: 15 నెలలు (ప్రామాణిక CloudWatch నిలుపుదల)
  • RUM డేటా: 30 రోజులు (1 సంవత్సరం వరకు కాన్ఫిగర్ చేయగలరు)
  • Synthetics ఫలితాలు: 90 రోజులు (కాన్ఫిగర్ చేయగలరు)

డేటా తొలగింపు:

  • CloudWatch డేటా నిలుపుదల విధానాలను ఉపయోగించండి
  • ఆటోమేటెడ్ డేటా లైఫ్‌సైకిల్ నిర్వహణను అమలు చేయండి
  • అవసరమైతే AWS Support ద్వారా డేటా తొలగింపు అభ్యర్థించండి

సమ్మతి ఫ్రేమ్‌వర్క్‌లు

మద్దతు ఉన్న ప్రమాణాలు:

  • SOC 2: భద్రత, లభ్యత మరియు గోప్యత నియంత్రణలు
  • PCI DSS: చెల్లింపు కార్డ్ పరిశ్రమ డేటా భద్రతా ప్రమాణాలు
  • HIPAA: ఆరోగ్య బీమా పోర్టబిలిటీ మరియు జవాబుదారీ చట్టం
  • GDPR: సాధారణ డేటా రక్షణ నిబంధన
  • FedRAMP: ఫెడరల్ రిస్క్ మరియు ఆథరైజేషన్ మేనేజ్‌మెంట్ ప్రోగ్రామ్

సమ్మతి మార్గదర్శకత్వం

సాధారణ సమ్మతి సందర్భాల కోసం నిర్దిష్ట మార్గదర్శకత్వం.

HIPAA సమ్మతి

అవసరాలు:

  • ట్రేస్‌లలో రక్షిత ఆరోగ్య సమాచారాన్ని (PHI) ఎన్‌క్రిప్ట్ చేయండి
  • యాక్సెస్ నియంత్రణలు మరియు ఆడిట్ లాగింగ్ అమలు చేయండి
  • AWS తో Business Associate Agreements (BAA) ఉపయోగించండి
  • తగిన డేటా నిలుపుదల విధానాలను కాన్ఫిగర్ చేయండి

అమలు:

  • ట్రేస్ అట్రిబ్యూట్‌లు మరియు స్పాన్ పేర్లలో PHI ని మాస్క్ చేయండి
  • కస్టమర్-నిర్వహిత KMS కీలను ఉపయోగించండి
  • ఆడిట్ లాగింగ్ కోసం CloudTrail ను ఎనేబుల్ చేయండి
  • డేటా ఐసోలేషన్ కోసం VPC ఎండ్‌పాయింట్‌లను కాన్ఫిగర్ చేయండి

PCI DSS సమ్మతి

ముఖ్య పరిగణనలు:

  • ట్రేస్‌లు లేదా లాగ్‌లలో కార్డ్‌హోల్డర్ డేటాను ఎప్పుడూ చేర్చకండి
  • నెట్‌వర్క్ సెగ్మెంటేషన్ అమలు చేయండి
  • రవాణాలో మరియు విశ్రాంతిలో డేటా కోసం ఎన్‌క్రిప్షన్ ఉపయోగించండి
  • సాధారణ భద్రతా అంచనాలు మరియు పెనెట్రేషన్ టెస్టింగ్

ఉత్తమ పద్ధతులు:

  • ఇన్‌స్ట్రుమెంటేషన్ ముందు సున్నితమైన డేటాను టోకెనైజ్ చేయండి
  • Application Signals ఏజెంట్‌ల కోసం ప్రైవేట్ సబ్‌నెట్‌లను ఉపయోగించండి
  • కనీస-అధికార IAM విధానాలను అమలు చేయండి
  • సేకరించిన డేటా యొక్క సాధారణ సమీక్ష

GDPR సమ్మతి

డేటా సబ్జెక్ట్ హక్కులు:

  • వ్యక్తిగత డేటాను యాక్సెస్ చేసే హక్కు
  • సవరణ హక్కు
  • తొలగింపు హక్కు ("మరచిపోయే హక్కు")
  • డేటా పోర్టబిలిటీ హక్కు

అమలు:

  • టెలిమెట్రీ డేటాలో PII సేకరణను తగ్గించండి
  • డేటా అజ్ఞాతీకరణ సాంకేతికతలను అమలు చేయండి
  • వ్యాపార అవసరాల ఆధారంగా డేటా నిలుపుదలను కాన్ఫిగర్ చేయండి
  • డేటా ప్రాసెసింగ్ కార్యకలాపాలను డాక్యుమెంట్ చేయండి

భద్రతా మానిటరింగ్

భద్రతా సంఘటనల కోసం మీ Application Signals డిప్లాయ్‌మెంట్‌ను మానిటర్ చేయండి.

భద్రతా ఉత్తమ పద్ధతులు

  • API కార్యకలాప లాగింగ్ కోసం CloudTrail ను ఎనేబుల్ చేయండి
  • IAM విధాన మార్పులు మరియు యాక్సెస్ నమూనాలను మానిటర్ చేయండి
  • అసాధారణ డేటా యాక్సెస్ నమూనాల కోసం అలర్ట్‌లను సెటప్ చేయండి
  • సాధారణ భద్రతా అంచనాలు మరియు దుర్బలత్వ స్కాన్‌లు
  • బహుళ-కారకాల ప్రమాణీకరణ (MFA) అమలు చేయండి

సంఘటన ప్రతిస్పందన

  • భద్రతా సంఘటనల కోసం సంఘటన ప్రతిస్పందన ప్రణాళిక కలిగి ఉండండి
  • రాజీ పడితే Application Signals ను ఎలా నిష్క్రియం చేయాలో తెలుసుకోండి
  • ఎస్కలేషన్ విధానాలను డాక్యుమెంట్ చేయండి
  • సాధారణ సంఘటన ప్రతిస్పందన పరీక్ష
