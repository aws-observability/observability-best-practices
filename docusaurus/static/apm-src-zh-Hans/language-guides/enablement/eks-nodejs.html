Overview

This guide provides step-by-step instructions to deploy a Node.js Express application on Amazon EKS with AWS Application Signals auto-instrumentation. This is the EKS equivalent of the Lambda Node.js function, reusing the same business logic and endpoints.

What you'll accomplish:
  • Deploy a Node.js Express app on EKS
  • Enable AWS Application Signals auto-instrumentation
  • View your service in CloudWatch Application Signals console
  • Monitor service metrics, traces, and operations
Time to complete: 30-45 minutes

Prerequisites

Required Tools

  • AWS CLI v2 (latest version)
  • Docker Desktop
  • kubectl (latest version)
  • Node.js 18+ (for local development)
  • npm (for local development)
  • AWS Account with appropriate permissions
  • Text editor

AWS Permissions Required

  • EKS: Full access (create clusters, node groups, add-ons)
  • IAM: Create roles and policies
  • CloudWatch: Logs and Application Signals access
  • ECR: Push/pull container images
  • VPC: Create and manage VPC resources

Knowledge Assumed

  • Basic AWS CLI usage
  • Basic Docker commands
  • Basic kubectl commands
  • Basic understanding of EKS concepts
  • Basic Node.js/Express knowledge

Architecture

EKS Observability Add-on (Standard Approach): This guide uses the official EKS observability add-on, which is the recommended and standard way to enable Application Signals on EKS clusters. The add-on automatically handles telemetry collection without requiring manual CloudWatch Agent configuration.

Your EKS cluster will run:

  1. Node.js Express Application: Your web API with auto-instrumentation annotations
  2. EKS Observability Add-on: Official AWS add-on for automatic tracing and metrics collection
  3. CloudWatch Agent (managed by add-on): Automatically deployed and configured by the add-on

Flow:

EKS Cluster
  → Node.js Express App (auto-instrumented via annotation)
  → EKS Observability Add-on
  → CloudWatch Agent (managed)
  → CloudWatch Application Signals
Key Benefits of EKS Observability Add-on:
  • No manual CloudWatch Agent configuration required
  • Automatic auto-instrumentation for supported languages
  • Built-in support for Application Signals
  • Simplified deployment and management

Step 1: Create the Application

1.1 Create Project Structure

mkdir eks-nodejs
cd eks-nodejs

1.2 Create the Node.js Application

Create index.js with the same endpoints as the Lambda function:

const express = require('express');
const axios = require('axios');

const app = express();
const port = process.env.PORT || 8080;

// Middleware
app.use(express.json());

// Simple endpoints
app.get('/', (req, res) => {
    res.json({
        message: 'AWS Application Signals EKS Node.js Demo',
        timestamp: Date.now(),
        service: 'nodejs-eks-service',
        runtime: 'nodejs18.x'
    });
});

app.get('/health', (req, res) => {
    res.json({
        status: 'healthy',
        timestamp: Date.now(),
        service: 'nodejs-eks-service'
    });
});

app.get('/api/hello/:name', (req, res) => {
    const { name } = req.params;
    // Simulate some processing time
    setTimeout(() => {
        res.json({
            message: `Hello, ${name}!`,
            timestamp: Date.now(),
            service: 'nodejs-eks-service'
        });
    }, Math.random() * 100 + 10);
});

app.get('/api/slow', (req, res) => {
    // Simulate slow operation (500-1500ms)
    const delay = Math.random() * 1000 + 500;
    setTimeout(() => {
        res.json({
            message: 'Slow operation completed',
            timestamp: Date.now(),
            service: 'nodejs-eks-service',
            duration_ms: Math.floor(delay)
        });
    }, delay);
});

app.get('/api/error', (req, res) => {
    if (Math.random() > 0.5) {
        throw new Error('Simulated error for testing Application Signals error tracking');
    }
    res.json({
        message: 'No error occurred',
        timestamp: Date.now(),
        service: 'nodejs-eks-service'
    });
});

app.get('/api/external', async (req, res) => {
    try {
        const response = await axios.get('https://httpbin.org/delay/1', {
            timeout: 10000
        });
        res.json({
            message: 'External call completed',
            external_status_code: response.status,
            timestamp: Date.now(),
            service: 'nodejs-eks-service'
        });
    } catch (error) {
        res.status(500).json({
            error: 'External call failed',
            message: error.message,
            timestamp: Date.now(),
            service: 'nodejs-eks-service'
        });
    }
});

app.listen(port, '0.0.0.0', () => {
    console.log(`Server running on port ${port}`);
});

Step 3: Deploy to EKS

3.1 Create EKS Cluster

# Create EKS cluster
eksctl create cluster --name appsignals-cluster --region us-east-1 --nodegroup-name workers --node-type t3.medium --nodes 2 --nodes-min 1 --nodes-max 3

# Create namespace
kubectl create namespace appsignals-demo

3.2 Install EKS Observability Add-on (Required)

Important: The EKS observability add-on is the standard and recommended way to enable Application Signals on EKS. This add-on automatically handles telemetry collection and provides auto-instrumentation capabilities. 
# Install the EKS observability add-on (REQUIRED for Application Signals)
aws eks create-addon --cluster-name appsignals-cluster --addon-name amazon-cloudwatch-observability --region us-east-1

# Verify the add-on is installed
aws eks describe-addon --cluster-name appsignals-cluster --addon-name amazon-cloudwatch-observability --region us-east-1
What the EKS Observability Add-on provides:
  • Automatic CloudWatch Agent deployment
  • Built-in support for Application Signals
  • Auto-instrumentation for Java, Python, Node.js, and .NET
  • No manual configuration required

3.3 Build and Push Docker Image

# Create ECR repository
aws ecr create-repository --repository-name my-appsignals-nodejs-app --region us-east-1

# Get login token
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 188524202814.dkr.ecr.us-east-1.amazonaws.com

# Build and tag image
docker build -t my-appsignals-nodejs-app .
docker tag my-appsignals-nodejs-app:latest 188524202814.dkr.ecr.us-east-1.amazonaws.com/my-appsignals-nodejs-app:latest

# Push image
docker push 188524202814.dkr.ecr.us-east-1.amazonaws.com/my-appsignals-nodejs-app:latest

Step 4: Generate Traffic and Monitor

4.1 Generate Traffic

Create generate-traffic.sh:

#!/bin/bash

# Port forward the service
kubectl port-forward service/nodejs-eks-service 8080:80 -n appsignals-demo &
PORT_FORWARD_PID=$!

# Wait for port forward to be ready
sleep 10

BASE_URL="http://localhost:8080"

echo "Generating traffic for Node.js EKS Application Signals demo..."

# Generate traffic for 5 minutes
for i in {1..300}; do
    echo "Request $i"

    # Root endpoint
    curl -s "$BASE_URL/" > /dev/null

    # Health check
    curl -s "$BASE_URL/health" > /dev/null

    # Hello endpoint
    curl -s "$BASE_URL/api/hello/User$i" > /dev/null

    # Slow endpoint
    curl -s "$BASE_URL/api/slow" > /dev/null

    # Error endpoint
    curl -s "$BASE_URL/api/error" > /dev/null

    # External call
    curl -s "$BASE_URL/api/external" > /dev/null

    sleep 1
done

echo "Traffic generation complete!"

# Clean up port forward
kill $PORT_FORWARD_PID

Make it executable and run:

chmod +x generate-traffic.sh
./generate-traffic.sh

4.2 Monitor in CloudWatch

  1. Open CloudWatch Console

    • Go to AWS Console → CloudWatch

  2. View Service Map

    • Look for "nodejs-eks-service" service

  3. Check Metrics

    • View latency, error rate, and throughput
    • Compare with Lambda metrics

  4. Analyze Traces

    • Click on service to see detailed traces
    • View operation-level metrics

Troubleshooting

Common Issues

  1. Application won't start

    • Check pod status: kubectl get pods -n appsignals-demo
    • Check pod logs: kubectl logs -n appsignals-demo -l app=nodejs-eks-service
    • Verify EKS observability add-on is installed

  2. No data in Application Signals

    • Verify auto-instrumentation annotation is present
    • Check environment variables are set correctly
    • Ensure EKS observability add-on is running

  3. Port forward issues

    • Check service is running: kubectl get svc -n appsignals-demo
    • Verify pods are ready: kubectl get pods -n appsignals-demo
