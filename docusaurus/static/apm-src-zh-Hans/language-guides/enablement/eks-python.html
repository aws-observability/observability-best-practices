概述

本指南提供分步说明，指导您在 Amazon EKS 上部署带有 AWS Application Signals 自动插桩的 Python FastAPI 应用。这是 Lambda Python 函数的 EKS 等效实现，复用了相同的业务逻辑和端点。

完成目标：
  • 在 EKS 上部署 Python FastAPI 应用
  • 启用 AWS Application Signals 自动插桩
  • 在 CloudWatch Application Signals 控制台中查看您的服务
  • 监控服务 metrics、traces 和操作
完成时间： 30-45 分钟

前置条件

必备工具

  • AWS CLI v2（最新版本）
  • Docker Desktop
  • kubectl（最新版本）
  • Python 3.11+ (for local development)
  • pip（用于本地开发）
  • 具有适当权限的 AWS 账户
  • 文本编辑器

  • EKS：完全访问（创建集群、节点组、插件）
  • IAM：创建角色和策略

  • ECR：推送/拉取容器镜像
  • VPC：创建和管理 VPC 资源
  • S3：用于测试的读取权限

前置知识

  • AWS CLI 基础使用

架构

EKS 可观测性插件（标准方式）： This guide uses the official EKS observability add-on, which is the recommended and standard way to enable Application Signals on EKS clusters. The add-on automatically handles telemetry collection 不使用 requiring manual CloudWatch Agent configuration.

您的 EKS 集群将运行：

  1. Python FastAPI Application：带有自动插桩注解的 Web API
  2. EKS 可观测性插件：用于自动追踪和 metrics 收集的官方 AWS 插件
  3. CloudWatch Agent（由插件管理）：由插件自动部署和配置

流程：

EKS Cluster
  → Python FastAPI App (auto-instrumented via annotation)
  → EKS Observability Add-on
  → CloudWatch Agent (managed)
  → CloudWatch Application Signals
EKS 可观测性插件的主要优势：
  • 无需手动配置 CloudWatch Agent
  • 支持语言的自动插桩
  • 内置 Application Signals 支持
  • 简化部署和管理

步骤 1： 创建应用

1.1 创建项目结构

mkdir eks-python
cd eks-python

1.2 Create the Python Application

Create app.py with the same endpoints as the Lambda function:

"""
AWS Application Signals EKS Python Service
FastAPI application with Application Signals auto-instrumentation
"""
import time
import random
import boto3
import requests
from fastapi import FastAPI, HTTPException
from fastapi.responses import JSONResponse
from typing import Dict, Any
import uvicorn

# Initialize FastAPI app
app = FastAPI(
    title="AWS Application Signals EKS Python Demo",
    description="Python service running on EKS with Application Signals",
    version="1.0.0"
)

# Initialize AWS clients
s3_client = boto3.client('s3')


@app.get("/")
async def root():
    """Root endpoint"""
    return {
        "message": "AWS Application Signals EKS Python Demo",
        "timestamp": time.time(),
        "service": "python-eks-service"
    }


@app.get("/health")
async def health():
    """Health check endpoint"""
    return {
        "status": "healthy",
        "timestamp": time.time(),
        "service": "python-eks-service"
    }


@app.get("/api/hello/{name}")
async def hello(name: str):
    """Personalized greeting with simulated processing"""
    # Simulate some processing time
    time.sleep(random.uniform(0.01, 0.1))
    
    return {
        "message": f"Hello, {name}!",
        "timestamp": time.time(),
        "service": "python-eks-service"
    }


@app.get("/api/slow")
async def slow():
    """Intentionally slow endpoint for latency testing"""
    time.sleep(random.uniform(0.5, 1.5))
    
    return {
        "message": "Slow operation completed",
        "timestamp": time.time(),
        "duration": "1-1.5 seconds",
        "service": "python-eks-service"
    }


@app.get("/api/error")
async def error():
    """Endpoint that randomly raises errors for testing error rates"""
    if random.random() > 0.5:
        raise HTTPException(
            status_code=500, 
            detail="Simulated error for testing Application Signals error tracking"
        )
    
    return {
        "message": "No error occurred",
        "timestamp": time.time(),
        "service": "python-eks-service"
    }


@app.get("/api/external")
async def external():
    """Makes external HTTP call for distributed tracing"""
    try:
        # This will be automatically traced by OpenTelemetry
        response = requests.get("https://httpbin.org/delay/1", timeout=5)
        
        return {
            "message": "External call completed",
            "external_status_code": response.status_code,
            "timestamp": time.time(),
            "service": "python-eks-service"
        }
    except Exception as e:
        raise HTTPException(
            status_code=500,
            detail=f"External call failed: {str(e)}"
        )


@app.get("/api/s3")
async def s3():
    """Makes AWS S3 call for testing AWS SDK instrumentation"""
    try:
        # This will be automatically traced by OpenTelemetry
        response = s3_client.list_buckets()
        bucket_names = [bucket['Name'] for bucket in response['Buckets']]
        
        return {
            "message": "S3 call completed successfully",
            "bucket_count": len(bucket_names),
            "buckets": bucket_names[:5],  # Show first 5 buckets
            "timestamp": time.time(),
            "service": "python-eks-service"
        }
    except Exception as e:
        raise HTTPException(
            status_code=500,
            detail=f"S3 call failed: {str(e)}"
        )


@app.get("/api/metrics")
async def metrics():
    """Custom metrics endpoint"""
    # Simulate some business logic
    processing_time = random.uniform(0.1, 0.5)
    time.sleep(processing_time)
    
    # Simulate different response times based on some logic
    if processing_time > 0.3:
        status = "slow"
    else:
        status = "fast"
    
    return {
        "message": "Metrics endpoint",
        "processing_time": processing_time,
        "status": status,
        "timestamp": time.time(),
        "service": "python-eks-service"
    }


if __name__ == "__main__":
    uvicorn.run(
        "app:app",
        host="0.0.0.0",
        port=8000,
        reload=True,
        log_level="info"
    )

1.4 创建 Dockerfile

Create Dockerfile:

# Use Python 3.11 slim image
FROM python:3.11-slim

# Set working directory
WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Copy requirements first for better caching
COPY requirements.txt .

# Install Python dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY app.py .

# Create non-root user
RUN useradd --create-home --shell /bin/bash app && chown -R app:app /app
USER app

# Expose port
EXPOSE 8000

# Health check
HEALTHCHECK --interval=30s --timeout=30s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# Run the application
CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]

步骤 3： 部署到 EKS

3.1 创建 EKS 集群

# Create EKS cluster
eksctl create cluster --name appsignals-cluster --region us-east-1 --nodegroup-name workers --node-type t3.medium --nodes 2 --nodes-min 1 --nodes-max 3

# Create namespace
kubectl create namespace appsignals-demo

3.2 安装 EKS 可观测性插件（必需）

重要： EKS 可观测性插件是在 EKS 上启用 Application Signals 的标准和推荐方式。该插件自动处理遥测数据收集并提供自动插桩功能。 
# Install the EKS observability add-on (REQUIRED for Application Signals)
aws eks create-addon --cluster-name appsignals-cluster --addon-name amazon-cloudwatch-observability --region us-east-1

# Verify the add-on is installed
aws eks describe-addon --cluster-name appsignals-cluster --addon-name amazon-cloudwatch-observability --region us-east-1
EKS 可观测性插件提供：
  • 自动部署 CloudWatch Agent
  • 内置 Application Signals 支持
  • 支持 Java、Python、Node.js 和 .NET 的自动插桩
  • 无需手动配置

3.3 构建并推送 Docker 镜像

# Create ECR repository
aws ecr create-repository --repository-name my-appsignals-python-app --region us-east-1

Create deployment.yaml:

apiVersion: apps/v1

  replicas: 2

步骤 4： 生成流量并监控

4.1 生成流量

Create generate-traffic.sh:

#!/bin/bash

# Port forward the service
kubectl port-forward service/python-eks-service 8000:80 -n appsignals-demo &
PORT_FORWARD_PID=$!

# Wait for port forward to be ready
sleep 10

BASE_URL="http://localhost:8000"

echo "Generating traffic for Python EKS Application Signals demo..."

# Generate traffic for 5 minutes
for i in {1..300}; do
    echo "Request $i"

    # Root endpoint
    curl -s "$BASE_URL/" > /dev/null

    # Health check
    curl -s "$BASE_URL/health" > /dev/null

    # Hello endpoint
    curl -s "$BASE_URL/api/hello/User$i" > /dev/null

    # Slow endpoint
    curl -s "$BASE_URL/api/slow" > /dev/null

    # Error endpoint
    curl -s "$BASE_URL/api/error" > /dev/null

    # External call
    curl -s "$BASE_URL/api/external" > /dev/null

    # S3 call
    curl -s "$BASE_URL/api/s3" > /dev/null

    # Metrics endpoint
    curl -s "$BASE_URL/api/metrics" > /dev/null

    sleep 1
done

echo "Traffic generation complete!"

# Clean up port forward
kill $PORT_FORWARD_PID

故障排除

常见问题

  1. 应用无法启动

    • 检查 Pod 状态： kubectl get pods -n appsignals-demo
    • 检查 Pod 日志： kubectl logs -n appsignals-demo -l app=python-eks-service
    • 验证 EKS 可观测性插件已安装

  2. Application Signals 中无数据

    • 验证自动插桩注解是否存在
    • Check 环境变量s are set correctly
    • 确保 EKS 可观测性插件正在运行

  3. 端口转发问题

    • 检查服务是否运行： kubectl get svc -n appsignals-demo
    • 验证 Pod 是否就绪： kubectl get pods -n appsignals-demo
