概述

本指南提供分步说明，指导您部署启用 AWS Application Signals 的 Java Lambda 函数。这是使用 AWS Lambda 控制台内置层支持的最简单方法。

完成目标：
  • 部署启用 Application Signals 的 Java Lambda 函数
  • 通过 Lambda Layer 启用 AWS Application Signals 自动插桩
  • 在 CloudWatch Application Signals 控制台中查看您的服务
  • 监控服务 metrics、traces 和操作
完成时间： 15-20 分钟

前置条件

必备工具

  • AWS CLI v2（最新版本）
  • Java 17+
  • Maven 3.8+
  • 具有适当权限的 AWS 账户
  • 文本编辑器

所需 AWS 权限

  • Lambda：创建和管理函数
  • IAM：创建角色和策略
  • CloudWatch：Logs 和 Application Signals 访问
  • S3：基本读取权限（用于测试）

架构

您的 Lambda 函数将通过以下方式自动插桩：

  1. AWS Lambda Layer for OpenTelemetry - 提供自动插桩
  2. Application Signals 集成 - 内置于层中
  3. 自动追踪 - HTTP 请求、AWS SDK 调用、函数执行

步骤 1： 启用 Application Signals in Your Account

重要： 每个 AWS 账户每个区域只需启用一次 Application Signals。

方案 1：通过 AWS 控制台启用（推荐）

  1. 前往 AWS 控制台 → CloudWatch → Application Signals
  2. 选择您的区域（例如 us-east-1)
  3. Click "Get Started" or "启用 Application Signals"
  4. 等待 2-3 分钟激活

步骤 2： 准备 Java Lambda 函数

2.1 创建应用目录

mkdir -p ~/my-lambda-java-appsignals
cd ~/my-lambda-java-appsignals

2.2 创建项目结构

mkdir -p src/main/java/com/example/appsignals/controller

2.3 创建 pom.xml

使用以下依赖项创建 Maven POM 文件：

  • AWS Lambda Core
  • AWS Lambda Events
  • AWS SDK for S3
  • Spring Boot Web Starter (for local testing)
  • JSON Processing
  • HTTP Client

重要： 不要添加 OpenTelemetry 依赖项。它们由 AWS Lambda Layer 提供。

步骤 3： 构建 Lambda 部署包

3.1 设置变量

export AWS_REGION="us-east-1"
export AWS_ACCOUNT_ID="YOUR_ACCOUNT_ID"
export FUNCTION_NAME="lambda-java"

3.2 构建 JAR 文件

mvn clean package

步骤 4： 为 Lambda 创建 IAM 角色

使用以下策略创建 IAM 角色：

  • AWSLambdaBasicExecutionRole
  • CloudWatchLambdaApplicationSignalsExecutionRolePolicy (CRITICAL!)
  • AmazonS3ReadOnlyAccess (for testing)

4.1 创建 IAM 角色

aws iam create-role \
  --role-name lambda-java-execution-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'

4.2 附加所需策略

aws iam attach-role-policy \
  --role-name lambda-java-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

aws iam attach-role-policy \
  --role-name lambda-java-execution-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLambdaApplicationSignalsExecutionRolePolicy

aws iam attach-role-policy \
  --role-name lambda-java-execution-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

4.3 获取角色 ARN

ROLE_ARN=$(aws iam get-role \
  --role-name lambda-java-execution-role \
  --query 'Role.Arn' \
  --output text)

echo "Role ARN: $ROLE_ARN"

Step 5: 部署 Lambda 函数

5.1 创建 Lambda 函数

aws lambda create-function \
  --function-name lambda-java \
  --runtime java17 \
  --role "$ROLE_ARN" \
  --handler com.example.appsignals.ApplicationSignalsApplication::handleRequest \
  --zip-file fileb://target/lambda-java-appsignals-1.0.0.jar \
  --timeout 30 \
  --memory-size 512

5.2 Add AWS Lambda Layer for OpenTelemetry

对于 us-east-1:

LAYER_ARN="arn:aws:lambda:us-east-1:615299751070:layer:AWSOpenTelemetryDistroJava:18"

5.3 配置环境变量

aws lambda update-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --environment Variables='{
    "AWS_LAMBDA_EXEC_WRAPPER": "/opt/otel-instrument",
    "OTEL_SERVICE_NAME": "'${FUNCTION_NAME}'",
    "OTEL_RESOURCE_ATTRIBUTES": "service.name='${FUNCTION_NAME}',deployment.environment=lambda:'${AWS_REGION}'"
  }'

Step 6: 测试 Lambda 函数

6.1 测试基本功能

aws lambda invoke \
  --function-name ${FUNCTION_NAME} \
  --payload '{"path": "/", "httpMethod": "GET"}' \
  --region ${AWS_REGION} \
  response.json

6.2 生成流量 for Application Signals

for i in {1..50}; do
  aws lambda invoke \
    --function-name ${FUNCTION_NAME} \
    --payload "{\"path\": \"/api/hello/User${i}\", \"httpMethod\": \"GET\"}" \
    --region ${AWS_REGION} \
    /dev/null
done

Step 7: 生成流量 for Testing

To test Application Signals, you need to generate traffic to your Lambda function. We provide two traffic generator options:

7.1 Python 流量生成器（推荐）

创建 Python 脚本以生成真实的流量模式：

#!/usr/bin/env python3
import boto3
import json
import time
import random
import sys

def generate_traffic(function_name, region='us-east-1', profile='eksworkstream', num_requests=50):
    # Initialize Lambda client
    session = boto3.Session(profile_name=profile)
    lambda_client = session.client('lambda', region_name=region)
    
    # Test endpoints with different characteristics
    endpoints = [
        {"path": "/", "method": "GET", "name": "Root", "weight": 20},
        {"path": "/health", "method": "GET", "name": "Health", "weight": 15},
        {"path": "/api/hello/User", "method": "GET", "name": "Hello", "weight": 30},
        {"path": "/api/slow", "method": "GET", "name": "Slow", "weight": 10},
        {"path": "/api/error", "method": "GET", "name": "Error", "weight": 10},
        {"path": "/api/external", "method": "GET", "name": "External", "weight": 8},
        {"path": "/api/s3", "method": "GET", "name": "S3", "weight": 5},
        {"path": "/api/database", "method": "GET", "name": "Database", "weight": 2},
    ]
    
    # Create weighted list for random selection
    weighted_endpoints = []
    for endpoint in endpoints:
        weighted_endpoints.extend([endpoint] * endpoint["weight"])
    
    success_count = 0
    error_count = 0
    
    for i in range(1, num_requests + 1):
        endpoint = random.choice(weighted_endpoints)
        
        if endpoint["name"] == "Hello":
            user_name = f"User{random.randint(1, 1000)}"
            path = f"/api/hello/{user_name}"
        else:
            path = endpoint["path"]
        
        payload = {"path": path, "httpMethod": endpoint["method"]}
        
        try:
            response = lambda_client.invoke(
                FunctionName=function_name,
                InvocationType='RequestResponse',
                Payload=json.dumps(payload)
            )
            
            response_payload = json.loads(response['Payload'].read())
            status_code = response_payload.get('statusCode', 200)
            
            if status_code == 200:
                success_count += 1
            else:
                error_count += 1
                
            print(f"Request {i}: {endpoint['name']} | Status: {status_code}")
            time.sleep(random.uniform(0.1, 0.5))
            
        except Exception as e:
            error_count += 1
            print(f"Request {i}: {endpoint['name']} | Error: {str(e)}")
    
    print(f"Summary: {success_count} successful, {error_count} errors")

if __name__ == "__main__":
    function_name = sys.argv[1] if len(sys.argv) > 1 else "lambda-java"
    num_requests = int(sys.argv[2]) if len(sys.argv) > 2 else 50
    generate_traffic(function_name, num_requests=num_requests)

7.2 Shell 脚本流量生成器

用于流量生成的替代 shell 脚本：

#!/bin/bash
FUNCTION_NAME="lambda-java"
NUM_REQUESTS=50

endpoints=(
    "root:/:GET:20"
    "health:/health:GET:15"
    "hello:/api/hello:GET:30"
    "slow:/api/slow:GET:10"
    "error:/api/error:GET:10"
    "external:/api/external:GET:8"
    "s3:/api/s3:GET:5"
    "database:/api/database:GET:2"
)

for i in $(seq 1 $NUM_REQUESTS); do
    endpoint_info=${endpoints[$((RANDOM % ${#endpoints[@]}))]}
    IFS=':' read -r name path method weight <<< "$endpoint_info"
    
    if [ "$name" = "hello" ]; then
        user_name="User$((RANDOM % 1000 + 1))"
        full_path="/api/hello/$user_name"
    else
        full_path="$path"
    fi
    
    payload="{\"path\": \"$full_path\", \"httpMethod\": \"$method\"}"
    
    aws lambda invoke \
        --function-name $FUNCTION_NAME \
        --payload "$payload" \
        /tmp/response.json
    
    echo "Request $i: $name"
    sleep 0.2
done

7.3 使用说明

Python 流量生成器：

# Save as traffic-generator.py and run:
python3 traffic-generator.py lambda-java
python3 traffic-generator.py lambda-java 100  # 100 requests

Shell 脚本流量生成器:

# Save as generate-traffic.sh, make executable, and run:
chmod +x generate-traffic.sh
./generate-traffic.sh

  • 加权随机选择： 更真实的流量模式

  • 错误模拟： 包含用于故障测试的错误端点
  • 性能测试： 包含用于延迟测试的慢速端点

故障排除

问题：Shell 环境问题

问题： 命令因以下错误而失败： parse error near 'cursor_snap_ENV_VARS...' or command not found: dump_zsh_state

原因： Cursor IDE 环境变量注入正在破坏 shell 环境

解决方案：

  1. 使用新的终端窗口 在 Cursor IDE 之外
  2. 重启 Cursor IDE 完全关闭后重试
  3. 使用 VS Code 代替 Cursor IDE
  4. 使用 Lambda 控制台方法 （推荐在 shell 问题时使用）

问题：角色 ARN 为空

问题： aws: error: argument --role: 需要一个参数

原因： The ROLE_ARN variable is empty, indicating IAM role creation failed

调试步骤：

  1. Check if role 是否存在： 
    aws iam get-role --role-name lambda-java-execution-role
  2. 如果缺失则创建角色： 
    aws iam create-role \
  --role-name lambda-java-execution-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'
  3. 手动获取角色 ARN： 
    aws iam get-role \
  --role-name lambda-java-execution-role \
  --query 'Role.Arn' \
  --output text
  4. 在 create-function 命令中直接使用 ARN

问题：命令语法错误

常见错误：

  • 在同一行使用句号分隔多个命令
  • 使用重复的参数 (e.g., --role twice)
  • 变量缺少引号s

❌ 错误 - 不要这样做：

# WRONG: Commands on same line with period
aws iam create-role \ ... }. aws lambda create-function \ ...

# WRONG: Duplicate --role parameter
aws lambda create-function \
  --role ${ROLE_ARN} \
  --role lambda-java-execution-role

# WRONG: Missing quotes around variable
--role $ROLE_ARN

✅ 正确 - 应该这样做：

# Step 1: Create IAM role (separate command)
aws iam create-role \
  --role-name lambda-java-execution-role \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'

# Step 2: Attach policies (separate commands)
aws iam attach-role-policy \
  --role-name lambda-java-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

aws iam attach-role-policy \
  --role-name lambda-java-execution-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLambdaApplicationSignalsExecutionRolePolicy

# Step 3: Get role ARN
ROLE_ARN=$(aws iam get-role \
  --role-name lambda-java-execution-role \
  --query 'Role.Arn' \
  --output text)

# Step 4: Create Lambda function (with quotes around variable)
aws lambda create-function \
  --function-name lambda-java \
  --runtime java17 \
  --role "$ROLE_ARN" \
  --handler com.example.appsignals.ApplicationSignalsApplication::handleRequest \
  --zip-file fileb://target/lambda-java-appsignals-1.0.0.jar \
  --timeout 30 \
  --memory-size 512

问题：Maven 构建失败

常见 Maven 问题：

  • Shade 插件错误： 使用简化的 pom.xml 提供的
  • Java 版本不匹配： 确保已安装并配置 Java 17
  • 依赖冲突： 如果不需要，请移除 Spring Boot 依赖

问题：服务未出现在 Application Signals 中

检查：

  • 验证您的账户已启用 Application Signals
  • 检查 OpenTelemetry 层是否已附加
  • 验证环境变量设置正确
  • 生成流量后等待 5-10 分钟
  • 确保 IAM 策略 CloudWatchLambdaApplicationSignalsExecutionRolePolicy 已附加

问题：Lambda 函数错误

常见问题：

  • 缺少 IAM 策略 - Ensure CloudWatchLambdaApplicationSignalsExecutionRolePolicy 已附加
  • 错误的层 ARN - 验证您所在区域的层 ARN
  • Missing 环境变量 - Ensure AWS_LAMBDA_EXEC_WRAPPER=/opt/otel-instrument
  • Handler 不匹配 - 确保 handler 为 com.example.appsignals.ApplicationSignalsApplication::handleRequest
  • JAR 文件未找到 - 确保 Maven 构建成功完成

Issue: 权限拒绝错误

检查您的 AWS 凭证：

  • 验证 AWS 配置文件： aws configure list
  • 检查调用者身份： aws sts get-caller-identity
  • 确保您具有所需的 IAM 权限

关键配置详情

关键环境变量

变量 值 用途
AWS_LAMBDA_EXEC_WRAPPER /opt/otel-instrument 启用 OpenTelemetry 插桩
OTEL_SERVICE_NAME ${FUNCTION_NAME} Application Signals 中的服务名称
