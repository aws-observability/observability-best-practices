概述

本指南提供分步说明，指导您部署启用 AWS Application Signals 的 Python Lambda 函数。这是使用 AWS Lambda 控制台内置层支持的最简单方法。

完成目标：
  • 部署启用 Application Signals 的 Python Lambda 函数
  • 通过 Lambda Layer 启用 AWS Application Signals 自动插桩
  • 在 CloudWatch Application Signals 控制台中查看您的服务
  • 监控服务 metrics、traces 和操作
完成时间： 15-20 分钟

前置条件

必备工具

  • AWS CLI v2（最新版本）
  • Python 3.11+
  • 具有适当权限的 AWS 账户
  • 文本编辑器

所需 AWS 权限

  • Lambda：创建和管理函数
  • IAM：创建角色和策略
  • CloudWatch：Logs 和 Application Signals 访问
  • S3：基本读取权限（用于测试）

前置知识

  • AWS CLI 基础使用
  • Basic understanding of Lambda concepts

架构

您的 Lambda 函数将通过以下方式自动插桩：

  1. AWS Lambda Layer for OpenTelemetry - 提供自动插桩
  2. Application Signals 集成 - 内置于层中
  3. 自动追踪 - HTTP 请求、AWS SDK 调用、函数执行
Flow:
Lambda Function (auto-instrumented)
  → Sends traces/metrics to Application Signals
  → View in CloudWatch Application Signals console

步骤 1： 启用 Application Signals in Your Account

重要： 每个 AWS 账户每个区域只需启用一次 Application Signals。

方案 1：通过 AWS 控制台启用（推荐）

  1. 前往 AWS 控制台 → CloudWatch → Application Signals
  2. 选择您的区域（例如 us-east-1)
  3. Click "Get Started" or "启用 Application Signals"
  4. 等待 2-3 分钟激活

方案 2：验证是否已启用

# Check if service-linked role exists
aws iam get-role \
  --role-name AWSServiceRoleForCloudWatchApplicationSignals \
  --region us-east-1

# If role exists, Application Signals is enabled

输出应显示： 如果已启用则显示角色 ARN，否则显示错误。

步骤 2： 准备 Python Lambda 函数

2.1 创建应用目录

mkdir -p ~/my-lambda-appsignals-app
cd ~/my-lambda-appsignals-app

2.2 Create lambda_function.py

"""
AWS Application Signals Lambda Function
Adapted from ECS FastAPI implementation for Lambda
"""
import json
import time
import random
import boto3
import requests
from typing import Dict, Any


def lambda_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """
    Main Lambda handler with Application Signals auto-instrumentation
    
    The AWS Lambda Layer for OpenTelemetry automatically instruments:
    - HTTP requests (requests library)
    - AWS SDK calls (boto3)
    - Lambda function execution
    - Custom spans and metrics
    """
    
    # Parse the event to determine which endpoint to call
    path = event.get('path', '/')
    http_method = event.get('httpMethod', 'GET')
    
    try:
        if path == '/' or path == '':
            return root_handler(event, context)
        elif path == '/health':
            return health_handler(event, context)
        elif path.startswith('/api/hello/'):
            name = path.split('/')[-1]
            return hello_handler(name, event, context)
        elif path == '/api/slow':
            return slow_handler(event, context)
        elif path == '/api/error':
            return error_handler(event, context)
        elif path == '/api/external':
            return external_handler(event, context)
        elif path == '/api/s3':
            return s3_handler(event, context)
        else:
            return {
                'statusCode': 404,
                'headers': {
                    'Content-Type': 'application/json',
                    'Access-Control-Allow-Origin': '*'
                },
                'body': json.dumps({
                    'error': 'Not Found',
                    'path': path,
                    'timestamp': time.time()
                })
            }
            
    except Exception as e:
        print(f"Error processing request: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'Internal Server Error',
                'message': str(e),
                'timestamp': time.time()
            })
        }


def root_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Root endpoint handler"""
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': 'AWS Application Signals Lambda Demo',
            'timestamp': time.time(),
            'function_name': context.function_name,
            'request_id': context.aws_request_id
        })
    }


def health_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Health check endpoint"""
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'status': 'healthy',
            'timestamp': time.time(),
            'function_name': context.function_name,
            'memory_limit': context.memory_limit_in_mb,
            'remaining_time': context.get_remaining_time_in_millis()
        })
    }


def hello_handler(name: str, event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Personalized greeting with simulated processing"""
    # Simulate some processing time
    time.sleep(random.uniform(0.01, 0.1))
    
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': f'Hello, {name}!',
            'timestamp': time.time(),
            'function_name': context.function_name
        })
    }


def slow_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Intentionally slow endpoint for latency testing"""
    time.sleep(random.uniform(0.5, 1.5))
    
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': 'Slow operation completed',
            'timestamp': time.time(),
            'duration': '1-1.5 seconds'
        })
    }


def error_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Endpoint that randomly raises errors for testing error rates"""
    if random.random() > 0.5:
        raise Exception("Simulated error for testing Application Signals error tracking")
    
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': 'No error occurred',
            'timestamp': time.time()
        })
    }


def external_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Makes external HTTP call for distributed tracing"""
    try:
        # This will be automatically traced by OpenTelemetry
        response = requests.get("https://httpbin.org/delay/1", timeout=5)
        
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'message': 'External call completed',
                'external_status_code': response.status_code,
                'timestamp': time.time()
            })
        }
    except Exception as e:
        raise Exception(f"External call failed: {str(e)}")


def s3_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Makes AWS S3 call for testing AWS SDK instrumentation"""
    try:
        # This will be automatically traced by OpenTelemetry
        s3_client = boto3.client('s3')
        
        # List buckets (this is a safe operation that doesn't require specific permissions)
        response = s3_client.list_buckets()
        bucket_names = [bucket['Name'] for bucket in response['Buckets']]
        
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'message': 'S3 call completed successfully',
                'bucket_count': len(bucket_names),
                'buckets': bucket_names[:5],  # Show first 5 buckets
                'timestamp': time.time()
            })
        }
    except Exception as e:
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'S3 call failed',
                'message': str(e),
                'timestamp': time.time()
            })
        }

步骤 3： Create Lambda Deployment Package

3.1 设置变量

# Replace with your values
export AWS_REGION="us-east-1"

获取您的账户 ID：

export AWS_ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
echo "Account ID: $AWS_ACCOUNT_ID"

3.2 创建部署包

# Create deployment package
pip install -r requirements.txt -t ./package
cp lambda_function.py ./package/
cd package
zip -r ../lambda-deployment.zip .
cd ..

步骤 4： 为 Lambda 创建 IAM 角色

4.1 创建信任策略

cat > /tmp/lambda-trust-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "lambda.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
EOF

4.2 创建执行角色

aws iam create-role \
  --role-name ${FUNCTION_NAME}-execution-role \
  --assume-role-policy-document file:///tmp/lambda-trust-policy.json

# Attach basic Lambda execution policy
aws iam attach-role-policy \
  --role-name ${FUNCTION_NAME}-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

# Attach Application Signals policy (CRITICAL!)
aws iam attach-role-policy \
  --role-name ${FUNCTION_NAME}-execution-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLambdaApplicationSignalsExecutionRolePolicy

# Attach S3 read policy for testing
aws iam attach-role-policy \
  --role-name ${FUNCTION_NAME}-execution-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

Step 5: 部署 Lambda 函数

5.1 创建 Lambda 函数

# Get the role ARN
ROLE_ARN=$(aws iam get-role \
  --role-name ${FUNCTION_NAME}-execution-role \
  --query 'Role.Arn' \
  --output text)

echo "Role ARN: $ROLE_ARN"

# Create the Lambda function
aws lambda create-function \
  --function-name ${FUNCTION_NAME} \
  --runtime python3.11 \
  --role ${ROLE_ARN} \
  --handler lambda_function.lambda_handler \
  --zip-file fileb://lambda-deployment.zip \
  --timeout 30 \
  --memory-size 512 \
  --region ${AWS_REGION}

5.2 Add AWS Lambda Layer for OpenTelemetry

获取您所在区域的正确层 ARN：

对于 us-east-1:

LAYER_ARN="arn:aws:lambda:us-east-1:615299751070:layer:AWSOpenTelemetryDistroPython:18"

对于 other regions, 检查 the AWS documentation.

# Add the layer
aws lambda update-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --layers ${LAYER_ARN} \
  --region ${AWS_REGION}

5.3 配置环境变量

aws lambda update-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --environment Variables='{
    "AWS_LAMBDA_EXEC_WRAPPER": "/opt/otel-instrument",
    "OTEL_SERVICE_NAME": "'${FUNCTION_NAME}'",
    "OTEL_RESOURCE_ATTRIBUTES": "service.name='${FUNCTION_NAME}',deployment.environment=lambda:'${AWS_REGION}'"
  }' \
  --region ${AWS_REGION}

Step 7: 验证 Application Signals

7.1 检查 CloudWatch Logs

# Check Lambda function logs
aws logs tail /aws/lambda/${FUNCTION_NAME} --region ${AWS_REGION} --since 5m --format short

查找:

  • 应用日志显示执行成功
  • No OpenTelemetry-related errors

7.2 检查 Application Signals 服务发现

# Wait 5-10 minutes after traffic generation
aws application-signals list-services \
  --region ${AWS_REGION} \
  --start-time $(date -u -v-2H +%s) \
  --end-time $(date -u +%s) \
  --query 'ServiceSummaries[*].{Name:KeyAttributes.Name,Environment:KeyAttributes.Environment}'

预期结果： 您的服务应显示为：

  • Name: ${FUNCTION_NAME}
  • 环境： lambda:${AWS_REGION}

7.3 检查服务操作

aws application-signals list-service-operations \
  --region ${AWS_REGION} \
  --start-time $(date -u -v-2H +%s) \
  --end-time $(date -u +%s) \
  --key-attributes Name=${FUNCTION_NAME},Type=Service,Environment=lambda:${AWS_REGION} \
  --query 'ServiceOperations[*].Name'

预期操作：

  • lambda_handler
  • 自动插桩生成的各种内部操作

7.4 查看 Metrics

# Check Application Signals metrics
aws cloudwatch list-metrics \
  --namespace ApplicationSignals \
  --region ${AWS_REGION} \
  --dimensions Name=Service,Value=${FUNCTION_NAME} \
  --query 'Metrics[*].MetricName' \
  --output text | tr '\t' '\n' | sort -u

预期 metrics：

  • LambdaDuration
  • LambdaErrors
  • LambdaInvocations
  • Latency
  • Error
  • Fault

故障排除

问题：服务未出现在 Application Signals 中

检查：

# 1. Verify Application Signals is enabled
aws iam get-role \
  --role-name AWSServiceRoleForCloudWatchApplicationSignals

# 2. Check if the layer is attached
aws lambda get-function \
  --function-name ${FUNCTION_NAME} \
  --query 'Configuration.Layers'

# 3. Check environment variables
aws lambda get-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --query 'Environment.Variables'

问题：Lambda 函数错误

检查：

# Check function logs
aws logs tail /aws/lambda/${FUNCTION_NAME} --region ${AWS_REGION} --since 10m

常见问题：

  1. 缺少 IAM 策略 - Ensure CloudWatchLambdaApplicationSignalsExecutionRolePolicy 已附加
  2. 错误的层 ARN - 验证您所在区域的层 ARN
  3. Missing 环境变量 - Ensure AWS_LAMBDA_EXEC_WRAPPER=/opt/otel-instrument

问题：X-Ray 中无 Traces

常见原因：

  1. Missing X-Ray permissions - The Application Signals policy includes X-Ray permissions
  2. Layer not working - Verify the layer 已附加 and 环境变量 is set
  3. No traffic - 生成一些请求以创建 traces

FAQ

Q: Can I use this with other Python frameworks?
A: Yes! The auto-instrumentation works with any Python code, including Flask, Django, etc.

Q: Do I need to modify my application code?
A: No! The Lambda layer provides automatic instrumentation for HTTP requests and AWS SDK calls.

Q: Can I add custom traces/spans?
A: Yes! Import OpenTelemetry API (提供的 by the layer):

from opentelemetry import trace
tracer = trace.get_tracer(__name__)

Q: What about cold starts?
A: The layer adds minimal overhead to cold starts. Application Signals metrics help you monitor cold start performance.

Q: Can I use this with Lambda containers?
A: Yes! The same layer can be added to container-based Lambda functions.
