调试指南

快速分诊工作流（推荐）

  1. 从 SLO 开始（如果已定义）：识别不健康的服务/操作/依赖项。
  2. 使用服务视图：按高延迟或故障率排序，找出主要问题来源。
  3. 打开服务详情：检查操作和依赖项边缘；选择一个延迟/故障数据点。
  4. 跳转到关联的追踪/span：检查追踪时间线，找到缓慢/失败的下游调用。
  5. 通过日志/运行时指标确认：在相关时关联错误日志和 JVM/CPU/GC。

参见：使用 Application Signals 监控运行状况。

使用 Transaction Search 查找"偶发峰值"

对于间歇性峰值，Transaction Search 帮助您按属性分析 span 并深入查看受影响的用户/请求。常见模式：

  • 分组分析按业务属性（例如，客户 ID）查找受影响的用户。
  • 筛选按服务/操作和持续时间（p99）来隔离异常值。
  • 打开追踪从异常 span 开始，跟踪依赖项调用路径。

参见：排查应用程序问题（Transaction Search）。

常见的"为什么看不到数据？"检查项

  • 账户启用：Application Signals 必须在每个账户/区域启用一次（创建服务关联角色）。
  • 时间范围：服务/操作仅在所选时间窗口内有活动时才会显示。
  • 服务命名：确保 service.name / OTEL_RESOURCE_ATTRIBUTES 设置正确（非 EKS 通常需要显式的服务/环境名称）。
  • 管道健康：CloudWatch Agent/收集器正在运行并接收 OTLP；验证本地端点和 IAM 权限。

参见：排查 Application Signals 安装问题。
