生成式 AI Observability

Application Signals + Amazon Bedrock

CloudWatch Application Signals 支持对与 Amazon Bedrock 模型交互的生成式 AI 应用程序进行故障排查。它为 Bedrock API 调用提供开箱即用的遥测数据，以便您调查：

  • 模型配置问题
  • 使用量和成本驱动因素（令牌使用量）
  • 模型延迟
  • 响应的停止/完成原因

细粒度指标和链路追踪

对于 Bedrock 调用，Application Signals 可以生成并关联具有资源级别详情的性能指标和链路追踪 span（例如：模型 ID、护栏 ID、知识库 ID 和 Bedrock 代理 ID）。

参见：示例：对与 Amazon Bedrock 交互的生成式 AI 应用进行故障排查。

GenAI 语义属性 + Transaction Search

Application Signals 可以发出 OpenTelemetry GenAI 语义约定属性（例如 gen_ai.request.model 和令牌使用量字段）。您可以使用 CloudWatch Transaction Search 来分析这些属性，比较不同模型之间的提示词、令牌使用量和延迟。

  • Transaction Search
  • 搜索和分析 span
