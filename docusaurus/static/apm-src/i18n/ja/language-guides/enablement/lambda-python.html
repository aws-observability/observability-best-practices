概要

このガイドでは、 ステップバイステップの手順 を提供します。AWS Application Signals を有効にした Python Lambda 関数をデプロイするための手順です。AWS Lambda コンソールと組み込みレイヤーサポートを使用した最もシンプルなアプローチです。

達成できること：
  • Application Signals を使用した Python Lambda 関数のデプロイ
  • Lambda Layer による AWS Application Signals 自動計装の有効化
  • CloudWatch Application Signals コンソールでのサービスの確認
  • サービスのメトリクス、トレース、およびオペレーションの監視
完了までの時間： 15〜20 分

前提条件

必要なツール

  • AWS CLI v2（最新バージョン）
  • Python 3.11+
  • 適切な権限を持つ AWS アカウント
  • テキストエディタ

必要な AWS 権限

  • Lambda: 関数の作成と管理
  • IAM: ロールとポリシーの作成
  • CloudWatch: ログおよび Application Signals へのアクセス
  • S3: 基本的な読み取りアクセス（テスト用）

前提知識

  • AWS CLI の基本的な使用方法
  • Lambda の概念に関する基本的な理解

アーキテクチャ

Lambda 関数は以下を使用して自動的に計装されます：

  1. OpenTelemetry 用 AWS Lambda Layer - 自動計装を提供
  2. Application Signals との統合 - レイヤーに組み込み済み
  3. 自動トレーシング - HTTP リクエスト、AWS SDK 呼び出し、関数の実行
Flow:
Lambda Function (auto-instrumented)
  → Sends traces/metrics to Application Signals
  → View in CloudWatch Application Signals console

ステップ 1: アカウントで Application Signals を有効にする

重要： Application Signals は、AWS アカウントごとにリージョンごとに1回有効化する必要があります。

オプション1：AWS コンソールから有効化する（推奨）

  1. AWS コンソール → CloudWatch → Application Signals に移動する
  2. リージョンを選択する（例： us-east-1)
  3. 「Get Started」または「Enable Application Signals」をクリックする
  4. 有効化されるまで2〜3分待つ

オプション2：すでに有効化されているか確認する

# Check if service-linked role exists
aws iam get-role \
  --role-name AWSServiceRoleForCloudWatchApplicationSignals \
  --region us-east-1

# If role exists, Application Signals is enabled

出力結果は以下のようになります： 有効化されている場合はロール ARN、されていない場合はエラーが表示されます。

ステップ2：Python Lambda 関数を準備する

2.1 アプリケーションディレクトリを作成する

mkdir -p ~/my-lambda-appsignals-app
cd ~/my-lambda-appsignals-app

2.2 作成する lambda_function.py

"""
AWS Application Signals Lambda Function
Adapted from ECS FastAPI implementation for Lambda
"""
import json
import time
import random
import boto3
import requests
from typing import Dict, Any


def lambda_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """
    Main Lambda handler with Application Signals auto-instrumentation
    
    The AWS Lambda Layer for OpenTelemetry automatically instruments:
    - HTTP requests (requests library)
    - AWS SDK calls (boto3)
    - Lambda function execution
    - Custom spans and metrics
    """
    
    # Parse the event to determine which endpoint to call
    path = event.get('path', '/')
    http_method = event.get('httpMethod', 'GET')
    
    try:
        if path == '/' or path == '':
            return root_handler(event, context)
        elif path == '/health':
            return health_handler(event, context)
        elif path.startswith('/api/hello/'):
            name = path.split('/')[-1]
            return hello_handler(name, event, context)
        elif path == '/api/slow':
            return slow_handler(event, context)
        elif path == '/api/error':
            return error_handler(event, context)
        elif path == '/api/external':
            return external_handler(event, context)
        elif path == '/api/s3':
            return s3_handler(event, context)
        else:
            return {
                'statusCode': 404,
                'headers': {
                    'Content-Type': 'application/json',
                    'Access-Control-Allow-Origin': '*'
                },
                'body': json.dumps({
                    'error': 'Not Found',
                    'path': path,
                    'timestamp': time.time()
                })
            }
            
    except Exception as e:
        print(f"Error processing request: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'Internal Server Error',
                'message': str(e),
                'timestamp': time.time()
            })
        }


def root_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Root endpoint handler"""
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': 'AWS Application Signals Lambda Demo',
            'timestamp': time.time(),
            'function_name': context.function_name,
            'request_id': context.aws_request_id
        })
    }


def health_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Health check endpoint"""
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'status': 'healthy',
            'timestamp': time.time(),
            'function_name': context.function_name,
            'memory_limit': context.memory_limit_in_mb,
            'remaining_time': context.get_remaining_time_in_millis()
        })
    }


def hello_handler(name: str, event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Personalized greeting with simulated processing"""
    # Simulate some processing time
    time.sleep(random.uniform(0.01, 0.1))
    
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': f'Hello, {name}!',
            'timestamp': time.time(),
            'function_name': context.function_name
        })
    }


def slow_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Intentionally slow endpoint for latency testing"""
    time.sleep(random.uniform(0.5, 1.5))
    
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': 'Slow operation completed',
            'timestamp': time.time(),
            'duration': '1-1.5 seconds'
        })
    }


def error_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Endpoint that randomly raises errors for testing error rates"""
    if random.random() > 0.5:
        raise Exception("Simulated error for testing Application Signals error tracking")
    
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps({
            'message': 'No error occurred',
            'timestamp': time.time()
        })
    }


def external_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Makes external HTTP call for distributed tracing"""
    try:
        # This will be automatically traced by OpenTelemetry
        response = requests.get("https://httpbin.org/delay/1", timeout=5)
        
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'message': 'External call completed',
                'external_status_code': response.status_code,
                'timestamp': time.time()
            })
        }
    except Exception as e:
        raise Exception(f"External call failed: {str(e)}")


def s3_handler(event: Dict[str, Any], context: Any) -> Dict[str, Any]:
    """Makes AWS S3 call for testing AWS SDK instrumentation"""
    try:
        # This will be automatically traced by OpenTelemetry
        s3_client = boto3.client('s3')
        
        # List buckets (this is a safe operation that doesn't require specific permissions)
        response = s3_client.list_buckets()
        bucket_names = [bucket['Name'] for bucket in response['Buckets']]
        
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'message': 'S3 call completed successfully',
                'bucket_count': len(bucket_names),
                'buckets': bucket_names[:5],  # Show first 5 buckets
                'timestamp': time.time()
            })
        }
    except Exception as e:
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'S3 call failed',
                'message': str(e),
                'timestamp': time.time()
            })
        }

ステップ4：Lambda 用の IAM ロールを作成する

4.1 信頼ポリシーを作成する

cat > /tmp/lambda-trust-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "lambda.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
EOF

4.2 実行ロールを作成する

aws iam create-role \
  --role-name ${FUNCTION_NAME}-execution-role \
  --assume-role-policy-document file:///tmp/lambda-trust-policy.json

# Attach basic Lambda execution policy
aws iam attach-role-policy \
  --role-name ${FUNCTION_NAME}-execution-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

# Attach Application Signals policy (CRITICAL!)
aws iam attach-role-policy \
  --role-name ${FUNCTION_NAME}-execution-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLambdaApplicationSignalsExecutionRolePolicy

# Attach S3 read policy for testing
aws iam attach-role-policy \
  --role-name ${FUNCTION_NAME}-execution-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

ステップ5：Lambda 関数をデプロイする

5.1 Lambda 関数を作成する

# Get the role ARN
ROLE_ARN=$(aws iam get-role \
  --role-name ${FUNCTION_NAME}-execution-role \
  --query 'Role.Arn' \
  --output text)

echo "Role ARN: $ROLE_ARN"

# Create the Lambda function
aws lambda create-function \
  --function-name ${FUNCTION_NAME} \
  --runtime python3.11 \
  --role ${ROLE_ARN} \
  --handler lambda_function.lambda_handler \
  --zip-file fileb://lambda-deployment.zip \
  --timeout 30 \
  --memory-size 512 \
  --region ${AWS_REGION}

5.2 OpenTelemetry 用の AWS Lambda Layer を追加する

リージョンに対応した正しいレイヤー ARN を取得する：

対象： us-east-1:

LAYER_ARN="arn:aws:lambda:us-east-1:615299751070:layer:AWSOpenTelemetryDistroPython:18"

他のリージョンについては、 AWS ドキュメント.

# Add the layer
aws lambda update-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --layers ${LAYER_ARN} \
  --region ${AWS_REGION}

5.3 環境変数を設定する

aws lambda update-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --environment Variables='{
    "AWS_LAMBDA_EXEC_WRAPPER": "/opt/otel-instrument",
    "OTEL_SERVICE_NAME": "'${FUNCTION_NAME}'",
    "OTEL_RESOURCE_ATTRIBUTES": "service.name='${FUNCTION_NAME}',deployment.environment=lambda:'${AWS_REGION}'"
  }' \
  --region ${AWS_REGION}

AWS コンソールを使用する場合：

8.1 コンソールから関数を作成する

  1. AWS Lambda コンソールに移動する
  2. 「関数の作成」をクリックする
  3. 「一から作成」を選択する
  4. 関数名： ${FUNCTION_NAME}
  5. ランタイム：Python 3.11
  6. 関数を作成する

8.2 コードをアップロードする

  1. 関数コードセクションで、デフォルトのコードを以下のコードに置き換えてください lambda_function.py
  2. 「Deploy」をクリックします

8.3 Application Signals を有効にする

  1. Configuration → Monitoring and operations tools に移動します
  2. 「Edit」をクリックします
  3. 「CloudWatch Application Signals and X-Ray」セクションで：
    • 「Automatically collect application traces and standard application metrics with Application Signals」にチェックを入れます
    • 「Automatically collect Lambda service traces for end to end visibility with X-Ray」にチェックを入れます
  4. 「Save」をクリックします
以上です！ コンソールが自動的に：
  • OpenTelemetry 用の AWS Lambda Layer を追加します
  • を設定します AWS_LAMBDA_EXEC_WRAPPER 環境変数
  • 必要な IAM ポリシーをアタッチします

トラブルシューティング

問題：サービスが Application Signals に表示されない

確認事項：

# 1. Verify Application Signals is enabled
aws iam get-role \
  --role-name AWSServiceRoleForCloudWatchApplicationSignals

# 2. Check if the layer is attached
aws lambda get-function \
  --function-name ${FUNCTION_NAME} \
  --query 'Configuration.Layers'

# 3. Check environment variables
aws lambda get-function-configuration \
  --function-name ${FUNCTION_NAME} \
  --query 'Environment.Variables'

問題：Lambda 関数のエラー

確認事項：

# Check function logs
aws logs tail /aws/lambda/${FUNCTION_NAME} --region ${AWS_REGION} --since 10m

よくある問題：

  1. IAM ポリシーが不足している - 以下を確認してください CloudWatchLambdaApplicationSignalsExecutionRolePolicy がアタッチされていること
  2. レイヤーの ARN が間違っている - お使いのリージョンのレイヤー ARN を確認してください
  3. 環境変数が不足している - 以下を確認してください AWS_LAMBDA_EXEC_WRAPPER=/opt/otel-instrument

問題：X-Ray にトレースが表示されない

よくある原因：

  1. X-Ray の権限が不足している - Application Signals ポリシーには X-Ray の権限が含まれています
  2. レイヤーが機能していない - レイヤーがアタッチされており、環境変数が設定されていることを確認してください
  3. トラフィックがない - トレースを生成するためにリクエストを送信してください

主な学習内容

なぜこれが機能するのか

  1. AWS Lambda Layer for OpenTelemetry: AWS公式のインストルメンテーションライブラリを提供
  2. 自動インストルメンテーション: 基本的なトレースにコード変更は不要
  3. 組み込みのApplication Signals: LayerにはApplication Signals統合が含まれる
  4. シンプルな設定: Layerを追加して環境変数を設定するだけ

FAQ

Q: 他のPythonフレームワークでも使用できますか？
A: はい！自動インストルメンテーションはFlask、Djangoなど、あらゆるPythonコードで動作します。

Q: アプリケーションコードを変更する必要がありますか？
A: 不要です！Lambda レイヤーは、HTTP リクエストと AWS SDK 呼び出しに対して自動インストルメンテーションを提供します。

Q: カスタムトレース/スパンを追加できますか？
A: はい！レイヤーが提供する OpenTelemetry API をインポートしてください：

from opentelemetry import trace
tracer = trace.get_tracer(__name__)

Q: コールドスタートはどうなりますか？
A: レイヤーがコールドスタートに与えるオーバーヘッドは最小限です。Application Signals のメトリクスを使用して、コールドスタートのパフォーマンスを監視できます。

Q: Lambda コンテナで使用できますか？
A: はい！同じレイヤーをコンテナベースの Lambda 関数に追加できます。
