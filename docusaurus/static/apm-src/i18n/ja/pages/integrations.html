インテグレーション

AWS サービスインテグレーション

CloudWatch Application Signals は AWS サービスとシームレスに統合し、クラウドインフラストラクチャ全体にわたる包括的なオブザーバビリティを提供します。

API Gateway

API Gateway からバックエンドサービスへのリクエストを追跡し、API パフォーマンスを監視します。

インテグレーション機能:

  • API コールの自動トレーシング
  • API エンドポイントのレイテンシー監視
  • エラーレートの追跡
  • リクエスト/レスポンスの相関

セットアップ:

API Gateway ステージ設定で X-Ray トレーシングを有効にします。

Application Load Balancer (ALB)

ロードバランサーを通じたトラフィックフローを追跡し、アプリケーションパフォーマンスと相関させます。

インテグレーションのメリット:

  • リクエストルーティングの可視化
  • ターゲットグループのパフォーマンスメトリクス
  • ヘルスチェックの監視
  • 負荷分散の分析

DynamoDB

データベース操作を監視し、データアクセスパターンにおけるパフォーマンスのボトルネックを特定します。

トレーシング機能:

  • クエリ操作のタイミング
  • スロットルおよびエラーの追跡
  • テーブルおよびインデックスのパフォーマンス
  • クロスリージョンレプリケーションの監視

Amazon RDS

データベースパフォーマンスとコネクションプーリングに関するインサイトを取得します。

データベース監視:

  • クエリ実行時間
  • コネクションプールの使用率
  • データベースインスタンスのメトリクス
  • スロークエリの特定

Amazon S3

オブジェクトストレージ操作とデータ転送パフォーマンスを追跡します。

ストレージ操作:

  • オブジェクトの Put/Get トレーシング
  • 転送レイテンシーの監視
  • エラーレート分析
  • クロスリージョン転送追跡

SNS/SQS

イベント駆動型アーキテクチャにおけるメッセージ処理とキューパフォーマンスを監視します。

メッセージングインサイト：

  • メッセージのパブリッシュ/サブスクライブトレーシング
  • キュー深度モニタリング
  • メッセージ処理レイテンシ
  • デッドレターキュー分析

AWS Step Functions

ステートマシン処理におけるワークフロー実行を可視化し、ボトルネックを特定します。

ワークフローモニタリング：

  • 状態遷移トレーシング
  • 実行時間分析
  • エラー状態の特定
  • 並列実行モニタリング

Amazon EventBridge

イベント駆動型システムを通じたイベントルーティングと処理を追跡します。

イベント処理：

  • イベントルーティングの可視化
  • ルール評価タイミング
  • ターゲット呼び出しトレーシング
  • イベント変換モニタリング

AWS Lambda

サーバーレス関数のパフォーマンスとコールドスタートの影響を監視します。

サーバーレスインサイト：

  • 関数呼び出しトレーシング
  • コールドスタート時間
  • メモリおよびCPU使用率
  • エラーとタイムアウトの追跡

Amazon ECS/EKS

マイクロサービス監視のためのコンテナオーケストレーションプラットフォーム統合。

コンテナモニタリング：

  • サービスメッシュトレーシング
  • Pod間通信
  • リソース使用率
  • デプロイメント追跡

サードパーティ統合

サードパーティのツールおよびプラットフォームでApplication Signalsを拡張します。

OpenTelemetry

OpenTelemetryエコシステムを活用して、ベンダー中立のオブザーバビリティを実現します。

統合オプション:

  • カスタムインストルメンテーションライブラリ
  • サードパーティエージェントのサポート
  • マルチバックエンドデータエクスポート
  • コミュニティコレクターへの貢献

APMベンダー

他のAPMソリューションからの移行、または統合を行います。

サポートされている統合:

  • サードパーティAPMツールへのデータエクスポート
  • ハイブリッドクラウドのオブザーバビリティ
  • マルチベンダーの相関分析
  • データ移行ユーティリティ

モニタリングとアラート

Application Signalsを既存のモニタリングスタックと連携させます。

統合パートナー:

  • PagerDuty: インシデント対応とアラート
  • Slack: チーム通知
  • Datadog: 統合モニタリングプラットフォーム
  • New Relic: アプリケーションパフォーマンスモニタリング
  • Splunk: ログ分析と相関

CI/CD統合

オブザーバビリティを開発パイプラインに組み込みます。

DevOpsツール:

  • GitHub Actions: 自動テストとデプロイ
  • Jenkins: ビルドパイプラインの統合
  • GitLab CI: DevSecOpsワークフロー
  • Terraform: コードとしてのインフラストラクチャ
  • AWS CDK: クラウドインフラストラクチャのプロビジョニング

カスタム統合

APIとSDKを使用して独自のインテグレーションを構築します。

CloudWatch APIs

CloudWatch APIを使用して、Application SignalsがCloudWatchに公開するテレメトリを基に、カスタムダッシュボード、アラーム、およびメトリクス駆動のワークフローを構築します。

利用可能なAPI:

  • PutMetricData - カスタムメトリクスの送信
  • GetMetricData - メトリクスデータの取得
  • PutDashboard / GetDashboard - ダッシュボードの管理
  • PutMetricAlarm / DescribeAlarms - アラームの管理

AWS X-Ray APIs

Application SignalsはトレースストアとしてAWS X-Rayを使用します。直接的なトレースの取り込み/クエリ統合が必要な場合は、X-Ray APIを使用してください。

  • PutTraceSegments - トレースセグメントの取り込み
  • GetTraceSummaries / BatchGetTraces - トレースのクエリ

Webhooksとコールバック

リアルタイム通知を受信し、カスタムワークフローと統合します。

インテグレーションパターン:

  • カスタム通知システム向けのアラートwebhook
  • Lambdaを使用したイベント駆動処理
  • カスタムダッシュボードの更新
  • 自動修復ワークフロー

SDKとライブラリ

インストルメンテーションは通常、OpenTelemetryおよび/またはAWS Distro for OpenTelemetry (ADOT)を通じて行われます。AWS API呼び出しには、サポートされているOpenTelemetryインストルメンテーションと共にAWS SDKを使用してください。

サポートされている言語:

  • JavaScript/Node.js: AWS SDK for JavaScript
  • Python: Boto3 AWS SDK
  • Java: AWS SDK for Java
  • .NET: AWS SDK for .NET
  • Go: AWS SDK for Go
