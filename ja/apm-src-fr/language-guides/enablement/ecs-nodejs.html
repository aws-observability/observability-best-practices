Overview

This guide provides step-by-step instructions to deploy a Node.js Express application on Amazon ECS with AWS Application Signals enabled. No CDK required - pure AWS CLI and Docker.

What you'll accomplish:
  • Deploy a Node.js Express app on ECS Fargate
  • Enable AWS Application Signals auto-instrumentation
  • View your service in CloudWatch Application Signals console
  • Monitor service metrics, traces, and operations
Time to complete: 30-45 minutes

Prerequisites

Required Tools

  • AWS CLI v2 (latest version)
  • Docker Desktop
  • Node.js 18+ (for local development)
  • npm (for local development)
  • AWS Account with appropriate permissions
  • Text editor

AWS Permissions Required

  • ECS: Full access (create clusters, services, task definitions)
  • ECR: Push images
  • IAM: Create roles and policies
  • CloudWatch: Logs and Application Signals access
  • EC2: VPC, subnet, and security group access
  • SSM: Parameter Store write access

Knowledge Assumed

  • Basic AWS CLI usage
  • Basic Docker commands
  • Basic understanding of ECS concepts
  • Basic Node.js/Express knowledge

Architecture

Your ECS task will contain 3 containers:

  1. Init Container (init): Copies ADOT Node.js auto-instrumentation files
  2. CloudWatch Agent (ecs-cwagent): Collects traces and metrics
  3. Application Container (app): Your Node.js Express application

Flow:

Init Container (runs once, copies files) 
  → Application starts (auto-instrumented via NODE_OPTIONS)
  → Sends traces/metrics to CloudWatch Agent (localhost:4316)
  → CloudWatch Agent sends to Application Signals

/**

 * This application demonstrates auto-instrumentation with AWS Application Signals
 * on ECS. The ADOT Collector handles all instrumentation automatically when configured.

const cors = require('cors');

const axios = require('axios');

Step 3: Deploy to ECS

3.1 Create ECR Repository

# Get your AWS account ID
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
echo "AWS Account ID: $ACCOUNT_ID"

# Create ECR repository
aws ecr create-repository --repository-name my-appsignals-nodejs-app --region us-east-1

# Get login token
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin $ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com

3.2 Build and Push Docker Image

# Build the image
docker build -t my-appsignals-nodejs-app .

# Tag the image
docker tag my-appsignals-nodejs-app:latest $ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com/my-appsignals-nodejs-app:latest

# Push the image
docker push $ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com/my-appsignals-nodejs-app:latest

3.7 Create VPC and Security Groups

# Get default VPC
VPC_ID=$(aws ec2 describe-vpcs --filters "Name=is-default,Values=true" --query 'Vpcs[0].VpcId' --output text)
echo "VPC ID: $VPC_ID"

# Get subnets
SUBNET_IDS=$(aws ec2 describe-subnets --filters "Name=vpc-id,Values=$VPC_ID" --query 'Subnets[].SubnetId' --output text)
echo "Subnet IDs: $SUBNET_IDS"

# Create security group
SECURITY_GROUP_ID=$(aws ec2 create-security-group \
    --group-name my-appsignals-nodejs-sg \
    --description "Security group for Node.js ECS Application Signals demo" \
    --vpc-id $VPC_ID \
    --query 'GroupId' \
    --output text)
echo "Security Group ID: $SECURITY_GROUP_ID"

# Allow HTTP traffic
aws ec2 authorize-security-group-ingress \
    --group-id $SECURITY_GROUP_ID \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0

# Allow all outbound traffic
aws ec2 authorize-security-group-egress \
    --group-id $SECURITY_GROUP_ID \
    --protocol -1 \
    --cidr 0.0.0.0/0

3.8 Create ECS Service

# Create ECS service
aws ecs create-service \
    --cluster my-appsignals-nodejs-cluster \
    --service-name my-appsignals-nodejs-service \
    --task-definition my-appsignals-nodejs-app-task \
    --desired-count 1 \
    --launch-type FARGATE \
    --network-configuration "awsvpcConfiguration={subnets=[$SUBNET_IDS],securityGroups=[$SECURITY_GROUP_ID],assignPublicIp=ENABLED}" \
    --region us-east-1

Step 4: Generate Traffic and Monitor

4.1 Get Service Endpoint

# Get the public IP of the running task
TASK_ARN=$(aws ecs list-tasks --cluster my-appsignals-nodejs-cluster --service-name my-appsignals-nodejs-service --query 'taskArns[0]' --output text)
echo "Task ARN: $TASK_ARN"

# Get task details
TASK_DETAILS=$(aws ecs describe-tasks --cluster my-appsignals-nodejs-cluster --tasks $TASK_ARN --query 'tasks[0].attachments[0].details[?name==`networkInterfaceId`].value' --output text)
echo "Network Interface ID: $TASK_DETAILS"

# Get public IP
PUBLIC_IP=$(aws ec2 describe-network-interfaces --network-interface-ids $TASK_DETAILS --query 'NetworkInterfaces[0].Association.PublicIp' --output text)
echo "Public IP: $PUBLIC_IP"

4.2 Generate Traffic

Create generate-traffic.sh:

#!/bin/bash

BASE_URL="http://$PUBLIC_IP:80"

echo "Generating traffic for Node.js ECS Application Signals demo..."

# Generate traffic for 5 minutes
for i in {1..300}; do
    echo "Request $i"

    # Root endpoint
    curl -s "$BASE_URL" > /dev/null

    # Health check
    curl -s "$BASE_URL/health" > /dev/null

    # Hello endpoint
    curl -s "$BASE_URL/api/hello/User$i" > /dev/null

    # Slow endpoint
    curl -s "$BASE_URL/api/slow" > /dev/null

    # Error endpoint
    curl -s "$BASE_URL/api/error" > /dev/null

    # External call
    curl -s "$BASE_URL/api/external" > /dev/null

    # S3 call
    curl -s "$BASE_URL/api/s3" > /dev/null

    # Database simulation
    curl -s "$BASE_URL/api/database" > /dev/null

    # Complex operation
    curl -s "$BASE_URL/api/complex" > /dev/null

    sleep 1
done

echo "Traffic generation complete!"

  1. Task fails to start

    • Check ECR repository exists and image is pushed
    • Verify IAM roles have correct permissions
    • Check CloudWatch logs for error details

  2. No data in Application Signals

    • Verify CloudWatch agent is running
    • Check environment variables are set correctly
    • Ensure ADOT auto-instrumentation is working

  3. Service not accessible

    • Check security group allows inbound traffic on port 80
    • Verify task is running and healthy
    • Check VPC and subnet configuration
