Bonnes pratiques

Bonnes pratiques d'instrumentation

Une instrumentation efficace est la clé pour obtenir des informations précieuses à partir de vos données d'observabilité. Suivez ces bonnes pratiques pour vous assurer que votre instrumentation fournit une valeur maximale tout en minimisant la surcharge.

Quoi instrumenter

Concentrez-vous sur l'instrumentation des parties les plus critiques de votre application :

Priorité élevée :

  • Points de terminaison d'API publics et opérations orientées utilisateur
  • Requêtes de base de données et appels de services externes
  • Flux de travail et transactions critiques pour l'entreprise
  • Chemins d'erreur et gestion des exceptions

Priorité moyenne :

  • Communication interne entre services
  • Tâches en arrière-plan et tâches planifiées
  • Opérations de cache

Priorité basse :

  • Fonctions utilitaires simples
  • Service de contenu statique
  • Points de terminaison de vérification de santé

Conventions de nommage

Un nommage cohérent facilite la recherche et le filtrage de vos traces :

Noms des spans :

  • Utilisez des minuscules avec des tirets : http-get-user-profile
  • Incluez le type d'opération : db-query-users, redis-get-session
  • Soyez descriptif mais concis : évitez les noms génériques comme « process » ou « handle »

Noms des services :

  • Utilisez un nommage de service cohérent entre les environnements
  • Incluez l'environnement dans le nom du service pour les configurations multi-environnements
  • Suivez les conventions de nommage de votre organisation

Attributs et étiquettes

Utilisez des attributs pour ajouter du contexte à vos spans et traces :

Attributs standards :

  • http.method - Méthode HTTP (GET, POST, etc.)
  • http.url - URL de la requête (attention aux informations personnelles)
  • http.status_code - Code de statut de la réponse HTTP
  • db.system - Type de base de données (mysql, postgres, etc.)
  • db.name - Nom de la base de données
  • db.operation - Opération de base de données (SELECT, INSERT, etc.)

Attributs personnalisés :

  • user.id - Identifiant utilisateur (haché si information personnelle)
  • business.operation - Type d'opération métier
  • version - Version de l'application
  • environment - Environnement de déploiement

Considérations relatives aux informations personnelles et à la sécurité

Protégez les données sensibles dans votre télémétrie :

Ne jamais inclure :

  • Mots de passe, clés API ou jetons d'authentification
  • Informations personnellement identifiables (PII) comme les e-mails, numéros de téléphone
  • URL complètes avec des paramètres de requête contenant des données sensibles
  • Numéros de carte de crédit ou informations financières

Alternatives sûres :

  • Utiliser des identifiants hachés ou tokenisés
  • Inclure uniquement la partie chemin des URL
  • Utiliser des indicateurs d'état génériques au lieu de valeurs spécifiques
  • Implémenter le masquage des données au niveau de l'instrumentation

Stratégies d'échantillonnage

L'échantillonnage aide à contrôler les coûts tout en maintenant la couverture d'observabilité.

Choisir une stratégie d'échantillonnage

Échantillonnage basé sur le taux :

  • Échantillonner un pourcentage fixe de requêtes (par ex., 10 %)
  • Simple à implémenter et à comprendre
  • Peut manquer des événements importants mais rares

Échantillonnage en queue :

  • Prendre les décisions d'échantillonnage après la collecte des données de trace
  • Peut prioriser les traces avec des erreurs ou une latence élevée
  • Plus complexe mais plus intelligent

Échantillonnage basé sur des règles :

  • Échantillonner en fonction de conditions spécifiques (utilisateur, point de terminaison, etc.)
  • Garantir que les chemins critiques sont toujours tracés
  • Nécessite une définition soignée des règles

Taux d'échantillonnage recommandés

  • Développement : 100 % d'échantillonnage pour une visibilité complète
  • Pré-production : 50-100 % d'échantillonnage pour les tests
  • Production : 10-20 % pour une couverture rentable
  • Services à fort trafic : 1-5 % pour contrôler le volume
  • Chemins critiques : 100 % d'échantillonnage quelles que soient les autres règles

Optimisation des performances

Minimiser l'impact sur les performances de l'instrumentation.

Réduire la surcharge

  • Utiliser le traitement asynchrone des spans lorsque possible
  • Regrouper les exportations de spans pour réduire les appels réseau
  • Configurer des tailles de tampon appropriées
  • Surveiller les métriques de performance de l'instrumentation

Gestion des ressources

  • Définir des délais d'expiration raisonnables pour les exportations de spans
  • Implémenter des disjoncteurs pour les défaillances backend
  • Utiliser le pooling de connexions pour les collecteurs
  • Configurer des politiques de nouvelle tentative appropriées

Alertes et surveillance

Mettez en place des alertes efficaces pour rester au fait de la santé de vos applications.

Métriques clés à surveiller

  • Latence : Temps de réponse P95/P99
  • Taux d'erreur : Pourcentage de requêtes échouées
  • Débit : Requêtes par seconde
  • Saturation : Utilisation des ressources
  • Volume de traces : Nombre de traces collectées

Alertes basées sur les SLO

Définissez des objectifs de niveau de service et alertez lorsqu'ils sont menacés :

  • SLO de latence : 95 % des requêtes < 500ms
  • SLO d'erreur : 99,9 % de taux de réussite
  • SLO de disponibilité : 99,95 % de temps de fonctionnement

Gestion des coûts

Contrôlez les coûts d'observabilité tout en maintenant la valeur.

Stratégies d'optimisation des coûts

  • Implémenter des taux d'échantillonnage appropriés
  • Utiliser des politiques de rétention des données
  • Filtrer les spans et attributs inutiles
  • Archiver les anciennes données vers un stockage moins coûteux
  • Surveiller et optimiser le volume de données

Politiques de rétention des données

  • Stockage chaud : Données récentes (7-30 jours) pour le dépannage actif
  • Stockage tiède : Données historiques (30-90 jours) pour l'analyse des tendances
  • Stockage froid : Données archivées (90+ jours) pour la conformité et les requêtes rares
