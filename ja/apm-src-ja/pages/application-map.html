CloudWatch APM ドキュメント

Application Map

概要

CloudWatch Application Map は、アプリケーションのトポロジーと運用上の健全性を可視化します。Application Signals における従来の「Service Map」という用語に代わるものです。

クライアント、カナリア、サービス、および依存関係間の接続を表示し、チームが実行フローを把握して、障害やレイテンシーの発生箇所を迅速に特定するのに役立ちます。

主な機能

  • SLO/SLI の健全性: どのサービスが目標を達成または違反しているか、また影響がダウンストリームにどのように伝播するかを確認できます。
  • ドリルダウン: ノードやエッジを選択してメトリクスを表示し、サービスの詳細に移動できます。
  • グループ化とフィルタリング: デフォルトのグループ（関連サービス、環境）を使用するか、タグや OTEL リソース属性を使用してカスタムグループを定義できます。
  • 未インストルメント済みサービス: Application Signals は、まだインストルメントされていないサービスを表示して、ロールアウトの優先順位付けを支援できます。
  • クロスアカウントの可視性: クロスアカウントオブザーバビリティが設定されている場合、複数のアカウントにまたがるサービスを可視化できます。
