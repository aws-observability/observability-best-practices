CloudWatch APM 문서

기능 및 역량

Application Signals: 제공하는 기능

CloudWatch Application Signals는 CloudWatch에서 AWS의 애플리케이션 성능 모니터링(APM) 경험입니다. 활성화 후, 자동으로 애플리케이션 텔레메트리를 수집하고 서비스, 오퍼레이션, 종속성별로 구성하여 운영 상태를 빠르게 분류하고 장기적인 성능 목표를 추적할 수 있습니다.

  • 표준 애플리케이션 메트릭: 호출량, 가용성, 지연 시간, 장애 및 오류.
  • 서비스 및 종속성 토폴로지: 발견된 서비스와 관계에서 생성된 Application Map.
  • 심층 문제 해결: 서비스 상태에서 상관 트레이스 및 관련 텔레메트리로 드릴다운.

Application Map (토폴로지 + 상태)

Application Map은 애플리케이션의 토폴로지를 시각화하고 운영 상태를 강조 표시합니다. 클라이언트, 카나리, 서비스 및 종속성 간의 연결을 보여주어 운영자가 실행 흐름을 이해하고 문제가 전파되는 위치를 식별할 수 있습니다.

수행할 수 있는 작업

  • SLO/SLI 상태 확인: 목표를 충족하지 못하는 서비스(및 업스트림/다운스트림 영향)를 빠르게 파악합니다.
  • 드릴다운: 노드/엣지를 선택하여 상세 메트릭을 열고 서비스 세부 정보로 이동합니다.
  • 그룹화 및 필터링: 기본 그룹화(관련 서비스, 환경)를 사용하고 AWS 태그 또는 OpenTelemetry 리소스 속성을 사용하여 사용자 정의 그룹화를 만듭니다.
  • 교차 계정 가시성: 여러 AWS 계정에 걸쳐 서비스를 봅니다(교차 계정 관찰성을 통해 설정한 경우).

서비스, 오퍼레이션 및 종속성

Application Signals는 팀이 프로덕션 시스템을 디버깅하는 방식에 맞는 모델로 텔레메트리를 구조화합니다:

  • 서비스: 배포 가능한 단위(예: ECS 서비스, EKS 워크로드, Lambda 함수).
  • 오퍼레이션: 서비스 내의 요청 핸들러(예: HTTP 라우트 또는 RPC 메서드).
  • 종속성: 서비스에서 만드는 다운스트림 호출(예: 다른 서비스 또는 AWS 종속성).

이 구조를 통해 "어떤 오퍼레이션이 성능 저하되었는가?", "업스트림 문제인가 다운스트림 문제인가?", "어떤 종속성이 지연 시간이나 장애에 기여하고 있는가?"와 같은 질문에 쉽게 답할 수 있습니다.

서비스 수준 목표 (SLO)

Application Signals가 수집하는 텔레메트리에서 직접 SLO를 생성하고 모니터링합니다. SLO는 목표에 대한 성능과 안정성을 추적하고 "위험" 상태의 서비스를 조기에 표면화하는 데 도움이 됩니다.

  • 주요 SLI에 대한 SLO 정의: 비즈니스 목표와 관련된 가용성, 지연 시간 또는 기타 CloudWatch 메트릭.
  • 상태 추적: 서비스 목록 및 Application Map에서 SLI 상태를 확인합니다.
  • 알림: SLO 상태/소진율에 연결된 알람을 생성하여 장애가 발생하기 전에 팀이 대응할 수 있도록 합니다.

계측 (OpenTelemetry 호환)

Application Signals는 OpenTelemetry와 호환되며 플랫폼과 언어에 따라 여러 가지 애플리케이션 계측 방법을 지원합니다.

  • 지원 언어: Java, Python, Node.js 및 .NET (추가 런타임은 OpenTelemetry 제로 코드 계측 경로를 통해 지원).
  • 지원 시스템: Amazon EKS, Kubernetes, Amazon ECS, Amazon EC2 (및 CloudWatch Agent + AWS Distro for OpenTelemetry에 의존하는 활성화 패턴).
  • 일반적인 설정 패턴: 환경에 따라 AWS Distro for OpenTelemetry (ADOT) SDK/콜렉터 또는 OpenTelemetry SDK + 콜렉터.

RUM 및 Synthetics와의 통합

Application Signals는 CloudWatch RUM 및 CloudWatch Synthetics와 통합됩니다. 해당 클라이언트/카나리에 대해 트레이싱을 활성화하면 Application Map에 표시되어 사용자 경험과 사전 예방 테스트를 호출하는 백엔드 서비스에 연결하는 데 도움이 됩니다.

  • RUM 클라이언트: 프론트엔드 성능 및 JS 오류를 백엔드 서비스 호출에 연결합니다.
  • Synthetics 카나리: 중요한 흐름을 검증하고 사용자보다 먼저 문제를 감지하는 데 도움을 줍니다.
