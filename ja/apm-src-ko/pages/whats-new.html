새로운 기능

이 페이지에서 다루는 내용

이 사이트는 정적 문서 스캐폴드입니다. 콘텐츠의 정확성을 유지하기 위해 여기의 "새로운 기능"은 주요 공개 마일스톤을 요약하고 세부 사항에 대해서는 공식 AWS 소스를 안내합니다.

  • AWS 발표: AWS "새로운 기능"
  • CloudWatch 문서: Application Signals

2024년 6월: Application Signals 정식 출시

CloudWatch Application Signals가 정식 출시(GA)되었습니다. 호출량, 가용성, 지연 시간, 장애 및 오류와 같은 표준 애플리케이션 메트릭을 자동으로 수집하여 빠른 서비스 수준 분류 및 문제 해결에 중점을 둡니다.

AWS 발표 참조: GA 발표.

2024년 9월: 요청 기반 SLO

AWS가 CloudWatch Application Signals에 대한 요청 기반 서비스 수준 목표(SLO) 지원을 발표했습니다.

AWS 발표 참조: 요청 기반 SLO.

진행 중: Application Map 및 분류 워크플로

CloudWatch Application Map은 Application Signals의 "서비스 맵" 용어를 대체합니다. 토폴로지(클라이언트, 카나리, 서비스, 종속성)를 시각화하고 SLO/SLI 상태를 표시하여 팀이 인시던트를 분류하는 데 도움을 줍니다.

문서 참조: Application Map.
