Application Map

What it is

The CloudWatch Application Map visualizes your application topology and operational health. It replaces the legacy “Service Map” terminology for Application Signals.

It shows connections between clients, canaries, services, and dependencies, helping teams understand execution flow and quickly locate where faults and latency originate.

Key capabilities

  • SLO/SLI health: see which services are meeting or breaching objectives and how impact propagates downstream.
  • Drill-down: select nodes/edges to view metrics and navigate to service details.
  • Grouping and filtering: use default groupings (Related services, Environment) or define custom groups using tags or OTEL resource attributes.
  • Un-instrumented services: Application Signals can display services that aren’t instrumented yet to help prioritize rollout.
  • Cross-account visibility: visualize services across multiple accounts when cross-account observability is configured.
