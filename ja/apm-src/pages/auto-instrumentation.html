Auto Instrumentation

What “auto instrumentation” means

Auto instrumentation adds OpenTelemetry instrumentation at runtime (without changing your application code) so the service emits the telemetry Application Signals uses for discovery and health monitoring.

Common AWS setup components

  • AWS Distro for OpenTelemetry (ADOT) SDK and/or collector
  • CloudWatch Agent to collect/export telemetry in supported setups
  • Platform enablement (EKS add-on / ECS sidecar / EC2 agent setup / Lambda layers)

Start here: Application Signals overview.

Supported languages and systems

Application Signals supports Java, Python, Node.js, and .NET applications and is supported/tested on Amazon EKS, Kubernetes, Amazon ECS, and Amazon EC2.

See: Supported systems.

Notes and gotchas

  • Node.js ESM: ESM support is limited; CommonJS is the recommended path unless you follow the ESM-specific guidance in the docs.
  • Python in containers: you may need PYTHONPATH configured for OTel autoinstrumentation.
