Generative AI observability

Application Signals + Amazon Bedrock

CloudWatch Application Signals supports troubleshooting generative AI applications that interact with Amazon Bedrock models. It provides out-of-the-box telemetry for Bedrock API calls so you can investigate:

  • Model configuration issues
  • Usage and cost drivers (token usage)
  • Model latency
  • Stopped/finish reasons for responses

Fine-grained metrics and traces

For Bedrock calls, Application Signals can generate and correlate performance metrics and trace spans with resource-level detail (for example: model ID, guardrails ID, knowledge base ID, and Bedrock agent ID).

See: Example: troubleshoot generative AI apps interacting with Amazon Bedrock.

GenAI semantic attributes + Transaction Search

Application Signals can emit OpenTelemetry GenAI semantic convention attributes (for example, gen_ai.request.model and token usage fields). You can analyze these with CloudWatch Transaction Search to compare prompts, token usage, and latency across models.
