ऑटो इंस्ट्रूमेंटेशन

"ऑटो इंस्ट्रूमेंटेशन" का क्या अर्थ है

ऑटो इंस्ट्रूमेंटेशन रनटाइम पर OpenTelemetry इंस्ट्रूमेंटेशन जोड़ता है (आपके एप्लिकेशन कोड को बदले बिना) ताकि सर्विस वह टेलीमेट्री उत्सर्जित करे जिसका उपयोग Application Signals डिस्कवरी और हेल्थ मॉनिटरिंग के लिए करता है।

सामान्य AWS सेटअप कंपोनेंट्स

  • AWS Distro for OpenTelemetry (ADOT) SDK और/या कलेक्टर
  • CloudWatch Agent समर्थित सेटअप में टेलीमेट्री एकत्र/निर्यात करने के लिए
  • प्लेटफ़ॉर्म इनेबलमेंट (EKS ऐड-ऑन / ECS साइडकार / EC2 एजेंट सेटअप / Lambda लेयर्स)

यहां से शुरू करें: Application Signals अवलोकन।

समर्थित भाषाएं और सिस्टम

Application Signals Java, Python, Node.js, और .NET एप्लिकेशन को सपोर्ट करता है और Amazon EKS, Kubernetes, Amazon ECS, और Amazon EC2 पर समर्थित/परीक्षित है।

देखें: समर्थित सिस्टम।

नोट्स और सावधानियां

  • Node.js ESM: ESM सपोर्ट सीमित है; CommonJS अनुशंसित पथ है जब तक कि आप डॉक्स में ESM-विशिष्ट मार्गदर्शन का पालन नहीं करते।
  • कंटेनरों में Python: OTel ऑटोइंस्ट्रूमेंटेशन के लिए आपको PYTHONPATH कॉन्फ़िगर करने की आवश्यकता हो सकती है।
