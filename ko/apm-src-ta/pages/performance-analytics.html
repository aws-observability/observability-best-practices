CloudWatch APM ஆவணங்கள்

செயல்திறன் பகுப்பாய்வு

நிலையான பயன்பாட்டு மெட்ரிக்ஸ்

Application Signals நிலையான பயன்பாட்டு மெட்ரிக்ஸை CloudWatch-க்கு publish செய்கிறது (ApplicationSignals namespace). இந்த மெட்ரிக்ஸ் சேவைகள், செயல்பாடுகள் மற்றும் சார்புகள் முழுவதும் விரைவான பகுப்பாய்வுக்கு வடிவமைக்கப்பட்டுள்ளன.

  • தாமதம் (ms)
  • தவறு (HTTP 5xx மற்றும் span status errors)
  • பிழை (HTTP 4xx)
  • கிடைக்கும் தன்மை console-ல் \( (1 - \text{Faults}/\text{Total}) \times 100 \) என கணக்கிடப்படுகிறது, இங்கு Total SampleCount(Latency)-லிருந்து பெறப்படுகிறது. 4xx responses இந்த கிடைக்கும் தன்மை கணக்கீட்டில் வெற்றிகரமாக கருதப்படுகின்றன.

பார்க்கவும்: Application Signals மூலம் சேகரிக்கப்படும் மெட்ரிக்ஸ்.

பரிமாணங்கள் (service vs dependency)

மெட்ரிக்ஸ் செயல்திறனை பிரிக்க உதவும் பரிமாணங்களுடன் வெளியிடப்படுகின்றன:

  • சேவை மெட்ரிக்ஸ்: Environment, Service, Operation
  • சார்பு மெட்ரிக்ஸ்: Environment, Service, Operation மற்றும் RemoteService / RemoteOperation (மற்றும் platform-ஐ பொறுத்து பிற remote dimensions)

இது "எந்த செயல்பாடு பின்னடைந்தது?" மற்றும் "தாமதம் ஒரு dependency call-ஆல் இயக்கப்படுகிறதா?" போன்ற கேள்விகளுக்கு பதிலளிப்பதை எளிதாக்குகிறது.

Runtime மெட்ரிக்ஸ் (ஆதரிக்கப்படும் இடங்களில்)

Application Signals ஆதரிக்கப்படும் runtimes-க்கு runtime மெட்ரிக்ஸையும் சேகரிக்க முடியும் (எடுத்துக்காட்டாக, JVM GC, memory, CPU; Python process metrics; .NET runtime metrics). Node.js-க்கு Runtime metrics சேகரிக்கப்படுவதில்லை.

OTEL_AWS_APPLICATION_SIGNALS_RUNTIME_ENABLED=false மூலம் runtime metrics collection-ஐ முடக்கலாம் (platform-குறிப்பிட்ட படிகள் docs-ல் உள்ளன).
