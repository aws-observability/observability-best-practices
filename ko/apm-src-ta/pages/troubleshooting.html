CloudWatch APM ஆவணங்கள்

பிழைத்திருத்தம்

பொதுவான சிக்கல்கள்

CloudWatch Application Signals-ஐ செயல்படுத்தும்போது அடிக்கடி எதிர்கொள்ளும் சிக்கல்கள் மற்றும் அவற்றை எவ்வாறு தீர்ப்பது என்பதை இந்த பிரிவு உள்ளடக்குகிறது.

CloudWatch-ல் Traces தோன்றவில்லை

சாத்தியமான காரணங்கள்:

  • சேவைக்கு Application Signals இயக்கப்படவில்லை
  • தவறான IAM permissions
  • Agent சரியாக configure செய்யப்படவில்லை அல்லது இயங்கவில்லை
  • Sampling rate மிகவும் குறைவாக அமைக்கப்பட்டுள்ளது
  • CloudWatch-க்கு network connectivity சிக்கல்கள்

தீர்வுகள்:

  1. AWS console-ல் Application Signals இயக்கப்பட்டுள்ளதை சரிபாருங்கள்
  2. IAM permissions-ல் cloudwatch:PutMetricData மற்றும் xray:PutTraceSegments உள்ளதை சரிபாருங்கள்
  3. Agent இயங்குகிறது மற்றும் சரியாக configure செய்யப்பட்டுள்ளது என்பதை உறுதிப்படுத்துங்கள்
  4. Testing-க்கு sampling rate-ஐ அதிகரிக்கவும் (தற்காலிகமாக 100% அமைக்கவும்)
  5. VPC security groups மற்றும் NACLs CloudWatch-க்கு outbound traffic அனுமதிக்கிறதா என்று சரிபாருங்கள்

காணாமல் போன அல்லது முழுமையற்ற Traces

பொதுவான சிக்கல்கள்:

  • சேவைகளுக்கிடையே Context propagate ஆகவில்லை
  • சில சேவைகள் instrumented செய்யப்படவில்லை
  • Sampling spans-ஐ drop செய்கிறது
  • Agent configuration சிக்கல்கள்

Debugging படிகள்:

  1. Uninstrumented சேவைகளை அடையாளம் காண service map-ஐ சரிபாருங்கள்
  2. Trace context headers pass ஆகிறதா என்று சரிபாருங்கள் (எ.கா., x-amzn-trace-id)
  3. Configuration errors-க்கு agent logs-ஐ review செய்யுங்கள்
  4. Sampling சிக்கல்களை நிராகரிக்க 100% sampling-உடன் test செய்யுங்கள்
  5. Partial traces-ஐ கண்டறிய trace search பயன்படுத்துங்கள்

அதிக Instrumentation Overhead

அறிகுறிகள்:

  • அதிகரித்த CPU பயன்பாடு
  • அதிக memory consumption
  • அதிகரித்த தாமதம்
  • Application செயல்திறன் சீரழிவு

Optimization strategies:

  • Sampling rate-ஐ குறைக்கவும் (production-க்கு 10%-உடன் தொடங்குங்கள்)
  • Tail-based-க்கு பதிலாக head-based sampling பயன்படுத்துங்கள்
  • பொருத்தமான span limits கட்டமைக்கவும்
  • Batch span exports
  • Instrumentation metrics-ஐ கண்காணிக்கவும்

RUM Data தோன்றவில்லை

சரிபார்ப்புப் பட்டியல்:

  • RUM web client சரியாக install மற்றும் load செய்யப்பட்டுள்ளது
  • Application ID மற்றும் configuration சரியானது
  • RUM domains-க்கு CORS headers configure செய்யப்பட்டுள்ளன
  • Content Security Policy RUM scripts-ஐ அனுமதிக்கிறது
  • Browser console JavaScript errors காட்டவில்லை

சரிபார்ப்பு:

// Check if RUM is loaded
console.log(window.cwr);

// Verify configuration
console.log(cwr('getConfig'));

Synthetic Canaries தோல்வியடைகின்றன

பொதுவான காரணங்கள்:

  • Canary VPC-லிருந்து network connectivity சிக்கல்கள்
  • Authentication அல்லது authorization தோல்விகள்
  • Application மாற்றங்கள் test scripts-ஐ உடைக்கின்றன
  • Canary runtime அல்லது dependencies outdated

Debugging:

  1. CloudWatch Logs-ல் canary logs-ஐ review செய்யுங்கள்
  2. Canary scripts-ஐ locally test செய்யுங்கள்
  3. VPC routing மற்றும் security groups-ஐ சரிபாருங்கள்
  4. தேவைப்பட்டால் canary runtime-ஐ update செய்யுங்கள்
  5. Application endpoints accessible என்பதை verify செய்யுங்கள்

Debugging வழிகாட்டிகள்

சிக்கலான சிக்கல்களை கண்டறிவதற்கான படிப்படியான வழிகாட்டிகள்.

Trace Context Propagation-ஐ Debugging செய்தல்

படி 1: Instrumentation-ஐ சரிபாருங்கள்

உங்கள் call chain-ல் உள்ள அனைத்து சேவைகளும் சரியாக instrumented செய்யப்பட்டுள்ளதை சரிபாருங்கள்:

# Check for trace headers in requests
curl -H "x-amzn-trace-id: Root=1-5759e988-bd862e3fe1be46a994272793;Parent=53995c3f42cd8ad8;Sampled=1" \
     http://your-service/endpoint

படி 2: Agent Configuration-ஐ சரிபாருங்கள்

Agent configuration files மற்றும் environment variables-ஐ verify செய்யுங்கள்.

படி 3: End-to-End Test செய்யுங்கள்

ஒரு test request செய்து அனைத்து சேவைகள் வழியாக trace செய்யுங்கள்.

செயல்திறன் சிக்கல் கண்டறிதல்

Bottleneck-ஐ அடையாளம் காணுங்கள்:

  1. High-latency சேவைகளுக்கு service map-ஐ சரிபாருங்கள்
  2. மெதுவான spans-க்கு trace timelines-ஐ ஆராயுங்கள்
  3. Resource metrics-ஐ (CPU, memory, disk I/O) review செய்யுங்கள்
  4. Logs-ல் error patterns-ஐ பாருங்கள்
  5. Database query செயல்திறனை சரிபாருங்கள்

பொதுவான Bottlenecks:

  • Database query optimization தேவை
  • External API calls timeout ஆகின்றன
  • Resource constraints (CPU/memory)
  • Thread pool exhaustion
  • சேவைகளுக்கிடையே network தாமதம்

Agent பிழைத்திருத்தம்

Agent நிலையை சரிபாருங்கள்:

# For Java applications
jps -l | grep opentelemetry

# For Python applications
ps aux | grep opentelemetry

# Check agent logs
tail -f /var/log/aws-opentelemetry-agent.log

பொதுவான Agent சிக்கல்கள்:

  • JVM-க்கு தவறான Java agent version
  • காணாமல் போன dependencies அல்லது libraries
  • Configuration file syntax errors
  • Log files-க்கு எழுதும் permission சிக்கல்கள்

கருவிகள் மற்றும் கட்டளைகள்

பிழைத்திருத்தத்திற்கு பயனுள்ள கருவிகள் மற்றும் கட்டளைகள்.

CloudWatch CLI கட்டளைகள்

# List Application Signals services
aws application-signals list-services

# Get service metrics
aws cloudwatch get-metric-data --metric-data-queries '[
  {
    "Id": "m1",
    "MetricStat": {
      "Metric": {
        "Namespace": "AWS/ApplicationSignals",
        "MetricName": "Latency"
      },
      "Period": 300,
      "Stat": "Average"
    }
  }
]'

# Query traces
aws xray get-trace-summaries --start-time 2024-01-01T00:00:00Z --end-time 2024-01-02T00:00:00Z

Log Analysis

Telemetry data-ஐ பகுப்பாய்வு செய்ய CloudWatch Logs Insights பயன்படுத்துங்கள்:

# Find errors in application logs
fields @timestamp, @message
| filter @message like /ERROR/
| sort @timestamp desc
| limit 100

# Correlate traces with logs
fields @timestamp, traceId, @message
| filter ispresent(traceId)
| sort @timestamp desc
