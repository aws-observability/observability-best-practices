CloudWatch APM 文档

安全与合规

安全

CloudWatch Application Signals 实施了多层安全措施来保护您的 Observability 数据，并确保符合行业标准。

数据加密

静态加密：

  • 所有链路追踪数据使用 AES-256 加密
  • RUM 数据使用 AWS Key Management Service (KMS) 加密
  • Synthetic Monitoring 数据以加密方式存储
  • 您可以使用客户托管密钥进行额外控制

传输加密：

  • 所有数据传输使用 TLS 1.2 或更高版本
  • 对 CloudWatch 服务的 API 调用均已加密
  • RUM 数据收集使用 HTTPS
  • 代理到收集器之间的通信已加密保护

身份和访问管理

IAM 权限：

在授予 Application Signals 访问权限时，请使用最小权限原则：

只读访问：
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
完全访问：
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

数据隐私和 PII 保护

数据脱敏：

  • 对敏感信息实施数据掩码处理
  • 对链路追踪属性中的 PII 使用令牌化
  • 配置数据保留策略
  • 定期审计收集的数据

最佳实践：

  • 切勿在链路追踪中包含密码、API 密钥或令牌
  • 对电子邮件地址和电话号码进行哈希或掩码处理
  • 从包含敏感数据的 URL 中删除查询参数
  • 使用通用标识符代替个人信息

网络安全

VPC 配置：

  • 在您的 VPC 内部署代理以增强安全性
  • 使用 VPC 端点进行 CloudWatch API 调用
  • 配置安全组以限制流量
  • 实施网络 ACL 进行额外控制

代理安全：

  • 以最小必要权限运行代理
  • 及时更新代理的安全补丁
  • 监控代理日志中的安全事件
  • 尽可能使用只读访问

数据隐私

了解 Application Signals 如何处理您的数据隐私要求。

数据收集和使用

我们收集的数据：

  • 应用程序性能指标和链路追踪
  • 服务依赖关系信息
  • 错误信息和堆栈跟踪
  • 系统和基础设施元数据

我们不收集的数据：

  • 应用程序业务数据或用户内容
  • 数据库内容或文件系统数据
  • 用户凭证或认证令牌
  • 个人身份信息（除非明确包含）

数据保留

默认保留期限：

  • 链路追踪数据：30 天（可配置最长 1 年）
  • 指标：15 个月（标准 CloudWatch 保留期）
  • RUM 数据：30 天（可配置最长 1 年）
  • Synthetics 结果：90 天（可配置）

数据删除：

  • 使用 CloudWatch 数据保留策略
  • 实施自动化数据生命周期管理
  • 如需要，可通过 AWS Support 请求数据删除

合规框架

支持的标准：

  • SOC 2：安全性、可用性和保密性控制
  • PCI DSS：支付卡行业数据安全标准
  • HIPAA：健康保险流通与责任法案
  • GDPR：通用数据保护条例
  • FedRAMP：联邦风险和授权管理计划

合规指南

针对常见合规场景的具体指导。

HIPAA 合规

要求：

  • 对链路追踪中的受保护健康信息 (PHI) 进行加密
  • 实施访问控制和审计日志
  • 与 AWS 签订业务关联协议 (BAA)
  • 配置适当的数据保留策略

实施：

  • 在链路追踪属性和 span 名称中掩码 PHI
  • 使用客户托管的 KMS 密钥
  • 启用 CloudTrail 进行审计日志记录
  • 配置 VPC 端点实现数据隔离

PCI DSS 合规

关键考虑事项：

  • 切勿在链路追踪或日志中包含持卡人数据
  • 实施网络分段
  • 对传输中和静态数据使用加密
  • 定期进行安全评估和渗透测试

最佳实践：

  • 在插桩前对敏感数据进行令牌化
  • 为 Application Signals 代理使用私有子网
  • 实施最小权限 IAM 策略
  • 定期审查收集的数据

GDPR 合规

数据主体权利：

  • 访问个人数据的权利
  • 更正权
  • 删除权（"被遗忘权"）
  • 数据可移植权

实施：

  • 最小化遥测数据中的 PII 收集
  • 实施数据匿名化技术
  • 根据业务需求配置数据保留
  • 记录数据处理活动

安全监控

监控您的 Application Signals 部署中的安全事件。

安全最佳实践

  • 启用 CloudTrail 进行 API 活动日志记录
  • 监控 IAM 策略变更和访问模式
  • 为异常数据访问模式设置告警
  • 定期进行安全评估和漏洞扫描
  • 实施多因素认证 (MFA)

事件响应

  • 制定安全事件的事件响应计划
  • 了解在遭到入侵时如何禁用 Application Signals
  • 记录升级流程
  • 定期进行事件响应测试
