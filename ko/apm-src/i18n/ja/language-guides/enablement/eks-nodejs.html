概要

このガイドでは、 ステップバイステップの手順 Amazon EKS 上に Node.js Express アプリケーションを AWS Application Signals の自動計装とともにデプロイする方法を説明します。これは Lambda Node.js 関数の EKS 版であり、同じビジネスロジックとエンドポイントを再利用しています。

達成できること：
  • EKS 上に Node.js Express アプリをデプロイする
  • AWS Application Signals の自動計装を有効にする
  • CloudWatch Application Signals コンソールでサービスを確認する
  • サービスのメトリクス、トレース、オペレーションを監視する
完了までの時間： 30〜45 分

アーキテクチャ

EKS Observability アドオン（標準的なアプローチ）： このガイドでは、EKSクラスターでApplication Signalsを有効にするための推奨される標準的な方法である、公式のEKSオブザーバビリティアドオンを使用します。このアドオンは、CloudWatch Agentを手動で設定することなく、テレメトリの収集を自動的に処理します。

EKSクラスターで実行されるもの：

  1. Node.js Expressアプリケーション：自動インストルメンテーションアノテーションを含むWeb API
  2. EKSオブザーバビリティアドオン：トレースとメトリクス収集を自動化する公式AWSアドオン
  3. CloudWatch Agent（アドオンによる管理）：アドオンによって自動的にデプロイおよび設定される

フロー：

EKS Cluster
  → Node.js Express App (auto-instrumented via annotation)
  → EKS Observability Add-on
  → CloudWatch Agent (managed)
  → CloudWatch Application Signals
EKSオブザーバビリティアドオンの主なメリット：
  • CloudWatch Agentの手動設定が不要
  • サポートされている言語への自動インストルメンテーション
  • Application Signalsの組み込みサポート
  • デプロイと管理の簡素化

ステップ1：アプリケーションの作成

1.1 プロジェクト構造の作成

mkdir eks-nodejs
cd eks-nodejs

1.2 Node.jsアプリケーションの作成

作成 index.js Lambda関数と同じエンドポイントを持つ：

const express = require('express');
const axios = require('axios');

const app = express();
const port = process.env.PORT || 8080;

// Middleware
app.use(express.json());

// Simple endpoints
app.get('/', (req, res) => {
    res.json({
        message: 'AWS Application Signals EKS Node.js Demo',
        timestamp: Date.now(),
        service: 'nodejs-eks-service',
        runtime: 'nodejs18.x'
    });
});

app.get('/health', (req, res) => {
    res.json({
        status: 'healthy',
        timestamp: Date.now(),
        service: 'nodejs-eks-service'
    });
});

app.get('/api/hello/:name', (req, res) => {
    const { name } = req.params;
    // Simulate some processing time
    setTimeout(() => {
        res.json({
            message: `Hello, ${name}!`,
            timestamp: Date.now(),
            service: 'nodejs-eks-service'
        });
    }, Math.random() * 100 + 10);
});

app.get('/api/slow', (req, res) => {
    // Simulate slow operation (500-1500ms)
    const delay = Math.random() * 1000 + 500;
    setTimeout(() => {
        res.json({
            message: 'Slow operation completed',
            timestamp: Date.now(),
            service: 'nodejs-eks-service',
            duration_ms: Math.floor(delay)
        });
    }, delay);
});

app.get('/api/error', (req, res) => {
    if (Math.random() > 0.5) {
        throw new Error('Simulated error for testing Application Signals error tracking');
    }
    res.json({
        message: 'No error occurred',
        timestamp: Date.now(),
        service: 'nodejs-eks-service'
    });
});

app.get('/api/external', async (req, res) => {
    try {
        const response = await axios.get('https://httpbin.org/delay/1', {
            timeout: 10000
        });
        res.json({
            message: 'External call completed',
            external_status_code: response.status,
            timestamp: Date.now(),
            service: 'nodejs-eks-service'
        });
    } catch (error) {
        res.status(500).json({
            error: 'External call failed',
            message: error.message,
            timestamp: Date.now(),
            service: 'nodejs-eks-service'
        });
    }
});

app.listen(port, '0.0.0.0', () => {
    console.log(`Server running on port ${port}`);
});

ステップ3：EKSへのデプロイ

3.1 EKSクラスターの作成

# Create EKS cluster
eksctl create cluster --name appsignals-cluster --region us-east-1 --nodegroup-name workers --node-type t3.medium --nodes 2 --nodes-min 1 --nodes-max 3

# Create namespace
kubectl create namespace appsignals-demo

3.2 EKSオブザーバビリティアドオンのインストール（必須）

重要： EKSオブザーバビリティアドオンは、EKSでApplication Signalsを有効にするための標準かつ推奨される方法です。このアドオンはテレメトリの収集を自動的に処理し、自動インストルメンテーション機能を提供します。 
# Install the EKS observability add-on (REQUIRED for Application Signals)
aws eks create-addon --cluster-name appsignals-cluster --addon-name amazon-cloudwatch-observability --region us-east-1

# Verify the add-on is installed
aws eks describe-addon --cluster-name appsignals-cluster --addon-name amazon-cloudwatch-observability --region us-east-1
EKSオブザーバビリティアドオンが提供するもの：
  • CloudWatch Agentの自動デプロイ
  • Application Signalsの組み込みサポート
  • Java、Python、Node.js、および.NETへの自動インストルメンテーション
  • 手動設定が不要

3.3 Dockerイメージのビルドとプッシュ

# Create ECR repository
aws ecr create-repository --repository-name my-appsignals-nodejs-app --region us-east-1

# Get login token
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 188524202814.dkr.ecr.us-east-1.amazonaws.com

docker build -t my-appsignals-nodejs-app .

# Push image
docker push 188524202814.dkr.ecr.us-east-1.amazonaws.com/my-appsignals-nodejs-app:latest

ステップ4: トラフィックの生成とモニタリング

4.1 トラフィックの生成

作成 generate-traffic.sh:

#!/bin/bash

# Port forward the service
kubectl port-forward service/nodejs-eks-service 8080:80 -n appsignals-demo &
PORT_FORWARD_PID=$!

# Wait for port forward to be ready
sleep 10

BASE_URL="http://localhost:8080"

echo "Generating traffic for Node.js EKS Application Signals demo..."

# Generate traffic for 5 minutes
for i in {1..300}; do
    echo "Request $i"

    # Root endpoint
    curl -s "$BASE_URL/" > /dev/null

    # Health check
    curl -s "$BASE_URL/health" > /dev/null

    # Hello endpoint
    curl -s "$BASE_URL/api/hello/User$i" > /dev/null

    # Slow endpoint
    curl -s "$BASE_URL/api/slow" > /dev/null

    # Error endpoint
    curl -s "$BASE_URL/api/error" > /dev/null

    # External call
    curl -s "$BASE_URL/api/external" > /dev/null

    sleep 1
done

echo "Traffic generation complete!"

# Clean up port forward
kill $PORT_FORWARD_PID

実行可能にして実行する:

chmod +x generate-traffic.sh
./generate-traffic.sh

4.2 CloudWatchでのモニタリング

  1. CloudWatchコンソールを開く

    • AWSコンソール → CloudWatchに移動する
    • "Application Signals"に移動する

  2. サービスマップを表示する

    • "nodejs-eks-service"サービスを探す
    • サービスの依存関係と正常性を確認する

  3. メトリクスを確認する

    • レイテンシー、エラーレート、スループットを表示する
    • Lambdaメトリクスと比較する

  4. トレースを分析する

    • サービスをクリックして詳細なトレースを表示する
    • オペレーションレベルのメトリクスを表示する

トラブルシューティング

よくある問題

  1. アプリケーションが起動しない

    • Podのステータスを確認する: kubectl get pods -n appsignals-demo
    • Podのログを確認する: kubectl logs -n appsignals-demo -l app=nodejs-eks-service
    • EKSオブザーバビリティアドオンがインストールされていることを確認する

  2. Application Signalsにデータが表示されない

    • 自動インストルメンテーションのアノテーションが存在することを確認する
    • 環境変数が正しく設定されていることを確認する
    • EKSオブザーバビリティアドオンが実行中であることを確認する

  3. ポートフォワードの問題

    • サービスが実行中であることを確認する: kubectl get svc -n appsignals-demo
    • Podの準備ができていることを確認する: kubectl get pods -n appsignals-demo
