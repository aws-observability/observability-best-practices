Overview

This guide provides step-by-step instructions to deploy a Python FastAPI application on Amazon EC2 with AWS Application Signals enabled. This is the EC2 equivalent of the Lambda Python function, reusing the same business logic and endpoints.

What you'll accomplish:
  • Deploy a Python FastAPI app on EC2
  • Enable AWS Application Signals auto-instrumentation
  • View your service in CloudWatch Application Signals console
  • Monitor service metrics, traces, and operations
Time to complete: 30-45 minutes

Prerequisites

Required Tools

  • AWS CLI v2 (latest version)
  • Docker Desktop
  • Python 3.11+ (for local development)
  • pip (for local development)
  • AWS Account with appropriate permissions
  • Text editor

AWS Permissions Required

  • EC2: Full access (create instances, security groups, key pairs)
  • IAM: Create roles and policies
  • CloudWatch: Logs and Application Signals access
  • VPC: Create and manage VPC resources
  • SSM: Parameter Store write access

Knowledge Assumed

  • Basic AWS CLI usage
  • Basic Docker commands
  • Basic understanding of EC2 concepts
  • Basic Python/FastAPI knowledge

Architecture

Your EC2 instance will run:

  1. Python FastAPI Application: Your web API with Application Signals instrumentation
  2. OpenTelemetry Auto-instrumentation: Automatic tracing and metrics collection
  3. CloudWatch Agent: Collects traces and metrics (optional, can use direct OTLP)

Flow:

EC2 Instance
  → Python FastAPI App (auto-instrumented)
  → OpenTelemetry Collector (optional)
  → CloudWatch Application Signals

1.4 Create OpenTelemetry Configuration

Create otel_config.py:

"""
OpenTelemetry configuration for Python EC2 Application Signals
This file sets up automatic instrumentation for the FastAPI application
"""
import os
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk.resources import Resource
from opentelemetry.semantic_conventions.resource import ResourceAttributes
from opentelemetry.instrumentation.fastapi import FastAPIInstrumentor
from opentelemetry.instrumentation.requests import RequestsInstrumentor
from opentelemetry.instrumentation.boto3 import Boto3Instrumentor

def setup_otel():
    """Set up OpenTelemetry instrumentation"""

    # Configure resource
    resource = Resource.create({
        ResourceAttributes.SERVICE_NAME: "ec2-python",
        ResourceAttributes.SERVICE_NAMESPACE: "aws-observability",
        ResourceAttributes.SERVICE_VERSION: "1.0.0",
        ResourceAttributes.DEPLOYMENT_ENVIRONMENT: "ec2",
    })

    # Set up tracer provider
    trace.set_tracer_provider(TracerProvider(resource=resource))
    tracer = trace.get_tracer(__name__)

    # Configure OTLP exporter
    otlp_endpoint = os.getenv("OTEL_EXPORTER_OTLP_ENDPOINT", "https://otlp.nr-data.net")
    otlp_headers = os.getenv("OTEL_EXPORTER_OTLP_HEADERS", "api-key=your-api-key")

    # Parse headers
    headers = {}
    if otlp_headers:
        for header in otlp_headers.split(","):
            if "=" in header:
                key, value = header.split("=", 1)
                headers[key.strip()] = value.strip()

    # Create OTLP exporter
    otlp_exporter = OTLPSpanExporter(
        endpoint=f"{otlp_endpoint}/v1/traces",
        headers=headers
    )

    # Add span processor
    span_processor = BatchSpanProcessor(otlp_exporter)
    trace.get_tracer_provider().add_span_processor(span_processor)

    print("OpenTelemetry configured for ec2-python")

def instrument_app(app):
    """Instrument the FastAPI application"""
    # Instrument FastAPI
    FastAPIInstrumentor.instrument_app(app)

    # Instrument requests library
    RequestsInstrumentor().instrument()

    # Instrument boto3
    Boto3Instrumentor().instrument()

    print("Application instrumented with OpenTelemetry")

1.5 Create Main Entry Point

Create main.py:

"""
Main entry point for the EC2 Python Application Signals application
This file sets up OpenTelemetry instrumentation and starts the FastAPI app
"""
import os
import asyncio
from app import app
from otel_config import setup_otel, instrument_app

# Set up OpenTelemetry before importing the app
setup_otel()

# Instrument the FastAPI application
instrument_app(app)

if __name__ == "__main__":
    import uvicorn

    # Set environment variables for Application Signals
    os.environ.setdefault("OTEL_SERVICE_NAME", "ec2-python")
    os.environ.setdefault("OTEL_RESOURCE_ATTRIBUTES", "service.name=ec2-python,service.namespace=aws-observability,service.version=1.0.0,deployment.environment=ec2")
    os.environ.setdefault("OTEL_EXPORTER_OTLP_ENDPOINT", "https://otlp.nr-data.net")
    os.environ.setdefault("OTEL_EXPORTER_OTLP_HEADERS", "api-key=your-api-key")
    os.environ.setdefault("OTEL_TRACES_EXPORTER", "otlp")
    os.environ.setdefault("OTEL_METRICS_EXPORTER", "otlp")
    os.environ.setdefault("OTEL_LOGS_EXPORTER", "otlp")

    print("Starting EC2 Python Application Signals Demo...")
    print(f"Service: ec2-python")
    print(f"Environment: {os.getenv('OTEL_RESOURCE_ATTRIBUTES', 'default')}")

    # Start the application
    uvicorn.run(app, host="0.0.0.0", port=8080)

# Install dependencies if not present
if [ ! -d "venv" ]; then
    echo "Creating virtual environment..."
    python3 -m venv venv
fi

# Activate virtual environment
source venv/bin/activate

# Install dependencies if requirements.txt is newer than installed packages
if [ requirements.txt -nt venv/pyvenv.cfg ] || [ ! -f venv/pyvenv.cfg ]; then
    echo "Installing dependencies..."
    pip install -r requirements.txt
fi

# Start the application
echo "Starting Python application on port 8080..."
python main.py

Step 3: Deploy to EC2

3.1 Create EC2 Instance

# Create a key pair
aws ec2 create-key-pair --key-name ec2-python-key --query 'KeyMaterial' --output text > ec2-python-key.pem
chmod 400 ec2-python-key.pem

# Create security group
aws ec2 create-security-group \
    --group-name ec2-python-sg \
    --description "Security group for Python EC2 Application Signals demo"

# Get security group ID
SG_ID=$(aws ec2 describe-security-groups --group-names ec2-python-sg --query 'SecurityGroups[0].GroupId' --output text)

# Allow HTTP traffic
aws ec2 authorize-security-group-ingress \
    --group-id $SG_ID \
    --protocol tcp \
    --port 8080 \
    --cidr 0.0.0.0/0

# Allow SSH
aws ec2 authorize-security-group-ingress \
    --group-id $SG_ID \
    --protocol tcp \
    --port 22 \
    --cidr 0.0.0.0/0

# Launch EC2 instance
aws ec2 run-instances \
    --image-id ami-0c02fb55956c7d316 \
    --count 1 \
    --instance-type t3.medium \
    --key-name ec2-python-key \
    --security-group-ids $SG_ID \
    --user-data '#!/bin/bash
yum update -y
yum install -y docker
systemctl start docker
systemctl enable docker
usermod -a -G docker ec2-user
yum install -y git wget
# Install Python 3.11
yum install -y python3.11 python3.11-pip python3.11-venv'

Troubleshooting

Common Issues

  1. Application won't start

    • Check Python 3.11 is installed: python3 --version
    • Verify port 8080 is open in security group
    • Check application logs

  2. No data in Application Signals

    • Verify environment variables are set
    • Check OpenTelemetry configuration
    • Ensure IAM role has proper permissions

  3. S3 calls failing

    • Attach IAM role with S3 read permissions
    • Verify AWS credentials are configured
