Custom metrics (from apm-center samples)

Some sample services in apm-center include an endpoint used to exercise “custom metrics” behavior during testing. For example, the EKS Python sample provides GET /api/metrics:

Open the EKS Python enablement guide

@app.get("/api/metrics")
async def metrics():
    """Custom metrics endpoint"""
    # Simulate some business logic
    processing_time = random.uniform(0.1, 0.5)
    time.sleep(processing_time)

    # Simulate different response times based on some logic
    if processing_time > 0.3:
        status = "slow"
    else:
        status = "fast"

    return {
        "message": "Metrics endpoint",
        "processing_time": processing_time,
        "status": status,
        "timestamp": time.time(),
        "service": "python-eks-service"
    }
