API Application Signals

Acces programmatique aux donnees et fonctionnalites d'Application Signals.

Operations sur les services

ListServices

Renvoie une liste des services decouverts par Application Signals.

Syntaxe de la requete :
GET /services?nextToken=string&maxResults=integer&startTime=long&endTime=long
Parametres :
  • nextToken - Jeton pour la pagination
  • maxResults - Nombre maximum de resultats (1-100)
  • startTime - Heure de debut en horodatage Unix
  • endTime - Heure de fin en horodatage Unix
Reponse :
{
  "services": [
    {
      "name": "string",
      "namespace": "string",
      "logGroupName": "string"
    }
  ],
  "nextToken": "string"
}

GetService

Renvoie des informations detaillees sur un service specifique.

Syntaxe de la requete :
GET /services/{serviceName}?startTime=long&endTime=long
Parametres de chemin :
  • serviceName - Nom du service
Reponse :
{
  "service": {
    "name": "string",
    "namespace": "string",
    "logGroupName": "string",
    "telemetry": {
      "apm": {
        "status": "ENABLED|DISABLED",
        "lastUpdated": "timestamp"
      },
      "rum": {
        "status": "ENABLED|DISABLED",
        "lastUpdated": "timestamp"
      },
      "synthetics": {
        "status": "ENABLED|DISABLED",
        "lastUpdated": "timestamp"
      }
    }
  }
}

Reference SDK

Documentation SDK specifique a chaque langage.

SDK JavaScript

Installation :

npm install @aws-sdk/client-application-signals

Utilisation :

import { ApplicationSignalsClient, ListServicesCommand } from "@aws-sdk/client-application-signals";

const client = new ApplicationSignalsClient({ region: "us-east-1" });

const command = new ListServicesCommand({
  startTime: new Date(Date.now() - 24 * 60 * 60 * 1000),
  endTime: new Date()
});

const response = await client.send(command);

SDK Python (Boto3)

Installation :

pip install boto3

Utilisation :

import boto3
from datetime import datetime, timedelta

client = boto3.client('application-signals', region_name='us-east-1')

response = client.list_services(
    startTime=datetime.utcnow() - timedelta(days=1),
    endTime=datetime.utcnow()
)

SDK Java

Dependance Maven :

<dependency>
  <groupId>software.amazon.awssdk</groupId>
  <artifactId>application-signals</artifactId>
  <version>2.20.0</version>
</dependency>

Utilisation :

ApplicationSignalsClient client = ApplicationSignalsClient.builder()
    .region(Region.US_EAST_1)
    .build();

ListServicesRequest request = ListServicesRequest.builder()
    .startTime(Instant.now().minus(1, ChronoUnit.DAYS))
    .endTime(Instant.now())
    .build();

ListServicesResponse response = client.listServices(request);

SDK .NET

Package NuGet :

dotnet add package AWSSDK.ApplicationSignals

Utilisation :

using Amazon.ApplicationSignals;
using Amazon.ApplicationSignals.Model;

var client = new AmazonApplicationSignalsClient(RegionEndpoint.USEast1);

var request = new ListServicesRequest
{
    StartTime = DateTime.UtcNow.AddDays(-1),
    EndTime = DateTime.UtcNow
};

var response = await client.ListServicesAsync(request);

Codes d'erreur

Codes d'erreur courants et leur signification.

Erreurs Application Signals

  • ValidationException - Parametres d'entree invalides
  • ResourceNotFoundException - La ressource demandee n'existe pas
  • ThrottlingException - Debit de requetes depasse
  • InternalServerException - Erreur interne du service
  • AccessDeniedException - Permissions insuffisantes

Erreurs X-Ray

  • InvalidRequestException - Requete mal formee
  • ThrottledException - Limite de debit depassee
  • ResourceNotFoundException - Trace introuvable

Erreurs CloudWatch

  • InvalidParameterValueException - Valeur de parametre invalide
  • MissingRequiredParameterException - Parametre requis manquant
  • LimitExceededException - Quota depasse
