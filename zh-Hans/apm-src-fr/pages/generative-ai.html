Observabilité de l'IA générative

Application Signals + Amazon Bedrock

CloudWatch Application Signals prend en charge le dépannage des applications d'IA générative qui interagissent avec les modèles Amazon Bedrock. Il fournit une télémétrie prête à l'emploi pour les appels API Bedrock afin que vous puissiez investiguer :

  • Problèmes de configuration des modèles
  • Facteurs d'utilisation et de coût (utilisation des jetons)
  • Latence des modèles
  • Raisons d'arrêt/de fin des réponses

Métriques et traces détaillées

Pour les appels Bedrock, Application Signals peut générer et corréler des métriques de performance et des spans de traces avec un niveau de détail au niveau des ressources (par exemple : ID du modèle, ID des garde-fous, ID de la base de connaissances et ID de l'agent Bedrock).

Voir : Exemple : dépanner les applications d'IA générative interagissant avec Amazon Bedrock.

Attributs sémantiques GenAI + Transaction Search

Application Signals peut émettre des attributs de convention sémantique OpenTelemetry GenAI (par exemple, gen_ai.request.model et les champs d'utilisation des jetons). Vous pouvez les analyser avec CloudWatch Transaction Search pour comparer les prompts, l'utilisation des jetons et la latence entre les modèles.
