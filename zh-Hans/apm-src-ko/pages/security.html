CloudWatch APM 문서

보안 및 규정 준수

보안

CloudWatch Application Signals는 관찰성 데이터를 보호하고 산업 표준을 준수하기 위해 여러 계층의 보안을 구현합니다.

데이터 암호화

저장 시 암호화:

  • 모든 트레이스 데이터는 AES-256 암호화를 사용하여 암호화됩니다
  • RUM 데이터는 AWS Key Management Service(KMS)로 암호화됩니다
  • 합성 모니터링 데이터는 암호화된 상태로 저장됩니다
  • 추가 제어를 위해 고객 관리 키를 사용할 수 있습니다

전송 중 암호화:

  • 모든 데이터 전송은 TLS 1.2 이상을 사용합니다
  • CloudWatch 서비스에 대한 API 호출은 암호화됩니다
  • RUM 데이터 수집은 HTTPS를 사용합니다
  • 에이전트와 콜렉터 간 통신은 보안 처리됩니다

자격 증명 및 접근 관리

IAM 권한:

Application Signals에 대한 접근 권한을 부여할 때 최소 권한 원칙을 사용합니다:

읽기 전용 접근:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:ListServices",
        "application-signals:GetService",
        "application-signals:ListOperations",
        "application-signals:GetOperation",
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics",
        "xray:GetTraceSummaries",
        "xray:GetTraceGraph",
        "xray:GetServiceGraph"
      ],
      "Resource": "*"
    }
  ]
}
전체 접근:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "application-signals:*",
        "cloudwatch:PutMetricData",
        "xray:PutTraceSegments",
        "xray:PutTelemetryRecords"
      ],
      "Resource": "*"
    }
  ]
}

데이터 개인정보 보호 및 PII 보호

데이터 정제:

  • 민감한 정보에 대한 데이터 마스킹 구현
  • 트레이스 속성의 PII에 대해 토큰화 사용
  • 데이터 보존 정책 구성
  • 수집된 데이터의 정기 감사

모범 사례:

  • 트레이스에 비밀번호, API 키 또는 토큰을 절대 포함하지 마세요
  • 이메일 주소와 전화번호를 해싱하거나 마스킹하세요
  • 민감한 데이터가 포함된 URL에서 쿼리 파라미터를 제거하세요
  • 개인 정보 대신 일반 식별자를 사용하세요

네트워크 보안

VPC 구성:

  • 향상된 보안을 위해 VPC 내에 에이전트를 배포하세요
  • CloudWatch API 호출에 VPC 엔드포인트를 사용하세요
  • 트래픽을 제한하도록 보안 그룹을 구성하세요
  • 추가 제어를 위해 네트워크 ACL을 구현하세요

에이전트 보안:

  • 최소한의 필요 권한으로 에이전트를 실행하세요
  • 보안 패치로 에이전트를 최신 상태로 유지하세요
  • 보안 이벤트에 대해 에이전트 로그를 모니터링하세요
  • 가능한 경우 읽기 전용 접근을 사용하세요

데이터 개인정보 보호

Application Signals가 데이터 개인정보 보호 요구 사항을 처리하는 방법을 이해합니다.

데이터 수집 및 사용

수집하는 데이터:

  • 애플리케이션 성능 메트릭 및 트레이스
  • 서비스 종속성 정보
  • 오류 정보 및 스택 트레이스
  • 시스템 및 인프라 메타데이터

수집하지 않는 데이터:

  • 애플리케이션 비즈니스 데이터 또는 사용자 콘텐츠
  • 데이터베이스 내용 또는 파일 시스템 데이터
  • 사용자 자격 증명 또는 인증 토큰
  • 개인 식별 정보(명시적으로 포함하지 않는 한)

데이터 보존

기본 보존 기간:

  • 트레이스 데이터: 30일 (최대 1년까지 구성 가능)
  • 메트릭: 15개월 (표준 CloudWatch 보존)
  • RUM 데이터: 30일 (최대 1년까지 구성 가능)
  • Synthetics 결과: 90일 (구성 가능)

데이터 삭제:

  • CloudWatch 데이터 보존 정책 사용
  • 자동화된 데이터 라이프사이클 관리 구현
  • 필요한 경우 AWS Support를 통해 데이터 삭제 요청

규정 준수 프레임워크

지원되는 표준:

  • SOC 2: 보안, 가용성 및 기밀성 통제
  • PCI DSS: 결제 카드 산업 데이터 보안 표준
  • HIPAA: 건강 보험 이식성 및 책임에 관한 법률
  • GDPR: 일반 데이터 보호 규정
  • FedRAMP: 연방 위험 및 인증 관리 프로그램

규정 준수 지침

일반적인 규정 준수 시나리오에 대한 구체적인 지침입니다.

HIPAA 규정 준수

요구 사항:

  • 트레이스에서 보호 건강 정보(PHI)를 암호화
  • 접근 제어 및 감사 로깅 구현
  • AWS와 비즈니스 협력 계약(BAA) 사용
  • 적절한 데이터 보존 정책 구성

구현:

  • 트레이스 속성 및 스팬 이름에서 PHI 마스킹
  • 고객 관리 KMS 키 사용
  • 감사 로깅을 위해 CloudTrail 활성화
  • 데이터 격리를 위해 VPC 엔드포인트 구성

PCI DSS 규정 준수

주요 고려 사항:

  • 트레이스 또는 로그에 카드 소유자 데이터를 절대 포함하지 마세요
  • 네트워크 분리 구현
  • 전송 중 및 저장 시 데이터에 암호화 사용
  • 정기적인 보안 평가 및 침투 테스트

모범 사례:

  • 계측 전에 민감한 데이터 토큰화
  • Application Signals 에이전트에 프라이빗 서브넷 사용
  • 최소 권한 IAM 정책 구현
  • 수집된 데이터의 정기 검토

GDPR 규정 준수

데이터 주체 권리:

  • 개인 데이터에 대한 접근권
  • 정정권
  • 삭제권 ("잊힐 권리")
  • 데이터 이식권

구현:

  • 텔레메트리 데이터에서 PII 수집 최소화
  • 데이터 익명화 기술 구현
  • 비즈니스 요구에 따라 데이터 보존 구성
  • 데이터 처리 활동 문서화

보안 모니터링

보안 이벤트에 대해 Application Signals 배포를 모니터링합니다.

보안 모범 사례

  • API 활동 로깅을 위해 CloudTrail 활성화
  • IAM 정책 변경 및 접근 패턴 모니터링
  • 비정상적인 데이터 접근 패턴에 대한 알림 설정
  • 정기적인 보안 평가 및 취약점 스캔
  • 다중 인증(MFA) 구현

인시던트 대응

  • 보안 이벤트에 대한 인시던트 대응 계획 수립
  • 침해 시 Application Signals를 비활성화하는 방법 숙지
  • 에스컬레이션 절차 문서화
  • 정기적인 인시던트 대응 테스트
