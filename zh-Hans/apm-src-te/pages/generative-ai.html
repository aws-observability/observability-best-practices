జనరేటివ్ AI observability

Application Signals + Amazon Bedrock

CloudWatch Application Signals Amazon Bedrock మోడల్‌లతో ఇంటరాక్ట్ అయ్యే జనరేటివ్ AI అప్లికేషన్‌లను ట్రబుల్‌షూట్ చేయడానికి మద్దతు ఇస్తుంది. ఇది Bedrock API కాల్‌ల కోసం అవుట్-ఆఫ్-ది-బాక్స్ టెలిమెట్రీని అందిస్తుంది, తద్వారా మీరు ఈ క్రింది వాటిని పరిశోధించవచ్చు:

  • మోడల్ కాన్ఫిగరేషన్ సమస్యలు
  • వినియోగం మరియు ఖర్చు డ్రైవర్లు (టోకెన్ వినియోగం)
  • మోడల్ లేటెన్సీ
  • ప్రతిస్పందనల కోసం ఆగిన/ముగింపు కారణాలు

సూక్ష్మ-స్థాయి మెట్రిక్స్ మరియు ట్రేస్‌లు

Bedrock కాల్‌ల కోసం, Application Signals రిసోర్స్-స్థాయి వివరాలతో (ఉదాహరణకు: మోడల్ ID, guardrails ID, knowledge base ID, మరియు Bedrock agent ID) పనితీరు మెట్రిక్స్ మరియు ట్రేస్ స్పాన్‌లను ఉత్పత్తి చేసి సంబంధం కలుపగలదు.

చూడండి: ఉదాహరణ: Amazon Bedrock తో ఇంటరాక్ట్ అయ్యే జనరేటివ్ AI యాప్‌లను ట్రబుల్‌షూట్ చేయడం.

GenAI సెమాంటిక్ అట్రిబ్యూట్‌లు + Transaction Search

Application Signals OpenTelemetry GenAI సెమాంటిక్ కన్వెన్షన్ అట్రిబ్యూట్‌లను (ఉదాహరణకు, gen_ai.request.model మరియు టోకెన్ వినియోగ ఫీల్డ్‌లు) విడుదల చేయగలదు. ప్రాంప్ట్‌లు, టోకెన్ వినియోగం మరియు మోడల్‌ల మధ్య లేటెన్సీని పోల్చడానికి మీరు వీటిని CloudWatch Transaction Search తో విశ్లేషించవచ్చు.

  • Transaction Search
  • స్పాన్‌లను శోధించడం మరియు విశ్లేషించడం

మరింత చదవడం

  • AWS ప్రకటన: Application Signals Bedrock కు మద్దతు ఇస్తుంది
  • బ్లాగ్: Amazon CloudWatch AppSignals తో Amazon Bedrock observability ను మెరుగుపరచండి
