2024年6月：Application Signals の一般提供開始

CloudWatch Application Signals が一般提供（GA）となりました。コール数、可用性、レイテンシー、障害、エラーなどの標準的なアプリケーションメトリクスを自動収集することで、サービスレベルのトリアージとトラブルシューティングの迅速化に重点を置いています。

AWS アナウンスを参照： GA アナウンス.

2024年9月：リクエストベースの SLO

AWS は CloudWatch Application Signals に対するリクエストベースのサービスレベル目標（SLO）サポートを発表しました。

AWS アナウンスを参照： リクエストベースの SLO.

継続中：Application Map とトリアージワークフロー

CloudWatch の Application Map は、Application Signals における「Service Map」という用語に置き換わるものです。トポロジー（クライアント、カナリア、サービス、依存関係）を可視化し、SLO/SLI の健全性を表示することでチームのインシデントトリアージを支援します。

ドキュメントを参照： Application Map.
