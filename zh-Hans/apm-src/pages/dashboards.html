Dashboards

Built-in Application Signals views

Application Signals provides an application-centric workflow in the CloudWatch console:

  • Services: list services, spot high fault/latency, see SLI health when SLOs exist.
  • Service detail: drill into operations, dependencies, and correlated traces.
  • Application Map: visualize topology and propagate health across dependencies.

See: Operational health monitoring with Application Signals.

CloudWatch Dashboards (custom)

You can also build CloudWatch dashboards using metrics emitted by Application Signals (namespace ApplicationSignals) and correlate with alarms/logs as needed.

  • Service metrics: latency, faults, errors across Service/Operation/Environment dimensions.
  • Dependency metrics: remote service/operation dimensions for downstream calls.
  • Runtime metrics (where supported): correlate GC/CPU/memory with latency spikes.

See: Metrics collected by Application Signals.
