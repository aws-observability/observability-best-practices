This site is a static docs scaffold. To keep the content accurate, “What’s New” here summarizes notable public milestones and points to authoritative AWS sources for details.

  • AWS announcements: AWS “What’s New”
  • CloudWatch docs: Application Signals

June 2024: Application Signals general availability

CloudWatch Application Signals became generally available (GA). It focuses on fast service-level triage and troubleshooting by automatically collecting standard application metrics such as call volume, availability, latency, faults, and errors.

See AWS announcement: GA announcement.

September 2024: Request-based SLOs

AWS announced request-based service level objectives (SLOs) support for CloudWatch Application Signals.

See AWS announcement: Request-based SLOs.

Ongoing: Application Map and triage workflow

The CloudWatch Application Map replaces the “Service Map” terminology for Application Signals. It visualizes topology (clients, canaries, services, dependencies) and can display SLO/SLI health to help teams triage incidents.

See docs: Application Map.
